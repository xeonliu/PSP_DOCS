SceKernelLMOption
 [構造体]
sceKernelLoadModule()のオプション構造体

構造

typedef struct SceKernelLMOption {
	SceSize size;
	SceUID mpidtext;
	SceUID mpiddata;
	unsigned int flags;
	char position;
	char access;
	char creserved[2];
} SceKernelLMOption;
メンバ

  size     sizeof(SceKernelLMOption)を指定します。  
  mpidtext     モジュールのプログラムパートをロードする先のメモリ  
  パーティションを指定します。  
  mpiddata     モジュールのデータパートをロードする先のメモリ  
  パーティションを指定します。  
  現在、有効なオプションではありません。  
  flags     現在意味がある設定はありません。  
  position     モジュールの配置方針を以下のいずれかで指定します。  
  SCE_KERNEL_LM_POS_Low (= 0):  
  　モジュールはできるだけ若いアドレスに配置  
  SCE_KERNEL_LM_POS_High (= 1):  
  　モジュールはできるだけ大きいアドレスに配置  
  access     ロード方式を指定します。  
  現在、有効なオプションではありません。  
  creserved     予約領域です。現在、有効ではありません。  

解説

LoadModuleの細かな制御を指定するためのオプション構造体です。

mpidtextにSCE_KERNEL_UNKNOWN_PARTITION(= 0)を指定すると、モジュールに応じた適切なメモリパーティションへロードすることができます。

以下のようにメンバの値を設定すると、LoadModuleの引数にこのオプション構造体を指定しなかった場合と同じ動作となります。

  mpidtext = SCE_KERNEL_UNKNOWN_PARTITION;
  position = SCE_KERNEL_LM_POS_Low;

SceKernelSMOption
 [構造体]
sceKernelStartModule() / sceKernelStopModule()のオプション構造体

構造


typedef struct SceKernelSMOption {
	SceSize size;
	SceUID mpidstack;
	SceSize stacksize;
	int priority;
	unsigned int attribute;
} SceKernelSMOption;
メンバ

  size     構造体のサイズを指定します。  
  mpidstack     起動スレッドのスタックのパーティションを指定します。  
  0を指定した場合、モジュールのデータセグメントと  
  同一パーティションにスタックを確保します。  
  stacksize     スタックサイズを指定します。  
  0を指定した場合、システム標準値を設定します。  
  priority     起動スレッドの優先度を指定します。  
  attribute     起動スレッドの属性を指定します。  

解説

sceKernelStartModule(), sceKernelStopModule()の細かな設定を行うためのオプション構造体です。

SceKernelModuleInfo
 [構造体]
sceKernelQueryModuleInfo()のインフォメーション構造体

構造

#define SCE_KERNEL_MAX_MODULE_SEGMENT  4

typedef struct SceKernelModuleInfo {
	SceSize size;
	char nsegment;

メンバ

  size     構造体のサイズを指定します。  
  nsegment     モジュールの持つセグメントの数  
  reserved     予約領域です。現在、有効ではありません。  
  segmentaddr     セグメントの開始アドレス  
  segmentsize     セグメントのサイズ  
  entry_addr     エントリ関数のアドレス  
  gp_value     gpの値  
  text_addr     テキストセグメントのアドレス

  data_size     データセグメントのアドレス  
  bss_size     データセグメントのサイズ  
  modattribute     モジュール属性  
  modversion     モジュールのバージョン  
  modname     モジュール名  
  terminal     文字列の終端です。常に0です。  

解説

sceKernelQueryModuleInfo()によってモジュールの情報を取得するためのインフォメーション構造体です。

sceKernelLoadModule
 [モジュール管理]
プログラムモジュールをファイルパスからロードする

形式

#include <kernel.h>
SceUID sceKernelLoadModule(
	const char *fileName,
	int flag,
	const SceKernelLMOption *opt
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  fileName     ロードするモジュールのファイル名  
  flag     現在有効な設定はありません。  
  opt     ロード時の挙動を指定するための  
  SceKernelLMOption構造体へのポインタを指定します。  
  不要な場合はNULLを指定します。  
  構造体を指定する場合、構造体のsizeフィールドに構造体のサイズを  
  設定してください。  

返り値

  modid (modid>0)     ロードしたモジュールのUID  
SCE_KERNEL_ERROR_ILLEGAL_CONTEXT
SCE_KERNEL_ERROR_ILLEGAL_OBJECT
SCE_KERNEL_ERROR_NO_MEMORY
SCE_KERNEL_ERROR_NOFILE
SCE_KERNEL_ERROR_FILEERR
SCE_KERNEL_ERROR_MODULE_MGR_BUSY
SCE_KERNEL_ERROR_ILLEGAL_SIZE
SCE_KERNEL_ERROR_ILLEGAL_PERM_CALL

解説

ファイル名で指定されるプログラムモジュールをロードします。
この後、sceKernelStartModule()を呼び出して、プログラムモジュールを起動する必要があります。

引数optにSceKernelLMOption構造体を指定する場合には、構造体のsizeフィールドに必ず構造体のサイズを設定してください。設定していない場合にはSCE_KERNEL_ERROR_ILLEGAL_SIZEのエラーが返ります。

sceKernelStartModule
 [モジュール管理]
ロードされた未起動のプログラムモジュールを起動する

形式

#include <kernel.h>
int sceKernelStartModule(
	SceUID modid,
	SceSize args,
	const void *argp,
	int *modresult,
	const SceKernelSMOption *opt
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  modid     ロードしたモジュール識別子  
  args     モジュールの開始エントリに与えるパラメータのサイズ  
  argp     モジュールの開始エントリに与えるパラメータへのポインタ  
  modresult     モジュールの開始エントリの返り値  
  opt     起動スレッドの挙動を設定します。  
  不要な場合はNULLを指定します。  
  構造体を指定する場合、構造体のsizeフィールドに構造体のサイズを  
  設定してください。  

返り値

  modid (modid>0)     スタートし常駐したモジュールのUID  
SCE_KERNEL_ERROR_OK
SCE_KERNEL_ERROR_ERROR
SCE_KERNEL_ERROR_ILLEGAL_ADDR
SCE_KERNEL_ERROR_ILLEGAL_CONTEXT
SCE_KERNEL_ERROR_LIBRARY_NOTFOUND
SCE_KERNEL_ERROR_MODULE_MGR_BUSY
SCE_KERNEL_ERROR_ILLEGAL_SIZE

解説

sceKernelLoadModule()でロードしたプログラムモジュールを起動します。一回、スタートしたモジュールは、ストップ、アンロードを行い、再度ロードされるまでスタートできません。args,argpはモジュール固有のスタートエントリに渡されます。modresultはモジュールのスタートエントリの呼び出し結果です。常駐モジュールの場合はモジュール識別子、非常駐モジュールの場合はSCE_KERNEL_ERROR_OKを返します。スタートエントリ内部で再度モジュール操作を行うと、デッドロックになるので注意してください。

引数optにSceKernelSMOption構造体を指定する場合には、構造体のsizeフィールドに必ず構造体のサイズを設定してください。設定していない場合にはSCE_KERNEL_ERROR_ILLEGAL_SIZEのエラーが返ります。

sceKernelStopModule
 [モジュール管理]
既にロードされ起動済のプログラムモジュールを停止する

形式

#include <kernel.h>
int sceKernelStopModule(
	SceUID modid,
	SceSize args,
	const void *argp,
	int *modresult,
	const SceKernelSMOption *opt
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  modid     ロードしたモジュール識別子  
  args     モジュールの停止エントリに与えるパラメータのサイズ  
  argp     モジュールの停止エントリに与えるパラメータへのポインタ  
  modresult     モジュールの停止エントリの返り値  
  opt     停止スレッドの挙動を設定します。不要な場合はNULLを指定します。  
  構造体を指定する場合、構造体のsizeフィールドに構造体のサイズを  
  設定してください。  

返り値

  modid (modid>0)     ストップに失敗したモジュールのUID  
SCE_KERNEL_ERROR_OK
SCE_KERNEL_ERROR_ERROR
SCE_KERNEL_ERROR_ILLEGAL_ADDR
SCE_KERNEL_ERROR_ILLEGAL_CONTEXT
SCE_KERNEL_ERROR_MODULE_MGR_BUSY
SCE_KERNEL_ERROR_ALREADY_STOPPING
SCE_KERNEL_ERROR_ALREADY_STOPPED
SCE_KERNEL_ERROR_ILLEGAL_SIZE

解説

sceKernelStartModule()で起動したモジュールのストップエントリを呼び出します。
ストップエントリ成功時に、モジュールを停止します。
ストップエントリの呼び出しに成功したが、ストップに失敗した場合は、モジュール識別子を返します。ストップエントリがストップに成功した場合は、SCE_KERNEL_ERROR_OKを返します。
ストップエントリ内部で再度モジュール操作を行うとデッドロックになります。

引数optにSceKernelSMOption構造体を指定する場合には、構造体のsizeフィールドに必ず構造体のサイズを設定してください。設定していない場合にはSCE_KERNEL_ERROR_ILLEGAL_SIZEのエラーが返ります。

sceKernelUnloadModule
 [モジュール管理]
プログラムモジュールの削除

形式

#include <kernel.h>
int sceKernelUnloadModule(
	SceUID modid
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  modid     モジュール識別子  

返り値

  modid (modid>0)     アンロードに成功したモジュールのUID  
SCE_KERNEL_ERROR_ERROR
SCE_KERNEL_ERROR_ILLEGAL_ADDR

SCE_KERNEL_ERROR_MODULE_MGR_BUSY

解説

プログラムモジュールを削除します。削除するモジュールはスタートしていないかストップしてある必要があります。削除に成功したモジュールのモジュール識別子を返します。

APIを呼び出したプログラムモジュールを停止、削除します。STOPエントリに対するパラメータを与えることができます。
削除するモジュールはスタートしていなければいけません。
また呼び出しを行ったスレッドを停止、削除します。
機能時には、sceKernelStopUnloadSelfModuleWithStatus()のサブセットであり、そちらをご利用ください。下位互換性のため残しております。

引数pOptにSceKernelSMOption構造体を指定する場合には、構造体のsizeフィールドに必ず構造体のサイズを設定してください。設定していない場合には、SCE_KERNEL_ERROR_ILLEGAL_SIZEのエラーが返ります。

sceKernelQueryModuleInfo
 [モジュール管理]
プログラムモジュールの情報を取得

形式

#include <kernel.h>
int sceKernelQueryModuleInfo (
	SceUID uid,
	SceKernelModuleInfo *pInfo
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフである

引数

  uid     モジュール識別子  
  pInfo     モジュール情報を取得するためのインフォメーション構造体。  
  構造体のsizeフィールドに構造体のサイズを設定してください。  

返り値

SCE_KERNEL_ERROR_OK
SCE_KERNEL_ERROR_ERROR
SCE_KERNEL_ERROR_ILLEGAL_ADDR
SCE_KERNEL_ERROR_ILLEGAL_CONTEXT
SCE_KERNEL_ERROR_UNKNOWN_MODULE
SCE_KERNEL_ERROR_CANNOT_GET_MODULE_INFORMATION
SCE_KERNEL_ERROR_ILLEGAL_SIZE

解説

プログラムモジュールの情報を取得します。
情報を取得する対象となるプログラムモジュールは、sceKernelLoadModule()またはsceKernelLoadModuleByID()でロードされたものを指定します。アンロードされたモジュールの状態を取得できません。

SceKernelModuleInfo構造体のsizeフィールドに必ず構造体のサイズを設定してください。設定していない場合には、SCE_KERNEL_ERROR_ILLEGAL_SIZEのエラーが返ります。

sceKernelGetModuleIdByAddress
 [モジュール管理]

#include <kernel.h>
int sceKernelGetModuleIdByAddress (
	const void* base
);
呼出条件

割り込みハンドラから呼び出し不可

指定したアドレスに対応するモジュールのモジュール識別子を返します。baseにはユーザ空間のアドレスしか指定できません。

sceKernelGetModuleIdList
 [モジュール管理]
ユーザ空間に存在するモジュール番号のリストを取得

形式

#include <kernel.h>
int sceKernelGetModuleIdList (
	SceUID *readbuf,
	int readbufsize,
	int *count,
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフである

引数

  readbuf     モジュール識別子を格納するバッファ  
  readbufsize     上記のバイトサイズ  
  count     モジュール数を格納するアドレス  

返り値

SCE_KERNEL_ERROR_OK
SCE_KERNEL_ERROR_ERROR
SCE_KERNEL_ERROR_ILLEGAL_CONTEXT
SCE_KERNEL_ERROR_ILLEGAL_ADDR

解説

ユーザモジュールの識別子のリストを返します。モジュール数がreadbufsizeに収まらない場合は、収まる分だけ返します。
