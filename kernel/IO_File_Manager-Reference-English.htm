SceIoStat
 [Structure]
Structure for obtaining status

Structure

struct SceIoStat {
	SceMode st_mode;
	unsigned int st_attr;
	SceOff st_size;
	ScePspDateTime st_ctime;
	ScePspDateTime st_atime;
	ScePspDateTime st_mtime;
	unsigned int st_private[6];
};
Members

  st_mode     File type (file/dir) and mode (R/W/X)   
  st_attr     Device-dependent attribute   
  st_size     File size   
  st_ctime     Creation time   
  st_atime     Last access time   
  st_mtime     Last modification time   
  st_private[6]     Other  

Description

The following structure is used by the sceIoGetstat(), sceIoChstat() functions.
This structure is defined in iofilemgr_stat.h. Be sure to include iofilemgr_stat.h when you use the SceIoStat structure.
The st_mode field of SceIoStat contains 32 bits of information representing the file type and access permissions.
Bits 0-11 contain flags representing the file access permissions.

  Value     Symbol     Meaning  
  0001     SCE_STM_XOTH     other execute permission   
  0002     SCE_STM_WOTH     other write permission   
  0004     SCE_STM_ROTH     other read permission   
  0010     SCE_STM_XGRP     group execute permission  
  0020     SCE_STM_WGRP     group write permission   
  0040     SCE_STM_RGRP     group read permission   
  0100     SCE_STM_XUSR     user execute permission  
  0200     SCE_STM_WUSR     user write permission  
  0400     SCE_STM_RUSR     user read permission   

Bits 12-15 store file type information.
  Value     Symbol     Meaning  
  0x1000     SCE_STM_FDIR     directory  
  0x2000     SCE_STM_FREG     regular file  
  0x4000     SCE_STM_FLNK     symbolic link   

All time data is set to NULL if the device has no time data.

sceIoOpen / sceIoOpenCB
 [Function]
Open file

Syntax

#include <iofilemgr.h>
#include <kerror.h>

SceUID sceIoOpen(
	const char *filename,
	int flag,
	SceMode mode
);
SceUID sceIoOpenCB(
	const char *filename,
	int flag,
	SceMode mode
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

 filename   Specifies the filename to be opened. The filename must have the following format.  
 Device name:[Filename within device]  
 Example  
 "tty0:" 
 "tty:" (same as "tty0:") 
 "tty1:" 
 "host0:hostfilename" 
 "disc0:/PSP_GAME/USRDIR/demo01.pmf" 
 flag   Specifies one of SCE_O_RDONLY, SCE_O_WRONLY,  
 SCE_O_RDWR for the mode of the file to be opened. 
 SCE_O_RDONLY   Open for read only 
 SCE_O_WRONLY   Open for write only 
 SCE_O_RDWR   Open for read and write 
 In addition, depending on the device, the following can also be logically ORed with mode. 
 SCE_O_APPEND   Append write mode. Writes are always  
 done to the end of the file. 
 SCE_O_CREAT   Create the file if it does not already exist 
 SCE_O_TRUNC   When the file is opened for writing, 
  discard the contents and reset the size to 0 
 SCE_O_EXCL   Report an error if a file with the same  
 name exists when SCE_O_CREAT is specified 
 SCE_O_FDEXCL   Report an error if a thread other than the thread that opened the file operates the file descriptor 
 SCE_O_PWLOCK   Disable control of the power state from the time the file is opened until it is closed. 
 mode   When flag is set to SCE_O_CREAT, mode specifies the mode of the file to be  
 created. This value is reflected in the st_mode field of the SceIoStat structure  
 when the file status is obtained from sceIoGetstat(). 

Return Value

  fd(fd>=0)     File descriptor  
SCE_KERNEL_ERROR_MFILE
SCE_KERNEL_ERROR_NODEV

  Other negative values      Driver error code  
(normally an error macro from kerror.h)

Description

Opens a file. If the open is successful, returns a small positive integer (the file descriptor).
sceIoOpenCB() is a service call equivalent to sceIoOpen() except that a function has been added for checking for the existence of a callback notification when a wait state is entered. If a callback is reported while waiting for the completion of processing, the wait state is temporarily exited, the callback function is executed, and then the wait state is entered again.

Note
If SCE_O_PWLOCK is specified, a power lock will be in effect from the time the file is opened until sceIoClose() is called to close the respective fd. While the power lock is active, the user will no longer be able to turn off the power by operating the POWER/HOLD switch. Consequently, usability will be reduced if SCE_O_PWLOCK is carelessly used.
As mentioned above, if a file is opened with SCE_O_PWLOCK, a POWER/HOLD switch operation will be delayed until sceIoClose() is called to cancel the lock state. However, if an error occurs in a function such as sceIoRead() and the operation is retried, the power lock may end up staying active for a long time, and the power may turn off at an unexpected time from the user's viewpoint.

Also, after suspend/standby processing has begun, if sceIoOpen() is called to open a file with SCE_O_PWLOCK, the thread may be blocked before the file is opened.

Always use the power lock discreetly. Avoid using it carelessly.

sceIoWaitAsync / sceIoWaitAsyncCB / sceIoPollAsync
 [Function]
Get execution result of asynchronous I/O processing

Syntax

#include <iofilemgr.h>
#include <kerror.h>

int sceIoWaitAsync(
	SceUID fd,
	SceIores *result
);
int sceIoWaitAsyncCB(
	SceUID fd,
	SceIores *result
);
int sceIoPollAsync(
	SceUID fd,
	SceIores *result
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  fd     File descriptor returned by sceIoOpen() when the file was opened  
  result     Pointer to variable for storing the result of file I/O processing that was executed asynchronously  

Return Value

  0     Asynchronous processing completed  
  1     Asynchronous processing still executing  
SCE_KERNEL_ERROR_BADF
SCE_KERNEL_ERROR_NOASYNC

  Other negative values      Error code  
(normally an error macro from kerror.h)

Description

Gets the execution result of file I/O processing being executed asynchronously.
sceIoWaitAsync() waits until asynchronous I/O processing is completed and returns 0 after storing the result in the variable pointed to by the result argument.
sceIoWaitAsyncCB() is a service call equivalent to the sceIoWaitAsync() service call with a function added for checking for the existence of a callback notification when the PSP™ enters a wait state. If a callback is reported while the PSP™ is waiting for the completion of asynchronous I/O processing, the PSP™ temporarily exits the wait state to execute the callback function and then enters the wait state again.
sceIoPollAsync() is a service call equivalent to the sceIoWaitAsync() service call with the function for entering a wait state if asynchronous I/O processing is not completed removed. If asynchronous I/O processing is not completed, 1 is returned for the return value. In this case, the value of the variable pointed to by the result argument is not changed.

When 0 is returned by the function, it means that asynchronous I/O processing that was started by sceIoReadAsync() or sceIoWriteAsync() has completed. In that case, the asynchronous I/O result can be obtained by any of the sceIoWaitAsync(), sceIoWaitAsyncCB(), or sceIoPollAsync() APIs.
Other asynchronous I/O processing cannot be invoked for the same file descriptor until the current asynchronous I/O processing has completed.

sceIoSetAsyncCallback
 [Function]
Set callback for asynchronous processing notification

	SceUID fd,
	SceUID cbid,
	int arg
);

Multithread safe

Arguments

  arg     Argument to be passed to the callback function  

Return Value

  0     Success  
SCE_KERNEL_ERROR_BADF

  Other negative values      Error code  
(normally an error macro from kerror.h)

Description

Sets the callback ID of the thread manager that notifies when the asynchrnous processing of the file I/O has been completed.
The callback must be generated in advance with sceKernelCreateCallback(). When the asynchronous I/O processing is completed, sceKernelNotifyCallback() will notify to the specified callback ID. After that, the thread that generated the specified callback will check the existence of the notification with sceKernelCheckCallback(), or by executing a wait API with callback calling (such as sceKernelSleepThreadCB()), the callback function set when the callback was generated will be called.
In order to complete the asynchronous I/O processing, it is necessary to execute a function such as sceIoPollAsync() within the callback function that has been called and receive the execution result of the file I/O processing.
