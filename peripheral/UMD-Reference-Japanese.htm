SCE_UMD_ALIAS_NAME
 [定数定義]
UMD®メディアエイリアス名

定義

#include	<mediaman.h>
#define  SCE_UMD_ALIAS_NAME		"disc0:"
解説

PSP®のUMD®メディアドライブのエイリアス名を表す定数です。

"disc"という名前で、ユニット番号0:となっています。
PSP®でエイリアス名を指定する場合は、この定数をお使いください。
sceUmdActivate()関数等で使用されます。

関連項目

sceUmdActivate()

UMD®ドライブ状態
 [定数定義]
UMD®ドライブ状態

定義

#include	<mediaman.h>
  マクロ     説明  
  SCE_UMD_MEDIA_OUT     メディアが抜かれた。  
  SCE_UMD_MEDIA_IN     メディアが挿された。  
  SCE_UMD_MEDIA_CHG     前のメディアと異なるメディアの挿入。  
  （コールバック非通知）  
  SCE_UMD_READY     ドライブレディ状態である。  
  （ブロックアクセスが可能な状態）  
  SCE_UMD_NOT_READY     ドライブレディ状態でない。  
  SCE_UMD_READABLE     読み込み可能状態である。  
  （ファイルアクセスが可能な状態）  


解説

UMD®へのアクセス可能・不可能状態を表した定数です。

UMD®メディアの挿入状態も含みます。
UMD®ドライブ状態の多くは、エッジトリガで通知されます。
起動時のUMD®メディアの有無は、sceUmdCheckMedium()関数によって確認できます。
その他のUMD®ドライブ状態を取得する方法は、sceUmdRegisterUMDCallBack()関数でコールバックを使用する方法と、sceUmdGetDriveStat()関数で状態を取得する方法があります。

SceUmdSeekParam
 [構造体]
シーク設定情報

構造

#include <umddevctl.h>
typedef struct {
	SceOff sk_off;
	SceInt32 sk_reserved;
	SceInt32 sk_whence;
}SceUmdSeekParam;
メンバ

  sk_off     ファイルオフセット(byte)  
  sk_reserved     パディング(未使用)  
  sk_whence     移動方法  

解説

ファイルのシーク情報を格納する構造体です。

sceIoIoctl *()関数のSCE_ISOFS_SEEK_FILE、SCE_UMD_SEEK_FILEコマンドの設定パラメータとして使用します。
ISO9660ファイルシステムに対するsk_offメンバに設定される値は、バイト単位、ブロックデバイスに対するsk_offメンバに設定される値は、セクタ単位として処理されます。
sk_whenceメンバは、下の表のようになります。

  マクロ     説明  
  SCE_SEEK_SET     sk_off はファイルの先頭からの距離。  
  SCE_SEEK_CUR     sk_off は現在のポインタからの相対距離。  
  SCE_SEEK_END     sk_off はファイルの末尾からの距離。

sceUmdActivate
 [関数]
UMD®ドライブのアクティベート処理

形式

#include <mediaman.h>
int sceUmdActivate(
	const int mode,
	const char *aliasname
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  mode     電源モード  
  aliasname     "disc:0"又はSCE_UMD_ALIAS_NAME  

返り値

成功時には、SCE_KERNEL_ERROR_OKが返ります。
エラーの場合は、負の値が返ります。

解説

UMD®/DVDドライブのアクティベートを行う関数です。

ドライブのアクティベート処理では、UMD®/DVDにアクセスするために、内部処理としてsceIoAssign()を実行し、ファイルシステム"isofs0:"、ブロックデバイス"umd0:"、エイリアス名"disc0:"でドライブをアサインします。
現在、引数modeには、SCE_UMD_MODE_POWERONを指定してください。このモードは、UMD®ドライブの電源状態がONでなかった場合、電源投入します。なお、電源投入は、UMD®がDRIVE SELECTされている場合のみ行われます。DVDの電源制御は、ここでは行われません。

本関数を利用する場合は、mediaman_stub.aをリンクしてください。

sceUmdRegisterUMDCallBack
 [関数]
UMD®状態遷移時に呼ばれるコールバックの登録

形式

#include <mediaman.h>
SceUID sceUmdRegisterUMDCallBack(
	SceUID cb_id
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  cb_id     コールバックID  

返り値

成功した場合は、SCE_KERNEL_ERROR_OKが返ります。
エラーの場合は、負の値が返ります。

解説

UMD®状態遷移時に呼ばれるコールバックの登録を行う関数です。

sceKernelCreateCallback()で取得した、コールバックIDを引数cd_idに設定してください。
コールバックは、UMD®の状態遷移時に即座に呼ばれるわけではなく、sceKernelCheckCallback()、およびsceKernelXXXXXXCB()関数実行時に、コールバックされます。

sceKernelCreateCallback()で登録するコールバックのプロトタイプは、以下のようになっています。

int callback(int count, int arg, void *param);
UMD®の状態遷移時に呼ばれるコールバックでは、第二引数argからどの状態になったのかを判断するための値が設定されます。値は、論理和(OR)で設定されています。
値は、以下のようになります。

  マクロ     説明  
  SCE_UMD_MEDIA_OUT     メディアが抜かれた。  
  SCE_UMD_MEDIA_IN     メディアが挿された。  
  SCE_UMD_READY     ドライブレディ状態である。  
  SCE_UMD_NOT_READY     ドライブレディ状態でない。  
  SCE_UMD_READABLE     読み込み可能状態である。  


各状態の詳細については、「UMD-Overview」の「UMD®ドライブ状態」を参照してください。
コールバックの詳細については、「スレッドマネージャ」の「コールバック関数」の説明を参照してください。
本関数を利用する場合は、mediaman_stub.aをリンクしてください。

sceUmdGetErrorStat
 [関数]
UMD®のエラーステータス状態の取得

形式

#include <mediaman.h>
int sceUmdGetErrorStat(
	void
);
呼出条件

割り込みハンドラからの呼び出し可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

なし

返り値

エラーの場合は、UMD®エラー状態を負の値で返します。
エラーがない場合は、SCE_KERNEL_ERROR_OKを返します。

解説

UMD®エラー状態を取得する関数です。

sceUmdActivate()関数などは、内部処理の終了を待たずに処理を戻す関数であるため、内部でのアサイン処理が正しく終了したか判断できません。その場合、本関数を使うことで確認することができます。
例えば、メディアが挿入されたにもかかわらず、UMD®ドライブ状態が、NOT_READY状態の場合、エラーを戻している可能性があります。
また、ディスクの入れ替えが禁止されている状態で、別のディスクが入れられた場合には、本関数はSCE_UMD_ERROR_REPLACEDを返します。

本関数を利用する場合は、mediaman_stub.aをリンクしてください。

sceUmdWaitDriveStat / sceUmdWaitDriveStatWithTimer / sceUmdWaitDriveStatCB
 [関数]
UMD®ドライブステータスイベント待ち

形式

#include <mediaman.h>
int sceUmdWaitDriveStat(
	const unsigned int state
);
int sceUmdWaitDriveStatWithTimer(
	const unsigned int state,
	const int timer
);
int sceUmdWaitDriveStatCB(
	const unsigned int state,
	const int timer
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  state     ステータスイベント  
  timer     タイマ(μs)  

返り値

成功した場合は、SCE_KERNEL_ERROR_OKが返ります。
エラーの場合は、負の値が返ります。

解説

stateに設定したUMD®ドライブステータス（UMD®ドライブ状態）イベントが発生するまで、本関数を呼んだスレッドをウェイトする関数です。

ステータスイベントは、ドライブステータスと一致しています。
sceUmdWaitDriveStat()は、指定したステータスになるまでウェイトします。
sceUmdWaitDriveStatWithTimer()は、指定したステータスになるか、指定タイムアウトまでウェイトします。
sceUmdWaitDriveStatCB()は、sceUmdWaitDriveStatWithTimer()の動作中に加え、コールバック通知の有無をチェックを行います。

引数timerには、設定した値のマイクロ秒単位以上経過すると、タイムアウトで処理を抜けます。
引数stateには、以下のマクロが指定可能です。

  マクロ     説明  
  SCE_UMD_MEDIA_OUT     メディアが抜かれた。  
  SCE_UMD_MEDIA_IN     メディアが挿された。  
  SCE_UMD_MEDIA_CHG     前のメディアと異なるメディアが挿入された。  
  （コールバック非通知）  
  SCE_UMD_READY     ドライブレディ状態である。  
  SCE_UMD_NOT_READY     ドライブレディ状態でない。  
  SCE_UMD_READABLE     読み込み可能状態である。  


上記以外のステータスイベントを設定すると直ちにエラーとなります。
本関数を利用する場合は、mediaman_stub.aをリンクしてください。
