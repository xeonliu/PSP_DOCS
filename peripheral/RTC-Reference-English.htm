RTC Service Reference

SceRtcTick
 [Structure]
Time information in ticks

Structure

#include <psptypes.h>
#include <rtcsvc.h>
typedef struct SceRtcTick {
	SceULong64 tick;
} SceRtcTick;
Members

  tick     Cumulative number of Ticks from 0001/01/01 00:00:00  

Description

This is a structure for handling time information in a uniform format. It is used for performing operations such as addition or subtraction on time values. This structure can be converted to and from the ScePspDateTime structure by using sceRtcGetTick() and sceRtcSetTick().

See Also

ScePspDateTime, sceRtcGetTick(), sceRtcSetTick()

sceRtcGetCurrentTick
 [Function]
Get current time (UTC) in Tick representation

Syntax

#include <rtcsvc.h>
int sceRtcGetCurrentTick (
	SceRtcTick *pTick
);

#define sceRtcGetCurrentTickUTC(_tick)	sceRtcGetCurrentTick(_tick)
Calling Conditions

Can be called from an interrupt handler (does not depend on interrupt-disabled or -enabled state)
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pTick     Pointer to SceRtcTick for receiving the current time (UTC)  

Return Value

If an error occurs, a negative value is returned.

  Value     Result  
  SCE_OK     Processing succeeded  
  < 0     An error occurred  


Description

This function gets the current UTC time. Since the time that is obtained is the Tick representation of the SceRtcTick structure, it can be converted to the ScePspDateTime structure format by using sceRtcSetTick(). To reflect a time-difference offset, use sceRtcTickAddMinutes() to add the offset in terms of minutes to the Tick representation interval.

Note

The sceRtcGetCurrentTick() function should only be used to display the date and time. It should not be used for thread scheduling.

During a suspend/resume, using the sceRtcGetCurrentTick() function will result in a discontinuity in the time that is obtained. Since the real-time clock is resynchronized during suspend/resume, time may appear to move backward. To obtain a continuous system clock for purposes such as thread scheduling, the sceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), or sceKernelGetSystemTimeWide() functions should be used instead.

See Also

sceRtcSetTick(), sceRtcTickAddMinutes(), sceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), sceKernelGetSystemTimeWide()


sceRtcGetCurrentClock
 [Function]
Get current time in specified time zone

Syntax

#include <rtcsvc.h>
int sceRtcGetCurrentClock (
	ScePspDateTime *pClock
	int iTimeZone
);
#define sceRtcGetCurrentClockUTC(_p)	sceRtcGetCurrentClock(_p,0)
Calling Conditions

Can be called from an interrupt handler (does not depend on interrupt-disabled or -enabled state)
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pClock     Pointer to ScePspDateTime for receiving the current time  
  iTimeZone     Time zone offset (in minutes)  

Return Value

If an error occurs, a negative value is returned.
  Value     Result  
  SCE_OK     Processing succeeded  
  < 0     An error occurred  


Description

This function gets the current time in ScePspDateTime representation based on the specified time zone offset. Since this function is implemented internally by sceRtcGetCurrentTick() and sceRtcSetTick(), it is equivalent to the following code.


// Get current UTC in ScePspDateTime format
ScePspDateTime *sceRtcGetCurrentClock(ScePspDateTime *pTime, int iZoneTime)
{
	SceRtcTick tick;
	sceRtcGetCurrentTick(&tick);
	sceRtcTickAddMinutes(&tick, &tick, (SceLong64)iTimeZone);
	sceRtcSetTick(pTime, &tick);
	return (pTime);
}

If 0 is specified as the iTimeZone value, the UTC time is obtained.

Note

The sceRtcGetCurrentClock() function should only be used to display the date and time. It should not be used for thread scheduling.

During a suspend/resume, using the sceRtcGetCurrentClock() function will result in a discontinuity in the time that is obtained. Since the real-time clock is resynchronized during suspend/resume, time may appear to move backward. To obtain a continuous system clock for purposes such as thread scheduling, the sceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), or sceKernelGetSystemTimeWide() functions should be used instead.

See Also

sceRtcGetCurrentTick(), sceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), sceKernelGetSystemTimeWide()

Because the time that is obtained is the Tick representation of the SceRtcTick structure, it can be converted to the ScePspDateTime structure format using sceRtcSetTick(). To reflect a time-difference offset, use sceRtcTickAddMinutes() to add the offset in minutes to the Tick representation interval.

Only use the sceRtcGetCurrentNetworkTick() function to display the date and time. Do not use it for thread scheduling.

During a suspend/resume, using the sceRtcGetCurrentNetworkTick() function will result in discontinuity in the time that is obtained. Because the real-time clock is resynchronized during suspend/resume, time may appear to reverse. To obtain a continuous system clock for purposes such as thread scheduling, use the sceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), or sceKernelGetSystemTimeWide() function instead.

sceRtcSetAlarmTick
 [Function]
Set RTC alarm time (UTC) in Tick representation

Syntax

#include <rtcsvc.h>
int sceRtcSetAlarmTick (
	const SceRtcTick *pAlarmTick
);
#define sceRtcSetAlarmTickUTC(_tick)	sceRtcSetAlarmTick(_tick)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Not multithread safe

Arguments

  pAlarmTick     Pointer to SceRtcTick representing the alarm time (UTC) to be set  
  If NULL is specified, the alarm is canceled  

Return Value

If an error occurs, a negative value is returned.
  Value     Result  
  SCE_OK     Processing was successful  


Description

This function sets the RTC alarm time in UTC. When an RTC alarm is set, you can cause the PSP™ to transition to sleep state by using the scePowerRequestSuspend() function. The system will automatically resume when the RTC alarm time is reached.
The RTC alarm time can be set up to 24 hours in advance. A time in the past cannot be set.

If the application transitions because of sceKernelLoadExec() or if it is terminated by sceKernelExitGame(), the RTC alarm setting will be automatically canceled.
The alarm setting is also canceled if the user invokes standby mode by using the POWER/HOLD switch.

Note

The RTC alarm feature enables the PSP™ to resume at a specified date and time according to an alarm. Do not use the sceRtcSetAlarmTick() function for any other purpose such as thread scheduling.

See Also

sceRtcGetAlarmTick()

sceRtcGetLastReincarnatedTime
 [Function]
Get time, in Ticks, recovered after the last battery shut-off

Syntax

#include <rtcsvc.h>
int sceRtcGetLastReincarnatedTime (
	SceRtcTick *pTick
);
Calling Conditions

Can be called from an interrupt handler(does not depend on interrupt-disabled or -enabled state)
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pTick     Pointer to the SceRtcTick to receive the time (UTC) recovered after the last battery shut-off  

Return value

If an error occurs, a negative number is returned.
  Value     Result  
  SCE_OK     Success  


Description

Returns the time, in Ticks, from the last time that a time reset occurred due to battery shut-off.
The time obtained by the sceRtcGetLastReincarnatedTime() function is based on the time which was set by the user via the PSP™ system software. In other words, the sum of the time obtained by the sceRtcGetLastReincarnatedTime() function and the accumulative time obtained by the sceRtcGetAccumulativeTime() function matches sceRtcGetCurrentTick().

Note that regardless of whether the battery shut-off event represented a historical time in the past, the result of the the sceRtcGetLastReincarnatedTime() function will change in relative terms if the user has manipulated the clock. By storing the value of the sceRtcGetLastReincarnatedTime() function and determining whether or not it has changed, it is possible to detect whether or not the battery shut off, the time was manipulated by the user, the battery was moved to another PSP™, etc.
When drafting specifications which utilize this functionality, be careful that it does not become detrimental to the user.

See Also

sceRtcGetAccumulativeTime()

Note that the even if the user has set the clock, there will be no change in the result of the sceRtcGetLastAdjustedTime() function if the clock was merely set again to the same time. In these cases as well, the value of the accumulative time obtained by the sceRtcGetAccumulativeTime() function will still increase uniformly, so it is possible to detect that the clock has been turned back by the user.

When drafting specifications which utilize this functionality, be careful that it does not become detrimental to the user.
