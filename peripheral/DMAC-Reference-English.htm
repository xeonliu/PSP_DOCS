PSP™ Programmer Tool Runtime Library Release 6.3.0

DMAC Driver Reference


sceDmacMemcpy
 [Function]
Transfer memory data using DMAC

Syntax

#include <dmacdrv.h>
int sceDmacMemcpy (
	void *dst,
	const void *src,
	unsigned int uSize
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  dst     Destination address  
  src     Source address  
  uSize     Transfer count (number of transfer bytes)  

Return Value

When the function completes successfully, zero is returned.
If an error occurs, a negative number is returned.

Description

This function performs a memory transfer using the DMAC.

Although the writeback method is used for the ALLEGREX CPU D-cache, coherency is not guaranteed by using a mechanism such as hardware cache snooping. Note that when a memory transfer is performed using the sceDmacMemcpy() function, D-cache writeback of the source area and invalidation of the destination area must be performed manually.
sceDmacMemcpy() performs memory transfers using the maximum width of 128 bits. Efficiency is greatest when the pointers indicated by the dst and src arguments are aligned on 16-byte boundaries.

Based on the relationship between the pointers indicated by the dst and src arguments and the number of transfer bytes indicated by the uSize argument, non-cache memory accesses may be performed by the CPU in the neighborhoods of the transfer starting position and endpoint. Make sure that the transfer destination area indicated by the dst and uSize arguments does not overlap the transfer source area indicated by the src and uSize arguments.

Since the DMAC is a resource that is shared within the system, processing is blocked if another thread is using the DMAC by calling a function such as sceDmacMemcpy().
