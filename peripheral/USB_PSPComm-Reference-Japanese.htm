SceUsbPspcmBind
 [構造体]
通信開始要求の情報

定義

#include <usbpspcm.h>
typedef struct SceUsbPspcmBind {
	SceSize size;
	int addr;
	SceUInt *timeout;
} SceUsbPspcmBind;
メンバ

  size     構造体のサイズ  
  addr     通信したい相手のアドレス  
  timeout     タイムアウト時間  

解説

sceIoDevctl()のSCE_USBPSPCM_BINDコマンドの引数として使用する構造体です。

引数addrには通信したい相手のアドレスを指定し、引数sizeには必ずsizeof(SceUsbPspcmBind)を代入しておいてください。また引数timeoutのタイムアウト時間は、通信相手がハングアップした場合など異常時の対策として設けています。状況にもよりますが、通常はタイムアウト時間として2、3秒といった秒単位の時間を指定してください。なお、timeoutにNULLを指定すると無制限に待たせることができます。

sceIoRead
 [関数]
データの受信

形式

#include <iofilemgr.h>
#include <usbpspcm.h>
SceSSize sceIoRead(
	SceUID fd,
	void *buf,
	SceSize nbyte
);
呼出条件

スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  fd     受信対象のファイルディスクリプタ  
  buf     受信したデータを格納するバッファ  
  nbyte     受信するサイズ(Byte)  

返り値

正常終了の場合は受信したサイズを返します。
エラーの場合は以下のエラーコードかkerror.hやpsperror.hに定義されたエラーコードを返します。

SCE_USBPSPCM_ERROR_NOTAVAIL, SCE_USBPSPCM_ERROR_RESETTING
SCE_USBPSPCM_ERROR_RESET_END, SCE_USBPSPCM_ERROR_CLOSED
SCE_USBPSPCM_ERROR_INVAL, SCE_USBPSPCM_ERROR_INPROGRESS

解説

sceIoOpen()関数で開いたデバイスファイルを使ってデータを受信する関数です。

ライブラリ内でキャッシュのインバリデートを行いますので、引数bufは64バイトアラインさせてください。
通信相手からリセットされた場合、リセット中はSCE_USBPSPCM_ERROR_RESETTINGを返し、リセット終了後はSCE_USBPSPCM_ERROR_RESET_ENDをRead / Write / Ioctlのいずれかに対して一度だけ返します。
通信相手が通信を終了した場合には、デバイスファイルをクローズするまでSCE_USBPSPCM_ERROR_CLOSEDを返します。

通常モード(NORMAL)では、1回のWriteデータを一塊のデータとして扱っているため、一塊のデータの最後のデータのReadが終了した時点で必ずReadが終了します。一塊のデータの残りサイズと引数nbyteが等しい場合には、次のReadが0バイトで終了します。そのため、アプリケーションでは、返り値の受信したサイズが引数nbyteより小さかった時点で、一塊のデータを受信し終わったと判断することができます。
ReadモードがRD_SEQモードの場合には、一塊のデータの残りサイズと引数nbyteが等しい場合でも、次のReadは0バイトで終了せず、続けて次の一塊のデータを読むことができます。

使用例

#define BUF_SIZE    0x1000

SceUChar buf[BUF_SIZE] __attribute__ ((aligned (64)));  // 64バイトアライン
SceSSize len;

// データの受信
if ((len = sceIoRead(fd, buf, BUF_SIZE)) < 0) {
	// エラー処理
}

挿抜処理スレッドを登録するコマンドです。

登録できるスレッドIDは一つだけです。既にスレッドが登録されている場合にはSCE_USBPSPCM_ERROR_ALREADYを返します。
PSP®の挿抜が行われると登録したスレッドが起動され、そのスレッドの引数としてSceUsbPspcmConnect構造体が渡されます。構造体のaddrメンバは挿抜が行われたPSP®のアドレスを表し、statメンバは挿抜状態を表します。statメンバのSCE_USBPSPCM_OWNビットがセットされている場合には、それが自分自身の挿抜であることを表します。
スレッドIDを登録した時点で既にUSBケーブルが挿されている場合には、まず自分自身が挿されていることの通知が行われた後、他のPSP®が挿されていればその通知が順次行われます。
自分自身が抜かれた場合には他のPSP®との接続は必ず切断されるため、自分自身の切断通知のみが行われ、他のPSP®の切断通知は行われませんので注意してください。

挿抜処理スレッドが終了するまでは、次の挿抜通知は行われません。そのため、このスレッド内では待ち状態になるAPIや非常に時間のかかる処理は行わないようにしてください。

使用例

// 挿抜処理スレッド
int connection_detect_thread(SceSize argsize, void *arg)
{
	SceUsbPspcmConnect *conn;

	conn = (SceUsbPspcmConnect *)arg;

	if (conn->stat & SCE_USBPSPCM_OWN) {
		// 自身の挿抜の場合
		if (conn->stat & SCE_USBPSPCM_CONNECTED) {
			int addr = conn->addr;  // 自身のアドレス
			// 自身が挿された場合の処理
		} else if (conn->stat & SCE_USBPSPCM_DISCONNECTED) {
			// 自身が抜かれた場合の処理
		} else if (conn->stat & SCE_USBPSPCM_INVAL_TITLE) {
			// タイトルチェックに失敗した場合の処理
		} else if (conn->stat & SCE_USBPSPCM_BUSY) {
			// USBホストが忙しい場合の処理
		} else if (conn->stat & SCE_USBPSPCM_WAIT) {
			// タイトルチェック待ちの場合
		} else if (conn->stat & SCE_USBPSPCM_VERSION_ERROR) {
			// ライブラリのバージョンが合っていない場合
		} else {
			// "PSP"の接続台数が多すぎた場合
		}
	} else {
		// 他の"PSP"の挿抜の場合
		int addr = conn->addr;  // 挿抜"PSP"のアドレス
		if (conn->stat & SCE_USB_CONNECTED) {
			// 他の"PSP"が挿された場合の処理
		} else {
			// 他の"PSP"が抜かれた場合の処理
		}
	}

	
}

	SceUID thid;
	int ret;

	thid = sceKernelCreateThread("ConnectionDetect", connection_detect_thread,

				     0x1000,

		// エラー処理
	}

	ret = sceIoDevctl(SCE_USBPSPCM_DEVNAME,

	}

 [Ioctlコマンド]
Read/Writeモードの設定

呼出条件

スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）

引数

  argp     Read/Writeモードを表すSceUInt型変数へのポインタ  
  arglen     sizeof(SceUInt)  
  bufp     NULL  
  buflen     0  

返り値

正常終了の場合は0を返します。
エラーの場合は以下のエラーコードかkerror.hやpsperror.hに定義されたエラーコードを返します。

SCE_USBPSPCM_ERROR_INPROGRESS, SCE_USBPSPCM_ERROR_CLOSED
SCE_USBPSPCM_ERROR_INVAL

解説

sceIoRead(), sceIoWrite()関数でのRead/Writeモードを設定するコマンドです。

Read/Writeモードは以下の4通りの指定ができます。
  • SCE_USBPSPCM_RDWR_NORMAL
  • SCE_USBPSPCM_RDWR_RD_SEQ
  • SCE_USBPSPCM_RDWR_WR_SEQ
  • SCE_USBPSPCM_RDWR_RD_SEQ | SCE_USBPSPCM_RDWR_WR_SEQ

デフォルトでは通常モード(SCE_USBPSPCM_RDWR_NORMAL)に設定されています。このモードでは1回のWriteデータを一塊のデータとして扱い、Readは一塊のデータを読み終わった時点で終了します。もし、一塊のデータの残りのサイズと、sceIoRead()関数の引数nbyteの値が等しかった場合には、次のReadが0バイトで終了します。受信側アプリケーションでは、sceIoRead()関数の引数nbyteに指定した値よりも、返り値の受信データサイズの方が小さかった時点で、一塊のデータを受信し終わったと判断することができます。そのため、送られるデータサイズが不明な場合にも、確実に一塊のデータの終了を知ることができます。

RD_SEQモード(SCE_USBPSPCM_RDWR_RD_SEQ)は、一塊のデータの残りのサイズと、sceIoRead()関数の引数nbyteの値が等しかった場合にも、次のReadが0バイトで終了せず、続けて次の一塊のデータを読むことができます。
あらかじめ受信するデータの最大サイズが分かっている場合には、一塊のデータの残りのサイズと、sceIoRead()関数の引数nbyteの値が等しかった場合でも、そこで一塊のデータが終了であることが分かります。このような場合にRD_REQモードに設定することで、冗長な0バイトのReadを省略することができます。

WR_SEQモード(SCE_USBPSPCM_RDWR_WR_SEQ)は、一塊のデータを複数回のWriteに分割して送信することができます。大きなサイズのデータを一塊のデータとして扱いたい場合に使用してください。
このモードでは、１回のWriteのサイズが4KByteの整数倍である場合に限り、ライブラリは一塊のデータがまだ終わりではなく、続きのデータがあるものと判断します。そのため、送信側アプリケーションでは、1回のWriteのサイズを4KByteの整数倍になるようにしてください。
また、一塊のデータの最後のWriteだけは、通常モードに戻してからWriteするようにしてください。もし、WR_SEQモードのままで、最後のWriteサイズが4KByteの整数倍であった場合、ライブラリは続きのデータがあるものと判断してしまい、受信側で一塊のデータの最後を正しく判断することができなくなります。

通信相手が通信を終了した場合には、デバイスファイルをクローズするまでSCE_USBPSPCM_ERROR_CLOSEDを返します。

使用例

SceUInt mode = SCE_USBPSPCM_RDWR_RD_SEQ;

// Readモード変更
if (sceIoIoctl(fd, SCE_USBPSPCM_RDWR_MODE, &mode, sizeof(SceUInt), NULL, 0) < 0) {
	// エラー処理
}
