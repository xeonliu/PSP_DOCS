SCE_USBGPS_STATE_WORKING
 [Constant Definitions]
GPS receiver state

Definition

#include <usbgps.h>
#define SCE_USBGPS_STATE_WORKING  	 3
Description

This constant represents the GPS receiver state, which can be acquired by using sceUsbGpsGetState(). It represents the state in which the GPS receiver has been started up and position information and GPS satellite information can be acquired by using sceUsbGpsGetData(). However, whether or not position information that is acquired by using sceUsbGpsGetData() is valid can be verified by using the mode and quality members of the SceUsbGpsPositionData structure. The other members of the SceUsbGpsPositionData structure are invalid when mode is SCE_USBGPS_POSITION_MODE_NOT_AVAILABLE or when quality is SCE_USBGPS_POSITION_QUALITY_NOT_AVAILABLE.

SceUsbGpsPositionAngle
 [Structure]
Structure representing latitude and longitude in degrees, minutes, seconds, and milliseconds

Definition

#include <usbgpsh>
typedef struct SceUsbGpsPositionAngle {
 	SceShort16 degree;
 	SceUShort16 minute;
 	SceUShort16 second;
 	SceUShort16 milisecond;
} SceUsbGpsPositionAngle;
Members

  degree     Latitute and longitude degrees component. If greater than zero, it represents north latitude or east longitude, and if it is less than zero, it represents south latitude or west longitude.  
  minute     Latitude and longitude minutes component  
  second     Latitude and longitude seconds component  
  milisecond     Latitude and longitude milliseconds component  

Description

This structure represents the latitude and longitude in terms of degrees, minutes, seconds, and milliseconds. North or south latitude and east or west longitude can be distinguished according to the sign of the degree member. Also, sceUsbGpsFloatToPositionAngle() and sceUsbGpsPositionAngleToFloat() can mutually convert latitude and longitude between degrees in SceFloat32 format and degree / minute / second / millisecond format.

See Also

sceUsbGpsFloatToPositionAngle(), sceUsbGpsPositionAngleToFloat()

SceUsbGpsPositionData
 [Structure]
Position information

Definition

#include <usbgps.h>
typedef struct SceUsbGpsPositionData {
	ScePspDateTime datetime;
	SceFloat32 hdop;
	SceUChar8 quality;
	SceUChar8 mode;
	SceUChar8 reserve0[2];
	SceFloat32 latitude;
	SceFloat32 longitude;
	SceFloat32 altitude;
	SceFloat32 reserve1;
	SceFloat32 speed;
	SceFloat32 course;
} SceUsbGpsPositionData;
Members

  datetime     Returns the date and time that satellite information was obtained from the GPS receiver in UTC format.  
  hdop     Horizontal dilution of precision rate  
  quality     Positioning quality  
  mode     Positioning mode  
  reserve0     Reserved (Do not evaluate the contents)  
  latitude     Latitude (degrees)  
  longitude     Longitude (degrees)  
  altitude     Average height from sea level (meters)  
  reserve1     Reserved (Do not evaluate the contents)  
  speed     Ground surface speed (kph)  
  course     Direction of travel (degrees)  
Description

This structure represents position information.
It is used as an argument of sceUsbGpsGetData().

Notes

  • In GPS positioning, positioning information is affected by the arrangement of GPS satellites in the sky for both single-point positioning and relative positioning. One of the numerical values indicating the degree of this effect is hdop (horizontal dilution of precision rate). The precision decreases as this value gets larger.
  • altitude (average height from sea level) is valid only when mode (positioning mode) is SCE_USBGPS_POSITION_MODE_3D.
  • course (direction of travel) is valid only when moving.
See Also

sceUsbGpsGetData (), sceUsbGpsFloatToPositionAngle()

sceUsbStart
 [Function]
Start USB driver

Syntax

#include <usb.h>
int sceUsbStart(
	const char *name,
	unsigned int args,
	const void *argp
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  name     Name of USB driver to be started  
  args     0  
  argp     NULL  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, one of the following error codes defined in usb.h or an error code defined in kerror.h or psperror.h is returned.

SCE_USB_ERROR_ALREADY, SCE_USB_ERROR_NOTAVAIL
SCE_USB_ERROR_NOENT, SCE_USB_ERROR_INPROGRESS
SCE_USB_ERROR_INVAL

Description

This is an API provided by the USB bus driver. It starts the USB driver having the name specified by name.
To use this API, link usb_stub.a.

To use the USBGPS receiver, start the USB drivers in the following order.
  1. USB bus driver (SCE_USB_BUS_DRIVER_NAME)
  2. USB accessory base driver (SCE_USBACC_NAME)
  3. USB GPS driver (SCE_USBGPS_NAME)

Example

// Start USB bus driver
if (sceUsbStart(SCE_USB_BUS_DRIVER_NAME, 0, NULL) < 0) {
	// Error handling
}

// Start USB accessory base driver
if (sceUsbStart(SCE_USBACC_NAME, 0, NULL) < 0) {
	// Error handling
}

// Start USB GPS driver
if (sceUsbStart(SCE_USBGPS_NAME, 0, NULL) < 0) {
	// Error handling
}
See Also

sceUsbStop()

sceUsbStop
 [Function]
Stop USB driver

Syntax

#include <usb.h>
int sceUsbStop(
	const char *name,
	unsigned int args,
	const void *argp
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  name     Name of USB driver to be stopped  
  args     0  
  argp     NULL  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, one of the following error codes defined in usb.h or an error code defined in kerror.h or psperror.h is returned.

SCE_USB_ERROR_ALREADY, SCE_USB_ERROR_NOTAVAIL
SCE_USB_ERROR_NOENT, SCE_USB_ERROR_INPROGRESS

Description

This is an API provided by the USB bus driver. It stops the USB driver having the name specified by name.
To use this API, link usb_stub.a.
To stop a USB driver, you must first deactivate the USB device by using the sceUsbDeactivate() function.

To use the GPS receiver, stop USB drivers in the reverse order that was used with sceUsbStart().

Example

// Stop USB GPS driver
if (sceUsbStop(SCE_USBGPS_NAME, 0, NULL) < 0) {
	// Error handling
}

// Stop USB accessory base driver
if (sceUsbStop(SCE_USBACC_NAME, 0, NULL) < 0) {
	// Error handling
}
// Stop USB bus driver
if (sceUsbStop(SCE_USB_BUS_DRIVER_NAME, 0, NULL) < 0) {
	// Error handling
}
See Also

sceUsbActivate
 [Function]
Activate USB device

Syntax

#include <usb.h>
int sceUsbActivate(
	unsigned int idProduct
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  idProduct     Product ID  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, one of the following error codes defined in usb.h or an error code defined in kerror.h or psperror.h is returned.

SCE_USB_ERROR_ALREADY, SCE_USB_ERROR_NOENT, SCE_USB_ERROR_NOSPC
SCE_USB_ERROR_NOMEM, SCE_USB_ERROR_INPROGRESS
SCE_USB_ERROR_INVAL, SCE_USB_ERROR_NOTAVAIL

Description

This is an API provided by the USB bus driver. It collects together plugged-in USB drivers and activates them as one USB device. Until these drivers are activated as a USB device, they are not recognized by the host even if a USB cable is connected.
To use this API, link usb_stub.a.
To activate drivers as a USB device, you must start the required USB drivers in advance.

To use the GPS receiver, specify SCE_USB_PID_GPS, which is defined in usbgps.h, for the product ID argument.

Example

if (sceUsbActivate(SCE_USB_PID_GPS) < 0) {
	// Error handling
} else {
	printf("success to activate\n");
}

sceUsbGpsGetData
 [Function]
Get position information and GPS satellite information

Syntax

#include <usbgps.h>
int sceUsbGpsGetData (
	SceUsbGpsPositionData *pos,
	SceUsbGpsSatelliteData *sat
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pos     Pointer to structure for receiving position information  
  sat     Pointer to structure for receiving GPS satellite information  

Return Value

When the function completes normally, 0 is returned.

If an error occurs, one of the following error codes or an error code defined in kerror.h or psperror.h is returned.
SCE_USBACC_ERROR_NOT_ACCESSORY
SCE_USBGPS_ERROR_NOT_AVAILABLE
SCE_USBGPS_ERROR_NOT_STARTED
SCE_USBGPS_ERROR_NOT_OPENED
SCE_USBGPS_ERROR_PRIV_REQUIRED
SCE_USBGPS_ERROR_BUSY

Description

This function gets the position information and GPS satellite information that was acquired from GPS receiver. The position information and GPS satellite information are updated at intervals of at least 1 second. If the USB jack is not connected to anything or if a non-GPS accessory is connected, SCE_USBGPS_ERROR_NOT_AVAILABLE is returned. If something other than a PSP™ USB accessory is connected, SCE_USBACC_ERROR_NOT_ACCESSORY is returned. If a USB GPS accessory is connected and initialization processing is being performed, SCE_USBGPS_ERROR_BUSY is returned. If the GPS receiver initialization processing has completed and the GPS receiver is operating, 0 is returned as the return value.
