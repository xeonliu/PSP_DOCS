SceFmacVideoParam
 [Structure]
Initialization parameters

Structure

#include <fmacvideo.h>
typedef struct SceFmacVideoParam {
	SceSize size;
	char *pTitle;
	char *pSubtitle;
	char *pFilename;
	char *pSuffix;
	SceUShort16 WriteRule;
	SceUShort16 Numbering;
	SceUShort16 MaxNumber;
	SceUShort16 Wrtworksize;
	char *pWrtworkbuff;
} SceFmacVideoParam;
Members

 size   Size of this structure (value of sizeof(SceFmacVideoParam)) 
 pTitle   Application title is entered. 
 This string is displayed immediately below the video by system software. 
 A maximum of 32 bytes can be specified. The character code is UTF-8. 
 pSubtitle   The video classification is entered.  
 This string is displayed immediately below "Video" - "Application title" by 
 system software. 
 A maximum of 32 bytes can be specified. It can be omitted by specifying  
 SCE_NULL. The character code is UTF-8. 
 pFilename   The filename of the video is entered. A maximum of 64 bytes can be  
 specified. 
 The character code is UTF-8. 
 pSuffix   The extension of the video file is entered. A maximum of 16 bytes can be  
 specified. 
 Since the library does not place a period between the filename and suffix,  
 do not forget to specify it. A period does not necessarily need to be placed  
 at the beginning. When the numbering feature is used, characters may  
 follow the numeric suffix. 
 When the numbering feature is not used, the extension may also be  
 contained in the filename, and this item may be omitted. It can be omitted  
 by specifying SCE_NULL. The character code is UTF-8. 
 WriteRule   Specifies the file writing rule. The following rules can be specified. 
 These rules cannot be specified at the same time. 
     SCE_FMAC_WRITERULE_STANDARD   Perform normal processing.  
 Specified when the following  
 features are not used. 
     SCE_FMAC_WRITERULE_NUMBERING   Enable the numbering  
 feature. A 3-digit number is  
 automatically appended at  
 the end of the filename when  
 adding a new file. 
     SCE_FMAC_WRITERULE_OVERWRITE   If the same filename exists, the file is forcibly overwritten. 
 Numbering   Specifies the starting number for the numeric suffix to be appended to the  
 filename when the numbering feature is enabled. Any number from 0 to  
 999 can be specified. 
 MaxNumber   Specifies the maximum number for the numeric suffix to be appended to  
 the filename when the numbering feature is enabled. Any number from 0  
 to 999 can be specified. No numbering will be performed beyond  
 MaxNumber. 
 Wrtworksize   Although the offset that is specified in sceFmacVideoAdd() must be  
 aligned at 64 bytes, if alignment adjustment is difficult to do in the  
 application, the alignment can be performed by the library, by specifying a 
 work area for pWrtworkbuff. The required work area size is  
 SCE_FMAC_PARAM_WRTALIGN_WORK. An error will occur if any  
 other value is specified. This can be omitted when the application can  
 specify offset with 64-byte alignment. If pWrtworkbuff is omitted, this  
 member is disabled. 
 pWrtworkbuff   Specifies the write alignment adjustment work area that is to be used by  
 the library. 
 Allocate an area with a size of Wrtworksize. This can be omitted. The  
 specified buffer address must have 64-byte alignment. 

Description

This structure is used when sceFmacVideoStart() is called to initialize the library. When the application uses pWrtworkbuff, save the area pointed to by pWrtworkbuff until sceFmacVideoTerm() or the next sceFmacVideoStart() is called. The other members need not be saved after the function returns.

sceFmacVideoInit
 [Function]
Initialization

Syntax

#include <fmacvideo.h>
int sceFmacVideoInit(
	void
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Not multithread safe

Arguments

None

Return Value

  Value     Description  
  SCE_FMAC_OK     Processing completed successfully.  
  SCE_FMAC_ERROR_WINIT     Initialization is being performed multiple times concurrently.  
  SCE_FMAC_ERROR_PRITOOHIGH     Priority of thread using this library is too high.  
  SCE_FMAC_ERROR_THREAD     Thread operation failed. Condition for which API was called is illegal.  


Description

This function initializes the library. The library APIs can only be used from the completion of library initialization (and startup) until the library is terminated.
A power service callback, FAT forMS callback, and thread for monitoring these callbacks are created and started by this API. After initialization successfully completes, these callbacks and thread will stay resident until the library is terminated (if an initialization error occurs, the callbacks and thread will not be created and started). The thread priority is set one level higher (one value less) than the thread that called this API. Note that multiple initializations cannot be performed.
After this API successfully completes, be sure to call sceFmacVideoTerm() to terminate the library when you have finished using it.
If the device is put to sleep or the Memory Stick™ is inserted/removed between initialization and termination, subsequent processing will result in an error. The library can be restarted using sceFmacVideoStart().

 [Function]
Start

#include <fmacvideo.h>
int sceFmacVideoStart(
	const SceFmacVideoParam *param

Calling Conditions

Can be called from a thread (must be called in an interrupt-enabled state)
Not multithread safe

Arguments

  param     Pointer to initialization parameters  

Return Value

  Value     Description  
  SCE_FMAC_OK     Processing completed successfully.  
  SCE_FMAC_ERROR_INIT     Library has not been initialized or started.  
  SCE_FMAC_ERROR_WRITERULE     File write rules are inconsistent.  
  SCE_FMAC_ERROR_OUTOFNUMB     Savable file count has been reached.  
  SCE_FMAC_ERROR_MSRELEASE     Memory Stick™ was inserted or removed.  
  SCE_FMAC_ERROR_SLEEP     Sleep is being canceled.  
  SCE_FMAC_ERROR_INVALIDNAME     Unusable character was specified in filename.  
  SCE_FMAC_ERROR_INVALIDADDR     Specified address is invalid.  
  SCE_FMAC_ERROR_INVALIDSIZE     Specified size is invalid.  
  SCE_FMAC_ERROR_PARAM     Parameter is invalid.  
  SCE_FMAC_ERROR_WPSWITCHON     Media is write protected.  
  SCE_FMAC_ERROR_NOMEDIUM     No media.  
  Other     See "FAT Filesystem For Memory Stick™ Reference."  


Description

This function initializes the library with the initialization parameter values and starts use of the library. This API can only be used from the time the library initialization is completed until the library is terminated.
If there is no write privilege because the flash media is write protected, for example, an error will occur.
If the device is put to sleep or the Memory Stick™ is inserted/removed between initialization and termination, subsequent processing will result in an error. The library can be restarted using this API. However, if this API is used immediately after a sleep/sleep cancellation error occurs, another error may occur. Although the application can automatically attempt a retry, a busy loop or other unrecoverable error may result.

This function terminates this library. After sceFmacVideoInit() successfully completes, be sure to call this function when you are finished using the library. sceFmacVideoWrite
 [Function]
Save new file

Syntax

#include <fmacvideo.h>
int sceFmacVideoWrite(
	const char *buff,
	SceSize size
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Not multithread safe

Arguments

  buff     Starting address of buffer to be written to file  
  size     Buffer size to be written to file  

Return Value

  Value     Description  
  SCE_FMAC_OK     Processing completed successfully.  
  Value greater than or equal to 1     Required capacity (kilobytes) for saving  
  SCE_FMAC_ERROR_INIT     Library has not been initialized or started.  
  SCE_FMAC_ERROR_MSRELEASE     Memory Stick™ was inserted or removed.  
  SCE_FMAC_ERROR_SLEEP     Sleep was started or canceled.  
  SCE_FMAC_ERROR_OUTOFNUMB     Savable file count has been reached.  
  SCE_FMAC_ERROR_INVALIDADDR     Specified address is invalid.  
  SCE_FMAC_ERROR_FILEEXIST     File already exists.  
  SCE_FMAC_ERROR_DFILEEXIST     File with same name as directory name exists.  
  SCE_FMAC_ERROR_FDIREXIST     Directory with same name as filename exists.  
  SCE_FMAC_ERROR_NOMEDIUM     No media.  
  Other     See "FAT Filesystem For Memory Stick™ Reference."  


Description

This function can be used to predetermine whether the specified size file can be saved according to items specified in sceFmacVideoStart(). If there is enough space on the device for saving the file, this function saves the specified buffer to a new specified size file. A directory is created if one does not exist. The starting address of the specified buffer must have 64-byte alignment.
If there is not enough space for saving the file, this function returns the required space in kilobytes as the return value. It calculates the required space by assuming that the directory will be created if one does not exist. Since this calculation also includes a calculation of the system reserved area, the application can use the value directly (when the file to be written consists of one batch of data).
If a file with the same name exists, processing will conform to the configured writing rules.
If the device is put to sleep or the Memory Stick™ is inserted/removed between initialization and termination, subsequent processing will result in an error. The library can be restarted using sceFmacVideoStart().

  whence     Reference point  
  * For details, see "IO/File Manager Document."  
  buff     Starting address of buffer to be written to file  
  size     Buffer size to be written to file  

Return Value

  Value     Description  
  SCE_FMAC_OK     Processing completed successfully.  
  SCE_FMAC_ERROR_INIT     Library has not been initialized or started.  
  SCE_FMAC_ERROR_MSRELEASE     Memory Stick™ was inserted or removed.  
  SCE_FMAC_ERROR_SLEEP     Sleep was started or canceled.  
  SCE_FMAC_ERROR_BADORDER     Writing order is illegal.  
  SCE_FMAC_ERROR_INVALIDADDR     Specified address is invalid.  
  SCE_FMAC_ERROR_NOMEDIUM     No media.  
  Other     See "FAT Filesystem For   
  Memory Stick™ Reference."  


Description

This function saves the specified buffer by appending its contents to the file with the specified size according to items specified in sceFmacVideoStart(). This API must be called following sceFmacVideoWrite(), which was called after sceFmacVideoStart(). Buffer data cannot be appended without data being saved to a new file first.
The starting address of the specified buffer used for writing must have 64-byte alignment.
Also, make sure the starting position has 64-byte alignment. When pWrtworkbuff is specified, the alignment for writing to the flash media will be adjusted by the library, but this requires some overhead. Make every effort to have the adjustment performed by the application.
If there is not enough space for saving the data, an error will occur. When using this API to append to a file, use sceFmacVideoCheck() in advance to check if the total data size can be saved to the file. If the device is put to sleep or the Memory Stick™ is inserted/removed between initialization and termination, subsequent processing will result in an error. The library can be restarted using sceFmacVideoStart().
