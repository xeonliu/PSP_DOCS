SceRtcTick
 [構造体]
Tick単位の時間情報

構造

#include <psptypes.h>
#include <rtcsvc.h>
typedef struct SceRtcTick {
	SceULong64 tick;
} SceRtcTick;
メンバ

解説

時刻情報を統一的な形式で扱うための構造体です。加減算など、時間の値に演算を行う場合に使います。ScePspDateTime構造体とはsceRtcGetTick(), sceRtcSetTick()を使って相互に変換することができます。

  SCE_OK     成功

解説

指定されたタイムゾーンオフセットに基づいて現在時刻をScePspDateTime表現で取得します。内部的にはsceRtcGetCurrentTick()とsceRtcSetTick()によって実現されているので、以下のコードと等価になります。

//J 現在時刻をScePspDateTime形式で取得(UTC)
//E get current UTC in ScePspDateTime format
ScePspDateTime *sceRtcGetCurrentClock(ScePspDateTime *pTime, int iZoneTime)
{
	SceRtcTick tick;
	sceRtcGetCurrentTick(&tick);
	sceRtcTickAddMinutes(&tick, &tick, (SceLong64)iTimeZone);
	sceRtcSetTick(pTime, &tick);
	return (pTime);
}

iTimeZoneの値として0を指定すると、UTC時刻が得られます。

注意

sceRtcGetCurrentClock()関数は日付および時刻を示すために使用されるための関数です。sceRtcGetCurrentClock()関数を、スレッドのスケジュール等に使用しないで下さい。

サスペンド/レジュームが発生したとき、sceRtcGetCurrentClock()関数の結果は非連続になります。リアルタイムクロックは再同期が行われますので、時刻の巻き戻りが発生する場合もあります。スレッドスケジューリングなど連続的なシステムクロックを取得する目的にはsceKernelGetSystemTime(), sceKernelGetSystemTimeLow(), sceKernelGetSystemTimeWide()関数を使用するようにしてください。

当該月の日数を取得

形式

#include <rtcsvc.h>
int sceRtcGetDaysInMonth (
	int year,
	int month
);
呼出条件

割り込みハンドラから呼び出し可能(割り込み禁止、許可状態に依存しない)
スレッドから呼び出し可能(割り込み禁止、許可状態に依存しない)
マルチスレッドセーフである

引数

  year     判定を行う年  
  month     判定を行う月  

返り値

指定された日数を返します。エラーの場合、負の値が返ります。
  値     結果  
  31     31日まである月  
  30     30日まである月  
  29     29日まである月  
  28     28日まである月  
  <0     年,月の指定が不正  


解説

year, monthで指定された月の日数を返します。
