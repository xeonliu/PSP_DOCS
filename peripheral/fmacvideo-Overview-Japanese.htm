フラッシュメディアアクセスビデオ

1 ライブラリ概要

1. 1 機能の概要と特長

フラッシュメディアアクセスビデオライブラリはMemory Stick™などのフラッシュメディアのビデオディレクトリにビデオデータをビデオファイルとして安全に保存する機能を提供します。
AVI生成ライブラリなどで生成したビデオデータやアプリケーション内に内包しているプロモーションビデオなどのビデオデータをシステムソフトウェアで認識可能なディレクトリ（/VIDEO/第一階層/第二階層(※1)/ビデオファイル）にディレクトリ構造やフラッシュメディアの残容量などをアプリケーションが配慮することなく、書き込むことができます。
※1 : 第二階層は省略可能です。

1. 2 特長

  • パワーロック処理を適宜行います。
  • Memory Stick™の挿抜を監視しています。
  • スリープ・スリープ解除を監視しています。
  • フラッシュメディアのディレクトリ構成（有無）をアプリケーションが意識することなく、ファイルを保存することができます。
  • フラッシュメディアの残容量をアプリケーションが計算することなく、ファイルを保存することができます。
  • フラッシュメディアの書き込み保護をアプリケーションがチェックすることなく、ファイルを保存することができます。
  • ファイルに追い番を自動的に付与してアプリケーションが意識することなく、複数のファイルを扱うことができます。

2 使用手順の概略

以下の手順でフラッシュメディアにアクセスします。
なお、各関数の呼出しに対して適宜エラーチェックを行ってください。

(1) 書き込みアライン調整作業領域を確保する

基本的には不要です。
ファイル追記保存関数sceFmacVideoAdd()で指定するoffsetは、64バイトにアラインしている必要がありますが、アプリケーションの実装上、アラインの調整が難しい場合は書き込みアライン調整作業領域を確保してください。この領域を指定することで本ライブラリ内で書き込みアラインを補完します。必要な作業領域のサイズはSCE_FMAC_PARAM_WRTALIGN_WORKです。使用開始アドレス（先頭アドレス）は64バイトアラインされている必要があります。この領域はライブラリの末期化までアプリケーション内で保持してください。

(2) ライブラリの初期化を行う

初期化関数sceFmacVideoInit()でライブラリを初期化します。この時、パワーサービスのコールバックとFAT Filesystem For Memory Stick™ (以下FAT forMS)のコールバック、それらのコールバック監視用のスレッドを生成・開始します。初期化成功後は末期化まで常駐します（初期化エラー時は生成・開始しません）。スレッドの優先度は初期化関数を呼び出したスレッドより1高く（値は1小さく）設定します。

(3) 初期化パラメータを適宜設定する

初期化パラメータ（SceFmacVideoParam）をアプリケーションの仕様に合わせて設定します。
指定できる文字列の文字コードはUTF-8です。文字列長はバイト数です（文字終端子も含みます）。文字数ではありませんのでご注意ください。

(4) ライブラリを開始する

開始関数sceFmacVideoStart()に対して、手順(3)で設定した初期化パラメータを指定します。

(5) 書き込むデータを用意する

書き込むデータを用意します。AVI生成ライブラリなどで生成したビデオデータやアプリケーション内に内包しているビデオデータなどをバッファリングします。バッファの使用開始アドレス（先頭アドレス）は64バイトアラインされている必要があります。

(6) フラッシュメディアに用意したデータを書き込む

ファイル新規保存関数sceFmacVideoWrite()でファイルとして書き込みます。一回で書き込めない場合は以後sceFmacVideoAdd()で追記してください。追記する際のオフセット値も64バイトアラインするようにしてください。(1)で書き込みアライン調整作業領域を確保している場合はオフセット値は64バイトアラインされていなくても動作しますが、多少のオーバーヘッドが発生しますので極力64バイトアラインで書き込めるようにしてください。
sceFmacVideoAdd()で追記する場合は、事前に保存判定関数sceFmacVideoCheck()で保存できるか確認してください。判定に使用するサイズは大体の値で構いません。多めに指定してください（例：生成されるAVIファイルの最大サイズ）。少なく指定した場合は正しく判定できませんので問題になります。また、初期化してから書き込むまでの間隔やバッファリングなどして書き込みと追記書き込みの間隔が長くなるような場合は、sceFmacVideoCheckStatus()でスリープやMemory Stick™の挿抜が発生していないかを定期的に確認してください。sceFmacVideoCheckStatus()で確認しなくとも次のAPI呼び出しでスリープや
Memory Stick™の挿抜は検出しますが、次のAPI呼び出しまでに長い時間を要す場合はその分判定が遅延しますのでユーザに不利益となります。（例えばAVI生成しながらMemory Stick™に保存する処理において、30秒分バッファリングしてから保存というような仕様の場合、スリープして、スリープ解除後に、最大30秒間エラーにならずに処理続行してから30秒後にエラーとなってしまいます。）

(7) スリープ・スリープ解除やMemory Stick™の挿抜を検出した場合

Memory Stick™にアクセスする全てのAPIはエラーとなります。再度(4)のsceFmacVideoStart()でライブラリを使用可能に復帰します。ただし、スリープ・スリープ解除エラー時、直ちにsceFmacVideoStart()を使用するとスリープ解除中エラーになることがあります。アプリケーションで自動リトライを行うことは構いませんが、復帰不能なエラー時の無限リトライやビジーループなどにはご注意ください。

(8) 終了処理

末期化関数sceFmacVideoTerm()を呼び出すことで、終了処理を行います。ライブラリ初期化成功後はライブラリの使用を終えるときに必ず呼び出してください。初期化時に生成・開始したパワーサービスとFAT ForMSのコールバックと、それらのコールバック監視用のスレッドを終了して削除します。

3 ファイルの保存について

3. 1 ディレクトリ・ファイル名に使用する文字

ファイルやディレクトリに使用できる文字コードはUTF-8(BOMなし)です。
使用できない文字は以下の通りです。
  • 半角記号 \ " / : < > | * ?
  • サロゲートペア（UTF-16で4byte使用する文字）

3. 2 ファイルの強制上書きについて

初期化時の「ファイルの書き込みルール」で「ファイルを上書き保存」に指定した場合、既にファイルが存在していても無視して上書きします。
アプリケーション内で別に上書き確認を行っている場合や、プロモーションビデオなど、上書きしても同じデータを再度書き込めるような場合にご使用いただけます。
ただし、アプリケーション内もしくはアプリケーションの解説書にデータを上書き保存する旨を明記することを推奨します。

3. 3 ファイルの追い番付与について

初期化時の「ファイルの書き込みルール」で「ファイルに追い番を付与して保存」に指定した場合、対象ディレクトリ内を検索して「空いている追い番を見つけて」ファイルに追い番を付与します。
ユーザが削除などを行って以下のように番号が空いている場合はその番号を使用して追い番を付与します。

[例]
/VIDEO/サンプル/ビデオ出力/テストムービー001.avi
/VIDEO/サンプル/ビデオ出力/テストムービー002.avi
/VIDEO/サンプル/ビデオ出力/テストムービー004.avi　←　003が抜けている
　　　　　　　　：
　　　　　　　　↓
/VIDEO/サンプル/ビデオ出力/テストムービー001.avi
/VIDEO/サンプル/ビデオ出力/テストムービー002.avi
/VIDEO/サンプル/ビデオ出力/テストムービー003.avi　←　003として保存した
/VIDEO/サンプル/ビデオ出力/テストムービー004.avi
　　　　　　　　：

対象ディレクトリの検索時、半角（1バイト）アルファベットは大文字と小文字の区別を行いません。例えば以下の場合は同じファイルがあるとして追い番を付与します。

[例]
/VIDEO/Sample/Output-Video/Test-Movie001.avi
/VIDEO/Sample/Output-Video/TEST-MOVIE002.avi

/VIDEO/SAMPLE/OUTPUT-VIDEO/TEST-MOVIE007.AVI
　　　　　　　　↓
/VIDEO/Sample/Output-Video/Test-Movie001.avi
/VIDEO/Sample/Output-Video/TEST-MOVIE002.avi
/VIDEO/Sample/OUTPUT-VIDEO/Test-Movie003.avi
/VIDEO/SAMPLE/Output-Video/Test-Movie004.avi

4 注意事項

4. 1 初期化パラメータについて

書き込みアライン調整作業領域を確保している場合は、本ライブラリの初期化正常終了から末期化まで、書き込みアライン調整作業領域は保持してください。また、直接アクセスすることもおやめください。

4. 2 保存容量のチェックについて

一度で書き出す処理の場合は事前のチェックは必要ありません。容量が足りない場合は書き出し関数の戻り値が必要サイズです。
複数のデータを数回に分けて書き出す場合は書き込めない旨のエラーが返るだけですので、アプリケーションで容量不足として必要容量などを表示することができません。複数のデータを書き出すような場合は、事前に大体のサイズを決めておき、sceFmacVideoCheck()でチェックしてください。なお、必要容量として多めに表示などを行うことは問題ありません。逆に、少なく指定した場合は正しく判定できませんのでご注意ください（例：生成されるAVIファイルの最大サイズ）。

4. 3 パワーロックについて

ライブラリ内でパワーロック処理を適宜行っているため、場面によってはスリープ/電源OFF（スタンバイ）処理の起動を遅延させて、フラッシュメディアアクセス中のユーザーによるサスペンドや電源オフによるメディア情報やデバイスそのものの破損などの危険からフラッシュメディアを保護しています。このライブラリを使用する際にアプリケーションでパワーロックを行わないでください。

4. 4 本ライブラリを使用してフラッシュメディアに書き込む場合

本ライブラリはシステムユーティリティではありませんが、簡易にかつ安全にフラッシュメディアにファイルを書き出すことができます。
アプリケーションに実装いただく場合は、アプリケーションの仕様（企画）に問題がなければ製品にご使用いただいても構いません。仕様（企画）の提示時に各リージョンにご相談ください。

4. 5 スリープもしくはMemory Stick™の挿抜によるエラー後の処理について

本ライブラリはライブラリ内でスリープ・スリープ解除及び、Memory Stick™の挿抜を監視しています。書き込み処理前及び分割書き込みの処理間などでスリープに移行した場合や、
Memory Stick™の挿抜を行った場合は、関数がエラーになります。これらの事象後に同じ
Memory Stick™が接続されている保障がないためです。
AVIファイルなど適切に書き出しを終了しないと「破損ファイル」になる場合もあります。
アプリケーションでは開始前の警告と実際にエラーになった時に、その旨表示するようにしてください。ユーザーのアクションを受けて、不要ファイルを削除しても構いません。
