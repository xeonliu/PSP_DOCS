SceFmacVideoParam
 [構造体]
初期化パラメータ

構造

#include <fmacvideo.h>
typedef struct SceFmacVideoParam {
	SceSize size;
	char *pTitle;
	char *pSubtitle;
	char *pFilename;
	char *pSuffix;
	SceUShort16 WriteRule;
	SceUShort16 Numbering;
	SceUShort16 MaxNumber;
	SceUShort16 Wrtworksize;
	char *pWrtworkbuff;
} SceFmacVideoParam;
メンバ

 size   この構造体のサイズ(sizeof(SceFmacVideoParam)の値) 
 pTitle   アプリケーションのタイトルを入れます。 
 この文字列がシステムソフトウェア上のビデオ直下に表示されます。 
 最大32バイト指定できます。文字コードはUTF-8です。 
 pSubtitle   ビデオの分類などを入れます。この文字列がシステムソフトウェア上の 
 「ビデオ」−「アプリケーションのタイトル」直下に表示されます。 
 最大32バイト指定できます。SCE_NULLを指定することで、 
 省略可能です。文字コードはUTF-8です。 
 pFilename   ビデオのファイル名を入れます。最大64バイト指定できます。 
 文字コードはUTF-8です。 
 pSuffix   ビデオファイルの拡張子を入れます。最大16バイト指定できます。 
 ファイル名とサフィックス間のピリオドはライブラリでは設定しませんので 
 忘れずに指定してください。必ずしも先頭にピリオドを置く必要は 
 ありません。ナンバリング機能を使用している場合は追い番の後に文字を 
 続けることができます。 
 ナンバリング機能を使用しない場合はファイル名に拡張子も含めて 
 この項目は省略しても構いません。SCE_NULLを指定することで、 
 省略可能です。文字コードはUTF-8です。 
 WriteRule   ファイルの書き込みルールを指定します。以下のルールがあります。 
 それぞれを同時に指定することはできません。 
     SCE_FMAC_WRITERULE_STANDARD   通常処理を行います。 
 以下の機能を使用しない時に 
 指定します。 
     SCE_FMAC_WRITERULE_NUMBERING   ナンバリング機能を有効に 
 します。 
 ファイル名の最後に自動的に 
 3桁の数値を付けて新規追加 
 します。 
     SCE_FMAC_WRITERULE_OVERWRITE   同じファイル名が存在する 
 場合に強制的に上書きします。 
 Numbering   ナンバリング機能が有効の時にファイル名に付ける追い番の開始番号を 
 指定します。0から999まで指定可能です。 
 MaxNumber   ナンバリング機能が有効の時にファイル名に付ける追い番の最大番号を 
 指定します。0から999まで指定可能です。この番号を超えて 
 ナンバリングは行いません。 
 Wrtworksize   sceFmacVideoAdd()で指定するoffsetは、64バイトにアラインしている 
 必要がありますが、アプリケーションの実装上、アラインの調整が難しい 
 場合はpWrtworkbuffに作業領域を指定することで本ライブラリ内で 
 補完します。必要な作業領域のサイズは 
 SCE_FMAC_PARAM_WRTALIGN_WORKです。これ以外の値はエラーに 
 なります。アプリケーションが64バイトアラインでoffsetを指定できる 
 場合は省略可能です。なお、pWrtworkbuffを省略している場合は 
 このメンバは無効です。 
 pWrtworkbuff   本ライブラリが使用する書き込みアライン調整作業領域を指定します。 
 Wrtworksize分の領域を確保してください。省略可能です。指定バッファの 
 アドレスは64バイトアラインされている必要があります。 

解説

sceFmacVideoStart()で本ライブラリを初期化するために使用する構造体です。アプリケーションがpWrtworkbuffを使用している場合は、pWrtworkbuffが指す領域をsceFmacVideoTerm()もしくは次のsceFmacVideoStart()まで保持してください。その他のメンバは関数リターン後は保持しなくても構いません。

初期化から末期化までの間にスリープやMemory Stick™を挿抜した場合は、以後の処理はエラーとなります。sceFmacVideoStart()を使用することで再開することができます。

スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフでない

引数

  pfilepath     使用中ファイルパスを受け取る領域の先頭アドレス  
  filepathsize     使用中ファイルパスを受け取る領域のバイト数  

返り値

  値     説明  
  SCE_FMAC_OK     正常終了  
  SCE_FMAC_ERROR_INIT     初期化・開始を行っていません。  
  SCE_FMAC_ERROR_INVALIDADDR     指定アドレスが不正です。  
  SCE_FMAC_ERROR_INVALIDSIZE     指定サイズが不正です。  


解説

sceFmacVideoStart()で設定した項目に従って、生成された使用中のファイルパスを取得します。
初期化から末期化もしくは再初期化までの間にスリープやMemory Stick™を挿抜した場合は、書き込み処理はエラーとなります。途中まで書き出されたファイルが不要な場合、アプリケーションにて削除など行う際に、本APIで処理中だったファイルパスを取得することができます。
