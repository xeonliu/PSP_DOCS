SceIoStat
 [構造体]
ファイル情報

構造

#include <iofilemgr_stat.h>
typedef struct {
	SceMode st_mode;
	Unsigned int st_attr;
	SceOff st_size;
	ScePspDateTime st_ctime;
	ScePspDateTime st_atime;
	ScePspDateTime st_mtime;
	unsigned int st_private[6];
}SceIoStat;
メンバ

  st_mode     ファイル種類およびアクセス権限  
  st_attr     ファイル属性(未使用)  
  st_size     ファイルサイズ(byte)  
  st_ctime     ファイル作成時間  
  st_atime     ファイルアクセス時間  
  st_mtime     ファイル変更時間  
  st_private     プライベートデータ  

解説

ファイルのステータス情報を格納する構造体です。

sceIoDread()関数およびsceIoGetstat()関数で取得することが可能です。
ISO9660ファイルシステムでは、st_private[0]メンバに、該当ファイルおよびディレクトリの先頭LBAが設定されます。

sceIoDread(), sceIoGetstat()

sceIoOpen
 [関数]
ファイルオープン

形式

#include <iofilemgr.h>
SceUID sceIoOpen (
	const char *filename, 
	int flag, 
	SceMode mode
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  filename     フルパス名を示すポインタ  
  パス名最大長：32bytes（NULLを含む）  
  ファイル名最大長：256bytes（NULLを含む）  
  flag     読み込みのみ指定可能  
  mode     未使用  

返り値

成功した場合は、ファイルディスクリプタ値が返ります。
エラーの場合は、負の値が返ります。

解説

ファイルおよび仮想ファイルをオープン処理をする関数です。

現在、引数filenameへのパス指定は、絶対パス指定しかできません。
ファイル名として使用可能な文字は、以下の文字種になります。
  • 大小英数文字
  • アンダーバー
  • ドット
  • セミコロン

同時オープン可能ファイル数は、32ファイルまでとなっています。

sceIoSetAsyncCallback
 [関数]
非同期処理通知用コールバックの設定

形式

#include <iofilemgr.h>
int sceIoSetAsyncCallback(
	SceUID fd, 
	SceUID cbid,
	int arg
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  fd     ファイルディスクリプタ  
  cbid     コールバックＩＤ  
  arg     コールバック関数に渡す引数  

返り値

成功した場合は、SCE_KERNEL_ERROR_OKが返ります。
エラーの場合は、負の値が返ります。

解説

ファイルI/Oの非同期処理が完了した際に通知を行う、スレッドマネージャのコールバックIDを設定する関数です。

コールバックは、あらかじめsceKernelCreateCallback()関数によって生成しておく必要があります。
非同期I/O処理が完了すると、指定したコールバックIDに対してsceKernelNotifyCallback()関数による通知が行われます。この後、コールバックを設定したスレッドがsceKernelCheckCallback()関数により通知の有無をチェックするか、SceXXXXCB()系関数のようなコールバック呼び出し付きの待ちAPIを実行することで、コールバック生成時に設定したコールバック関数が呼び出されます。
非同期I/O処理を完結させるためには、呼び出されたコールバック関数の中でsceIoPollAsync()関数、sceIoWaitAsync()関数などを実行してファイルI/O処理の実行結果を引き取る必要があります。

sceIoPollAsync / sceIoWaitAsync / sceIoWaitAsyncCB
 [関数]
非同期処理のステータス取得

形式

#include <iofilemgr.h>
int sceIoPollAsync(
	SceUID fd, 
	SceIores *iores
);
int sceIoWaitAsync(
	SceUID fd, 
	SceIores *iores
);
int sceIoWaitAsyncCB(
	SceUID fd, 
	SceIores *iores
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  fd     ファイルディスクリプタ  
  iores     ステータス  

返り値

非同期処理中の場合は、1を返します。
非同期処理の終了の場合は、0を返します。
エラーの場合は負の値を戻します。

解説

非同期I/O処理のステータスを取得する関数です。

sceIoWaitAsync()関数は、非同期I/O処理が完了するまで待ち、その結果を引数resultでポインタを渡す変数に格納してから返り値に0を返します。

sceIoWaitAsyncCB()関数は、sceIoWaitAsync()関数の動作中にコールバック通知の有無をチェックを行います。

sceIoPollAsync()関数は sceIoWaitAsync()関数から、非同期I/O処理が完了していない場合に待ち状態に入らず、すぐに処理を戻します。非同期I/O処理が完了していなかった場合は返り値に1を返します。この場合は、引数resultでポインタを渡した変数の値は変化しません。

非同期I/O関数を実行した場合は、上記の関数で処理状態を確認してください。
各関数が０を戻したとき、引数ioresには、非同期I/O関数に対応する同期I/O関数の戻り値が設定されています。
sceIoOpenAsync()であれば、sceIoOpen()の戻り値であるファイルディスクリプタ、sceIoReadAsync()であれば、sceIoRead()の戻り値である読み込みサイズが設定されています。
詳細は、各非同期I/O関数に対応する同期関数の説明を参照してください。

以下のコマンドが利用可能です。
  コマンド     説明  
  SCE_ISOFS_GET_PVD     PVDの取得。  
  SCE_ISOFS_GET_PTBL     パステーブルの取得。バッファサイズは、PVD  
  の132〜135bytes目から、LSBで取得可能。  
  SCE_ISOFS_GET_FILEPOINTER     ファイルオフセットの取得。  
  SCE_ISOFS_GET_LBA     ファイルの先頭LBAを取得。  
  SCE_ISOFS_GET_FSIZE     ファイルサイズの取得。  
  SCE_ISOFS_GET_SECTOR_SIZE     セクタサイズの取得。  
  SCE_ISOFS_SEEK_FILE     シーク処理。詳細後述。  
  SCE_ISOFS_UNCACHED_READ     強制ディスクアクセス読み込み。  
  SCE_ISOFS_READ_AHEAD     ファイル先読みリクエスト。  
  SCE_ISOFS_ACQ_READ_AHEAD     状態取得可能な先読みリクエスト。

SCE_ISOFS_GET_PVDについて
プライマリボリュームディスクリプタを取得します。2048bytes以上のバッファを設定してください。プライマリボリュームディスクリプタの詳細は
http://www.ecma-international.org/publications/files/ECMA-ST/Ecma-119.pdf
を参照してください。

SCE_ISOFS_GET_PTBLについて
パステーブルを取得します。パステーブルの詳細は、
http://www.ecma-international.org/publications/files/ECMA-ST/Ecma-119.pdf
を参照してください。

SCE_ISOFS_GET_FILEPOINTERについて
指定ファイルの現在のオフセットを取得します。

SCE_ISOFS_GET_LBAについて
指定ファイルの先頭LBAを取得します。

SCE_ISOFS_GET_FSIZEについて
指定ファイルのファイルサイズを取得します。

SCE_ISOFS_GET_SECTOR_SIZEについて
指定ファイルが保存されているデバイスのセクタサイズを取得します。

SCE_ISOFS_SEEK_FILEについて
sceIoLseek()関数が行うファイルオフセットの移動とともに、光ヘッドなどの物理的なシーク制御も行います。
コマンドパラメータには、必要なシークパラメータを設定してください。
ファイルオフセットは、バイト単位で設定されますが、光ヘッドの制御をともなう物理的なシークは、以下のようなLBAへシークします。

シーク位置(LBA) = ファイルが配置されている先頭のLBA + (ファイルオフセット/ セクタサイズ)

SCE_ISOFS_UNCACHED_READについて
UMD®/DVDのキャッシュ機能を利用せず、必ずディスクからデータを読み込むようにします。キャッシュ利用時に比べると、読み込み速度が低下しますので、使用する際は注意してください。また、このコマンド時成功時は、かならずディスクからデータを読み込むため、ドライブの電源状態は、アクティブに遷移します。
戻り値は、成功時は、sceIoRead*() 同様、０以上の読み込みサイズ（bytes）を戻します。

SCE_ISOFS_READ_AHEAD、SCE_ISOFS_ACQ_READ_AHEADについて
指定ファイルをumdcacheに先読みするようにリクエストします。
128KB未満のファイルは、ファイルの先頭LBAから64セクタ分がキャッシュされます。
6.4MB以上のサイズが指定された場合、キャッシュされません。
先読み開始位置(LBA)は、

先読み開始位置(LBA) = ファイルが配置されている先頭のLBA + (ファイルオフセット/ セクタサイズ)

となります。
SCE_ISOFS_ACQ_READ_AHEADコマンドは、リクエストしたファイルディスクリプタの先読み状態を、SCE_ISOFS_WAIT_STAT_READ_AHEAD, SCE_ISOFS_POLL_STAT_READ_AHEADコマンドから取得可能です。

SCE_ISOFS_WAIT_STAT_READ_AHEADについて
指定された先読みが、FINISH状態になるまで、コマンドを実行したスレッドをWAIT状態にします。
SCE_ISOFS_READ_AHEADコマンドをや先読みリクエストを行っていないファイルディスクリプタで、このコマンドを実行した場合、エラーやスレッドのWAIT状態とはならず、先読みはFINISH状態とみなされ、ただちに処理が戻ってきます。
SCE_ISOFS_CANCEL_STAT_READ_AHEADでWAIT状態がキャンセルされた場合、SCE_KERNEL_ERROR_WAIT_CANCELエラーが戻ります。

SCE_ISOFS_POLL_STAT_READ_AHEADについて
指定された先読みの状態を取得します。
SCE_ISOFS_READ_AHEADコマンドをや先読みリクエストを行っていないファイルディスクリプタで、このコマンドを実行した場合、エラーとはならず、先読みはFINISH状態とみなされ、ただちに処理が戻ってきます。

  値     説明  
  0     先読み処理は終了済  
  0x20     先読み処理を実行中  
  0x10     先読み処理を待機中  


SCE_ISOFS_CANCEL_STAT_READ_AHEADについて
SCE_ISOFS_WAIT_STAT_READ_AHEADコマンドで、先読みがFINISH状態になるまで、WAIT状態になっているスレッドを起床させます。
このコマンドは、先読み自体のキャンセルは行いません。

キャッシュの詳細は、「UMD®キャッシュ概要」を確認してください。
sceIoIoctlAsync
 [関数]
非同期処理の通常I/O処理以外でのデータ取得

形式

#include <iofilemgr.h>
int sceIoIoctlAsync(
	SceUID fd, 
	int cmd,
	const void *argp,
	SceSize arglen,
	void *bufp,
	SceSize buflen
);
呼出条件

割り込みハンドラからの呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  fd     ファイルディスクリプタ  
  cmd     コマンドコード  
  argp     コマンドパラメータ  
  arglen     コマンドパラメータサイズ  
  bufp     データバッファ  
  buflen     データバッファサイズ  

返り値

成功した場合は、SCE_KERNEL_ERROR_OKもしくは正の値が返ります。
エラーの場合は、負の値が返ります。

解説

通常、I/O処理から外れたデータの取得、および設定を行う場合に使用するファイルシステムコントロール関数を非同期におこなう関数です。

処理結果は、sceIoWaitAsync()関数、sceIoWaitAsyncCB()関数および、sceIoPollAsync()関数で確認することができます。
