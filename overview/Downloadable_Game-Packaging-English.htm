PSP™ Downloadable Game Package

PSP™ Downloadable Game Package Creation Guide

1 Introduction

This document describes the procedure for creating a PSP™ downloadable game package.
Precautions that need to be followed during programming and specifications of files (such as content information files) that need to be included in the package are described. The process from submitting the master disc to SCE, through QA testing, until the package is publicly released is explained. A chapter is also included on how to repackage existing applications that have already been released on UMD™.

1. 1 The PSP™ Downloadable Game Package

The PSP™ downloadable game package is a data packaging format that enables an application to be downloaded from the PlayStation®Store. After an application has been downloaded, it can be expanded on a Memory Stick™ and launched from the game list displayed by PSP™ system software. Access rights for the expanded application are managed by the NPDRM system so that only users with the proper authority are allowed to use the application.

Users can download a PSP™ downloadable game package from the PlayStation®Store to either a PlayStation®3, a PSP™, or a PC.
When a PSP™ downloadable game package is downloaded to a PlayStation®3, it is temporarily installed on the internal hard disk drive of the PlayStation®3, and then using system software, the user copies the package to the Memory Stick™ of a PSP™ via the USB connection.
When a PSP™ downloadable game package is downloaded to a PSP™, it will be expanded on the Memory Stick™ by system software. When a PSP™ downloadable game package is downloaded to a PC, a dedicated PC application called Media Go is used to install the package on to the Memory Stick™ of a PSP™ via the USB connection.
For the user to use a PlayStation®3 for downloading, PlayStation®3 content information files must be included in the PSP™ downloadable game package. When a PSP™ or PC is used for downloading, these PlayStation®3 content information files are not displayed.

Besides the PSP™ downloadable game package, another data format known as the PSP™ additional game data package can be used to distribute game data via the PlayStation®Store. For details, refer to the document entitled "PSP™ Additional Game Data Package Creation Guide".

1. 2 Workflow

The tasks for creating a PSP™ downloadable game package are normally performed in the following order.

(1) Create the application body

Create the program, data, and content information just as if you were creating a UMD™ game. After performing sufficient operational testing, use the UMD™ Image Generator to create a UMD™ image file.

(2) Create the files required for packaging

When PlayStation®3 content information files (required) and the game manual are included in the package, create the respective source image files and the configuration file.

(3) Perform debug authoring and testing

You can use the Simple Authoring Service (SAS) provided by SCE to create a debug package from the files created in (2). Application operation on a Memory Stick™ can be verified using this debug package and the development tool.

(4) Submit the master and perform QA testing

Assemble the UMD™ image file and files required for packaging in the predefined directory structure, burn them to a DVD-R, and submit the disc to SCE together with a checked TRC. SCE will create a PSP™ downloadable game package and perform QA testing. After QA testing is completed, the package will be made publicly available on the PlayStation®Store.

2 Creating the Application Body

This chapter describes how to create an application for distribution as a PSP™ downloadable game package. Precautions to note when developing the application are also mentioned.

2. 1 Programming Precautions

From a programming perspective, a PSP™ downloadable game is almost the same as a UMD™ game. However, downloadable games have certain specific restrictions and are especially susceptible to certain problems. For details about these restrictions and problems, refer to the "Notes on Creating PSP™ Downloadable Games" document.

2. 2 Operational Testing

A PSP™ downloadable game that will be distributed online must be of the same quality as a UMD™ game. Be sure to perform sufficient operational testing before mastering.

2. 3 The System File and Content Information Files

In the same way as when creating a UMD™ game, create a system file (PARAM.SFO) and content information files such as the still image icon (ICON0.PNG) for a PSP™ downloadable game. For detailed specifications and the creation procedure, refer to the "Game Master Disc Creation Procedure," which is included in the PSP™ runtime library documentation.

2. 4 Creating the UMD™ Image File

When operational testing is completed, assemble the program, data, and content information files in the predefined directory structure and use the UMD™ Image Generator to create the UMD™ image file. The UMD™ Image Generator outputs four or five files. Use the UMD™ Image Checker for Downloadable Game to check the output files and verify that the message "UMD Image Check result: OK" is displayed, which indicates that no error occurred.
For details about the procedure and the directory structure, refer to the "Game Master Disc Creation Procedure" document included in the PSP™ Runtime Library documentation, the "UMD™ Image Generator" document included in the Game Master Disc package and the "UMD™ Image Checker for Downloadable Game" document included in the Downloadable Game Creation package.
Although the UPDATE directory is automatically inserted by the UMD™ Image Generator, since it is ignored by system software, it does not need to be deleted. Note that the package size will not increase by leaving in this directory.

3 Files Required for Packaging

This chapter describes the following files, which are required when creating a PSP™ downloadable game package.
  • PlayStation®3 still image icon file
  • PlayStation®3 descriptive image file (high resolution / standard resolution)
  • Source image files for the game manual
  • Configuration file (NPDRM.CONF)
In addition to the above, BGM and video icons are also part of PlayStation®3 content information, however, for a PSP™ downloadable game package, only the still image icon and descriptive image files listed above should be included. Do not include any other PlayStation®3 content information in a PSP™ downloadable game package.

3. 3 Source Image Files for the Game Manual

The PSP™ downloadable game package can contain optional image files that contain information explaining how to run the game, enabling the user to browse the "Game Manual" on PSP™ system software screens. Although game manual image files can be omitted when building a downloadable game package, in this case, be sure to follow the SCE guidelines for each region.

Language switching is not supported for game manual image files. Also, to protect the game manual with NPDRM, it must be specified in the configuration file, which is described later.
The specifications for the source images for the game manual are given below.

Image Size
Horizontal : 480 pixels (fixed)
Vertical : 248 - 960 pixels

Image Format
PNG - 24 bits (cannot be interlaced)

File Size
1280K bytes (1280 x 1024 bytes) max. per page.
999 pages max.

Filename
Three digits representing the page number +".PNG": 001.PNG, 002.PNG, ...

Creation Method
Use a commercially available image editing tool.

3. 4 Configuration File

The configuration file is a text file containing the content ID and master package version. This information is used to uniquely identify the content in the PlayStation®Store. The filename of the configuration file should be NPDRM.CONF. The format of the configuration file is shown below.

Content-ID = IV0002-NPXS00004_00-0000111122223333
PackageVersion = 01.00
ProtectDocument = TRUE
HealthWarning = SCEJ

Content ID
This line should contain the string "Content-ID = " followed by the content ID. The content ID consists of the service ID issued by SCE followed by a hyphen and an arbitrary 16-digit label. For details about the content ID, refer to the "PlayStation®Network Overview" document. The content ID must be unique to each package.

Master Package Version
This line should contain the string "PackageVersion = " followed by the master package version. The master package version consists of two digits, a decimal point and two more digits. Choose values that conform to the master version numbering scheme described in the "Technical Requirements Checklist for PSP™ (PlayStation®Portable) Software" (TRC).

Game Manual Protection
To use NPDRM to protect the game manual, include the line "ProtectDocument = TRUE" in the configuration file. Like the downloadable game itself, once protected, the relevant game manual can only be browsed on a PSP™ that was authenticated with the user account used to purchase the downloadable game package. If "ProtectDocument = FALSE" is specified or if no specification is made, the game manual will not be protected by NPDRM.

Startup Warning
You can specify that a startup warning be displayed when the game is started up. Specify "HealthWarning = " followed by the specification word shown in the following table, which indicates the region or language where the downloadable game package will be released. See Chapter 7 for the corresponding text that is displayed.
If "HealthWarning = NONE" is specified or if no specification is made, no startup warning will be displayed.

  Specification Word     Description  
  SCEJ     Title for SCEJ  
  SCEA     Title for SCEA  
  SCEE_EN     Title for SCEE (English)  
  SCEE_FR     Title for SCEE (French)  
  SCEE_DE     Title for SCEE (German)  
  SCEE_IT     Title for SCEE (Italian)  
  SCEE_ES     Title for SCEE (Spanish)  
  SCEAsia     Title for SCEAsia  


Compression Rate Specification
During authoring of a downloadable game package, the data in the associated UMD™ image file is compressed. You can change the compression rate by specifying "ZIPThreshold=" followed by the compression rate (0 to 100). If there is no "ZIPThreshold=" specification, the compression rate will default to 80. If "ZIPThreshold=0" is specified, the data will not be compressed and there will no longer be any load for expanding the compressed data. However, the file size will increase and it will take longer for the user to download the package. Unless you specifically need to change the compression rate, do not include this specification.

Optimization Level Specification
The SCE authoring service may change the optimization level to improve processing performance when the system reads data in a downloadable game. If this change hinders the behavior of a program that is being developed, you can include the specification "OptimizeLevel=" followed by the optimization level that was used previously during authoring. If there is no impediment to program behavior, you should not enter this specification.
When the authoring service changes, refer to details that are publically available in the Technical Notes.

4 Debug Authoring and Testing

This chapter describes the files required for creating a debug package using the Simple Authoring Service (SAS) and testing of the debug package that is created.
For details on accessing and operating the Simple Authoring Service, refer to the "PlayStation®Network SMT Simple Authoring Service Users Guide."

4. 1 Creating a Debug Package

Directory Configuration
Group the files prepared in Chapter 3 into a ZIP archive file with the following directory structure.



Figure 1 Downloadable Game ZIP Archive Structure


Place the UMD™ image files directly under the root directory. Because the UMD™ Image Generator outputs four to five files, make sure to write them all. Also create a subdirectory named, "PS3_CONT" directly under the root directory, and place the PlayStation®3 content information files therein. Similarly, create a subdirectory named, "DOC," and place the original image files of the game manual in this subdirectory. Finally, place the configuration file "NPDRM.CONF" directly under the root directory.
Name the ZIP archive that you created, "contentID.ZIP."

4. 2 Authoring

Use the simple authoring service provided by SCE to perform authoring on the zip archive that you created, to create the PSP™ downloadable game data package. For details about how to use the simple authoring service, refer to the "PlayStation®Network SMT Simple Authoring Service Users Guide" document.
If there are any problems with the uploaded file, the following error codes will be reported. If any other error code is reported, notify SCE of the error code and message you received.

4. 3 Testing With the Development Tool

The following two files are obtained from the Simple Authoring Service.
  • ZIP package (*.ZIP) for testing application operation with the PSP™ development tool
  • PKG package (*.PKG) for verifying content information with the PlayStation®3 development tool
Expand the ZIP package in the root directory of the Memory Stick™.
If the Memory Stick™ is inserted in the PSP™ development tool, you can start the application from the game list displayed by the PSP™ system software to verify its operation. If the application on the Memory Stick™ was started from the system software, a program that references disc0: contents can be executed from the debugger even without UMD™.
Note that the ZIP package only runs on a PSP™ development tool.

Save the PKG package directly in the root directory of the Memory Stick™.
Insert the Memory Stick™ in the PlayStation®3 development tool and select the item "Install Package Files" from the games list displayed by system software; this will install the package. After the package is installed, verify that the PlayStation®3 content information is displayed correctly on the system software screen.
Also, if a PSP™ development tool is connected to the PlayStation®3 development tool by a USB connection, you can confirm the operation when the package is copied from the PlayStation®3 to the PSP™ by selecting the installed content. Sign in to the PlayStation®Network before performing this operation.
Note: If you do not have a PlayStation®3 development tool, check with the SCE Publisher Relations Department.

4. 4 Checking the Package

The make_package_npdrm command can be used to check the validity of the PKG package that was obtained from the simple authoring service.
To check the validity of the package, use the --check option.

Example) % make_package_npdrm  --check  IV0002-NPXS00004_00-0000111122223333.pkg
Also, the --list option can be used to display the files contained in the package.

Example) % make_package_npdrm  --list  IV0002-NPXS00004_00-0000111122223333.pkg

5 Submitting the Master and Performing QA Testing

5. 1 Creating the Submission Disc

Disc Layout
Arrange the files in the following directory structure and write them to a DVD-R.



Figure 2 Directory Structure of Submission Disc


Place the UMD™ image file in the root directory. Since the UMD™ Image Generator outputs four or five files, be sure to include all of them. Also, create a subdirectory named "PS3_CONT" under the root directory and place the PlayStation®3 content information files there. Similarly, create a subdirectory named "DOC" under the root directory and place the source image files for the game manual there. Finally, place the configuration file (NPDRM.CONF) in the root directory.

DVD Writing Software
Any DVD writing software can be used. Make sure that the DVD is written as a "data DVD."

Verify the DVD-R
Once the DVD-R has been successfully written, insert it into the DVD drive of the PSP™ development tool. Confirm that the application runs correctly and use the content information check feature of system software to verify that the content information is correct. For details, see the PSP™ runtime library document "Content Information Check Feature." Make sure that the system software of the PSP™ development tool has been updated in advance to the version specified by SCE.

5. 2 Submitting the Master

After the DVD-R has been checked as described above, submit it to SCE along with a checked "Technical Requirements Checklist for PSP™ (PlayStation®Portable) Software" (TRC) and other required documentation.
When a title is to be sold as both UMD™ media and as a downloadable game, the UMD™ master can be combined on the DVD-R disc that was created for the downloadable game.

5. 3 QA Testing

SCE will create the PSP™ downloadable game package by performing authoring using the submitted DVD-R and then performing QA testing.

Special QA Tests
Some QA tests require a special process. If you need the tests below performed for your application, please discuss this beforehand with the SCE Publisher Relations Department.
  • Testing compatibility between PSP™ system models
  • Testing output to an external display
  • Testing the use of accessories

5. 4 Public Availability

After QA testing has successfully completed, the PSP™ downloadable game package will be moved to the public server where it will become available to end users for downloading.

6 Repackaging UMD™ Games

This chapter describes the procedure for repackaging an existing (or "legacy") UMD™ game currently on the market as a PSP™ downloadable game. It primarily describes the differences in the process from earlier chapters.

6. 1 Creating the Application Body

If you wish to repackage an existing (or "legacy") UMD™ game as a PSP™ downloadable game package for distribution from the PlayStation®Store, you must first contact the SCE Publisher Relations Department. This is because additional steps that pertain to the PlayStation®Store such as the issuing of a service ID need to be performed.

Normally, you do not need to make any changes to the application body when repackaging a UMD™ game as a PSP™ downloadable game, however, the following point should be noted.

Multi-disc Titles
A multi-disc title cannot be repackaged as a PSP™ downloadable game.

6. 2 Creating Files Required for Packaging

Create the PlayStation®3 content information files, source image files for the game manual, and configuration file (NPDRM.CONF) just as if you were creating a new PSP™ downloadable game package.

6. 3 Creating the Submission Disc

Arrange the files in the following directory structure and write them to a DVD-R.



Figure 3 Directory Structure of Submission Disc (Repackaged Version)


The directory structure is the same as that for a new PSP™ downloadable game package except that the UMD™ image files are not included.
Any DVD writing software can be used. Make sure that the DVD is written as a "data DVD."

6. 4 QA Testing

After creating the DVD-R as described above, submit it to SCE along with the original UMD™ game. SCE will create a PSP™ downloadable game package, perform QA testing, and make the package publicly available.
For repackaging, your company will not be asked to perform testing of the actual game. If SCE has a problem with the PlayStation®3 content information on the disc you've submitted, the PSP™ downloadable game package may be returned to you and you may be asked to verify it.
