PSP™ Programmer Tool Runtime Library Release 6.3.0

Content Information Checking Function

1 Content Information Screen

This display shows content information for PSP™ system software (The illustration below is for UMD™ VIDEO) .


Content information consists of a number of image, video, and sound files.
It is used for PSP™GAME, UMD™VIDEO, UMD™AUDIO, and save data.

2 Content Information Checking Function

This is a function which checks the files which make up content information.
If an error results from this checking, then the location and the file where the error occurred is displayed.

2. 1 Checking Function Usage Procedure

Perform the following operations.

  1. Start up PSP™ system software on the development tool.
  2. Transition to the screen in which the content information is displayed.
  3. Press the triangle button to open the Options menu.
  4. Select "Check" in this menu.
  5. The checking function runs, and the results are displayed on the screen.

3 Checking Function Results

The following is an explanation of how to interpret the results of the checking function.

3. 1 "Status: OK" is displayed

The content information is correct. There are no items to correct.

3. 2 "Status: OK" is displayed, but lines in yellow starting with "Warning…" are also displayed

The content information itself is correct.
However, it is recommended that certain other information, files, etc. be specified. For such items, "Warning…" lines are displayed; please follow these recommendations as much as possible.

3. 3 "Status: Error" is displayed, and lines in red are also displayed

There are errors in the content information.
All of the errors are reported in the lines which follow; please correct these.
The structure of the error notifications is:

[Name of file containing error] - [Description of error]

There is also "Unknown file found: …"

3. 4 Precautions

Along with error descriptions, warnings, and so on, lines for confirming configuration details may also be displayed. For lines such as these, the characters are displayed in white. These are unrelated to the results of the checking function.
Depending on whether it is a PSP™ GAME or save data which is being checked, the warnings, error messages, etc. which are displayed may differ. In the explanations below, description lines which are actually displayed are described; whether each refers to a PSP™ GAME or to save data is noted between "<<" and ">>".

4 Warning… Lines

"Warning…" lines are used to indicate items, files, etc. for which checking did not result in an error, but for which configuration is recommended if at all possible.
These lines are displayed in yellow.
The types of warning lines are as follows.

4. 1 PARAM.SFO File Warnings

Warning PARAM.SFO - DISC_VERSION: …

<<PSP™ Game>>
The format of the disc version is incorrect. In order to move the mastering process with SCE forward in a smooth manner, please enforce proper formats as much as possible.

Warning PARAM.SFO - SAVEDATA_TITLE is required.

<<Save Data>>
The save data title does not exist. Please endeavor to use this as the subtitle.

Warning PARAM.SFO - SAVEDATA_DETAIL is required.

<<Save Data>>
Detailed information does not exist. Please endeavor to use this so that the nature of the save data can be understood without actually loading the data.

Warning PARAM.SFO - BOOTABLE: 0 (0x0)

<<PSP™ Additional Game Data>>
This warning is displayed if the check function was used on additional game data. BOOTABLE: 0 indicates normal processing, so no additional changes or settings are required. However, if BOOTABLE: 1 is displayed for additional game data, it means that an error occurred during creation of PARAM.SFO.

Warning PARAM.SFO - ATTRIBUTE: 1 (0x1)

<<PSP™ Downloadable Game>>
This warning is displayed for a downloadable game when additional game data might override content information. If this warning is displayed at any other time, it means that an error occurred during creation of PARAM.SFO.

5 Error Lines

Error lines display the cause of errors resulting from the checking function.
If left uncorrected, the information will not constitute correct content information, so please address these appropriately.
These lines are displayed in red.
The types of error lines are as follows.

5. 1 PARAM.SFO File Errors

PARAM.SFO - No File

<<PSP™ GAME, Save Data>>
PARAM.SFO does not exist. Please create it.

PARAM.SFO - Corrupted Data

<<PSP™ GAME, Save Data>>
There is a fatal error in PARAM.SFO. Please follow a valid method to recreate the file.

PARAM.SFO - DISC_ID Error: …
PARAM.SFO - PSP_SYSTEM_VER Error: …
PARAM.SFO - REGION Error: …
PARAM.SFO - CATEGORY: … is required.
PARAM.SFO - BOOTABLE: 1 is required.
PARAM.SFO - DISC_NUMBER: 1 is required.
PARAM.SFO - DISC_TOTAL: 1 is required.
PARAM.SFO - PARENTAL_LEVEL: 0 to 11 is required.
PARAM.SFO - TITLE is required.
PARAM.SFO - ATTRIBUTE Error: …
PARAM.SFO - ATTRIBUTE: 0 or 1 is required.
PARAM.SFO - USE_USB Error: …
PARAM.SFO - TARGET_APP_VER Error: …
PARAM.SFO - APP_VER Error: …
PARAM.SFO - GAMEDATA_ID Error: …

<<PSP™ GAME, Save Data>>
The respective items specified in PARAM.SFO are incorrect. Please replace them with correct values.

PARAM.SFO - SAVEDATA_DIRECTORY is incorrect.

<<Save Data Only>>
The directory in which the save data is stored is incorrect.

5. 2 Errors in Image Files ICON0.PNG, PIC0.PNG, and PIC1.PNG

ICON0.PNG - No File

<<PSP™ GAME, Save Data>>
ICON0.PNG does not exist. Please create it.

xxx.PNG - Corrupted Data

<<PSP™ GAME, Save Data>>
The image file is corrupted. Please recreate it.

xxx.PNG - png [W] x [H] is required.

<<PSP™ GAME, Save Data>>
The size of the image file is incorrect. The correct size is [W] x [H]. Please correct it.

xxx.PNG - png [D] bit is required.

<<PSP™ GAME, Save Data>>
The color count (color depth) of the image file is incorrect. The correct color count (color depth) is [D] bits. Please correct it.

5. 3 Errors in Video and Sound Files ICON1.PMF and SND0.AT3

ICON1.PMF - ICON1.PMF + SND0.AT3 > 500 KB
SND0.AT3 - ICON1.PMF + SND0.AT3 > 500 KB

<<PSP™ GAME, Save Data>>
The sum of the sizes of the animated icon file ICON1.PMF and the background music file SND0.AT3 must not exceed 500 kilobytes. Adjust the sizes to fit within this limit.

5. 4 Errors in Other Files

Unknown file found: …

<<PSP™ GAME, Save Data>>
A file which uses a system reserved name, or a file which is unnecessary was found. Please correct this by, for instance, changing the name or deleting the file as appropriate.

6 Configuration Content Display Lines

There are a number of settings for which it is convenient to perform verification using the tool, and using the checking function, these are displayed along with the results.
These lines have no effect on the results of the checking function. They are displayed in white.
Be careful not to confuse these with error and warning... lines.
These lines are as follows.

6. 1 PARAM.SFO-Related

ms0:/PSP/GAME/…
ef0:/PSP/GAME/...

«PSP™ GAME»
Displays the file path of the checked game.

DISC_ID: …

<<PSP™ GAME>>
Displays the part number of the disc.

DISC_VERSION: …

<<PSP™ GAME>>
Displays the version of the disc.

PSP_SYSTEM_VER: …

<<PSP™ GAME>>
Displays the version of the PSP™ system software which is required for start up.

USE_USB: …

<<PSP™ GAME>>
Displays whether or not a USB device is used (0 = a USB device is not used; 1 = a USB device is used).

APP_VER: …

<<PSP™ GAME>>
Displays the version of the patch.
For details, refer to the "PSP™ Patch Package Creation Guide."

TARGET_APP_VER: …

<<PSP™ GAME>>
Displays the differential target version.
For details, refer to the "PSP™ Patch Package Creation Guide."

PBOOT_TITLE: …

<<PSP™ GAME>>
Displays the title for a standalone patch package.
For details, refer to the "PSP™ Patch Package Creation Guide."

GAMEDATA_ID: …

<<PSP™ GAME>>
Displays parameters for using a common patch with multiple discs.
For details, refer to the "PSP™ Patch Package Creation Guide."

Version: …

<<Save Data>>
Displays the format of the save data (1.00: old format; 2.00: new format).

7 Checking Patched Games

When the checking function is used for games that have been patched, it checks the parameters of the game body and patch, and displays both results.
The results of the checking function for the game body are displayed following "---- EBOOT.PBP ----".
The results of the checking function for the patch are displayed following "---- PBOOT.PBP ----".
Note that checking function results are displayed for both of these.

The checking function also compares the parameters of the game body and patch.
The parameter comparison results are displayed following "---- COMPARE ----".
If the parameters differ, "different" is displayed in red.
The following are displayed for the parameter comparison result.

7. 1 Comparison Displays

DISC_ID: …

Displays the result of comparing the part numbers of the discs.

GAMEDATA_ID: DISC_ID: …

Displays the result of comparing GAMEDATA_ID and the part number of the disc.
When GAMEDATA_ID has been specified in the game body and the patch is placed in GAMEDATA_ID, the GAMEDATA_ID of the game body is compared with the DISC_ID of the patch.

REGION: …

Displays the result of comparing the regions.

PARENTAL_LEVEL: …

Displays the result of comparing the parental control levels.

USE_USB: …

Displays the result of comparing whether or not a USB device is used.
