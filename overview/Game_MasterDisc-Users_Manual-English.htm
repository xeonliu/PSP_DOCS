PSP™ Game Master Disc Package

Game Master Disc Creation Procedure

1 Workflow

The creation of a game UMD™ is described in broad terms below.

1. 1 Required File Creation

The following files are required in order to create a game UMD™.
  • The game program and data
  • System file (all characters in uppercase): PARAM.SFO
  • Content information files (all characters in uppercase): ICON0.PNG, ICON1.PMF, PIC0.PNG, PIC1.PNG, SND0.AT3

Note
The PARAM.SFO created with the previous version of the master disc creation tool cannot be used with the current version. Make sure to recreate it using the current version.

1. 2 DVD-R Creation

Write the files above to a DVD-R using the UMD™ Image Generator. In order to do this, these respective files must be placed in specific directories.

2 System File Creation

The system file is described below.

2. 1 What is the system file?

The system file is a file which stores parameters used for display by the PSP™ system software, such as the game title, user age restriction value, etc., as well as parameters used internally, such as the product number, disc serial number, etc. The parameters which are stored differ based on the type of content on the UMD™ (game, video, audio), the contents of the Memory Stick™, etc.
The filename of the system file is PARAM.SFO (all characters in uppercase), and the file is written in system file format.
It is created using SystemFileEditor.

2. 3 SystemFileEditor

The SystemFileEditor is a tool for setting the required parameters and creating the system file. For details regarding its use, refer to the "SystemFileEditor" document.

Config File
Use the UMDGame_SystemFile_Config.sfc file as the config file for the game UMD™ to be used by the SystemFileEditor.

Note on File Name
In SystemFileEditor, the output file name will not automatically become PARAM.SFO. Rather, it will be arbitrary name.sfo. When using the file in the game master disc, please change the file name to PARAM.SFO. In addition, a config file (.sfc) and system source file (.sfs) cannot be used as a system file. Please make sure that you change the .sfo file to PARAM.SFO and write it in the master disc.

Automatic Updating of PARAM.SFO by UMD™ Image Generator
When an image is created with UMD™ Image Generator, the PSP_SYSTEM_VER parameter in the PARAM.SFO file that was on the game master disc is automatically updated with the version of the SDK that was used to build the boot file. This operation is performed automatically and does not require manual intervention.

3 Content Information File Creation

The content information files are described below.

3. 1 What are content information files?

The images, videos, background music, etc. which represent and depict the actual content of a title are called content information. Content information is primarily used in PSP™ system software for display purposes. For information on screen configuration, see the "9 Appendix".

3. 2 File Structure

Content information is structured using multiple files. All filenames are in uppercase.
  • Still image icon: ICON0.PNG
  • Animated image icon: ICON1.PMF
  • Content description graphic: PIC0.PNG
  • Content information background graphic: PIC1.PNG
  • Background music: SND0.AT3

3. 3 Still Image Icon

The still image icon is displayed as a substitute icon when the animated image icon cannot be displayed or when an animated image icon is not specified.
Note that this file is displayed no matter what System Software language is selected by the user; localized still image icons are not supported.
It is a required file. Make sure to create it.

Image size
144 x 80 pixels

Graphics Format
PNG (24-bit)

Filename
ICON0.PNG (all characters in uppercase)

Creation Tool
Use a commercially available tool or the like.

3. 4 Animated Image Icon

An animation can be displayed as an icon.
It can be displayed with or without the sound.
When the animation ends, it will start again from the beginning, but the sound may be discontinued. To play the sound in a loop continuously, play it as background music. However, note that background music will not be synchronized with the icon. When an animation with the sound has BGM, only BGM will be played.
Note that this file is displayed no matter what System Software language is selected by the user; localized animated image icons are not supported.
This is treated as optional and thus can be omitted when not needed.

3. 5 Content Description Graphic

This is a graphic which describes the nature of the content. It is used to display the title graphic and text which can represent, for example, an overview of the content. It is a 32-bit graphic, and can include transparency. It will be displayed on top of and overlapping the content information background (see below).
Note that this file is displayed no matter what System Software language is selected by the user; localized image files are not supported.
This is treated as optional and thus can be omitted when not needed.

This is a graphic to be used as the background.
Note that this file is displayed no matter what System Software language is selected by the user; localized background image files are not supported.
This is treated as optional and thus can be omitted when not needed.

This is a sound file. If a loop range is specified, the music can be played in a loop without stopping.
Note that this file is played no matter what System Software language is selected by the user; localized BGM is not supported.

4 Game UMD™ Directory Structure

Files and directories on a UMD™ must conform to the ISO9660 Level 2 standard.
Under this standard, a game UMD™ has the following directory structure.



All files are placed under the PSP_GAME directory. Nothing need be written to the root directory (excluding the files which are automatically written by the UMD™ Image Generator).
The system file and the content information files are placed in the PSP_GAME directory. In terms of the directories under PSP_GAME, files related to game boot-up are placed in the SYSDIR directory, and game data files are placed in the USRDIR directory.

Note: Directory names and filenames should be as shown in the figure. For example, lowercase characters should not be included in the specified names. Filenames of the game data files are arbitrary.

5 SYSDIR Directory

Files related to game boot-up are placed in the SYSDIR directory.

5. 1 Specifying the Boot File in the UMD™ Image Generator

Specifying the boot file for the game which has been created is performed according to the following procedure.
  1. Place the boot file which has been created in the SYSDIR directory with the name BOOT.BIN.
  2. Specify BOOT.BIN file to be the boot file.
  3. EBOOT.BIN is created automatically in the SYSDIR directory.

For details, refer to the "UMD™ Image Generator" document.

5. 2 UPDATE Directory

This is the directory where data for PSP™ system software updates is saved. A special work area is not required. Do not put any files in this directory.

6 USRDIR Directory

Game data files and the like are placed in the USRDIR directory.
Subdirectories can be created arbitrarily and there are no particular restrictions, but directory depth, filename length, and the like must conform to ISO9660 Level 2.

6. 1 A Note on Directories

Be aware that the boot file (BOOT.BIN) and directory are in a different location from the USRDIR files and directory.

7 INSDIR Directory

Game data files and the like, which are installed by the game data installation utility, are placed in the INSDIR directory.
For details, refer to the "Installation Source Data Structure" chapter of the "Game Data Installation Utility" overview document.

8 DVD-R Creation using the UMD™ Image Generator

For information regarding the DVD-R creation procedure, refer to the "UMD™ Image Generator" document.

8. 1 Precautions

Layers
The following files should always be placed in Layer0. Please avoid having them extend to Layer1.
  • Game boot file: BOOT.BIN
  • System file: PARAM.SFO
  • Contents information file: ICON0.PNG, ICON1.PMF, PIC0.PNG, PIC1.PNG, SND0.AT3
Whether the file is to be placed in Layer0 or Layer1 can be checked in the Layout screen of the UMD™ Image Generator.

Size of BOOT.BIN
The maximum size of the game boot file BOOT.BIN is 24,641,535 (0x177FFFF) bytes.
Since a BOOT.BIN file that exceeds this size (greater than or equal to 24,641,536 (0x1780000) bytes) cannot be started up on an actual system (PSP-1000 or PSP-2000 series) or on a development system (DTP-T1000(A) or DTP-H1500(A)), be sure to create BOOT.BIN so that it does not exceed this maximum size.

9 Appendix

9. 1 Organization of the Content Information Screen Used by PSP™ System Software


Note: The layout of the data is not permanently guaranteed, so avoid creating content for which there are layout interdependencies.

9. 2 Display Order of Content Information in PSP™ System Software

When the UMD™ icon is focused, reading of all files consisting of the content information will begin. When the reading completes, the background of content information will be displayed. Then, a little later, display of the content explanation image (fade-in), play of the animation icon (fade-in), and play of the BGM are performed almost simultaneously. Completely concurrent playback of the animation icon and BGM is not guaranteed. Limited time lag for the playbacks may occur.
