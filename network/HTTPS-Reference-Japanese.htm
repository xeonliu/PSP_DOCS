httpsCallback
 [関数]
SSL通信が発生した時のコールバック関数

形式

#include <libhttps.h>
int *(*httpsCallback)(
	unsigned int verify_err,
	SceSslCert *const ssl_cert[],
	int cert_num,
	void *user_arg
);
引数


  verify_err     証明書のベリファイエラーが発生している時に、その原因がbit表現で  
  格納されます。設定される値はsceHttpsGetSslError()でエラー値が  
  SCE_HTTPS_ERROR_CERTの時のdetailの値と同一です。  
  詳細は、sceHttpsGetSslError()の説明をご覧ください。  
  ssl_cert     証明書チェーンを表す配列へのポインタ。  
  SceSslCertは、libssl.hで定義されています。  
  cert_num     証明書チェーンの証明書の数。  
  user_arg     sceHttpsSetSslCallback()で指定した値が格納されます。  
  使用しなくても構いません。  

返り値

  値     説明  
  0以上     SSL通信を許可  
  負の値     SSL通信を不許可  


解説

SSL通信発生時に呼び出されるコールバック関数の型定義です。
verify_err, ssl_cert, cert_numの情報から、SSL通信を開始してもよいなら正の値、SSL通信を中止したい時は負の値を返してください。ssl_certの証明書チェーンの情報は直接は参照できません。sceSslGetSerialNumber(), sceSslGetSubjectName(), sceSslGetNameEntryCount(), sceSslGetNameEntryInfo()などを用いて情報を取得してください。
また証明書チェーンのメモリはコールバック関数から抜けた後に開放されるため、証明書の情報が必要な場合はコールバック関数内で別メモリに保存しておく必要があります。

備考

libhttpをマルチスレッドで使用する場合は、当関数もマルチスレッドセーフである必要があります。

#include <libhttps.h>

解説

FLASHストレージに保存されているデフォルトCA証明書をロードし使用します。issuerに証明書発行者を、certsにロードする証明書を指定します。あらかじめlibsslの初期化およびsceHttpsInit()が呼ばれている必要があります。

使用例

/* FLASHストレージに含まれる全てのCA証明書をロードする場合 */
ret = sceHttpsLoadDefaultCert(SCE_HTTPS_ISSUER_ALL, 0);
if (ret < 0){
	printf("sceHttpsLoadDefaultCert() returns %x\n", ret);
	goto error;
}
..

/* FLASHストレージに含まれるCA証明書のうちSCE_HTTPS_ISSUER_RSAから
発行された全てのCA証明書をロードする場合 */
ret = sceHttpsLoadDefaultCert(	SCE_HTTPS_ISSUER_RSA,
 				SCE_HTTPS_ALL_CERTS);
if (ret < 0){
	printf("sceHttpsLoadDefaultCert() returns %x\n", ret);
	goto error;
}
..

/* FLASHストレージに含まれるCA証明書のうちSCE_HTTPS_ISSUER_RSAから
発行されたSCE_HTTPS_RSA_2048_V3証明書のみをロードする場合 */

ret = sceHttpsLoadDefaultCert(	SCE_HTTPS_ISSUER_RSA, 
				SCE_HTTPS_RSA_2048_V3);
if (ret < 0){
	printf("sceHttpsLoadDefaultCert() returns %x\n", ret);
	goto error;
}
..

/* FLASHストレージに含まれるCA証明書のうちSCE_HTTPS_ISSUER_RSAと
SCE_HTTPS_ISSUER_VERISIGNから発行された全てのCA証明書をロードする場合 */
ret = sceHttpsLoadDefaultCert(	SCE_HTTPS_ISSUER_RSA,
 				SCE_HTTPS_ALL_CERTS);
if (ret < 0){
	printf("sceHttpsLoadDefaultCert() returns %x\n", ret);
	goto error;
}
ret = sceHttpsLoadDefaultCert(	SCE_HTTPS_ISSUER_VERISIGN,
 				SCE_HTTPS_ALL_CERTS);
if (ret < 0){
	printf("sceHttpsLoadDefaultCert() returns %x\n", ret);
	goto error;
}
備考

独自のCA証明書を使用する場合は、sceHttpsInit()でセットする事ができます。
またissuerにSCE_HTTPS_ISSUER_ALLを指定した場合、アプリケーションが使用しているランタイムライブラリのバージョン当時の全てのルートCA証明書がロードされます。システムソフトウェアのアップデートでルートCA証明書の追加が行われた場合も、ロードされるルートCA証明書の数は変わりません。

sceHttpsGetSslError
 [関数]
SSL通信の詳細エラーコードの取得

形式

#include <libhttps.h>
#include <libhttp_error.h>

int sceHttpsGetSslError(
	int *request_id,
	int *err_num,
	unsigned int *detail
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止/許可状態に依存しない）
マルチスレッドセーフである

引数

  request_id     SSLレイヤーのエラー情報を取得するリクエストIDを指定します。  
  err_num     エラーコードを格納するためのアドレスを指定します。  
  エラーコードの値によっては、detailにより詳細な原因が格納されます。  
  detail     エラーコードの発生した原因の詳細を格納するためのアドレスを指定します。  
  エラーコードの値によっては使用されません。  
  また、複数の原因でエラーが起こっている場合は、それら原因のビット論理和  
  で表現された値が格納されます。  

err_numに格納されるエラーコードとdetailに格納される詳細値の対応は、下記の通りです。
  値     マクロ     意味  
  0x80435022     SCE_HTTPS_ERROR_OUT_OF_MEMORY     SSLライブラリの使用できる  
  メモリが不足している

解説

sceHttpSendRequest(), sceHttpReadData()で発生したSCE_HTTP_SSL_ERRORの詳細理由を取得します。request_idにSCE_HTTP_SSL_ERRORの発生したリクエストIDを指定すると、err_num, detailで指定したアドレスにエラーコードおよび理由を示す値が格納されます。detailを使用しないエラーコードの場合にはdetailには0が格納されます。

注意事項
HTTP Pipeline使用時には、sceHttpGetContentLength(), sceHttpGetStatusCode(), sceHttpGetAllHeader()等のレスポンスステータス取得関数でレスポンスヘッダを受信するため、それら関数でもSCE_HTTP_SSL_ERRORが発生することがありえることにご注意ください。またSSL通信を内部で行わないsceHttpsInit(), sceHttpsLoadDefaultCert()関数で発生するSCE_HTTP_SSL_ERRORからは詳細情報は取得できません。

sceHttpsSetSslCallback
 [関数]
SSL通信発生時のコールバック関数設定

形式

#include <libhttps.h>

int sceHttpsSetSslCallback(
	int id,
	int (*httpsCallback)(unsigned int, SceSslCert *const sslCert[],int,void* ),
	void *user_arg
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止/許可状態に依存しない）
マルチスレッドセーフである

引数

  id     対象となるテンプレート設定、コネクション設定のいずれかのID。  
  httpsCallback     コールバック関数へのポインタ  
  user_arg     ユーザが自由に設定してよい値。  
  コールバック関数が呼び出される時に設定した値が引数に使用される。  

返り値

正常終了の場合、SCE_OKを返します。

  値     マクロ     意味  
  0     SCE_OK     正常終了  
  0x80431100     SCE_HTTP_INVALID_ID     指定されたIDが不正  
  0x80431001     SCE_HTTP_BEFORE_INIT     ライブラリが初期化されていない  


解説

ssl通信が発生した時に呼び出されるコールバック関数をセットします。コールバック関数が設定されていない場合は、sceHttpsEnableOption(), sceHttpsDisableOption()で指定された動作で処理されます。
httpsCallbackにNULLを指定することで、コールバック関数を未設定の状態に戻すことができます。

備考

設定したコールバック関数は、HTTP通信処理関数を呼び出したスレッドのコンテキストで実行されます。HTTP Pipelineモードを使用した送受信を行わない限り、sceHttpSendRequest()もしくはsceHttpSendRequestInCacheFirstMode()を呼び出したスレッドのコンテキストで実行されます。
