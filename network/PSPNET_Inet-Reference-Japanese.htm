sceNetInetAccept
 [ソケットAPI]
TCPコネクションの確立が完了したソケットの取得

形式

#include <pspnet_inet.h>
#include <pspnet/sys/socket.h>

int sceNetInetAccept(
	int s,
	struct SceNetInetSockaddr *addr,
	SceNetInetSocklen_t *addrlen
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  s     ソケットディスクリプタ  
  addr     接続相手のアドレス構造体を格納する領域へのポインタ  
  addrlen     addrのアドレス構造体のサイズを格納する領域へのポインタ  

返り値

  値     結果  
  >0     コネクション確立済みのソケットディスクリプタ  
  -1     エラー  

エラーコードはsceNetInetGetErrno()またはsceNetInetGetPspError()で取得できます。

解説

ソケットsに対してTCPコネクションが確立した場合、sceNetInetAccept()を用いてコネクション確立済の新たなソケットディスクリプタを取得します。
addrlenには、addrで示されたポインタの領域のサイズを格納してから呼び出します。
正常終了時には、addrに接続相手のアドレス構造体が格納され、addrlenにそのアドレス構造体の大きさが格納されます。
ソケットが非ブロッキングでない場合、sceNetInetAccept()はコネクションが確立するまでブロックします。

使用例

struct SceNetInetSockaddrIn sin;
SceNetInetSocklen_t sockaddr_len;

/* TCP ソケットの生成 */
sid = sceNetInetSocket(SCE_NET_INET_AF_INET, SCE_NET_INET_SOCK_STREAM, 0);
if(sid < 0){
	/* エラー処理 */
}

/* ... バインド先の情報を sin にセット ... */

/* sin の内容で sid ソケットをバインド */
ret = sceNetInetBind(sid, (struct SceNetInetSockaddr *)&sin, sizeof(sin));
if(ret < 0){
	/* エラー処理 */
}

/* sid ソケットを、コネクション受け付け状態にする。backlog は 5 */
ret = sceNetInetListen(sid, 5);
if(ret < 0){
	/* エラー処理 */
}

while(1){
	/* 新規コネクション待ち */
	sockaddr_len = sizeof(sin);
	new_sid = sceNetInetAccept(sid, (struct SceNetInetSockaddr *)&sin, &sockaddr_len);
	if(new_sid < 0){
		/* エラー処理 */
	}

	/* ... 新規コネクション new_sid に対する処理 ... */
}
備考

エラー時にsceNetInetGetErrno()およびsceNetInetGetPspError()で取得される値は以下の通りです。

  sceNetInetGetPspError()     sceNetInetGetErrno()     説明  
  SCE_ERROR_ERRNO_EBADF     EBADF     ソケットディスクリプタが不正  
  SCE_ERROR_ERRNO_EINVAL     EINVAL     ソケットがコネクション  
  受け付け可能な状態  
  (LISTEN状態)ではない  
  SCE_ERROR_ERRNO_EOPNOTSUPP     EOPNOTSUPP     SCE_NET_INET_SOCK_STREAM  
  以外のソケットを指定した  
  SCE_ERROR_ERRNO_EAGAIN     EAGAIN     確立済コネクションが存在しない  
  (非ブロッキングの場合)  
  SCE_ERROR_ERRNO_EINTR     EINTR     sceNetInetSocketAbort()、  
  sceNetThreadAbort()によって  
  ブロックが解除された

sceNetInetBind
 [ソケットAPI]
ソケットへのアドレスのバインド

形式

#include <pspnet_inet.h>
#include <pspnet/sys/socket.h>

int sceNetInetBind(
	int s,
	const struct SceNetInetSockaddr *name,
	SceNetInetSocklen_t namelen
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  s     ソケットディスクリプタ  
  name     バインドするアドレス構造体へのポインタ  
  namelen     nameで示されたアドレス構造体のサイズ  

返り値

  値     結果  
  0     正常終了  
  -1     エラー  


エラーコードはsceNetInetGetErrno()またはsceNetInetGetPspError()で取得できます。

解説

ソケットsに対してnameで示されたローカルアドレス(IPアドレス・ポート番号)をバインドします。

使用例

#define LISTEN_PORT 80
struct SceNetInetSockaddrIn sin;

/* TCP ソケットの生成 */
sid = sceNetInetSocket(SCE_NET_INET_AF_INET, SCE_NET_INET_SOCK_STREAM, 0);
if(sid < 0){
	/* エラー処理 */
}

/* バインド先の情報を sin にセット */
memset(&sin, 0, sizeof(sin));
sin.sin_family = SCE_NET_INET_AF_INET;
sin.sin_addr.s_addr = SCE_NET_INET_INADDR_ANY;
sin.sin_port = sceNetHtons(LISTEN_PORT);

/* sin の内容で sid ソケットをバインド */
ret = sceNetInetBind(sid, (struct SceNetInetSockaddr *)&sin, sizeof(sin));
if(ret < 0){
	/* エラー処理 */
}
備考

エラー時にsceNetInetGetErrno()およびsceNetInetGetPspError()で取得される値は以下の通りです。

  sceNetInetGetPspError()     sceNetInetGetErrno()     説明  
  SCE_ERROR_ERRNO_EBADF     EBADF     ソケットディスクリプタが不正  
  SCE_ERROR_ERRNO_EADDRNOTAVAIL     EADDRNOTAVAIL     ローカルアドレスではない  
  SCE_ERROR_ERRNO_EADDRINUSE     EADDRINUSE     指定されたアドレスは使用中で  
  ある  
  SCE_ERROR_ERRNO_EINVAL     EINVAL     ソケットは既にバインドされて  
  いるソケットに指定された  
  アドレスファミリーと  
  name->sa_familyが等しくない  
  SCE_ERROR_ERRNO_ENOBUFS  
  SCE_ERROR_ERRNO_ENOMEM  		   ENOBUFS  
  ENOMEM  		   メモリ確保に失敗

sceNetInetSelect
 [ソケットAPI]
複数のソケットに対する同期I/O操作

形式

#include <pspnet_inet.h>
#include <pspnet/sys/select.h>

int sceNetInetSelect(
	int nfds,
	SceNetInetFdSet *readfds,
	SceNetInetFdSet *writefds,
	SceNetInetFdSet *exceptfds,
	struct SceNetInetTimeval *timeout
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  nfds     検査するソケットディスクリプタ番号の最大値+1  
  readfds     読み出し(受信)可能かどうかを検査する  
  ソケットディスクリプタセットへのポインタ  
  writefds     書き込み(送信)可能かどうかを検査する  
  ソケットディスクリプタセットへのポインタ  
  exceptfds     帯域外データが到着したかどうかを検査する  
  ソケットディスクリプタセットへのポインタ  
  timeout     タイムアウト時間を指定するtimeval構造体へのポインタ  

返り値

  値     結果  
  >0     ready状態になったディスクリプタの数  
  0     タイムアウト  
  -1     エラー  

エラーコードはsceNetInetGetErrno()またはsceNetInetGetPspError()で取得できます。

解説

readfds,writefds,exceptfdsで指定したそれぞれのソケットディスクリプタセットに対して、読み出し(受信)可能、書き込み(送信)可能、帯域外データの到着を検査し、ready状態になったソケットディスクリプタの数を返します。

ソケットディスクリプタセットは、以下のマクロを用いて作成します。

SceNetInetFdSet fds;		/* fdset 構造体 */
int s;				/* ソケットディスクリプタ */

SceNetInetFD_ZERO(&fds);	/* fdsの初期化 */

SceNetInetFD_SET(s, &fds);	/* fdsにsをセット */
SceNetInetFD_CLR(s, &fds);	/* fdsからsを外す */

本関数は、1つ以上のソケットディスクリプタがready状態になるか、もしくはtimeoutで指定された時間が経過するまでブロックします。
timeoutにNULLが指定された場合は、タイムアウトしません。
またtimeoutで指定された時間が0の場合は、ブロックせずに直ちに呼び出しから戻ります。

返り値が1以上の場合、readfds,writefds,exceptfdsにそれぞれの検査に対してready状態になったソケットディスクリプタがセットされます。これらのソケットディスクリプタセットに対して、SceNetInetFD_ISSET()マクロを用いてready状態になったソケットディスクリプタを判断します。

SceNetInetFD_ISSET(s, &fds); 	/* fdsにsがセットされている場合は0以外
				そうでない場合は0を返す */

使用例

int nfds;
SceNetInetFdSet readfds;
struct SceNetInetTimeval timeout;

/* readfds を初期化 */
SceNetInetFD_ZERO(&readfds);
/* readfds に sid1, sid2 をセット */
SceNetInetFD_SET(sid1, &readfds);
SceNetInetFD_SET(sid2, &readfds);

/* ... セットするソケットディスクリプタ中の最大値 + 1 を nfds にセットしておく ... */

/* sceNetInetSelect() のタイムアウト値を設定、ここでは10秒とする */
timeout.tv_sec  = 10;
timeout.tv_usec = 0;

/* sid1 もしくは sid2 の読み出し可能を待つ。タイムアウトは10秒 */
ret = sceNetInetSelect(nfds, &readfds, 0, 0, &timeout);
if(ret < 0){
	/* エラー処理 */
}
if(ret == 0){
	/* タイムアウト処理 */
}

/* sid1, sid2 のどちらかが読みだし可能 */
if(SceNetInetFD_ISSET(sid1, &readfds)){
	/* sid1 が読みだし可能 */
}
else if(SceNetInetFD_ISSET(sid2, &readfds)){
	/* sid2 が読みだし可能 */
}
備考

  sceNetInetGetPspError()     sceNetInetGetErrno()     説明

  プタセットが不正  
  SCE_ERROR_ERRNO_EINTR     EINTR     sceNetInetSocketAbort()、  
  sceNetThreadAbort()によって  
  ブロックが解除された  
  SCE_ERROR_ERRNO_EINVAL     EINVAL     タイムアウト時間が不正

sceNetInetShutdown
 [ソケットAPI]
TCPコネクションの細かなクローズ操作

形式

#include <pspnet_inet.h>
#include <pspnet/sys/socket.h>

int sceNetInetShutdown(
	int s,
	int how
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  s     ソケットディスクリプタ

返り値

  値     結果  
  0     正常終了  
  -1     エラー  

エラーコードはsceNetInetGetErrno()またはsceNetInetGetPspError()で取得できます。

解説

ソケットsのTCPコネクションのクローズ操作を行います。

SCE_NET_INET_SHUT_RDで読み出し側をクローズした場合、ソケットからの読み出しができなくなり、受信バッファのデータは破棄されます。またコネクションに到着するデータに対して、ACKは返されるがそのまま破棄されます。

SCE_NET_INET_SHUT_WRで書き込み側をクローズした場合、TCPコネクションはハーフクローズ(half close)の状態へ移行します。送信バッファに残っているデータは送信されますが、その後FINが送信されて終了処理が行われます。

SCE_NET_INET_SHUT_RDWRを指定した場合は、SCE_NET_INET_SHUT_RDを指定してsceNetInetShutdown()を呼び出してから、SCE_NET_INET_SHUT_WRを指定してsceNetInetShutdown()を呼び出すのと等価となります。

howの種類は以下の通りです。
  how     結果  
  SCE_NET_INET_SHUT_RD     読み出し側をクローズする  
  SCE_NET_INET_SHUT_WR     書き込み側をクローズする  
  SCE_NET_INET_SHUT_RDWR     読み出し側書き込み側両方をクローズする  


使用例

/* ソケット sid の読み出し、書き込み両方をクローズする */

if(ret < 0){
	/* エラー処理 */
}

/* ソケットディスクリプタを解放する */

if(ret < 0){
	/* エラー処理 */
}
備考

エラー時にsceNetInetGetErrno()およびsceNetInetGetPspError()で取得される値は以下の通りです。

  sceNetInetGetPspError()     sceNetInetGetErrno()     説明

SCE_NET_INET_SO_LINGER
 [ソケットオプション]
TCPコネクション終了処理の制御

データ型

struct SceNetInetLinger {
int l_onoff;
int l_linger;
}
値

l_onoff
  0     無効[default]  
  非0     有効(l_lingerを参照する)  

l_linger
  0     アボート  
  >0     リンガー時間の指定(10ミリ秒単位)  

解説

TCPコネクションの終了処理を制御します。ソケットタイプがSCE_NET_INET_SOCK_STREAM, SCE_NET_INET_SOCK_STREAM_P2Pの場合のみ有効です。

l_onoffが有効に設定されl_lingerが0の場合、TCPはクローズされたコネクションを直ちにアボートします。すなわち、送信バッファにあるデータを破棄し、RSTを送信します。

l_onoffが有効に設定されl_lingerが正数の場合、TCPはクローズされたコネクションに対してリンガー動作を行います。すなわち、送信バッファ中のデータをすべて送信して相手からACKを受信するか、リンガー時間経過するまで、待ち合わせ(ブロック)を行います。なお非ブロッキングソケットで待ち合わせ(ブロック)が必要な場合は、エラーとなりsceNetInetGetErrno()でEWOULDBLOCK（EAGAINと同値）が返ります。

備考

リンガー時間は32767(327.67秒)以下の値しか指定できません。

SCE_NET_INET_TCP_NODELAY
 [ソケットオプション]
TCPのNagleアルゴリズムの禁止

データ型

int
値

  0     Nagleアルゴリズムを使う[default]  
  非0     禁止  

解説

TCPのNagleアルゴリズムを禁止するかどうかを決定します。このオプションはソケットタイプがSCE_NET_INET_SOCK_STREAM, SCE_NET_INET_SOCK_STREAM_P2Pの場合のみ有効です。

Nagleのアルゴリズムでは、ACKを受け取っていない送信データがある場合、最大セグメントサイズ(MSS)より小さなデータを送信しません。
また受信データに対してACKを送信する場合、最大200ミリ秒待ってから送信する遅延ACKを行います。
したがって、小さなパケットを短い時間間隔で連続して送受信するようなアプリケーションの場合、Nagleのアルゴリズムによって応答時間が長くなるように感じられることがあります。

SCE_NET_INET_TCP_MSS_TO_ADVERTISE

unsigned short
値

解説

TCPコネクション確立時のSYNパケットに含める最大セグメントサイズ(MSS)オプション値の取得と設定を行います。このオプションはソケットタイプがSCE_NET_INET_SOCK_STREAM, SCE_NET_INET_SOCK_STREAM_P2Pの場合のみ有効です。

最大セグメントサイズとは、送信可能なTCPセグメントの最大値です。
