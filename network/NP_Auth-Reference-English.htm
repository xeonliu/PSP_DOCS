SceNpAuthRequestParameter
 [Structure]
Ticket request parameters

Structure

#include <np_auth.h>

#define SCE_NP_AUTH_LATEST_TICKET_VERSION_MAJOR     3
#define SCE_NP_AUTH_LATEST_TICKET_VERSION_MINOR     0
typedef 	struct SceNpAuthRequestParameter {
	SceSize size;
	SceNpTicketVersion version;
	const char *serviceId;
	const void *cookie;
	SceSize cookieSize;
	const char *entitlementId;
	SceUInt32 consumeCount;
	int (*ticketCb)(SceNpAuthRequestId, int, void *);
	void *cbArg;
} SceNpAuthRequestParameter;
Members

  size     Size of SceNpAuthRequestParameter structure (required)  
  version     Ticket version (required)  
  serviceId     Service ID (required)  
  cookie     Cookie (optional)  
  cookieSize     Cookie size (optional)  
  entitlementId     Entitlement ID for consumption processing (optional)  
  consumeCount     Consumed count (optional)  
  ticketCb     Ticket acquisition callback (required)  
  cbArg     User-specified callback argument (optional)  

Description

This structure is used for specifying parameters for the ticket to be acquired.
For a required item, be sure to specify a meaningful value. If an optional item is unnecessary, specify 0 or NULL.

For version, specify the version of the ticket to be obtained.

cookie and cookieSize are members for specifying data (cookie) that is added to the ticket. Any binary format data up to 1024 bytes long can be specified, and that data will be included as a cookie in a ticket that is issued from the NP server. If the ticket is sent to and used by an external server, the cookie can be used for any purpose such as session management between the application and external server.
If no cookie is needed, specify NULL for cookie and 0 for cookieSize.

entitlementId and consumedCount are members used to manage the consumption of a service entitlement for which the number of times the service entitlement can be used has been set. If these members are specified, the NP server checks the number of times the specified service entitlement can be used. If at least consumedCount available times remain, the NP server subtracts the consumedCount and issues a ticket that represents the state after this subtraction.
If no service entitlement needs to be consumed, specify NULL for entitlementId and 0 for consumedCount.

npauthhandler
 [Function]
Event handler prototype

Syntax

#include <np_auth.h>

int npauthhandler(
	SceNpAuthRequestId id,
	int result,
	void *arg
);
Arguments

  id     Request ID  
  result     Acquired result (Never 0; will always be positive or negative.)  
  arg     Passes the value of param->arg that was specified with the sceNpAuthCreateStartRequest() function.  

Description

This is an event handler for obtaining the result of the sceNpAuthCreateStartRequest() function. It is called from an internal thread. The gp value is set to the gp value of the module where the event handler that was set is located.
The ticket acquisition result is passed in result. If the ticket request was successful, a positive value representing the ticket size in bytes is passed in result. If the request failed, a negative value representing an error code is passed.
When a positive value is specified for result, the ticket can be obtained by calling sceNpAuthGetTicket() inside this API.
When result is a negative value, terminate this API immediately.
In either case, always specify 0 for the return value.

Example

static unsigned char *ticket=NULL;	// Buffer for getting ticket
static int ticket_len = 0;		// Ticket size
static int auth_result=0;		// Callback result

int
npauthhandler(SceNpAuthRequestId id, int result, void *arg)
{
	int ret = 0;
	(void)arg;

	// If an error occurs, result will be a negative value.
	if (result < 0) {
		auth_result = result;
		return 0;
	}
	
	// Get ticket.
	ticket_len = result;
	ticket = (unsigned char *)malloc(result);
	if (ticket == NULL) {
		auth_result = -1;
		return 0;
	}
	auth_result = sceNpAuthGetTicket(id, ticket, result);

	return 0;
}
Notes

Make sure that the event handler and a thread that calls an NP library API function will not be waiting for the same resource (such as sceKernelWaitSema() for the same semaphore). Otherwise, the thread that called the NP library API and the internal thread of the NP authentication library may get deadlocked.

See Also

SceNpAuthRequestParameter, sceNpAuthCreateStartRequest()

sceNpAuthGetEntitlementIdList
 [Function]
Get entitlement ID list

Syntax

#include <np_auth.h>

int sceNpAuthGetEntitlementIdList(
	const unsigned char *ticket,
	SceSize ticketSize,
	SceNpEntitlementId *entIdList,
	SceSize entIdListNum
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  ticket     Pointer to ticket data  
  ticketSize     Ticket data size  
  entIdList     Pointer to output array containing the entitlement ID list  
  entIdListNum     Number of elements in output entitlement ID list array  

Return Value

When the function completes normally, the number of entitlement IDs (non-negative value) contained in the ticket is returned.
If an error occurs, an error code is returned.

Description

This function counts the number of entitlements that are contained in the ticket specified by ticket and ticketSize and returns a list of the entitlement IDs.

This function is typically used as follows by calling it two times consecutively.
The first time it is called, NULL should be specified in entIdList. Since the return value will represent the total number of entitlements contained in the ticket, a buffer that can store at least that number of SceNpEntitlementIds should be prepared.
The second time this function is called, the address of that buffer should be specified in entIdList and the number of entitlement IDs that can be stored in that buffer should be specified in entIdListNum. This will store all entitlement IDs that are contained in the ticket in the buffer.

If the ticket contains more entitlements than the number specified by entIdListNum, only the specified number of entitlement IDs are stored in the buffer.
The maximum number of entitlements that can be contained in a ticket is 500 for ticket version 2.1 or 2500 for ticket version 3.0.

Example

unsigned char *ticket;
SceSize ticket_len, entIdListNum:
SceNpEntitlementId *entIdList;

// Assume ticket was obtained in buffer indicated by ticket and ticket_len

// If NULL is specified for the third argument, only the number of entitlements is returned
ret = sceNpAuthGetEntitlementIdList(ticket, ticket_len, NULL, 0);
if (ret < 0){
	// Error handling
}
entIdListNum = ret;

// Allocate required area for entIdList
entIdList = (SceNpEntitlementId *)malloc(entIdListNum * sizeof(SceNpEntitlementId));
if(entIdList == NULL){
	// Error handling
}

// Get list of entitlement IDs
ret = sceNpAuthGetEntitlementIdList(ticket, ticket_len, entIdList, entIdListNum);
if (ret < 0){
	// Error handling
}

Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  ticket     Pointer to ticket data  
  ticketSize     Ticket data size  
  entId     Target entitlement ID  
  ent     Entitlement information result output destination  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, an error code is returned.

Description

This function gets entitlement information for the entitlement specified by entId from the ticket specified by ticket and ticketSize and returns it in ent.
When NULL is specified for ent, this function only checks for the existence of entId in the ticket, and no entitlement information is returned. This is the method used to check the entitlement privilege, and sceNpAuthCheckEntitlementById() is provided as a special-purpose macro.

Example

unsigned char *ticket;
SceSize ticket_len:
SceNpEntitlement ent;
const char *entId;

// Assume ticket was obtained in buffer indicated by ticket and ticket_len 

// Assume target entitlement ID was stored in entID
ret = sceNpAuthGetEntitlementById(ticket, ticket_len, entId, &ent);
if (ret < 0){
	// Error handling
}

Description

struct SceNpMemoryStat stat;
