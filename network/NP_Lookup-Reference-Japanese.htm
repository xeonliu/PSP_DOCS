sceNpLookupCreateTransactionCtx
 [関数]
NPルックアップトランザクションコンテキストの作成

形式

#include <np/np_lookup.h>
int sceNpLookupCreateTransactionCtx(
	const SceNpCommunicationId *commId
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  commId     NPコミュニケーションIDへのポインタ  

返り値

正常終了の場合はNPルックアップトランザクションコンテキストのID(>0)を返します。
エラーの場合は負の値を返します。主なエラーコードを以下に示しますが、これ以外のエラーコードが返された場合でも破綻しないようにプログラミングしてください。

  マクロ     値     説明  
  SCE_NP_COMMUNITY_ERROR_NOT_INITIALIZED     0x80550702     np_serviceライブラリ初期化前  
  sceNpServiceInit()でnp_serviceライブラリ  
  初期化後に呼び出してください。  
  SCE_NP_COMMUNITY_ERROR_OUT_OF_MEMORY     0x80550703     メモリの空き容量が足りない  
  sceNpServiceInit()で指定した  
  メモリプールサイズを使用しています。  
  sceNpLookupDestroyTransactionCtx()の  
  呼び忘れがないか確認してください。  
  SCE_NP_COMMUNITY_ERROR_INSUFFICIENT_ARGUMENT     0x8055070c     commIdにNULLを指定した  
  sceNpLookupTitleSmallStorage()以外の通信  
  処理関数は正規のNPコミュニケーションIDを  
  指定しなくても動作しますが、運用管理のため  
  正式タイトルでNPルックアップライブラリの  
  機能を使用する場合は  
  PSP® Developer Networkから申請し  
  NPコミュニケーションIDを取得してください。  
  SCE_NP_COMMUNITY_ERROR_NO_LOGIN     0x80550705     NPにサインインしていない  


解説

NPルックアップライブラリで使用するトランザクションコンテキストを作成する関数です。
トランザクションコンテキストは、NP IDの取得やユーザプロファイルの取得などの通信処理関数1回ごとに使い捨てるコンテキストで、それらの関数を使用する前に必ず作成しておく必要があります。また、使用し終わったコンテキストはsceNpLookupDestroyTransactionCtx()で削除する必要があります。
sceNpServiceTerm()を呼ぶ前にすべてのトランザクションを明示的に削除してください。

sceNpLookupTitleSmallStorage
 [関数]
タイトルスモールストレージの取得（同期）

形式

#include <np/np_lookup.h>
int sceNpLookupTitleSmallStorage (
	int32_t transId,
	void *data,
	size_t maxSize,
	size_t *contentLength,
	void *option
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能(割り込み許可状態で呼ぶこと)
マルチスレッドセーフである

引数

  transId     NPルックアップトランザクションコンテキストID (IN)  
  data     タイトルスモールストレージデータを格納するためのメモリへのポインタ(OUT)  
  maxSize     dataで指定したメモリのサイズ。最大64KB (IN)  
  contentLength     dataに格納されたデータサイズ (OUT)  
  option     将来の拡張のためのオプション。必ずNULLを指定すること  

返り値

正常終了の場合は0を返します。
エラーの場合は負の値を返します。主なエラーコードを以下に示しますが、これ以外のエラーコードが返された場合でも破綻しないようにプログラミングしてください。

  マクロ     値     説明  
  SCE_NP_COMMUNITY_ERROR_NOT_INITIALIZED     0x80550702     np_serviceライブラリ初期化前  
  sceNpServiceInit()で  
  np_serviceライブラリ初期化後に  
  呼び出してください。  
  SCE_NP_COMMUNITY_ERROR_OUT_OF_MEMORY     0x80550703     メモリの空き容量が足りない  
  sceNpServiceInit()で指定した  
  メモリプールサイズを使用しています。  
  sceNpLookupDestroyTransactionCtx()  
  の呼び忘れがないか確認してください。  
  SCE_NP_COMMUNITY_ERROR_INVALID_ARGUMENT     0x80550704     optionにNULL以外を指定した  
  SCE_NP_COMMUNITY_ERROR_ABORTED     0x80550707     トランザクションが  
  sceNpLookupAbortTransaction()または  
  sceNpLookupDestroyTransactionCtx()  
  により中断された  
  SCE_NP_COMMUNITY_ERROR_BODY_TOO_LARGE     0x80550709     maxSizeで指定したサイズ以上の  
  データがサーバーに設置されていた  
  SCE_NP_COMMUNITY_ERROR_INSUFFICIENT_ARGUMENT     0x8055070c     dataにNULLを指定した  
  SCE_NP_COMMUNITY_ERROR_INVALID_ID     0x8055070e     transIdで指定したIDは存在しない  
  SCE_NP_COMMUNITY_ERROR_NO_LOGIN     0x80550705     NPにサインインしていない  


解説

NPコミュニケーションIDごとに設置できるデータ(最大64KBの1ファイル)をNPサーバーから取得するためのAPIです。サーバー上のデータがmaxSizeで指定したバイト数以下の時に、dataで指定したメモリにデータを格納しcontentLengthに取得したデータの総サイズを格納します。dataで指定するメモリはmaxSizeで指定したサイズ以上を必ず確保するようにしてください。
この関数は同期処理を行います。つまり、全データの取得が完了するまでブロックされます。この関数からリターンしたら、使用したトランザクションコンテキストをsceNpLookupDestroyTransactionCtx()を呼び出して破棄してください。

使用例

int32_t ret;
int32_t transId;
void *data=NULL;
size_t dataSize=0;

// transId,には適切な値が格納されているものとする
// 別スレッドでcellSysutilCheckCallback()がポーリングされているものとする

data = malloc(SCE_NET_NP_TSS_MAX_SIZE);
if (data == NULL){
	// エラー処理
}

ret = sceNpLookupTitleSmallStorage(transId, 
					data,
					SCE_NET_NP_TSS_MAX_SIZE,
					&dataSize,
					NULL);
If (ret < 0){
	// エラー処理
}
// 取得したデータの使用。バッファオーバーフローに注意
備考

サーバーにファイルが設置されていなかった場合は、0バイトのファイルが設置されていたものとして処理されます。具体的には、contentLengthに0が設定され正常終了として返り値に0が返ります。運用のミスによっても発生する状況ですので、この状況でも破綻しないようにプログラミングしてください。
