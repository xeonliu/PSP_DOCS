sceSslInit
 [関数]
libsslの初期化

形式

int sceSslInit (
	SceSize pool_size
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止/許可状態に依存しない)
マルチスレッドセーフでない

引数

  pool_size     SSL通信で使用するヒープメモリのサイズ  

返り値

  値     マクロ     意味  
  0     SCE_OK     正常終了  
  0x80435020     SCE_SSL_ALREADY_INITED     sceHttpEnd()を呼ぶことなく、  
  2回目のsceHttpInit()を呼んだ  
  0x804351FE     SCE_SSL_INVALID_VALUE     pool_sizeに0を指定した  


解説

libsslの初期化を行います。libhttpでhttpsの通信を行う前に呼び出しておく必要があります。pool_sizeで指定されたメモリサイズをSSL通信の利用するヒープサイズとして使用します。

使用例

#define SSL_POOLSIZE 	(150 * 1024U)
ret = sceSslInit(SSL_POOLSIZE);
if(ret < 0){
	/* エラー処理 */
}

sceSslGetSerialNumber
 [関数]
証明書のシリアルナンバー取得

形式

#include <libssl.h>
int sceSslGetSerialNumber (
	const SceSslCert *ssl_cert,
	const SceUChar8 **sbo_data,
	SceSize *sbo_len
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止/許可状態に依存しない)
マルチスレッドセーフである

引数

  ssl_cert     シリアルナンバーを取得したい証明書へのポインタを指定します。  
  証明書のポインタはsceHttpsSetSslCallback()で設定するコールバック関数で  
  取得されます。(IN)  
  sbo_data     シリアルナンバーの先頭アドレスを格納するためのメモリを指定します。  
  シリアルナンバーはビッグエンディアンで格納されます。(OUT)  
  sbo_len     シリアルナンバーの長さを格納するためのメモリのアドレスを指定します。(OUT)  

返り値

  値     マクロ     意味  
  0     SCE_OK     正常終了  
  0x80435001     SCE_SSL_BEFORE_INITED     ライブラリ初期化前  
  0x804351FE     SCE_SSL_INVALID_VALUE     ssl_cert, sbo_data, sbo_lenにNULLを  
  指定した  


解説

証明書オブジェクトのシリアルナンバーを取得するための関数です。証明書オブジェクトはHTTPライブラリのsceHttpsSetSslCallback()で設定するコールバック関数の引数として使用されます。シリアルナンバーが保存されているメモリ領域はコールバック関数の終了直後に開放されますので、保存が必要な場合はコールバック関数内で別メモリにコピーしてください。

sceSslGetSubjectName
 [関数]
証明書のサブジェクトネームを取得

形式

#include <libssl.h>
const SceSslCertName *sceSslGetSubjectName (
	const SceSslCert *ssl_cert
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止/許可状態に依存しない)
マルチスレッドセーフである

引数

  ssl_cert     サブジェクトネームを取得したい証明書へのポインタを指定します。  
  証明書のポインタはsceHttpsSetSslCallback()で設定するコールバック関数で  
  取得されます。(IN)  

返り値

正常終了のときSceSslCertNameオブジェクトへのポインタが返ります。
サブジェクトネームが取得できない時NULLが返ります。

解説

証明書オブジェクトのサブジェクトのネームオブジェクトを取得するための関数です。証明書オブジェクトはHTTPライブラリのsceHttpsSetSslCallback()で設定するコールバック関数の引数として取得されます。サブジェクトネームオブジェクトの詳細を取得するにはsceSslGetNameEntryCount()でサブジェクトネームエントリの数を取得した後にsceSslGetNameEntryInfo()を用いて各ネームエントリの情報を取得してください。

sceSslGetNameEntryInfo
 [関数]
ネームエントリの情報を取得

形式

#include <libssl.h>
int sceSslGetNameEntryInfo (
	const SceSslCertName *cert_name,
	int entry_num,
	const SceUChar8 **oidname,
	const SceUChar8 **value,
	SceSize *value_len
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止/許可状態に依存しない)
マルチスレッドセーフである

引数

  cert_name     エントリを取得するSSLネームオブジェクトのポインタを指定する。(IN)  
  entry_num     取得したいエントリの番号を指定する。SSLネームオブジェクトに格納されて  
  いるエントリの数はsceSslGetNameEntryCount()で取得できる。(IN)  
  oidname     エントリのオブジェクトIDを表すASCIZされた文字列。  
  NULLの時はUnknownなオブジェクトIDを意味する。(OUT)  
  value     エントリの値の先頭アドレスを格納するためのメモリを指定する。  
  ASCIZはされていない。(OUT)  
  value_len     valueのサイズを格納するためのメモリアドレスを指定。(OUT)  

返り値

正の値 cert_nameに含まれるネームエントリの個数。
  値     マクロ     意味  
  0     SCE_OK     正常終了  
  0x80435001     SCE_SSL_BEFORE_INITED     ライブラリ初期化前  
  0x804351FE     SCE_SSL_INVALID_VALUE     oid_name, value, value_lenにNULLを  
  指定した  


解説

SSLネームオブジェクトに含まれるネームエントリ数を取得するための関数です。SSLネームオブジェクトはsceSslGetSubjectName(), sceSslGetIssuerName()などで取得できます。当関数で取得した値が格納されているメモリはsceHttpsSetSslCallback()で設定したコールバック関数から抜けた直後に開放されるため、値を保存しておく場合はコールバック関数内で別メモリにコピーする必要があります。
