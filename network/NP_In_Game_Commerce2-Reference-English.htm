sceNpCommerce2CreateCtx
 [Context API]
Create a commerce 2 context

Syntax

#include <np/np_commerce2.h>
int sceNpCommerce2CreateCtx(
	SceUInt32 version,
	SceUInt32 *ctx_id
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  version     Version [IN]  
  ctx_id     Pointer to the area to store the context ID [OUT]  

Return Value

  Value     Description  
  0     Normal termination  
  Negative value     Error (See "List of Error Codes")  


Description

This function creates a commerce 2 context.
A commerce 2 context is required when calling a function that issues a request to the NP server. This means that a commerce 2 context is required when creating sessions, obtaining category content information or product information, and executing checkouts.

To version, specify SCE_NP_COMMERCE2_VERSION.

When this function ends normally, the context ID is returned to *ctx_id.

Examples

int ret;

ret = sceNpCommerce2CreateCtx(SCE_NP_COMMERCE2_VERSION, &ctx_id);
if ( ret < 0 ) {
    //Error handling
}
Notes

In the current specifications, the number of contexts that can be created at one time is one (SCE_NP_COMMERCE2_CTX_MAX).

sceNpCommerce2GetCategoryContentsGetResult
 [Category Content API]
Obtain category content data

Syntax

#include <np/np_commerce2.h>
int
sceNpCommerce2GetCategoryContentsGetResult(
	SceUInt32 req_id,
	void *buf,
	SceSize buf_size,
	SceSize *fill_size
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Aruments

  req_id     Request ID [IN]  
  buf     Buffer area to store the data obtained [IN]  
  buf_size     Size of area pointed to by buf [IN]  
  fill_size     Size of data obtained [OUT]  

Return Value

  Value     Description  
  0     Normal termination  
  Negative value     Error (See "List of Error Codes")  


Description

This function obtains category content data.
To req_id, specify the request ID specified in sceNpCommerce2GetCategoryContentsStart().
Create a memory area of SCE_NP_COMMERCE2_RECV_BUF_SIZE bytes, and specify its starting address to buf, and its size to buf_size.

This function is blocking. Note that it issues a request to the NP server and does not return until a response is received or an error occurs such as a timeout from libhttp, which is used internally. To abort this function's processing, use sceNpCommerce2AbortReq().

When this function ends normally, the size of the category content data obtained is stored to *fill_size. The category content data can then be taken out with a function such as sceNpCommerce2GetCategoryInfo(), but only after it is initialized with sceNpCommerce2InitGetCategoryContentsResult().

Warning
An account with a role that allows accesses with the NP IN-GAME commerce 2 library (such as TitleAdmin or TitleDev) is required during development on the development environment. It is also necessary to specify the source IP addresses. Without this setup, this function will return an SCE_NP_COMMERCE2_SERVER_ERROR_ACCESS_PERMISSION_DENIED or SCE_NP_COMMERCE2_SERVER_ERROR_UNKNOWN_ERROR error.

Notes

The data obtained with this function is of just the category content defined for the country/region to which the NP account belongs.
The available category content data can also be dependent on the product/SKU settings on the server, and eligibility rules based on past purchases. In principle, do not cache the category content data obtained (and the category content information taken out) locally, and instead obtain/take out the data when needed.

See Also

sceNpCommerce2GetCategoryContentsStart(), sceNpCommerce2AbortReq()

sceNpCommerce2GetContentInfo
 [Category Content API]
Take out information of content in the category

Syntax

#include <np/np_commerce2.h>

sceNpCommerce2GetContentInfo(
	const SceNpCommerce2GetCategoryContentsResult *result,
	unsigned int index,
	SceNpCommerce2ContentInfo *contentInfo
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  result     Pointer to initialized structure with category content data [IN]

  Value     Description

Description

This function takes out information of one content in the category (one subcategory or product) from the initialized category content data.
To index, specify the index number of the content to take out. To take out the first content, specify 0.

When this function ends normally, the specified content information is stored to *contentInfo.

Information of products can be taken out with sceNpCommerce2GetGameProductInfoFromContentInfo(). (This includes only partial product information.)

Notes

Description

This function obtains product data.
To req_id, specify the request ID specified in sceNpCommerce2GetProductInfoStart().
Create a memory area of SCE_NP_COMMERCE2_RECV_BUF_SIZE bytes, and specify its starting address to buf, and its size to buf_size.

This function is blocking. Note that it issues a request to the NP server and does not return until a response is received or an error occurs such as a timeout from libhttp, which is used internally. To abort this function's processing, use sceNpCommerce2AbortReq().

An account with a role that allows accesses with the NP IN-GAME commerce 2 library (such as TitleAdmin or TitleDev) is required during development on the development environment. It is also necessary to specify the source IP addresses. Without this setup, this function will return an SCE_NP_COMMERCE2_SERVER_ERROR_ACCESS_PERMISSION_DENIED or SCE_NP_COMMERCE2_SERVER_ERROR_UNKNOWN_ERROR error.

Notes

The data obtained with this function is of just the products defined for the country/region to which the NP account belongs.
The available product data can also be dependent on the product/SKU settings on the server, and eligibility rules based on past purchases. In principle, do not cache the product data obtained (and the product information taken out) locally, and instead obtain/take out the data when needed.

However, when product information is taken out with sceNpCommerce2GetGameProductInfoFromContentInfo(), SKU information is available only if there is one SKU in the product. This SKU information has dataType set to SCE_NP_COMMERCE2_GAME_SKU_DATA_TYPE_THIN.

sceNpCommerce2GetPrice
 [Other API]
Get price string

Syntax

#include <np/np_commerce2.h>
int sceNpCommerce2GetPrice(
	const SceNpCommerce2SessionInfo *s,
	char *buf,
	SceSize buflen,
	SceUInt32 price
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in interrupt-enabled state)
Multithread safe

Arguments

  s     Session information structure [IN]  
  buf     Buffer area for storing price string that is obtained [IN]  
  buflen     Size of area pointed to by buf [IN]  
  price     Price obtained from SKU information structure [IN]  

Return Value

  Value     Description  
  0     Normal termination  
  Negative value     Error (see "List of Error Codes")  


Description

Although the price member of the SKU information structure indicates the price, generally when the price is displayed, certain aspects of its representation, such as whether the currency symbol is in front of or behind the price or whether the decimal point is a comma or period, differ for each country or region.
This function provides a price representation that accounts for each of these country and regional differences. The price representation string that is obtained here can be used directly for display by the application.
Since this function assumes that account information for each country or region is obtained from the server when a session is created, a session must be created before calling this function.

Notes

It is recommended that this function be used to convert a price (numerical value) to a price format (string).
Use it to display a price for the user.
