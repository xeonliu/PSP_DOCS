PlayStation®Network Communication Service Overview

1 Overview of the Communication Service Functions

The PlayStation®Network ("NP") provides the following functions, which support gameplay among users on the NP.
  • Friend list and block list
  • Matchmaking functions and P2P communication functions for realizing gameplay among users on the NP

1. 1 Friend List and Block List

An NP user can register up to 50 other users as his/her friends. Registration requires the authorization of both parties.
An NP user can also register up to 50 other users in a block list for which friend registration requests are to be blocked.

Note
If the same account is used, the same friend list and block list will be used on the PlayStation®3 system.

1. 2 Profile

Each individual PlayStation®Network user has a personal profile. The profile contains information such as the player's avatar, self introduction, frequently used languages, and so on.

1. 3 Matching 2

Matching 2 functions support applications that implement online gameplay.
A user can create a session called a room, and search for a session created by another user and join it. Up to 64 users can join a room. Various types of information are attached to a session and its users. The application can reference and manipulate this information.
Information about a user joining or leaving a session is notified to the other users in the session. Users in the same session can also exchange messages.

1. 4 Signaling

Signaling realizes P2P communication in network game play. It provides functionalities such as NAT traversal functions, and functions to obtain connection information (including the IP address and port number used). For the communication protocol, the UDPP2P (extension of the UDP), and TCP over UDPP2P for realizing P2P communication with the TCP, are provided. These protocols also support secure communication by the use of encryptions and signature verifications.
Signaling is provided as part of the matching 2 functions.

1. 5 Title Small Storage

Title small storage is a server storage service that enables application-specific data to be placed on a server and referenced using scp.

2 NP Communication ID

The NP Communication ID is an ID for identifying games within the communication services of the NP. This NP Communication ID must be obtained prior to beginning title development. Store it in the SceNpCommunicationId structure and use it when calling library APIs.
Basically, one NP Communication ID is used per title. However, the following usage is also possible.
  • Use multiple NP Communication IDs in one title, and separately manage matchmaking per retail area by assigning one NP Communication ID to each retail area

Format
The format of the NP Communication ID is as follows.
(4 letters of the alphabet) + (5 numbers) + "_" (underscore) + (2 numbers)
Example: "ABCD12345_00"

3 Libraries Related to Communication Services

The libraries that are currently provided to enable the application to use communication services are as follows.
For details on the functions, refer to the overview and reference documents of each library.

3. 1 NP Roster Library

This library provides friend list and block list related functions.
  • Get friend list
  • Send message requesting friend registration
  • Reject friend registration request and remove friend
  • Approve friend registration request
  • Get block list
  • Add to block list
  • Remove from block list

3. 2 NP Matching 2 Utility

This system utility provides functions related to matchmaking.
  • Create room
  • Search room
  • Create room dynamically
  • Perform in-room text chat
    Vulgarity filter (to filter inappropriate language) can be applied
  • Exchange data between users in room
  • Restrict room entrance
  • Support gameplay
  • Search for a user using the Online ID
  • Establish P2P connection between room members
  • Get IP address and port number of room members

3. 3 NP Lookup Library

The NP lookup library provides functions related to profiles and title small storage.
  • Get the NP ID for a specific online ID
  • Get profile information for a specific user
  • Reference title small storage

4 Overview of the Application Development Process

The following steps involved in developing an application that uses the PlayStation®Network communication services are described in the subsequent sections.
  1. Get an NP Communication ID and register server settings
  2. Application Development and use of the PlayStation®Network server management tools
  3. Register server settings to the PlayStation®Network QA server
  4. Manage operation of the production environment server
For each development step, refer to the use case provided.

4. 1 Get an NP Communication ID and Register Server Settings

Role of the NP Communication ID
The NP Communication ID is used in PlayStation®Network communication services for identifying titles. The libraries that use the NP Communication ID are as follows.
  • NP Matching 2 library
  • NP Lookup library
These libraries use the NP Communication ID to guarantee accesses to the same server. Thus, when the communication services are to be used via these libraries, a unique NP Communication ID must basically be obtained and used per title.

Using an NP Communication ID that Is not Unique
When there is a particular reason to do so, a single title can use multiple NP Communication IDs, and multiple titles may use a single NP Communication ID.

Release the Same Title in Multiple Regions

Use a common NP Communication ID throughout all the regions to realize matching worldwide. To realize matching per region, use a different NP Communication ID in each region.

4. 2 Services that Require Server Settings

Some PlayStation®Network communication services require server settings to be made according to the specifications of each title. Register server settings to use the following libraries.
  • NP matching 2 library
  • NP lookup library (title small storage)

PlayStation®Network Communication Service Request
Select "PlayStation®Network communication service request" on the PlayStation®Network title registration page of the PSP™ Developer Network website. For a new registration, select "NP Communication ID has not been obtained" and enter the required information. The information required in this process is as follows.
  • Request type
  • Your company name
  • Title name
  • Publisher name
  • Contact information (contact person in your company and his/her email address)
  • Release date
  • Target region for your title
  • NP Communication ID
  • Services to be used
  • Matching 2 (including the setting for the number of worlds that will be used)
  • Title small storage
These settings are registered for each title and NP Communication ID. The completion of the server settings will be notified via e-mail to the registered contact person. This completes the preparation process required for developing an application that uses PlayStation®Network communication services.

To use the same NP Communication ID with multiple titles

To use a previously obtained NP Communication ID with another title, enter the information related to the new title in a similar manner as for a new registration and also enter the previously obtained NP Communication ID to be used. In the additional comment field, enter a comment indicating that this is a shared registration.

To change or add information after a new registration

To modify the title name, add a region, add a contact e-mail address, or change or add the service to be used and server settings, select "PlayStation®Network communication service request" on the PlayStation®Network title registration page of the PSP™ Developer Network website, select "NP Communication ID has not been obtained," and then enter the relevant NP Communication ID.

4. 3 Application Development and Use of the PlayStation®Network Server Management Tools

PlayStation®Network Server Management Tools
The PlayStation®Network Server Management Tools (SMT) is a group of web-based tools that supports the development of applications that use PlayStation®Network communication services. Tools corresponding to the following libraries are provided.
  • NP matching 2 utility
  • NP lookup library (title small storage)
Detailed specification settings can be made via the SMT for each service - including matching settings.
A function is also provided to test application operation: for example, server settings can be changed, and the registered data can be checked. For matching, a dummy room can be created and a room list can be obtained.

Note

For instructions on operating the SMT, refer to the "PlayStation®Network SMT Portal User's Guide" and the user guide provided for each tool. You can view and obtain these documents from the "PlayStation®Network Server Management Tools Users Guide" on the PlayStation®Network Server Management Tools page of the PSP™ Developer Network website.

Using the PlayStation®Network Server Management Tools
Select "Enter PlayStation®Network server management site" on the PlayStation®Network Server Management Tools page of the PSP™ Developer Network website, and the Server Management Tools Portal Menu will be displayed in a separate window. This menu is divided into "NP Communication ID Tools" and "Title ID Tools." Select "NP Communication ID Tools" when using communication services such as matching or title small storage.

Select Tool

This menu is used for selecting a tool. The NP Communication ID, Environment, and Tools which can be used with the currently logged in PSP™ Developer Network account are displayed. Select the desired operation to be performed.

Privilege Management

This menu is used for managing access privileges. Management privileges for the registered NP Communication ID and services are assigned to the PSP™ Developer Network account of the user who registered the server settings, and that person can assign management privileges or access privileges to other users (users in the same group on the PSP™ Developer Network).

Note

To grant privileges to another group on the PSP™ Developer Network or to a specific account that belongs to another group, contact the technical support team of SCE and provide them with your PSP™ Developer Network account information.

4. 4 Registering Server Settings to the PlayStation®Network QA Server

For title QA, the development server settings used in the communication services and set by the SMT must be reflected onto the QA server. Make sure to obtain these settings and to submit them for their registration onto the QA server at least one week before the master disc submittal.

Submit Server Settings for Registration to the QA Server
On the PlayStation®Network title registration page of the PSP™ Developer Network website, choose "Register PlayStation®Network QA Server". The information required in this process is as follows.
  • Date of QA master disc submittal
  • Title ID
  • Your company name
  • Title name
  • Publisher name
  • Contact information (contact person in your company and his/her email address)
  • Scheduled title release date
  • Target region for your title
  • NP Communication ID
  • Services used
  • Matching 2
When server settings are registered to the QA server, some of the settings will be locked and you will be unable to make subsequent changes to them using the SMT. Make sure to complete all the necessary changes to the server settings before submitting them to be registered onto the QA server.

Note

To change server settings after they were registered to the QA server, submit a request to unlock them. Submit a "PlayStation®Network communication service request" with a comment entered in the additional comment field indicating that the settings are to be unlocked.

Notes on Submitting Server Settings to be Registered onto the QA Server

Releasing the Same Title in Multiple Regions

If a common NP Communication ID is used in multiple regions, only perform QA server registration once. There is no need to separately submit server settings for each region's master disc submittal.

Resubmitting a Master Disc

When changing server settings using the SMT after they have been submitted once for master disc submittal, make sure you resubmit them to be registered onto the QA server again. Describe the changes you made in detail.
If server settings have not been changed, there is no need to resubmit them for QA server registration.

Using Title Small Storage

After accessing the TSSS upload server for development/QA and placing the required files in the QA directory, submit a request to register them to the QA server. For files that must be placed by the title release date, also access the public TSSS upload server and locate the required files by the required time. If it is difficult to place the files on the production environment server by this time, describe in detail the symptoms that will occur on the production environment server if these files have not been located there in the additional comment field of the QA server registration request. The verification of the task of reflecting these files on the production environment server will be adjusted based on that description.

4. 5 Managing Operation of the Production Environment Server

Once the master disc QA is completed, SCE will perform the operation to reflect the settings of the server onto the production environment server (the server for end users).
SCE basically manages the operation of this server. If you have any requests, contact the technical support team of SCE.

Using Title Small Storage
SCE does not reflect title small storage on the production environment server. At the required time, you should access the public TSSS upload server and upload the required files.
