PSP™ Programmer Tool Runtime Library Release 6.3.0

Game Sharing Authoring Procedure

1 About this Document

This document explains how to use the authoring server which is provided for programs which plan to implement game sharing.

2 Accessing the Authoring Server

2. 1 Access Procedure

In order to access the authoring server, it is first necessary to register an account, register the IP address which will normally be used to originate access to the server, and obtain a user ID and initial password. Those who intend to develop titles which will include game sharing functionality are asked to please contact Sony Computer Entertainment Inc.
The authoring server is accessed from a Web browser. In order to ensure security, connections are only allowed via SSL, and at the application level, authentication is performed on each individual user.

2. 4 Format of Files to Upload

Archive the data to be uploaded to the authoring server in tgz format.
The .tgz file should contain the following files:
  • A file to be authorized (with .prx extension) (not case-sensitive)
  • PARAM.SFO (not case-sensitive)
Also, by including an empty file named COMPRESS (all uppercase letters), the compression function will be enabled and the size of the authorized prx file can be reduced. When the compression function is enabled, the maximum size of the prx file to be uploaded is 14,680,064 bytes, and the maximum size of the generated file will be 8,388,608 bytes.

Upload file creation example


% touch COMPRESS
% tar cvzf upload.tar.gz  plain.prx  param.sfo  COMPRESS

Create the PARAM.SFO file by using the system file editor tool and specifying the WLANGame_SystemFile_Config.sfc file as the config file. For details, refer to the system file editor tool documentation.

2. 5 Format of Downloaded Files

The data downloaded from the authoring server is archived in tgz format.
The file structure is as follows.
  • Authorized file ([original filename]_authorized.prx
  • PARAM.SFO

3 Authoring Procedure

3. 1 Login

Upon successfully connecting to an authoring server, the login screen for user authentication opens.



Figure 1 Login Screen


Enter the previously registered user ID and password, and press the "Login" button.

3. 2 Main Menu

Upon successfully logging in, the main menu screen opens.



Figure 2 Main Menu


(1) Overall Screen Structure of the Application

Screens are basically divided into a "common header area" and a "content area."

Common Header Area

In the common header area, there is a link to the main menu (an icon shaped like a home), a link for logging out, a page title, and the username (which has a link to the Change Password screen).

Content Area

Different information is displayed for each individual page.

On the main menu screen, a list of categories for which authoring is possible is displayed in the upper part of the content area, and contact information for Sony Computer Entertainment Inc. is displayed in the lower part of the content area.
To perform game sharing authoring, select "Game Sharing" as the Category, and click the "Open List" link.

3. 3 Changing the Password

The Change Password screen will appear during login if the password has expired, or when the username in the common header area has been clicked.



Figure 3 Change Password Screen


The password can be changed by entering the current password in the "Password (Current)" field and the new password in the "Password (New)" and "Password (New Confirm)" fields, and then pressing the "Change Password" button.
In addition, passwords have an expiration date. When an attempt is made to log in using a password whose expiration date has passed, the Change Password screen will automatically appear. When this happens, change to a new password.

3. 4 Listing Files

When "Open List" on the Main Menu is clicked, the File List screen opens.



Figure 4 File List Screen


An input form for filtering the displayed files is displayed in the upper part of the content area, and a list of the files currently uploaded onto the server is displayed in the lower part of the content area.
Also, in the upper left of the input form are a "Back" link to return to the Main Menu, an "Upload" link to perform a file upload, and a "View authoring log" link to check the past authoring history. In the upper right are displayed the size of the currently uploaded files, the maximum size which can be uploaded to the server, and the percentage which is currently being used.
In order to perform game sharing authoring, the tgz file which has been created for this purpose must first be uploaded. Click the "Upload" link in the upper left of the screen.

Note: A "Status" display was added to the File List screen. This field changes according to the operation on the File Upload or File Conversion screen. Be sure to check the meaning of the "Status" field and the "Status" transitions as shown below. An automatic e-mail notification is sent for those steps marked with an asterisk (*).

  Status Field Display     Meaning  
  (Empty field)     When a file was uploaded  
  Requested     When a production authoring request was made  
  Rejected (*)     When a production authoring request was rejected by person in charge at SCE  
  Approved     When a production authoring request was approved by person in charge at SCE  
  Completed (*)     When a production authoring request was performed by person in charge at SCE  
  Canceled     When a request was canceled after a production authoring request was made  


Note: "Status" transitions


3. 5 Uploading Files

When the "Upload" link on the File List screen is clicked, the File Upload screen opens.



Figure 5 File Upload Screen


The authoring category to which the upload is about to be performed is displayed, along with its description, and a file input field for specifying the file to upload, along with an "Upload" button and a "Cancel" button.
When the "Browse" button to the right of the file input field is clicked, the File Selection dialog opens. From here, select the file to upload. When the compression function is enabled by including the COMPRESS file with the file to be uploaded, an error will occur if the size of the prx file to be uploaded exceeds 14,680,064 bytes.



Figure 6 File Selection Dialog


The file can now be uploaded by pressing the "Upload" button. During the upload, a pop-up such as that shown below is displayed, and the progress of the operation, along with messages explaining any errors and the like that occur, are displayed.



Figure 7 Progress Display Window


When the operation is complete, control returns to the File List screen. Close the pop-up window by clicking its "Close" button.

3. 6 Converting Files

After a file has been uploaded, it is displayed on the File List screen. If there are too many files to display all at once, either use the "Next" and "Prev" page navigation links or, alternatively, enter search criteria and perform filtering.



Figure 8 File List Screen (2)


On the left of the uploaded file list are "Convert" and "Delete" links for performing conversion and deletion operations, respectively. Click the "Convert" link in order to perform game sharing authoring. If the compression function is enabled, an error will occur if the size of the authorized prx file exceeds 8,388,608 bytes.




Figure 9 File Conversion Screen


The types of authoring which can be performed on the file to be converted are displayed in a type list. To perform authoring, click the "Process" button for the desired type. For game sharing authoring, the three types of authoring shown below can be selected.

  Authoring Type     Processing Details     Processing Procedure  
  Game Sharing Authoring for TOOL Only     Used to create a game sharing binary which can only be executed by the PSP™ development tool. This is supposed to be used for the purpose of debugging during development.     Press the "Process" button, and the same progress display pop-up window as for uploading will open. Wait for processing to complete, and then close the pop-up window by pressing its "Close" button.  
  Request for Production Authoring     Requests authoring for use in production software which will also run on the PSP™ console.     Press the "Process" button, and a pop-up window will appear to confirm the request. Press "Yes" or "No".  
  Cancel the request for Production Authoring  
  (added)  		   A request was made to cancel the requested production authoring processing.     When the Process button is pressed, a popup window appears to confirm whether or not this processing should actually be canceled. Press "Yes" or "No."  

3. 7 Downloading Files

When conversion is complete, the third file list screen is displayed.



Figure 10 File List Screen (3)


The "Plain", "TOOL Only" and "Production" columns which appear on the right side of the file list respectively correspond to "the original uploaded file," "the file which has been authorized and is in a format which can be sent to PSP™ system software on the PSP™ development tool," and "the file which has been authorized and is in a format which can be sent to the PSP™ console."
To download an authorized file which has been created via file conversion, click the "View Detail" link in either the "TOOL Only" column or the "Production" column.



Figure 11 File Detail Screen


The file size, creation date and time, etc. will be displayed; if these are correct, begin the download by pressing the "Download" button.



Figure 12 File Download (1)




Figure 13 File Download (2)


3. 8 Deleting Files

Files which are present on the server can be deleted by clicking the "Delete" link on the File List screen.
Note that when the "Status" on the File List screen is either "Requested" or "Approved", the file cannot be deleted. Perform file deletion after canceling the production authoring request.



Figure 14 File Deletion Confirmation Screen


3. 9 About the Maximum File Size Which can be Stored on the Server

An upper limit is set for the size of files which can be stored on the server, and once the capacity is reached, it will not be possible to perform any further uploading or authoring.
The amount of storage space which is currently being used is displayed in the upper right of the File List screen.
In order to conserve server resources, and from the perspective of security as well, it is recommended that, to the extent possible, files for which processing has ended be promptly deleted.

3. 10 Checking Authoring History

When the "View Authoring Log" link is clicked from the File List screen, the Authoring History Check screen opens.
