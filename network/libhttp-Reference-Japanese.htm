httpAuthInfoCallback
 [関数]
認証時のパスワード取得コールバック関数

形式

#include <libhttp.h>

int httpAuthInfoCallback(
	int request,
	SceHttpAuthType auth_type,
	const char *realm,
	char *username,
	char *password,
	SceBool need_entity,
	unsigned char **entity_body,
	SceSize *entity_size,
	SceBool *save,
	void *user_arg
);
引数


  request     呼び出し元のリクエストオブジェクトのID（IN）  
  auth_type     認証種別（IN）  
  SCE_HTTP_AUTH_BASIC        BASIC認証  
  SCE_HTTP_AUTH_DIGEST    DIGEST認証  
  realm     認証する空間を識別するために、サーバーに指定された文字列が格納される（IN）  
  username     保存されたユーザ名（IN）  
  入力されたユーザ名を格納するための領域（OUT）  
  SCE_HTTP_USERNAME_MAX_SIZEバイトを超えて設定してはいけない。  
  password     保存されたパスワード（IN）  
  入力されたパスワードを格納するための領域（OUT）  
  SCE_HTTP_PASSWORD_MAX_SIZEバイトを超えて設定してはいけない。  
  need_entity     entity_bodyが必要か否かを指定するフラグ（IN）  
  SCE_TRUE    entity_bodyが必要  
  SCE_FALSE    entity_bodyは不要(設定した場合、無視される)  
  entity_body     entity_bodyへのポインタを格納するためのメモリのアドレス（OUT）  
  usernameやpasswordと違いアプリケーションがメモリを確保する  
  必要がある事に注意。通常は指定する必要はありません。  
  entity_size     entity_bodyのサイズを格納するためのメモリへのポインタ（OUT）  
  通常は指定する必要はありません。  
  save     ユーザ名およびパスワードを保存するためのフラグ（OUT）  
  現在はSCE_FALSEを設定してください。  
  user_arg     sceHttpSetAuthInfoCallback()で指定したユーザ指定のポインタ（IN）  

返り値

0以上　認証を行う
負の値　認証を行わない

解説

BASIC／DIGEST認証時に呼び出されるコールバック関数の型定義です。
基本的にはusernameとpasswordにユーザー名とパスワードを格納し、返り値に0以上の値を返すことで認証に必要なヘッダが作成、送信されます。負の値を返したときは認証は行われません。
保存されたユーザ名およびパスワードがある場合は、usernameおよびpasswordに値が格納された状態で呼び出されます。

備考

need_entityにSCE_TRUEが指定されているときは、送信する予定のentity_bodyを設定する必要があります。これは、POSTメソッドでDIGEST認証を行い、かつサーバーがentity_bodyの情報を使用する種類の認証ヘッダしか受け付けない時のみに発生します。その場合、POSTで送信する予定のデータ全体が格納されたメモリの先頭アドレスをentity_bodyに、そのサイズをentity_sizeに設定してください。設定したメモリはrequestのsceHttpSendRequest()が終了するか、再び同じrequestで当関数が呼ばれるまで開放してはいけません。

httpRedirectCallback
 [関数]
リダイレクト時のコールバック関数

形式

#include <libhttp.h>

int httpRedirectCallback(
	int request,
	int status_code,
	int *method,
	const char *location,
	void *user_arg
);
引数


  request     呼び出し元のリクエストオブジェクトのID（IN）  
  status_code     リダイレクトのステータスコード.3xx等(IN)  
  method     リクエストオブジェクトに指定されているメソッド(IN/OUT)  
  location     リダイレクト先のLocationを表すASCIIZされた文字列が格納される(IN)  
  user_arg     sceHttpSetRedirectCallback()で指定したユーザ指定のポインタが格納される。  
  使用しなくてもよい(IN/OUT)  

返り値

0以上　リダイレクトを許可
負の値　リダイレクト不許可

解説

リダイレクト発生時に呼び出されるコールバック関数の型定義です。
request, status_code, method, locationの情報からリダイレクトしてもよいなら正の値、リダイレクトしてはいけないときは負の値を返して下さい。methodにはSceHttpMethodsで定義された値が格納されます。methodを書き換える事でリダイレクト後のメソッドを書き換える事ができます。ただしGET/HEADメソッドからPOSTメソッドに書き換えた場合リクエストボディは常に0バイトとなります。

備考

locationのメモリはコールバック関数を抜けた直後に開放されます。もし保持し続ける必要がある場合はコールバック関数内で別メモリにコピーしておく必要があります。またlibhttpをマルチスレッドで使用する場合は、当関数もマルチスレッドセーフである必要があります。

関連項目

sceHttpEnableRedirect(), sceHttpDisableRedirect(), sceHttpSetRedirectCallback()

sceHttpSetMallocFunction
 [関数]
ユーザ定義のメモリ管理関数の設定

形式

#include <libhttp.h>
int sceHttpSetMallocFunction(
	void *(*httpMallocFunction)(SceSize),
	void *(httpFreeFunction)(void *),
	void *(httpReallocFunction)(void*, SceSize)
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能

引数

  httpMallocFunction     ユーザ定義のmalloc関数  
  httpFreeFunction     ユーザ定義のfree関数  
  httpReallocFunction     ユーザ定義のrealloc関数(未使用)  

返り値

正常終了の場合、0が返ります。エラーの場合、負の値が返ります。

  値     マクロ     意味  
  0     SCE_OK     正常終了  
  0x804311FE     SCE_HTTP_INVALID_VALUE     指定された関数ポインタが不正  
  0x80431021     SCE_HTTP_BUSY     既にライブラリが使用開始されている

引数として、ユーザ定義のメモリ管理関数malloc、free、realloc関数のポインタを指定します。
ただし、現在libhttp内ではreallocは使用していないため、reallocはダミー関数でかまいません。

備考

本関数はsceHttpInit()の直後でのみ、呼ぶことが可能です。ライブラリを既に使用開始している状態では内部ヒープを使用しているため、外部のメモリ管理機能を使用することはできません。
また設定するmalloc関数はディスパッチ許可/不許可状態に関わらず使用できる必要があります。設定したコールバック関数はlibhttpの各関数を呼び出したスレッドのコンテキストで実行されます。

備考

HTTPの基本設定(以後テンプレート設定と呼びます)を作成します。libhttpでは設定をテンプレート設定、コネクション設定、リクエスト設定の3段階に分かれており、それぞれアクセス先のサーバーに依存しにくい部分の設定、アクセス先のサーバーごとの設定、リクエスト毎の設定という分類になっています。詳細は「HTTPライブラリOverveiw」をご参照ください。

  ステータスコード  		   ライブラリの挙動

  リダイレクトします。新たなURIはライブラリ内でキャッシュされ  
  今後同じURIを指定された時、自動的に新しいURIにアクセスします。  
  またlibhttp内部で新たなコネクション設定が作られます。  
  このコネクション設定はリダイレクトが発生した時に使用した

  となり、元のメソッドがPOSTであってもリクエストボディは使用され  
  ません。また、新たなURIはライブラリ内でキャッシュされません。  
  305     プロキシを使用していない状態で305が返った時、ライブラリはLocation  
  に指定されたProxyを利用し対象のURLへ再度アクセスします。  
  この情報はライブラリ内部でキャッシュされません。  
  307     302が返ったときと同様の挙動をします。
