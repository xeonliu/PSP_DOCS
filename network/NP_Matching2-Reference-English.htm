sceNpMatching2Init
 [Function]
Initialize the utility

Syntax

#include <np_matching2.h>

int sceNpMatching2Init(
	SceSize poolsize,
	int matching2_priority,
	SceSize matching2_stackSize,
	int signaling_priority
);
Calling Conditions

Cannot be called from an interrupt handler.
Can be called from a thread (must be called in an interrupt-enabled state).
Not multithread safe.

Arguments

  poolsize     Matching 2 internal memory pool size  
  matching2_priority     Priority of matching 2 internal thread  
  matching2_stackSize     Stack size of matching 2 internal thread  
  signaling_priority     Signaling thread priority  

Return Values

Returns 0 for normal termination.
Returns a negative value for errors. The main error codes are shown below. Note, however, that the application must not malfunction even if other error codes are returned.
  Macro     Value     Description  
  SCE_NP_MATCHING2_ERROR_ALREADY_INITIALIZED     0x80550c01     Already initialized  
  sceNpMatching2Init() may have already   
  been executed. Check the calling order.  
  SCE_NP_MATCHING2_ERROR_STACKSIZE_TOO_SHORT     0x80550ccb     Stack size of specified matching 2 internal   
  thread is too short. Specify a value greater  
  than or equal to SCE_NP_MATCHING2_INTERNAL_THREAD_LEAST_STACK_SIZE  
  for matching2_stackSize.  


Description

This function initializes the NP matching 2 utility. An internal thread of the NP matching 2 utility is created upon initialization. Specify the stack size and priority of this internal thread for matching2_stackSize and matching2_priority, respectively. Specify the memory pool size to be used for matching for poolsize. Specify the priority of the signaling thread for signaling_priority.

Examples

sceNpMatching2CreateContext
 [Function]
Create a context

Syntax

#include <np_matching2.h>

int sceNpMatching2CreateContext(
	const SceNpCommunicationId *commId,
	const SceNpCommunicationPassphrase *passPhrase,
	SceNpMatching2ContextId *ctxId,
	int option
);
Calling Conditions

Cannot be called from an interrupt handler.
Can be called from a thread (must be called in an interrupt-enabled state).
Multithread safe.

Arguments

  commId     NP Communication ID allocated to the application  
  passPhrase     Title passphrase allocated to the application  
  ctxId     Pointer to buffer to store the context ID  
  option     Context options  

Return Values

Returns 0 for normal termination.
Returns a negative value for errors. The main error codes are shown below. Note, however, that the application must not malfunction even if other error codes are returned.
  Macro     Value     Description  
  SCE_NP_MATCHING2_ERROR_NOT_INITIALIZED     0x80550c02     Not initialized  
  Execute sceNpMatching2Init() and   
  initialize the NP matching 2 utility.  
  SCE_NP_MATCHING2_ERROR_CONTEXT_MAX     0x80550c05     No more contexts can be created  
  Re-execute this function after deleting an   
  existing context.  
  SCE_NP_MATCHING2_ERROR_CONTEXT_ALREADY_EXISTS     0x80550c06     Context already exists  
  A context created by the same NP ID and   
  NP Communication ID already exists.  
  SCE_NP_MATCHING2_ERROR_INVALID_ARGUMENT     0x80550c04     Invalid argument  
  An argument required for executing the   
  function may not have been specified.   
  Check the argument values.  


Description

This function creates an NP matching 2 utility context. In order to use the NP matching 2 utility, it is necessary to create a context after initializing the utility. For commId and passPhrase, specify the NP Communication ID and title passphrase obtained upon user registration to NP.

The information used for user identification on the NP matching 2 system is the NP ID, Online Name, and Avatar URL. The NP ID is required; the Online Name and Avatar URL are optional. Applications can save on memory and network resources if the Online Name and Avatar URL are not used.

To use the Online Name and Avatar URL, specify context options for option. For information regarding context options, refer to the section on the constant SCE_NP_MATCHING2_CONTEXT_OPTION_xxx.

The context options are valid during the entire time the context is alive. The configuration cannot be changed while a context is alive.

When a context is successfully created, the context ID will be stored in the buffer indicated by ctxId. From this point onwards, this context ID will be used when calling NP matching 2 utility functions.

Examples

This example creates a context as follows.
  • Use the Online Name for user identification 

int ret;

//E Parameters required for creating a context
//  Assuming that appropriate values are stored
SceNpCommunicationId commId;
SceNpCommunicationPassphrase passPhrase;
int option;

//E Variable for storing the ID of the new context 
SceNpMatching2ContextId ctxId;

//E Use the Online Name for user identification
option = SCE_NP_MATCHING2_CONTEXT_OPTION_USE_ONLINENAME;

//E Create a context
ret = sceNpMatching2CreateContext(
    &commId, &passPhrase, &ctxId, option);
if (ret < 0) {
    //E Error handling
}
