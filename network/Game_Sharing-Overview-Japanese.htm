ゲームシェアリング概要

1 ゲームシェアリング概要

1. 1 概要

ゲームシェアリングはPSP®端末間において、外部起動用バイナリをアドホックモード通信によって送受信するための機能です。本ドキュメントでは、ゲームシェアリングのサーバー側の実装、及びゲームシェアリングに使用される外部起動用バイナリについて説明します。
外部起動用バイナリは、起動バイナリとヘッダ情報から構成される１つのファイルです。外部起動用バイナリの最大サイズは8メガバイト(8 × 1024 × 1024 byte)です。

ゲームシェアリングはあらかじめ定められたアドホックネットワーク内で行われます。以下にその例を示します。



図 1 ゲームシェアリングユーティリティ概要



  • ゲームシェアリングサーバー
    ゲームプログラムからゲームシェアリングユーティリティAPIを利用してクライアントからの接続を待機します。転送する外部起動用バイナリのサイズは（8 × 1024 × 1024）バイトに制限されています。
  • ゲームシェアリングクライアント
    PSP®システムソフトウェア上に実装されており、ユーザが「ゲーム」メニューの「ゲームシェアリング」を選択することによってサーバーを検索します。サーバー選択後、正常に外部起動用バイナリを受信したクライアントはそのデータを自動的に起動します。外部起動用バイナリはメモリ上に展開されます。PSP®の電源を切ると外部起動用バイナリは消失します。

1. 3 サンプル

ゲームシェアリングユーティリティのサーバー側の実装サンプルとして次のものがあります。

(1) devkit/sample/utility/gamesharing/adhoc_stick_server

ゲームシェアリングユーティリティのサーバーサンプルです。
カレントディレクトリにあるhost0:./upload.datファイルを外部起動用バイナリとして転送します。このサンプルはネットワークサンプルのadhoc_stickをベースに作られています。
サンプルを起動して、○ボタンを押すとゲームシェアリング用IBSSへ接続します。その後自動的にゲームシェアリングユーティリティが起動します。
サーバーはデータ転送後に、ゲーム用のIBSSへと接続し、adhoc_stickサンプルとして動作します。
ゲーム用IBSSへ接続している状態で○ボタンを押すと、ネットワークから切断し、初期画面に戻ります。

(2) devkit/sample/utility/gamesharing/adhoc_stick_gamedata

devkit/sample/gamesharing/adhoc_stick_server で使用される外部起動用バイナリ（upload.dat）のサンプルです。起動後にゲームシェアリングサーバーが指定したグループ名でアドホックネットワークに接続し、adhoc_stickサンプルとして動作します。

注）実際にゲームデータとして動作させるためにはコンパイルした後に外部起動用バイナリとしてパッケージング処理が必要になります。パッケージングの詳細については、「外部起動用バイナリについて」のドキュメントを参照してください。

2 ゲームシェアリングサーバー実装方法

ゲームシェアリングのサーバーの実装方法について説明します。実装に関しての詳細はサンプル(devkit/sample/utility/gamesharing/adhoc_stick_server）を参照してください。以下にサーバー側プログラムの流れを図示します。



図 2 ゲームシェアリング（サーバー側）プログラムの流れ


サーバー側のプログラムは大きく分けて、ネットワーク初期化、ゲームシェアリングネットワーク接続、ゲームシェアリングユーティリティ起動、外部起動用バイナリ転送、ゲームシェアリングネットワーク切断、の5つに分類できます。以下それぞれについて説明します。

2. 1 ネットワーク初期化

初めにゲームシェアリングユーティリティが使用するネットワークライブラリの初期化をゲームタイトルで行う必要があります。タイトルでもネットワークライブラリを使用している場合は一旦終了処理をしてから再初期化します。
必要なネットワークライブラリのモジュールはPSPNET共通モジュールセットおよびPSPNETアドホックモジュールセットです。
モジュールのロード手順についてはサンプルプログラム(devkit/sample/utility/gamesharing/adhoc_stick_server)を参照してください。

ネットワークライブラリの初期化処理ではゲームシェアリング用アドホックIDを指定することと、pspnet_adhoc_matchingライブラリの初期化が必要な点に注意してください。アドホックIDはtypeにSCE_NET_ADHOCCTL_ADHOCTYPE_SYSTEMを、dataに"000000001"を指定してください。その他のライブラリの初期化については「pspnetドキュメントおよびpspnet_adhoc_matchingドキュメント」を参照してください。

// ゲームシェアリング用のアドホックID を指定
static struct SceNetAdhocctlAdhocId gamesharing_id = {
    .type = SCE_NET_ADHOCCTL_ADHOCTYPE_SYSTEM,
    .data = {'0','0','0','0','0','0','0','0','1'}   /* "000000001" */
};

// pspnet_adhocctl モジュールの初期化
ret = sceNetAdhocctlInit(ADHOCCTL_STACKSIZE, ADHOCCTL_TPL, &gamesharing_id);
if (ret < 0) {
	// エラー処理
}

2. 2 ゲームシェアリングネットワーク接続

ゲームシェアリング用のアドホックネットワークにpspnet_adhocctlライブラリを用いて接続します。sceNetAdhocctlConnect()関数への引数としてゲームシェアリング用のグループ名を指定してください。グループ名は"GameShar"になります。詳細については「pspnetドキュメント」を参照してください。

static struct SceNetAdhocctlGroupName gamesharing_group_name = {
    .data = {'G', 'a', 'm', 'e', 'S', 'h', 'a', 'r'} /* "GameShar */
};

ret = sceNetAdhocctlConnect(&gamesharing_group_name);
if (ret < 0) {
	// エラー処理
}
while (1) {
	// 接続状態のポーリング
}
// 接続完了
また、ネットワークの接続にネットワーク接続ユーティリティを利用することもできます。

2. 3 ゲームシェアリングユーティリティ起動

ゲームシェアリングユーティリティはゲームシェアリングサーバーの機能を持ったユーティリティです。専用のAPIを呼び出すことによって、ゲームシェアリングサーバーが起動します。起動時にゲームデータ、グループ名を指定します。APIの詳細については、サンプルプログラム及び、「ユーティリティリファレンス」を参照してください。

正常にゲームシェアリングユーティリティが起動するとサーバー待機状態となります。この時、クライアント側からはサーバーリストを参照することができます。クライアントがダウンロード要求を送信しサーバー側がそれを受け入れると、ゲームシェアリング起動バイナリの送受信が開始されます。
外部起動用バイナリは専用のバイナリデータです。ゲームシェアリングではバイナリデータの形式について確認は行いません。指定されたデータサイズ分だけ送受信を行います。正しいデータ形式でない場合は、送受信完了後のゲーム起動にて失敗します。

ゲームシェアリングクライアントは、ゲーム起動時にサーバー側のMACアドレス、またサーバーが指定したグループ名を起動引数として渡します。これらのデータはゲーム起動後、サーバーとネットワーク対戦を行う場合などに使用することができます。

2. 4 ゲームシェアリングユーティリティ起動タイプ

ゲームシェアリングユーティリティの起動タイプには、クライアントとサーバーが1対1で通信を行うモードと、複数クライアントに対応したモードとの、2つのタイプがあります。
起動タイプは、SceUtilityGameSharingParam構造体のserverTypeで指定します。

  • 1対１モード
    1：1で通信を行うモードでは、クライアントからのリクエストをユーザが承諾するとデータ転送が開始されます。データ転送が完了した時点で、ユーティリティは自動的に終了します。
    以下に、1：1モードでのゲームシェアリングユーティリティの画面フローを示します。



図 3 １：１モードの画面フロー



  • 1対ｎモード
    1：nモードでは、クライアントからの送信に対して自動的にデータ転送が開始されます。この時、データ転送は最大で4並列で行われます。このモードでは、1：1モードのように、自動でユーティリティが終了することはありません。ユーティリティを終了させるためには、ユーティリティを起動したユーザが明示的にユーティリティを終了させる必要があります。

2. 5 外部起動用バイナリ転送

外部起動用バイナリを転送する方法として、次の2種類の方法があります。
バイナリ転送の方式は、SceUtilityGameSharingParam構造体のdataLocationTypeで指定します。
  • ファイルシステム上のデータを転送
    dataLocationTypeにSCE_UTILITY_GAMESHARING_LOCATION_TYPE_FILEを指定することで、ファイルシステム上のデータを転送します。ファイルパスをSceUtilityGameSharingParam構造体のfilePathに指定してください。
  • メモリ上のデータを転送
    dataLocationTypeにSCE_UTILITY_GAMESHARING_LOCATION_TYPE_MEMORYを指定することで、メモリ上のデータを転送します。データのバッファポインタとサイズをそれぞれ、SceUtilityGameSharingParam構造体のuploadDataとuploadDataSizeに指定してください。

2. 6 ゲームシェアリングネットワーク切断

ゲームシェアリングユーティリティの終了後は、通常通りネットワークを切断してください。詳細は「pspnetドキュメント」を参照してください。

3 外部起動用バイナリ

外部起動用バイナリは、ゲームシェアリングユーティリティによって送信され、PSP®システムソフトウェアのメニューから受信・起動されるバイナリデータです。外部起動用バイナリは、システムファイルおよびオーサリングされたゲームブートファイルから構成されます。
ここでは、ゲームブートファイルの実装方法について説明します。実装に関しての詳細はサンプル(devkit/sample/utility/gamesharing/adhoc_stick_gamedata）を参照してください。

3. 1 起動時の引数

バイナリが起動すると、プログラムのmain()関数の引数として次のような値が渡されます。
これらの値にはゲームシェアリングサーバーが指定した値が入っていますので、起動後にサーバーと通信対戦を行うような場合に利用することができます。
  main()への引数     データ内容  
  argv[1]     サーバーのMACアドレス  
  argv[2]     グループ名  

MACアドレスは、"XX:XX:XX:XX:XX:XX"という形式の文字列になります。

3. 2 モジュールのロード

モジュールをプログラムに内包して、バッファからロードすることによってモジュールを利用することができます。バッファからモジュールをロードするためには次のAPIを使用してください。

<modulemgr_gamesharing.h>
int sceKernelLoadModuleBufferUsbWlan(SceSize size, 
					void *base, 
					int flags, 
					const SceKernelLMOption *pOpt
					);

3. 3 デバッグ

オーサリング処理を行ったバイナリは、dstdbから起動することができなくなります。
オーサリング処理を行う前ならば、通常のプログラムと同様にdstdb等を使ってデバッグすることが可能です。
このとき、mstartコマンドでゲームシェアリングユーティリティサーバーのMACアドレスやグループ名を指定することでより本番に近い形で起動させることができます

例：
dstdb 0R0> mstart  sample.prx  00:00:00:00:00:00  GROUP

また、dstdbからプログラムを起動するとsceKernelLoadModuleBufferUsbWlan()を使用することができなくなります。モジュールをロードする場合は、通常のsceKernelLoadModule()を使用して、ホストやディスクからモジュールをロードしてください。

4 注意点

4. 1 PSP®システムソフトウェアバージョン

起動バイナリを受信するPSP®のシステムソフトウェアバージョンが、起動バイナリを作成したパッケージのリリースバージョンよりも古いものであった場合、この起動バイナリを起動させることはできません。
詳細については、「外部起動用バイナリについて」ドキュメントの注意事項を参照してください。

4. 2 ゲームシェアリングユーティリティ起動中のネットワークエラーについて

ゲームシェアリングをサーバーとして起動している間のネットワークのエラー表示は、ゲームシェアリングユーティリティが行うため、ゲームプログラム側で表示する必要はありません。ユーティリティの起動中にエラーイベントが発生した場合は画面表示をせずに、ユーティリティの呼出し後にネットワークの切断処理を行ってください。

4. 3 バッファからのモジュールロードについて

sceKernelLoadModuleBufferUsbWlan()を用いることで、バッファにあるモジュールをロードすることができます。ただし、ロードできるモジュールはdevkit/moduleおよびdevkit/kmoduleにあるものに限ります。また、sceKernelLoadModuleBufferUsbWlan()は、外部起動バイナリとして作られたプログラムからのみ、使用することができます。

4. 4 TOOLにおけるデバッグ情報表示

ツールでゲームシェアリングのデータを受信した場合、受信した外部起動バイナリに含まれているシステムファイル（param.sfo）の情報が表示されます。この機能はツール特有のもので、PSP®本体では表示されません。
表示される情報は次の通りです。

  Status     システムファイルのフォーマットが正しければ"OK"を、正式な  
  フォーマットに従っていない場合は"ERROR"を表示します。  
  DISC_ID     システムファイルに記載されている品番を表示します。  
  PSP_SYSTEM_VER     システムファイルに記載されているPSP®のシステムバージョン  
  を表示します。  
  REGION     システムファイルに記載されているリージョンを表示します。  
  SHARING_ID     システムファイルに記載されているSharingIDを表示します。
