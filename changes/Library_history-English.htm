5 Kernel-Related Library

5. 1 Release 6.3.0

  • On PSP-2000 or later, changing the clock frequency to 266MHz and 333MHz when WLAN is in use was enabled. On PSP-1000, as before the clock frequency is restricted to 222MHz while WLAN is in use.

    Whether the model can use 266MHz and 333MHz while WLAN is in use can be checked using scePowerCheckWlanCoexistenceClock(). This function identifies the model, but do not use it for any other purposes.

    On DTP-T1000, when the boot parameter bit is 11, it is possible to decide the behavior of the maximum clock frequency when WLAN is in use to be equivalent to that of PSP-1000 or PSP-2000 or later.

  • It fairly infrequently (once every several thousand times) failed to terminate a game application or to execute LoadExec. This problem has been fixed.

  • sceKernelPrintf() could not display a 64bit integer value correctly. This problem has been fixed.

    sceKernelCreateLwMutex()

5. 3 Release 6.0.0

  • Along with the easing of the restriction on the address alignment of sceKernelDcacheMemcpy(), a memory copy function sceKernelMemcpy() with improved speed has been added.
    sceKernelDcacheMemcpy() was deemed to be a simple memcpy by some compilers, however, no such restriction is placed on sceKernelMemcpy().

    Also, a memory filling function sceKernelMemset() to which the same improvement has been applied has been added.

  • A function sceKernelExtendThreadStack() has been added to temporarily extend a thread stack by reserving new memory.

5. 4 Release 3.9.5

  • The lightweight mutex (lwmutex) has been added.

    If a part of the work area is placed on the user memory space, the lightweight mutex can complete the processing within the memory space when it is successfully locked or when no other thread is activated by the unlock operation. This reduces the weight of the processing compared to the normal mutex because it can complete the processing without switching to the kernel mode using SYSCALL.

    The differences from the mutex are as shown below:
  • No automated unlock feature is provided for when a thread that still keeps the mutex is ended
  • No cancellation feature for when the mutex ownership is taken back is provided.

  • The sceKernelDcacheWritebackRange() and sceKernelDcacheWritebackInvalidateRange() type has been changed to the int type.
    0 is returned for the normal end. An error is returned when the size is set to 2GB(0x80000000u) or more. (No difference in the internal operation.)
