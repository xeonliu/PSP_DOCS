2. 3 devkit/sample/sound/mp4aacplay
devkit/sample/sound/mp3play
devkit/sample/sound/at3mp3play
devkit/sample/sound/mp3lowlevel
devkit/sample/sound/aacplay

これらのサウンドサンプルでは、メディア（UMD®, Memory Stick™）が抜かれた場合のファイルクローズ・再オープンを、ドライバに登録したコールバック内で行っていました。
この場合、特に、UMD®上のファイルオープンをコールバック内で行うと、コールバックが呼ばれたスレッドがオープン処理が完了するまでブロックされてしまいます。

この問題を解決するため、メディアが抜かれた場合のファイルクローズ・再オープンをBGM再生スレッドで行うように、サンプルを修正しました。

ただし、mp4aacplayサンプルでは、Memory Stick™以外に再生対象のMP4ファイルが配置されたときには音切れが発生してしまうため配置しないようにしてください。

4 Release 6.0.0

4. 1 sample/net/adhoc/adhoc_dialog

Release 6.0.0より前のサンプルではsceNetAdhocDialogJoin()使用時に、Join側のタイムアウト判定をScan 5回と定義されています。
この処理時間は、実質5秒程度となり、非常に短いタイムアウト判定になっていました。また、特にアドホックパーティを行う際に、タイムアウトしやすくなります。

Release 6.0.0ではScanを60回行うように変更しました。
