SceGumContext
 [構造体]
コンテキスト

構造

#include <libgum.h>
typedef struct {
	int buffer[32];
} SceGumContext;
解説

コンテキストの保存に使用する構造体です。
割り込みルーチンなどでlibgumを使用する場合、メインのコンテキストを破壊しないように保存し、リターンする前に復帰する必要があります。

関連項目

sceGumLoadContext(), sceGumStoreContext()

sceGumSetMatrixStack
 [関数]
行列スタック用バッファの設定

形式

#include <libgum.h>
void sceGumSetMatrixStack (
	ScePspFMatrix4 *m,
	int proj,
	int view,
	int world,
	int tex
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止 / 許可状態に依存しない)
マルチスレッドセーフでない

引数

  m     行列スタックとして使用するバッファのポインタ  
  proj     プロジェクション行列スタックの段数  
  view     ビュー行列スタックの段数  
  world     ワールド行列スタックの段数  
  tex     テクスチャ行列スタックの段数  

返り値

なし

解説

libgumで使用する行列スタックを初期化する関数です。
mで指定したScePspFMatrix4構造体の配列を行列スタックとして使用し、その配列のうちproj / view / world / texで指定した数の要素をそれぞれプロジェクション行列スタック / ビュー行列スタック / ワールド行列スタック / テクスチャ行列スタックに割当てます。
つまり、mで指定する配列のサイズはproj / view / world / texに指定する値の総和以上でなければなりません。

使用例

ScePspFMatrix4  mat_stack[2+2+8+2];	// 行列スタックバッファ
sceGumSetMatrixStack(mat_stack, 2, 2, 8, 2);	// 行列スタックの設定
sceGumMatrixMode
 [関数]
カレントマトリクスの種別選択

形式

#include <libgum.h>
void sceGumMatrixMode (
	int mode
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止 / 許可状態に依存しない)
マルチスレッドセーフでない

引数

  mode     行列の種別  

modeに指定するマクロは以下のとおりです。

  マクロ     説明  
  SCEGU_MATRIX_PROJECTION     プロジェクション行列  
  SCEGU_MATRIX_VIEW     ビュー行列  
  SCEGU_MATRIX_WORLD     ワールド行列  
  SCEGU_MATRIX_TEXTURE     テクスチャ行列  


返り値

なし

解説

カレントマトリクスの種別を選択する関数です。
カレントマトリクスをmodeで指定した種別の行列とみなし、このあとsceGumPushMatrix() / sceGumPopMatrix() / sceGumSetCurMatrix()を呼び出したときにそれぞれ種別に応じた適切な操作が行われるようにします。

使用例

呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止 / 許可状態に依存しない)

引数

  eye     視点の位置ベクトル  
  center     注視点の位置ベクトル

返り値

なし

解説

eye / center / up各引数に従って視点マトリクスを計算し、カレントマトリクスとの乗算を行って、結果をカレントマトリクスに設定する関数です。

eye.y = 0.1f;
eye.z = 5.0f;
center.x = 0.0f;		// 注視点の位置ベクトル (0.0,0.0,0.0)
center.y = 0.0f;
center.z = 0.0f;
up.x = 0.0f;		// 上向きベクトル(0.0,1.0,0.0)
up.y = 1.0f;
up.z = 0.0f;
sceGumLookAt( &eye, &center, &up );	// 視点マトリクスの設定

	float 			u, v;
	unsigned int 		rgba;
	ScePspFVector3	normal;
	ScePspFVector3	vector;

	// uv      rgba        normal          vertex
	{0.0f,0.0f, 0xffffffff, {0.0f,0.0f,1.0f}, {-1.0f,-1.0f,0.0f}},

ベジェ曲面を描画するコマンドシーケンスを、libguで作成中のディスプレイリストに挿入する関数です。カレントマトリクスが更新されていれば、新しいカレントマトリクスをセットするコマンドシーケンスをベジェ曲面描画の前に挿入します。
ベジェ曲面の制御点を表すデータ配列をvtxで指定し、インデックス配列をidxで指定し、U/V各方向の制御点数をucountおよびvcountで指定します。
typeには、vtxとidxに納めたデータの内容を指定します。テクスチャ座標のフォーマット / カラー値のフォーマット / 頂点ベクトルのフォーマット / インデックスタイプ / ウェイト値 / ウェイトの個数 / モーフィング頂点数を指定することができ、それぞれ適切なマクロ定数の論理和をとって指定します。頂点ベクトルは必ずデータに含まれているはずですので、頂点ベクトルのフォーマット指定(SCEGU_VERTEX_xxx)は省略できません。それ以外の指定は、該当するデータが含まれていなければ省略してかまいません。また、法線はライブラリ内部で自動計算した値が使われるため、法線ベクトルを指定しても無視されます。

使用例

sceGumPushMatrix();	// 現在のプッシュ
sceGumLoadIdentity();	// 単位行列のロード
sceGumDrawBezier( SCEGU_VERTEX_FLOAT, 4, 4, bezier );	// ベジェ曲面の描画
sceGumPopMatrix();	// カレントマトリクスのポップ
備考

引数vtxおよびidxが指すデータ領域は、該当するディスプレイリストの描画が終了するまで解放してはいけません。
曲面の細分割に関する設定はsceGuPatchDivide()とsceGuPatchPrim()で行います。
SCEGU_IMMEDIATEモードの場合は、sceGumDrawBezier()関数の呼び出しで即座に描画が開始されますので、引数idx, vtxで参照されるメモリエリアはあらかじめDキャッシュのコヒーレンシが確保されていなくてはなりません。必要な場合には、明示的にDキャッシュのライトバックを行ってください。

sceGumStoreContext
 [関数]

形式

#include <libgum.h>

	SceGumContext *ctx
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能(割り込み禁止 / 許可状態に依存しない)
マルチスレッドセーフでない

引数

  ctx     コンテキストのポインタ  

返り値

なし

解説

現在のコンテキストを読み出し、ctxで指定された変数に保存します。
メインルーチンと割り込みルーチンで同時にlibgumを使用する場合、割り込みルーチンでlibgumを使用する前にメインのコンテキストを破壊しないように保存し、リターンする前に復帰する必要があります。

使用例

SceGumContext	ctx;

sceGumStoreContext(&ctx);		// 現在のコンテキストを保存
sceGumLoadContext( &ctx );		// 現在のコンテキストを設定
