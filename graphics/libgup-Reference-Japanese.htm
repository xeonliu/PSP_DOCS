sceGupPutBeginObject
 [関数]
オブジェクトの作成開始

形式

#include <libgup.h>
static __inline__ void sceGupPutBeginObject (
	SceGupContext *pContext,
	int type,
	int count,
	const void *idx,
	const void *vtx
);
引数

  pContext     ディスプレイリストコンテキストへのポインタ  
  type     vtxおよびidxの指すデータタイプ  
  count     頂点の数(8の倍数)  
  idx     インデックス配列のポインタ  
  vtx     頂点配列のポインタ  

typeには、以下のマクロの論理和を指定します。

  マクロ     説明  
  SCE_GE_VERTEX_BYTE     頂点ベクトル(符号あり 8bit)  
  SCE_GE_VERTEX_SHORT     頂点ベクトル(符号あり 16bit)  
  SCE_GE_VERTEX_FLOAT     頂点ベクトル(浮動小数点)  
  SCE_GE_INDEX_UBYTE     符号なし 8bit インデックス  
  SCE_GE_INDEX_USHORT     符号なし 16bit インデックス  


返り値

なし

解説

引数pContextで示されるディスプレイリストに、バウンディングボックスを挿入してオブジェクトの作成を開始する関数です。このあと、sceGupPutEndObject()を呼び出すまでの間にディスプレイリストに挿入されたコマンドシーケンスがひとつのオブジェクトとなります。CMD_BBOX + CMD_BASE + CMD_BJUMPコマンドシーケンスにより、ディスプレイリストを描画する際に、行列変換後のバウンディングボックスが描画領域内に入るかどうかが判定され、もし領域外であれば、オブジェクトの描画処理がスキップされます(オブジェクトカリング)。
vtxとidxにはバウンディングボックスの各頂点を表す頂点ベクトル配列とインデックス配列を指定します。インデックス配列を使用しないときはidxにNULLを指定してください。
typeには、vtxとidxに指定したデータのタイプを指定します。
countには、バウンディングボックスの頂点数を指定します。8の倍数でなければなりません。

備考

バウンディングボックスは、最大32重までのネストできます。

使用例

// バウンディングボックスの挿入とオブジェクトの開始
sceGupPutBeginObject(pContext, SCE_GE_VERTEX_FLOAT, 8, NULL, vert);

// テクスチャ、カラー、法線、頂点付き三角形の描画
sceGupDrawArray(pContext, SCE_GE_PRIM_TRIANGLES,
		SCE_GE_TEXTURE_FLOAT|SCE_GE_COLOR_PF8888
		|SCE_GE_NORMAL_FLOAT|SCE_GE_VERTEX_FLOAT,
		3, NULL, model );

// オブジェクトの終了
sceGupPutEndObject(pContext);

  マクロ     説明

  SCEGU_PF5551     16bit 5:5:5:1 フォーマット

  SCEGU_PF8888     32bit 8:8:8:8 フォーマット

なし

解説

引数pContextで示されるディスプレイリストに、フレームバッファ上の矩形領域を各引数の指定に従ってコピーするコマンドシーケンスを挿入します。

sx,sy,dx,dyは0〜1023の値を、width,heightは1〜1023の値をとりますが、ソースおよびデスティネーションバッファの横幅をラップアラウンドするような転送、sx + widthが1023を超える転送、dx + widthが1023を超える転送は行えません。
sbw, dbwは、8の倍数で8〜1024の値をとります。
sbp, dbpは、16バイトアラインメントである必要があります。

フレームバッファ上にコピーされた内容をテクスチャとして参照するためには、参照に先立ってsceGupTexFlush()およびsceGupTexSync()を実行する必要があります。sceGupTexFlush()はテクスチャキャッシュの内容を無効化し、新しいデータをテクスチャとして認識させるために必要です。sceGupTexSync()はsceGupCopyImage()によって実行される矩形転送が完了するまでテクスチャ読み出しをブロックするために必要です。
フレームバッファ上にコピーされた内容をテクスチャ以外の目的(頂点データ,頂点インデックスデータ,ディスプレイリストなど)としてGraphics Engineが参照する場合は、ディスプレイリストがCMD_FINISH + CMD_ENDによっていったん終了される必要があり、libguではSCEGU_CALLモード, SCEGU_IMMEDIATEモードのsceGuFinish()による終了、またはsceGuSignalDrawSync()、libgupでのsceGupPutSignalDrawSync()による同期が必要になります。つまり、sceGupCopyImage()を含むディスプレイリストの中で、コピー先をそのままテクスチャ以外の目的で使用してはいけません。

使用例

sceGupCopyImage(pContext, SCEGU_PF5551, 0, 0, 128, 128, 512, (void *)0x10000000,
	0, 0, 512, (void *)0x20000000);
sceGupTexFlush(pContext);
sceGupTexSync(pContext);
