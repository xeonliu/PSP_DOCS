libgiq_picture

1 ライブラリ概要

1. 1 機能の特徴

libgiq_pictureはlibgim_pictureの軽量版です。libgim_pictureとほぼ同じAPIを提供し、より少ないCPU負荷でGIMファイルを表示することができます。

1. 2 関連ファイル

libgiq_pictureを使用するために必要なファイルは次のとおりです。

  カテゴリ     ファイル名  
  ヘッダファイル     giq/giq_picture.h  
  ライブラリファイル     libgiq_picture.a  


  • 標準のコンテキストで描画する場合は、libguおよびlibgumが必要です。
  • 内部関数を使用する場合は、giq/giq_protected.hをインクルードしてください。
  • ソースコードは、devkit/src/giq_picture/ にあります。

1. 3 サンプルプログラム

devkit/sample/graphics/picture_giq
libgim_pictureとlibgiq_pictureで画像データを表示するサンプルです。

2 使用手順

2. 1 基本的な処理

次に示すのは、libgiq_pictureを使用するプログラム例の主要部分です。

#include <giq/giq_picture.h>

// 初期化
・・・

// ピクチャデータをロード
SceGiqPicture *picture = sceGiqPictureCreate() ;
sceGiqPictureLoadFile( picture, data, size, 0 ) ;
SceGiqPicture *copy =sceGiqPictureDuplicate( picture, SCEGIQ_DUPLICATE_AUTOMATIC ) ;

for ( ; ; ) {
	//  ディスプレイリストを開始
	sceGuStart( SCEGU_LIST_CACHED, pkt, pkt_size ) ;

	//  ピクチャデータを表示
	sceGiqPictureAnimate( copy, 1.0f / 60.0f ) ;
	sceGiqPictureActivate( copy ) ;
	sceGumDrawArray( prim, format, 4, 0, verts ) ;
	. . .
	//  ディスプレイリストを終了
	sceGuFinish() ;
	sceGuSendList( SCEGU_QUEUE_TAIL, pkt, 0, 0, 0 ) ;
}

//  ピクチャデータを削除
sceGiqPictureDelete( picture ) ;
sceGiqPictureDelete( copy ) ;

(1) 初期化

必要に応じて、コールバック関数を登録します。
  • 通常のメモリを確保 / 解放する関数
  • DMA転送用のメモリを確保 / 解放する関数
  • 高速なeDRAM上のメモリを確保 / 解放する関数

(2) ピクチャデータをロード

  sceGiqPictureCreate()     : ピクチャ構造体を作成します。  
  sceGiqPictureLoadFile()     : ピクチャ構造体にデータを読み込みます。  
  sceGiqPictureDuplicate()     : ピクチャ構造体のコピーを作成します。  

(3) ディスプレイリストを開始

(4) ピクチャデータを表示

  sceGiqPictureAnimate()     : アニメーションの時間を進めます。  
  sceGiqPictureActivate()     : テクスチャ設定のディスプレイリストを作成します。  
  sceGumDrawArray()     : ポリゴン描画のディスプレイリストを作成します。  

(5) ディスプレイリストを終了

(6) ピクチャデータを削除

  sceGiqPictureDelete()     : ピクチャ構造体を解放します。  

2. 2 追加的な処理

必要に応じて、以下のような処理を追加してください。

(1) 各種パラメータの設定

  sceGiqPictureSetEnableBits()     : イネーブルビットを設定します。  
  sceGiqPictureSetImageCrop()     : イメージクロップを設定します。  
  ・・・       

(2) キャッシュのロード

  sceGiqPictureLoadCache()     : 高速メモリを確保します。  
  sceGiqPictureUnloadCache()     : 高速メモリを解放します。  

3 制限事項

3. 1 注意点

  • メモリブロックをまとめて確保するメモリクラスタは、暗黙に生成されます。

3. 2 内部データ

  • データ構造は以下のように変更されました。
  • 高速化用のImage / Paletteブロックは、リンクリストに登録されます。
  • アニメーションに関する情報は、Sequenceブロックに格納されます。
  • 描画データは以下のように変更されました。
  • テクスチャ設定パケットは、可能ならば暗黙に生成され、使用されます。
  • イメージのピクセルデータは、128バイトアラインメントで格納されます。

4 拡張された機能

4. 1 メモリマネージャ

メモリのアラインメントや解放遅延などの機能をメモリマネージャに実装しました。これらの処理をメモリマネージャに実行させることも、従来どおりユーザーが用意するメモリ管理関数に実行させることも可能です。

4. 2 メモリミューテックス

マルチスレッドによるアクセスからメモリマネージャを保護することが可能になりました。
スレッドセーフ性については第5章を参照してください。

4. 3 ピクチャ複製

従来は未対応であった、ピクセルデータを複製することが可能になりました。

5 スレッドセーフ性

5. 1 スレッドセーフ性

libgiq_pictureは、複数のピクチャが内部データを共有するように設計されているため、マルチスレッド処理には適していません。



もし、ピクチャの複製を行わず内部データを共有しない状態であっても、メモリマネージャがグローバル変数に格納されているため、何らかのアクセス保護が必要です。



マルチスレッド処理を行うためには、以下のような条件を満たす必要があります。
  • サブスレッドでアクセスするピクチャは他のピクチャと内部データを共有しない
  • メモリマネージャをミューテックスなどで保護する

5. 2 ファイルロード処理

たとえば、ファイルロード処理をサブスレッドで実行することは、ロード直後のピクチャが他のピクチャと内部データを共有しないことから、比較的実現が容易であると考えられます。

サブスレッドの処理は、たとえば以下のようになります。
  • メインスレッドと非同期でファイルロード処理を行う
  • メインスレッドと同期してファイルロード完了を通知する



サブスレッドのプログラムは、たとえば以下のようになります。

void *data = 0 ;
size_t size = 0 ;
LoadFile( "dog.gim", &data, &size ) ;

SceGiqPicture *picture = sceGiqPictureCreate() ;
sceGiqPictureLoadFile( picture, data, size, 0 ) ;
sceGiqPictureLoadCache( picture ) ;
