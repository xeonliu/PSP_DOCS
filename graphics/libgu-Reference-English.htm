sceGuStart
 [Function]
Start display list creation

Syntax

#include <libgu.h>
int sceGuStart ( 
	int mode, 
	void *p, 
	int size 
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  mode     Mode of display list to be created  
  p     Starting address of display list creation area  
  size     Size of display list creation area  

The following macros can be specified for mode.

  Macro     Description  
  SCEGU_IMMEDIATE     Draw while creating display list  
  SCEGU_CALL     A CALL display list is created without using a cache  
  SCEGU_LIST     An sceGuSendList() display list is created without using a cache  
  SCEGU_CALL_CACHED     A CALL display list is created via the D cache  
  SCEGU_LIST_CACHED     An sceGuSendList()display list is created via the D cache  


Return Value

If an error occurs, a negative value is returned.
  Value     Result  
  SCE_OK     Normal termination  
  < 0     An error occurred  


Description

This function starts display list creation. A display list is created by using mode setting functions and primitive functions following this function.
If SCEGU_IMMEDIATE is specified for mode, the display list command sequences are transferred to the Graphics Engine and drawn at the same time that the display list is being created.
When SCEGU_LIST is specified for mode, the display list is only created and no drawing is performed. After display list creation is completed by calling the sceGuFinish() function, the actual drawing can be performed later by calling the sceGuSendList() function.
When SCEGU_CALL is specified for mode, the display list is only created and no drawing is performed. After display list creation is completed by calling the sceGuFinish() function, be sure to use the sceGuCallList() function to indicate that this display list is to be called from another display list that will be its parent while the parent display list is being created. Then, this display list will be called and drawn when the parent display list is drawn.
If SCEGU_LIST_CACHED is specified for mode, although the processing is the same as when SCEGU_LIST is specified, the list is created via a cache address. Therefore, the D cache must be explicitly written back before drawing is performed by using sceGuSendList().
If SCEGU_CALL_CACHED is specified for mode, although the processing is the same as when SCEGU_CALL is specified, the list is created via a cache address. Therefore, the D cache must be explicitly written back before drawing is performed by using sceGuCallList().


For p, specify the starting address of the area for storing the created display list, and for size, specify the size of that area. Note that the display list is not a ring buffer. Since memory corruption occurs if the size is exceeded, be sure to use sceGuCheckList() appropriately during creation to check the size that has been used.

Since contexts are saved separately for each display list mode, sceGuStart() can be called multiple times as long as the value for mode is different. For example, while a display list is being created and drawn in SCEGU_IMMEDIATE mode, a separate display list can be created in SCEGU_LIST mode.
The following functions, which change the context, must be called for each mode when creating a display list.
  • sceGuScissor()
  • sceGuEnable(SCEGU_SCISSOR_TEST)
  • sceGuDisable(SCEGU_SCISSOR_TEST)
  • sceGuEnable(SCEGU_COLOR_DOUBLE)
  • sceGuDisable(SCEGU_COLOR_DOUBLE)
  • sceGuDepthRange()
  • sceGuDepthOffset()
  • sceGuSpriteMode()
  • sceGuClearColor()
  • sceGuClearStencil()
  • sceGuClearDepth()
  • sceGuTexMode()
  • sceGuTexFunc()
  • sceGuTexMapMode()
  • sceGuTexProjMapMode()

Example

static char disp_list[0x10000];	// Allocate area
sceGuStart( SCEGU_IMMEDIATE, (void *)disp_list, sizeof(disp_list) );	// Set area
See Also

sceGuFinish(), sceGuCallList(), sceGuSendList(), sceKernelDcacheWritebackAll(), sceKernelDcacheWritebackRange()

sceGuSendList
 [Function]
Queue display list

Syntax

#include <libgu.h>
int sceGuSendList ( 
	int mode,
	const void *p,
	SceGuContext *ctx 
	int nStackDepth,
	SceGeStack *stack
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  mode     Queuing mode  
  p     Display list  
  ctx     Pointer to context save area  
  nStackDepth     SIGNAL CALL stack depth  
  stack     SIGNAL CALL stack buffer  

The following macros can be specified for mode.

  Macro     Description  
  SCEGU_QUEUE_TAIL     Add at end of queue  
  SCEGU_QUEUE_HEAD     Add at beginning of queue  


Return Value

If an error occurs, a negative value is returned.

  Value     Result  
  SCE_OK     Normal termination  
  < 0     An error occurred  


Description

This function transfers a created display list to the Graphics Engine in order to begin drawing operations.
For p, specify the starting address of a display list that has been created in SCEGU_LIST mode.
If SCEGU_QUEUE_TAIL is specified for mode, the display list specified by p is added at the end of the drawing queue and will be drawn after drawing ends for other display lists that have already been queued.
If SCEGU_QUEUE_HEAD is specified for mode, the display list specified by p is added at the beginning of the drawing queue and will be drawn before other display lists that have already been queued.
If a pointer to the context save area is specified for ctx, the internal state of the Graphics Engine is saved immediately before the display list specified by p is drawn so that it will be restored after drawing ends. If it does not matter if the context is destroyed or if the display list specified by p will not change the context, specify NULL for ctx.

Example

static char disp_list[0x1000];
SceGeStack stk [32];

sceGuStart(SCEGU_LIST, disp_list, sizeof(disp_list));	// Start display list
	:
sceGuFinish();					// Terminate list
sceGuSendList(SCEGU_QUEUE_TAIL, disp_list, NULL, 32, stk);
						// Queue list
See Also

sceGuSignal
 [Function]
Insert signal interrupt

Syntax

#include <libgu.h>
void sceGuSignal (
	int mode,
	unsigned short id
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  mode     Signal interrupt mode  
  id     Signal ID  

The following macros can be specified for mode.

  Macro     Meaning  
  SCEGU_SIGNAL_WAIT     Wait for end of signal callback before restarting drawing  
  SCEGU_SIGNAL_NOWAIT     Continue drawing without waiting for end of signal callback  
  SCEGU_SIGNAL_PAUSE     Suspend drawing  
  Use sceGuContinue() to restart drawing  


Return Value

None

Description

This function inserts a signal interrupt command in the display list that is being created.
When the display list is drawn, an interrupt is generated for the CPU when this command is reached. mode specifies the drawing operation at that time. id, which is the signal ID, is passed to the signal callback.

Notes

When multiple signal interrupt commands are inserted in the same display list, their signal IDs must differ from each other.
Also, when 16 or more signal interrupt commands are inserted, an interrupt wait caused by sceGuSignalSync() may not work properly.

SCEGU_SIGNAL_WAIT is implemented using a signal initiated by the Graphics Engine Manager's SCE_GE_SIGNAL_SIGNAL_WAIT command. In this case, the SIGNAL handler is called while the display list scan is paused. Even though the display list is stopped, the Graphics Engine itself will continue drawing according to drawing commands that were invoked before the display list was stopped, which means that the drawing frame buffer cannot be read or written by the SIGNAL handler. When the SIGNAL handler ends, the display list operation is automatically restarted by the Graphics Engine Manager.

SCEGU_SIGNAL_NOWAIT is implemented using a signal initiated by the Graphics Engine Manager's SCE_GE_SIGNAL_SIGNAL_NOWAIT command. In this case, the SIGNAL handler is called after the Graphics Engine Manager has restarted the display list, so the drawing frame buffer cannot be read or written by the SIGNAL handler.

SGEGU_SIGNAL_PAUSE is implemented using a signal initiated by the Graphics Engine Manager's SCE_GE_SIGNAL_SIGNAL_PAUSE command. In this case, the SIGNAL handler is called after the Graphics Engine Manager has verified that drawing has temporarily completed, so the drawing frame buffer cannot be read or written by the SIGNAL handler. Also, another display list that has been created can be paused by specifying SCEGU_QUEUE_HEAD when calling sceGuSendList().
Since the display list is not automatically restarted by the Graphics Engine Manager even when the SIGNAL handler ends, operation must be explicitly restarted by calling sceGuContinue().

Example

static char disp_list[0x10000];	// Allocate area
sceGuStart( SCEGU_IMMEDIATE, (void *)disp_list, sizeof(disp_list) );	// Set area
sceGuSignal(SCEGU_SIGNAL_NOWAIT, 100);		// Signal interrupt
sceGuSignalSync(100);					// Signal wait
sceGuFinish();			// Set drawing end
See Also

sceGuStart(), sceGuFinish(), sceGuSignalSync()
