sceGiqPictureSetMemoryManager / sceGiqPictureSetMemoryManager2 / sceGiqPictureSetMemoryManager3
 [Function]
Set memory management functions

Syntax

void sceGiqPictureSetMemoryManager(
	void *(*malloc)( size_t ),
	void (*free)( void * ),
	size_t aligned_size,
	bool delay_release
) ;
void sceGiqPictureSetMemoryManager2(
	void *(*malloc)( size_t ),
	void (*free)( void * )
	size_t aligned_size,
	bool delay_release
) ;
void sceGiqPictureSetMemoryManager3(
	void *(*malloc)( size_t ),
	void (*free)( void * )
	size_t aligned_size,
	bool delay_release
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  malloc     Memory allocation function  
  free     Memory free function  
  aligned_size     Alignment  
  delay_release     Delayed release flag  

Return Value

None

Description

This function sets functions for allocating and freeing picture memory.

sceGiqPictureSetMemoryManager() sets normal memory management functions. Structures and arrays are stored in this memory. If it is omitted, the normal malloc and free are used.
sceGiqPictureSetMemoryManager2() sets DMA memory management functions. Packet and pixel data are stored in this memory. If omitted, the normal memory management functions are used.
sceGiqPictureSetMemoryManager3() sets eDRAM memory management functions. Packet and pixel data are stored in this memory. If omitted, eDRAM memory is not allocated.

For aligned_size, specify the minimum alignment guaranteed by the memory management functions. This enables unnecessary padding to be suppressed. If the minimum alignment is unknown, specify 1.

For delay_release, specify whether or not to perform delayed release. If delayed release is enabled, the release of memory will be delayed until sceGiqPictureCleanMemory() is called. This feature is used to release DMA memory or eDRAM memory at a safe time.
If this feature is unnecessary, specify false.

Example

sceGiqPictureSetMemoryManager( malloc, free, 16, true ) ;

Description

This function sets image cropping, i.e. the image display range.
For the picture specified by picture, the x, y, w, and h values specified by crop are set as the display range. x and y represent the position of the upper-left corner of the display range, and w and h represent the width and height of the display range. Because each of these values is proportional to the image width / height rounded up to the nearest power of 2, a value from 0.0f to 1.0f should be specified for each.

Arguments

  frame     Image frame number

Description

This function directly sets the basic image frame.
The basic image frame number is set to the value of the remainder when frame is divided by the number of frames in the image of the sequence specified by sequence.

sceGiqSequenceSetDissolve
 [Function]
Set dissolve value

Syntax

void sceGiqSequenceSetDissolve(
	SceGiqSequence *sequence,
	float dissolve
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  sequence     Sequence structure  
  dissolve     Dissolve value (0.0f to 1.0f)  

Return Value

None

Description

This function sets the dissolve value (i.e. the value representing the interpolation rate) when dissolving between two frames.
For the sequence specified by sequence, the value specified by dissolve is set as the dissolve value. The dissolve value can be in the range 0.0f to 1.0f. When it is 0.0f, it represents an interpolation rate in which the basic frame is 100% and the dissolve frame is 0%. When it is 1.0f, it represents an interpolation rate in which the basic frame is 0% and the dissolve frame is 100%.

Example

sceGiqSequenceSetDissolve( sequence, 0.5f ) ;
See Also
