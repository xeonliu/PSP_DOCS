PSP® Programmer Tool Runtime Library Release 6.6.0

libgmq_model

1 ライブラリ概要

1. 1 機能の特徴

libgmq_modelはlibgmo_modelの軽量版です。libgmo_modelとほぼ同じAPIを提供し、より少ないCPU負荷でGMOファイルを表示することができます。

1. 2 関連ファイル

libgmq_modelを使用するために必要なファイルは次のとおりです。

  カテゴリ     ファイル名  
  ヘッダファイル     gmq/gmq_model.h  
  giq/giq_picture.h  
  ライブラリファイル     libgmq_model.a  
  libgiq_picture.a  


  • テクスチャを表示するためlibgiq_pictureが必要です。
  • 標準のコンテキストで描画する場合は、libguおよびlibgumが必要です。
  • 内部関数を使用する場合は、gmq/gmq_protected.hをインクルードしてください。
  • ソースコードは、devkit/src/gmq_model/ にあります。

1. 3 サンプルプログラム

devkit/sample/graphics/model_gmq
libgmo_modelとlibgmq_modelでモデルデータを表示するサンプルです。

1. 4 参考資料

2 使用手順

2. 1 基本的な処理

次に示すのは、libgmq_modelを使用するプログラム例の主要部分です。

#include <gmq/gmq_model.h>

// 初期化
・・・

// モデルデータをロード
SceGmqModel *model = sceGmqModelCreate() ;
sceGmqModelLoadFile( model, data, size, 0 ) ;
SceGmqModel *copy =sceGmqModelDuplicate( model, SCEGMQ_DUPLICATE_AUTOMATIC ) ;
sceGmqModelSetCurrentMotion( copy, 0, 0.0f ) ;

for ( ; ; ) {
	//  ディスプレイリストを開始
	sceGuStart( SCEGU_LIST_CACHED, pkt, pkt_size ) ;

	//  モデルデータを表示
	sceGmqModelSetWorldMatrix( copy, &world_matrix ) ;
	sceGmqModelAnimate( copy, 1.0f / 60.0f ) ;
	sceGmqModelUpdate( copy ) ;
	sceGmqModelDraw( copy ) ;
	. . .
	//  ディスプレイリストを終了
	sceGuFinish() ;
	sceGuSendList( SCEGU_QUEUE_TAIL, pkt, 0, 0, 0 ) ;
}

//  モデルデータを削除
sceGmqModelDelete( model ) ;
sceGmqModelDelete( copy ) ;

(1) 初期化

必要に応じて、コールバック関数を登録します。
  • 通常のメモリを確保 / 解放する関数
  • DMA転送用のメモリを確保 / 解放する関数
  • 高速なeDRAM上のメモリを確保 / 解放する関数
  • テクスチャ画像の登録 / 取得を行う関数

(2) モデルデータをロード

  sceGmqModelCreate()     : モデル構造体を作成します。  
  sceGmqModelLoadFile()     : モデル構造体にデータを読み込みます。  
  sceGmqModelDuplicate()     : モデル構造体のコピーを作成します。  
  sceGmqModelSetCurrentMotion()     : 再生するモーションを選択します。  

(3) ディスプレイリストを開始

(4) モデルデータを表示

  sceGmqModelSetWorldMatrix()     : モデルの位置を設定します。  
  sceGmqModelAnimate()     : アニメーションの時間を進めます。  
  sceGmqModelUpdate()     : モデルのワールド行列を計算します。  
  sceGmqModelDraw()     : ディスプレイリストを作成します。  

(5) ディスプレイリストを終了

(6) モデルデータを削除

  sceGmqModelDelete()     : モデル構造体を解放します。  

2. 2 追加的な処理

必要に応じて、以下のような処理を追加してください。

(1) 各種パラメータの設定

  sceGmqModelSetEnableBits()     : イネーブルビットを設定します。  
  sceGmqModelSetEnableMask()     : イネーブルマスクを設定します。  
  ・・・       

(2) キャッシュのロード

  sceGmqModelLoadCache()     : 高速メモリを確保します。  
  sceGmqModelUnloadCache()     : 高速メモリを解放します。  

3 制限事項

3. 1 モデルデータの制限

  • ボーンは深さ優先順に格納されている
  • 1つのメッシュが使用するすべての頂点データは同じ頂点フォーマットを持つ

3. 2 注意点

  • メモリブロックをまとめて確保するメモリクラスタは、暗黙に生成されます。
  • 特定のパラメータを変更した場合は、以下の関数を呼び出す必要があります（詳しくはライブラリリファレンスを参照してください）。
  • sceGmqModelReconfigure()
  • sceGmqModelNormalize()

3. 3 内部データ

  • データ構造の変更のため、内部データにアクセスする一部のAPIは非対応か、または便宜上の処理を行うコンビニエンス関数になりました。
  • データ構造は以下のように変更されました。
  • Arraysブロックは、Meshブロックの下位ブロックとなります。
  • 高速化用のArraysブロックは、リンクリストに登録されます。
  • Materialブロックの情報は、Layerブロックに格納されます。
  • Animateコマンドは、FCurveブロックに統合されます。
  • 描画データは以下のように変更されました。
  • 描画プリミティブは、描画パケットに展開されて格納されます。
  • インデックス頂点は、非インデックス頂点に展開されて格納されます。

4 拡張された機能

4. 1 メモリマネージャ

メモリのアラインメントや解放遅延などの機能をメモリマネージャに実装しました。これらの処理をメモリマネージャに実行させることも、従来どおりユーザーが用意するメモリ管理関数に実行させることも可能です。

4. 2 メモリミューテックス

マルチスレッドによるアクセスからメモリマネージャを保護することが可能になりました。
スレッドセーフ性については第5章を参照してください。

4. 3 モデル複製

従来は未対応であった、頂点データやピクセルデータを複製することが可能になりました。

4. 4 リスト描画

ボーンの描画順序を直接指定したりZ値でソートすることが可能になりました。

4. 5 プリセット描画

パケットをあらかじめ作成しておくことでCPU負荷を軽減することが可能になりました。

4. 6 エッジ描画

モデルデータからエッジ情報を追加でロードし、稜線などのエッジを選択的に描画することが可能になりました。

4. 7 サブブロック名

モデルデータからサブブロック名を追加でロードし、アニメーションターゲットの名前解決を行うことが可能になりました。

4. 8 頂点ウェイトの省略

スキンメッシュが単一の頂点ウェイトを持つ場合に、頂点ウェイトを省略可能になりました。
ソリッドメッシュとして描画することで、GPU負荷を削減できる場合があります。
（その場合、メッシュに BlendSubset 1 Nが必要です）

頂点ウェイトのないメッシュは、以下のように処理されます。
  • BlendSubset なし : ソリッドメッシュ
  • BlendSubset 1 N : 頂点ウェイトが省略されたスキンメッシュ

なお、頂点ウェイトを省略したモデルのエッジ描画は未対応です。

5 スレッドセーフ性

5. 1 スレッドセーフ性

libgmq_modelは、複数のモデルが内部データを共有するように設計されているため、マルチスレッド処理には適していません。



もし、モデルの複製を行わず内部データを共有しない状態であっても、描画パラメータやメモリマネージャがグローバル変数に格納されているため、何らかのアクセス保護が必要です。



マルチスレッド処理を行うためには、以下のような条件を満たす必要があります。
  • サブスレッドでアクセスするモデルは他のモデルと内部データを共有しない
  • サブスレッドでアクセスするモデルに対して描画APIを呼び出さない
  • メモリマネージャをミューテックスなどで保護する

5. 2 ファイルロード処理

たとえばファイルロード処理をサブスレッドで実行することは、ロード直後のモデルが他のモデルと内部データを共有しないこと、また描画 API を呼び出すケースも少ないことから、比較的実現が容易であると考えられます。

サブスレッドの処理は、たとえば以下のようになります。
  • メインスレッドと非同期でファイルロード処理を行う
  • メインスレッドと同期してファイルロード完了を通知する



サブスレッドのプログラムは、たとえば以下のようになります。

void *data = 0 ;
size_t size = 0 ;
LoadFile( "dog.gmo", &data, &size ) ;

SceGmqModel *model = sceGmqModelCreate() ;
sceGmqModelSetOptionFlags( model, SCEGMQ_OPTION_IMPORT_PICTURES, 0 ) ;
sceGmqModelLoadFile( model, data, size, 0 ) ;
sceGmqModelLoadCache( model ) ;
外部テクスチャのインポートやプリセットパケットの生成処理は非同期部分で実行できません。外部テクスチャのインポートは他のモデルと内部データを共有します。またプリセットパケットの生成処理は内部的に描画APIを呼び出します。

そのため、上の例では、外部テクスチャを自動的にインポートしないようオプションフラグをクリアしてファイルロードを行っています。外部テクスチャをインポートするには、メインスレッドで明示的にsceGmqModelImportPictures()を呼び出してください。
