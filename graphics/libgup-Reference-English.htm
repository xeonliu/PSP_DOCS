);
Arguments

  pContext     Pointer to display list context  
  pStart     Start address of display list creation area  
  size     Size (in bytes) of display list creation area  

Return Value

None

Description

This function starts creation of a display list. After calling this function, create the display list using mode setting functions, primitive functions, etc.

sceGupPutBeginObject
 [Function]
Start creating object

Syntax

#include <libgup.h>
static __inline__ void sceGupPutBeginObject (
	SceGupContext *pContext,
	int type,
	int count,
	const void *idx,
	const void *vtx
);
Arguments

  pContext     Pointer to display list context  
  type     Data type pointed to by vtx and idx  
  count     Vertex count (multiple of 8)  
  idx     Index array pointer  
  vtx     Vertex array pointer  

For type, specify the logical OR of the following macros.

  Macro     Description  
  SCE_GE_VERTEX_BYTE     Vertex vector (signed 8-bit)  
  SCE_GE_VERTEX_SHORT     Vertex vector (signed 16-bit)  
  SCE_GE_VERTEX_FLOAT     Vertex vector (floating point)  
  SCE_GE_INDEX_UBYTE     Unsigned 8-bit index  
  SCE_GE_INDEX_USHORT     Unsigned 16-bit index  


Return Value

None

Description

This function inserts a bounding box in the display list indicated by the pContext argument, and starts creating an object. After that, the command sequence inserted in the display list will produce a single object until sceGupPutEndObject() is called. When drawing the display list using the CMD_BBOX + CMD_BASE + CMD_BJUMP command sequence, a check is made to determine whether or not the bounding box after matrix transformation falls inside or outside the drawing area. If it falls outside the drawing area, processing for drawing the object is skipped (object culling).
In vtx and idx, specify a vertex vector array and index array representing the vertices of the bounding box. When not using an index array, specify NULL in idx.
In type, specify the type of data specified in vtx and idx.
In count, specify the number of vertices in the bounding box. This must be a multiple of 8.

Notes

The bounding box can be nested to as many as 32 levels.

Example

// Insert bounding box, and start object
sceGupPutBeginObject(pContext, SCE_GE_VERTEX_FLOAT, 8, NULL, vert);

// Draw triangles with texture, color, normal line, and vertices
sceGupDrawArray(pContext, SCE_GE_PRIM_TRIANGLES,
		SCE_GE_TEXTURE_FLOAT|SCE_GE_COLOR_PF8888
		|SCE_GE_NORMAL_FLOAT|SCE_GE_VERTEX_FLOAT,
		3, NULL, model );

// End object
sceGupPutEndObject(pContext);

sceGupClear
 [Function]
Clear drawing buffer

Syntax

#include <libgup.h>
static __inline__ void sceGupClear (
	SceGupContext *pContext,
	int mask
);
Arguments

  pContext     Pointer to display list context  
  mask     Clearing mask  

The clearing target and method are specified in mask as the logical OR of the following macros.

  Macro     Description  
  SCEGU_CLEAR_COLOR     Clear color value  
  SCEGU_CLEAR_DEPTH     Clear depth buffer  
  SCEGU_CLEAR_STENCIL     Clear stencil bits  
  SCEGU_CLEAR_FAST     Divide into 8 parts and clear  
  SCEGU_CLEAR_ALL     Clear color value, depth buffer, and stencil bits  
  SCEGU_CLEAR_FASTALL     Divide into eight parts and clear color value,  
  depth buffer, and stencil bits  


Return Value

None

Description

This function inserts into the display list indicated by the pContext argument a command sequence which clears the drawing buffer and the depth buffer. In other words, in accordance with the specification for mask, a command sequence is inserted which fills the drawing buffer and the depth buffer using the color value specified using sceGupClearColor(), the depth value specified using sceGupClearDepth(), and the stencil bit value specified using sceGupClearStencil().
Normally, clearing is performed by drawing a 480x272 rectangle, but if SCEGU_CLEAR_FAST or SCEGU_CLEAR_FASTALL is specified in mask, clearing is performed by drawing eight 64x272 rectangles. This suppresses page breaks and allows clearing to be performed slightly more quickly, but increases the space used in the display list by 168 bytes.

Example

// Set clearing color
sceGupClearColor(pContext, 0x00800000);

// Set depth value for clearing
sceGupClearDepth(pContext, 0);

// Set value of stencil bits for clearing
sceGupClearStencil(pContext, 0);

// Clear drawing buffer
sceGupClear(pContext, SCEGU_CLEAR_ALL);

sceGupCopyImage
 [Function]
Copy pixel data

Syntax

#include <libgup.h>
static __inline__ void sceGupCopyImage (
	SceGupContext *pContext,
	int mode,
	int sx,
	int sy,
	int width,
	int height,
	int sbw,
	void *sbp,
	int dx,
	int dy,
	int dbw,
	void *dbp
);
Arguments

  pContext     Pointer to display list context  
  mode     Pixel format of data to transfer  
  sx     Transfer start location in source buffer  
  sy     Transfer start location in source buffer  
  width     Width (pixel count) of transfer area  
  height     Height (pixel count) of transfer area  
  sbw     Width of source buffer  
  sbp     Source buffer base pointer  
  dx     Transfer start location in destination buffer  
  dy     Transfer start location in destination buffer  
  dbw     Width of destination buffer  
  dbp     Destination buffer base pointer  

The macros specified in mode are as follows.
  Macro     Description  
  SCEGU_PF5650     16-bit 5:6:5:0 format  
  SCEGU_PF5551     16-bit 5:5:5:1 format  
  SCEGU_PF4444     16-bit 4:4:4:4 format  
  SCEGU_PF8888     32-bit 8:8:8:8 format  


Return Value

None

Description

This function inserts into the display list indicated by pContext a command sequence which copies a rectangular area in the frame buffer based on the values specified by the respective arguments.

Although sx, sy, dx, and dy can range from 0 to 1023, and width and height can range from 1 to 1023, transfers that cause the width of the source and destination buffers to wrap around, transfers in which sx + width is greater than 1023 and transfers in which dx + width is greater than 1023 cannot be performed.
sbw and dbw, which are multiples of 8, can have values from 8 to 1024.
sbp and dbp must be 16-byte aligned.

Note that the sceGupTexFlush() and sceGupTexSync() functions must be executed before data that was copied to the frame buffer can be referenced as a texture. sceGupTexFlush() is required to invalidate the texture cache and cause new data to be recognized as the texture. sceGupTexSync() is required to block texture loading until the rectangle transfer that is executed by sceGupCopyImage() has completed.
Before the Graphics Engine can reference data that was copied to the frame buffer for a purpose other than for use as a texture (for example, vertex data, vertex index data, display list, etc.), the display list must have finished by executing a CMD_FINISH + CMD_END command sequence. In libgu, this can be accomplished by calling sceGuFinish() in either SCEGU_CALL mode or SCEGU_IMMEDIATE mode. Also, drawing must be synchronized in libgu by calling sceGuSignalDrawSync() or in libgup by calling sceGupPutSignalDrawSync(). In other words, in a display list that contains sceGupCopyImage(), the data at the copy destination cannot be used directly for any other purpose except as a texture.

Example

sceGupCopyImage(pContext, SCEGU_PF5551, 0, 0, 128, 128, 512, (void *)0x10000000,
	0, 0, 512, (void *)0x20000000);
sceGupTexFlush(pContext);
sceGupTexSync(pContext);
See Also

sceGupTexFlush(), sceGupTexSync()
