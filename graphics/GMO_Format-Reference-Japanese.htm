引数

 頂点形式   enum   頂点データの要素です。以下の定数が使用可能です。 
 VERTEX   頂点ベクトル（浮動小数点数×3） 
 NORMAL   法線ベクトル（浮動小数点数×3） 
 COLOR   頂点カラー（32ビット整数×1） 
 POINTSIZE   ポイントサイズ（浮動小数点数×1） 
 TEXCOORDn   テクスチャ座標（浮動小数点数×2n）

 モーフ形式   enum   モーフ形状の頂点形式です。 

データ形式

まず基本形状の頂点データが格納されます。
頂点データは（1頂点のデータ×頂点数）で構成されます。
1頂点のデータの内容は、頂点形式に従います。

例：
  頂点、法線、カラー、ポイントサイズ、UV、ウェイト    // 1頂点のデータ  
  頂点、法線、カラー、ポイントサイズ、UV、ウェイト    // 1頂点のデータ  
  頂点、法線、カラー、ポイントサイズ、UV、ウェイト    // 1頂点のデータ  
  ・・・  


モーフ数が2以上のとき、続けてモーフ形状の頂点データが格納されます。
頂点データは（1頂点のデータ×頂点数×（モーフ数−1））で構成されます。
1頂点のデータの内容は、モーフ頂点形式に従います。

例：
  頂点、法線  
  頂点、法線  
  頂点、法線  
  ・・・  


頂点カラーは以下のようなビット構成で表現されます。

FCurve
 [ブロック]
関数データ

形式

FCurve "ブロック名" 補間形式 次元数 キー数 {
	ブロック本体
}
解説

Motionブロックにおいて、一つの関数データを表すサブブロックです。

サブブロック

なし

コマンド

なし

引数

 補間形式   enum   キーの内挿方法（以下の定数が使用可能） 
         CONSTANT   補間なし 
         LINEAR   線形補間 
         HERMITE   HERMITE補間 
         CUBIC   CUBIC補間 
         SPHERICAL   球面線形補間（クォータニオン用） 
         キーの外挿方法（以下の定数が使用可能） 
         HOLD   時刻を固定する 
         CYCLE   時刻を循環させる 
         SHUTTLE   時刻を往復させる 
         REPEAT   繰り返し（値を累積） 
         EXTEND   線形拡張 
 次元数   int   キーフレームの次元数です。フレーム値は含みません。 
 キー数   int   キーフレームの数です。 

データ形式

関数は複数のキーフレームの補間によって表現されます。
データは（キーフレーム×キー数）で構成されます。
キーフレームは、フレームと関数値で構成されます。
関数値はベクトルであり、次元数だけデータを持ちます。
キーフレームはフレームの小さい値から大きい値に（昇順に）並んでいる必要があります。

例：
  フレーム　関数値　関数値　関数値 ・・・    // キーフレーム  
  フレーム　関数値　関数値　関数値 ・・・    // キーフレーム  
  フレーム　関数値　関数値　関数値 ・・・    // キーフレーム  
  ・・・  


ただしHERMITE補間およびCUBIC補間の場合はハンドルを含むため、おのおの（次元数×3）、（次元数×5）だけデータを持ちます。

例：HERMITE補間
  フレーム　関数値　in_dy　out_dy　関数値　in_dy　out_dy ・・・  
  フレーム　関数値　in_dy　out_dy　関数値　in_dy　out_dy ・・・  
  フレーム　関数値　in_dy　out_dy　関数値　in_dy　out_dy ・・・  
  ・・・  

例：CUBIC補間
  フレーム　関数値　in_dx　in_dy　out_dx　out_dy　関数値 ・・・  
  フレーム　関数値　in_dx　in_dy　out_dx　out_dy　関数値 ・・・  
  フレーム　関数値　in_dx　in_dy　out_dx　out_dy　関数値 ・・・  
  ・・・  


in_dx, in_dyは左ハンドルを、out_dx, out_dyは右ハンドルを表します。

BoundingBox
 [共通コマンド]
バウンディングボックス

形式

BoundingBox 最小X 最小Y 最小Z 最大X 最大Y 最大Z
引数

  最小     float     頂点座標の最小値  
  最大     float     頂点座標の最大値  

解説

形状データをかこむ直方体を記述します。
モデル、ボーン、パート、およびメッシュで使用します。

各ブロックのバウンディングボックスの計算方法は次のようになります。
  • メッシュでは、メッシュに含まれる頂点のローカル座標バウンディングボックス
  • パートでは、パートに含まれる頂点のローカル座標バウンディングボックス
  • ボーンでは、描画パートのローカル座標バウンディングボックスを結合したもの
  • モデルでは、全てのボーンのワールド座標バウンディングボックスを結合したもの

  • メッシュでは、メッシュに含まれる頂点のローカル座標バウンディングポイント配列
  • パートでは、パートに含まれる頂点のローカル座標バウンディングポイント配列
  • ボーンでは、描画パートのローカル座標バウンディングポイント配列を結合したもの
  • モデルでは、全てのボーンのワールド座標バウンディングポイント配列を結合したもの

Animate
 [モーションコマンド]
アニメーション

形式

Animate "ブロック" コマンド 番号 "関数データ"
引数

  ブロック     ref     アニメーションするブロックへの参照  
  コマンド     enum     アニメーションするコマンドのタイプ  
  モード     enum     アニメーション制御モード  
  関数データ     ref     関数データへの参照  

解説

他のブロックのコマンドを、指定した関数データでアニメーションします。

ブロックには、モデル直下のブロックを指定できます。通常はボーンまたはマテリアルを指定します。（対応しているブロックは表示プログラムにより異なります。）

コマンドには、数値を引数とするコマンドを指定できます。通常は以下のようなコマンドを指定します。
Visibility / MorphWeights / MorphIndex / MultMatrix / Translate / Rotate* / Scale*
Diffuse / Specular / Emission / Ambient / TexCrop
（対応しているコマンドは表示プログラムにより異なります。）

モードには、アニメーションのサブブロック番号およびサブチャンネル番号を指定します。
サブブロックはINDEXn、サブチャンネルはOFFSETnで表します（n=0〜255)。
これらの値を論理和で指定することができます。通常は0にしてください。

関数データには、同じモーション内の関数データを指定します。
