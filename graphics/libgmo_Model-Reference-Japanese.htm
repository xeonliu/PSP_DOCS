解説

モデルのためのメモリを確保する関数と解放する関数を設定します。

sceGmoModelSetMemoryManager2()はDMAメモリ用の関数です。パケットと頂点データを格納します。省略時はsceGmoModelSetMemoryManager()で設定した関数が使用されます。
sceGmoModelSetMemoryManager3()はeDRAM用の関数です。頂点データを格納します。省略時はeDRAMの確保を行いません。

モデル用に確保されるメモリはVFPUを使用しない場合は4バイト単位のアラインメントが必要です。VFPUを使用する場合は16バイト単位のアラインメントが必要です。
DMAメモリ用、eDRAM用に確保されるメモリは4バイト単位のアラインメントが必要ですが、さらにDキャッシュライトバックによる予想外の挙動を回避するため、DMA転送されるメモリには64バイト単位のアラインメントが推奨されます。

使用例

sceGmoModelSetMemoryManager( malloc, free ) ;

引数

  size     メモリクラスタの容量  
  align     アラインメント（2のべき乗）  

返り値

確保されたメモリクラスタのアドレス（失敗したときは0）

解説

メモリクラスタの使用を開始します。
メモリクラスタは主にヒープメモリの細分化を避けるために使用します。
まず指定された容量のメモリクラスタがメモリ管理関数によって確保され、それ以降のライブラリ内部のメモリ確保がそのメモリクラスタに対して行われるようになります。確保されるメモリはすべてalignで指定される値でアラインメントされます。ただしメモリクラスタが不足した場合はメモリ管理関数によってメモリ確保が行われます。
メモリクラスタは内部のメモリがすべて解放されたときにメモリ管理関数により解放されます。

sceGmoModelOpenMemoryCluster2()はDMAメモリ用の関数です。
sceGmoModelOpenMemoryCluster3()はeDRAM用の関数です。

使用例

void *addr = sceGmoModelOpenMemoryCluster( 100000, 16 ) ;
SceGmoModel *model = sceGmoModelCreate( chunk ) ;
int size = sceGmoModelCloseMemoryCluster() ;

matrixにはプロジェクション行列とビュー行列の積を指定してください。そしてmarginにはクリップボリュームの大きさとスクリーンボリュームの大きさの比率を指定してください。ソフトウェアクリッピング処理が有効なとき、クリップボリューム内部とスクリーンボリューム外部の両方にまたがって位置するトライアングルがクリップされます。

  • 非インデックスの三角形ストリップのみに対応しています
    ( -O -Qオプションを指定してコンバートしてください)
  • バウンディングポイントを使用して判定を行っています
    ( --rebuild_bounding_points onを指定してコンバートしてください)
  • スキニング、モーフィング、16bit頂点カラーには対応していません
  • 半透明オブジェクトには使用できません
