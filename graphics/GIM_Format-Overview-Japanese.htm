PSP® GIM 2D Converter

GIMフォーマット・概要

1 はじめに

1. 1 機能概要

基本的なテクスチャ画像を格納するためのファイルフォーマットです。以下のような情報を格納することができます。
  • イメージ
  • パレット
  • シーケンス

1. 2 作成方法

一般的な画像ファイルを変換することで作成します。現在はBMP、TGA、TM2、DDS、AVI、PNGフォーマットに対応しています。


1. 3 表記について

このドキュメントではデータ型を以下のように表記します。バイトオーダーはリトルエンディアンです。

  型名     型     型名     型  
  char     符号つき8ビット整数     u_char     符号なし8ビット整数  
  short     符号つき16ビット整数     u_short     符号なし16ビット整数  
  int     符号つき32ビット整数     u_int     符号なし32ビット整数  
  float     32ビット浮動小数点数            


記述されている「アラインメント」は必要最小値です。詳しくは「PSP®ライブラリによる制限」の章を参照してください。

2 格納形式

まずファイル先頭にファイルヘッダが格納され、それに続くチャンクの階層構造に画像データなどが格納されます。

2. 1 ファイルヘッダ

ファイルヘッダにはファイルそのものに関する情報が格納されます。ファイルヘッダのデータ構造は以下のようになっています。

typedef struct {
	u_int signature ;	// ファイル識別子 ( '.GIM' )
	u_int version ;		// バージョン番号 ( '1.00' )
	u_int style ;		// データスタイル ( 'PSP' )
	u_int option ;		// オプション ( 0 )
} GimHeader ;

データスタイルはデータ表現が特定のAPIに最適化されていることを表します。'PSP' スタイルではPSP®ライブラリの制約にしたがった形式に限定されます。詳しくは「PSP®ライブラリによる制限」の章を参照してください。

2. 2 チャンク

チャンクには任意のデータを格納することができ、さらに階層構造を持つこともできます。チャンクのレイアウトは以下のようになっています。パディング領域は0のバイト列でうめられます。


チャンクヘッダのデータ構造は以下のようになっています。

typedef struct {
	u_short type ;		// チャンクタイプ
	u_short unused ; 	// 使用しません（つねに0）
	u_int next_offs ;	// 次のチャンクのオフセット
	u_int child_offs ;	// 下位チャンクのオフセット
	u_int data_offs ;	// チャンクデータのオフセット
} GimChunk ;

チャンクに含まれる各要素の先頭は4バイト単位でのアラインメントが必要です。そのためチャンクヘッダに含まれるオフセット値はすべて4の倍数となります。要素がない場合でもオフセット値は0にはなりません。アラインメントのためのパディングは0のバイト列でうめられます。

3 チャンク構造

チャンクの階層構造は以下のようになっています。

3. 1 ファイルチャンク

ファイルチャンクはファイル全体を表す最上位チャンクです。複数のピクチャチャンクを格納することができます。チャンクデータはありません。

3. 2 ピクチャチャンク

ピクチャチャンクは一つの画像データを表すチャンクです。ピクチャチャンクには以下のようなチャンクを格納することができます。各チャンクの個数は任意です。
  • イメージチャンク
  • パレットチャンク
  • シーケンスチャンク

チャンクデータはありません。

3. 3 イメージチャンク

イメージチャンクは同じピクセル数のイメージ配列を格納することができます。配列はミップマップやシーケンスとして使用できます。



ミップマップ使用時はレベルごとにピクセル数を(n+1)/2にすることができるほか、すべてのレベルに同じピクセル数のイメージを格納することもできます。

イメージチャンクデータのレイアウトは以下のようになっています。ただし各領域のうしろにはアラインメントのためのパディングが追加されることがあります。パディングは0のバイト列でうめられます。


イメージヘッダ
イメージヘッダのデータ構造は以下のようになっています。

typedef struct {
	u_short header_size ;	// ヘッダのサイズ（48）
	u_short unused ;	// 使用しません（0）
	u_short format ; 	// ピクセル形式（GIM_FORMAT_XXXX）
	u_short order ;		// ピクセル格納形式（GIM_ORDER_XXXX）
	u_short width ;		// 横幅のピクセル数
	u_short height ;	// 高さのピクセル数
	u_short bpp ;		// １ピクセルのビット数
	u_short pitch_align ;	// 横幅のバイト数のアラインメント
	u_short height_align ;	// 高さのピクセル数のアラインメント
	u_short dim_count ; 	// イメージ配列の次元数（2）
	u_short reserved ;	// 予約領域（0）
	u_short reserved2 ;	// 予約領域（0）
	u_int offsets ;		// オフセット配列へのオフセット
	u_int images ;		// 先頭イメージへのオフセット
	u_int total ;		// 合計サイズ
	u_int plane_mask ;	// プレーンマスク
	u_short level_type ;	// レベルのタイプ（GIM_TYPE_XXXX）
	u_short level_count ;	// レベル数
	u_short frame_type ;	// フレームのタイプ（GIM_TYPE_XXXX）
	u_short frame_count ;	// フレーム数
} GimImageHeader ;

header_sizeはヘッダのサイズを表します。48バイト以上必要です。
format、orderはイメージの形式を表します（詳細は別記）。
width、heightはイメージのピクセル数を表します（1以上）。
bppは1ピクセルの平均ビット数を表します（1以上）。
pitch_align、height_alignはアラインメントの単位を表します（1以上）。
dim_countはイメージ配列の次元数を表します（常に2）。
offsetsはイメージヘッダを起点としたオフセット配列へのオフセットです。
imagesはイメージヘッダを起点とした先頭イメージへのオフセットです。
totalはイメージチャンクデータ全体のサイズです。

plane_maskは少ないビット数のイメージを1枚のイメージに重ねて格納したときのプレーン構成を表します。隣接するプレーンごとにビット列を反転させます。最下位プレーンはプレーン数が偶数のときは0、奇数のときは１にします。formatがインデックス形式の場合のみ有効で、不要な場合は0にします。

例：


level_count、frame_countはイメージ数を表します（1以上）。
level_type、frame_typeはイメージ配列のタイプを表します。
level_typeにはGIM_TYPE_MIPMAPまたはMIPMAP2を指定します。
frame_typeにはGIM_TYPE_SEQUENCEを指定します（詳細は別記）。

ユーザデータ
任意のバイト列を格納することができます。

オフセット配列
イメージヘッダを起点とした各イメージへのオフセットをイメージ数だけ格納します。オフセットは4バイト整数です。4バイト単位でのアラインメントが必要です。

イメージ
ピクセル配列を格納します。16バイト単位でのアラインメントが必要です。

3. 4 パレットチャンク

パレットチャンクは、同じエントリ数のパレット配列を格納することができます。配列はミップマップやシーケンスとして使用できます。



パレットチャンクのチャンクデータのレイアウトおよびデータ構造はイメージチャンクと同じですが、以下のような制限があります。
  • heightは常に1となります。
  • formatはGIM_FORMAT_RGBAXXXXのみです。
  • plane_maskは常に0となります。
  • level_typeはGIM_TYPE_MIPMAP2のみです。

3. 5 シーケンスチャンク

シーケンスチャンクは簡単なアニメーション情報を格納することができます。
  • イメージフレーム番号
  • パレットフレーム番号
  • イメージ矩形

また、いろいろな設定情報を定数として格納することもできます。
  • アルファブレンディングモード
  • テクスチャファンクション
  • テクスチャフィルタ
  • テクスチャラップ

シーケンスチャンクデータのレイアウトは以下のようになっています。各領域は4バイトのアラインメントが必要です。各領域のうしろにはアラインメントのためのパディングが追加されることがあります。パディングは0のバイト列でうめられます。


シーケンスヘッダ
シーケンスヘッダのデータ構造は以下のようになっています。

typedef struct {
	u_short header_size ;	// ヘッダのサイズ（32）
	u_short unused ;	// 使用しません（0）
	u_short frame_start ;	// 開始フレーム
	u_short frame_end ;	// 終了フレーム
	u_short frame_rate ;	// フレームレート
	u_short frame_repeat ; 	// 繰り返しモード（GIM_REPEAT_XXXX）
	u_short n_constants ;	// 定数データ数
	u_short n_functions ;	// 関数データ数
	u_int constants ;  	// 定数データ配列へのオフセット
	u_int functions ;  	// 関数データ配列へのオフセット
	u_int keyframes ;  	// 先頭キーフレーム列へのオフセット
	u_int total ;		// 合計サイズ
} GimSequenceHeader ;

header_sizeはヘッダのサイズを表します。32バイト以上必要です。
frame_start、frame_endは再生範囲です（単位＝フレーム）。
frame_rateは再生速度です（単位＝フレーム／秒）。
frame_repeatは繰り返しモードです（詳細は別記）。
n_constantsは定数データの数を表します（0以上）。
n_functionsは関数データの数を表します（0以上）。
constantsはシーケンスヘッダを起点とした定数データ配列へのオフセットです。
functionsはシーケンスヘッダを起点とした関数データ配列へのオフセットです。
totalはシーケンスチャンクデータ全体のサイズです。

ユーザデータ
任意のバイト列を格納することができます。

定数データ
定数データはアニメーションされないパラメータの値を表します。定数データのデータ構造は以下のようになっています。

typedef struct {
	u_short param ;		// 対象パラメータ（GIM_PARAM_XXXX）
	short value ;		// 値
} GimConstant ;
paramは定数の値がセットされるパラメータを指定します（詳細は別記）。
valueは定数の値です。

関数データ
関数データはアニメーションされるパラメータをキーフレームで表します。関数データのデータ構造は以下のようになっています。

typedef struct {
	u_short param ;		// 対象パラメータ（GIM_PARAM_XXXX）
	u_short interp ;	// 補間方法（GIM_INTERP_XXXX）
	u_int keyframes ;	// キーフレーム列へのオフセット
} GimFunction ;

paramは関数の値がセットされるパラメータを指定します（詳細は別記）。
interpはキーフレーム間の補間方法を表します（詳細は別記）。
keyframesはシーケンスヘッダを起点としたキーフレーム列へのオフセットです。

キーフレーム
キーフレームはある時刻における関数の値を表すデータです。キーフレームのデータ構造は以下のようになっており、連続するキーはまとめて格納されます。

typedef struct {
	short count ;		// キーフレーム間隔
	short values[ n ] ; 	// キー値（可変個数）
} GimKeyframe ;

countが0以上の場合は次のキーフレームとの間隔を表し、valuesは1つだけ格納されます。countが負の場合は-countだけキーが連続することを表し、valuesは-countだけ格納されます。

例：


最後のキーフレームはcountを0にします。

3. 6 ファイル情報チャンク

ファイル情報チャンクはファイルに関する情報を格納することができます。チャンクデータはヌル文字('\0')で区切られた複数の文字列です。

文字列の意味は以下のように定義されています。
  • ProjectName(作成元のファイル名)
  • UserName(ファイルを作成したユーザー名)
  • SavedDate(ファイルを作成した日時)
  • Originator(ファイルを作成したアプリケーション名)
  • Comment(任意)

5 フォーマット定数

5. 1 ピクセル形式

GIM_FORMAT_RGBA5650


GIM_FORMAT_RGBA5551


GIM_FORMAT_RGBA4444


GIM_FORMAT_RGBA8888


GIM_FORMAT_INDEX4, INDEX8, INDEX16, INDEX32


GIM_FORMAT_DXTn
S3TCテクスチャ圧縮形式です。この形式ではイメージ内のピクセルは4×4のブロックごとに格納されます。詳しくはS3TCに関するドキュメントを参照してください。
なおイメージヘッダに含まれるbppは1ブロックの合計ビット数の1／16です。またpitch_alignは4ラインで必要とされるアラインメントの1／4です。
ブロック内のメモリレイアウトは以下のようになります。

  • GIM_FORMAT_DXT1, DXT1EXT


  • GIM_FORMAT_DXT3, DXT3EXT


  • GIM_FORMAT_DXT5, DXT5EXT


5. 2 ピクセル格納形式

GIM_ORDER_NORMAL
最も一般的なピクセル格納形式です。左から右、上から下の順で格納されます。8ビット未満のピクセルは格納先の下位ビットから格納されます。


GIM_ORDER_PSPIMAGE
PSP®においてテクスチャキャッシュに高速にロードできるピクセル格納形式です。横幅16バイト×高さ8ピクセルのブロックごとに格納されます。なおピクセル形式がS3TCのときはこの形式を指定できません。


5. 3 イメージ配列タイプ
