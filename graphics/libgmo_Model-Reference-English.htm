sceGmoModelSetMemoryManager / sceGmoModelSetMemoryManager2 / sceGmoModelSetMemoryManager3
 [Function]
Set memory management function

Syntax

void sceGmoModelSetMemoryManager(
	void *(*malloc)( size_t ),
	void (*free)( void * ) 
) ;
void sceGmoModelSetMemoryManager2(
	void *(*malloc)( size_t ),
	void (*free)( void * ) 
) ;
void sceGmoModelSetMemoryManager3(
	void *(*malloc)( size_t ),
	void (*free)( void * ) 
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  malloc     Memory allocation function  
  free     Memory release function  

Return value

None

Description

These functions set a function which allocates and a function which frees memory for a model.
sceGmoModelSetMemoryManager2() is the function for DMA memory. It stores packet and vertex data. If it is omitted, the function set by sceGmoModelSetMemoryManager() is used.
sceGmoModelSetMemoryManager3() is the function for eDRAM. It stores vertex data. If it is omitted, eDRAM allocation is not performed.

Fort the memory which is allocated for a model. if the VFPU is not used then 4-byte alignment is required. If the VFPU is used, 16-byte alignment is required. For memory which is allocated for DMA memory or eDRAM, although 4-byte alignment is required, 64-byte alignment of memory on which DMA transfers are to be performed is recommended in order to prevent unexpected behavior due to D-cache writebacks.

Example

sceGmoModelSetMemoryManager( malloc, free ) ;

Not multithread safe

Arguments

  get     Picture retrieval function  
  set     Picture registration function  

Return value

None

Description

These functions set functions which retrieve and register picture data by filename. By using these functions, when the texture of a model is to be incorporated in picture data, the registration function will be called. When the texture of a model is to not be contained in picture data, the retrieval function will be called, and external picture data can be linked to the texture. If omitted, these operations will not be performed.

sceGmoModelSetPictureManager2() is a function with sets a function which retrieves, and a function which registers, picture data by mapping type. If a texture has not been set to a layer of the model, the retrieval function is called, and the layer can be drawn using external picture data. Currently, there are no circumstances under which the registration function will be called.

Example

extern SceGimPicture *get_picture( const char *filename ) ;

sceGmoModelSetPictureManager( get_picture, 0 ) ;

	size_t size,
	size_t align

void *sceGmoModelOpenMemoryCluster2(
	size_t size,
	size_t align
) ;
void *sceGmoModelOpenMemoryCluster3(
	size_t size,
	size_t align
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  size     Size of memory cluster  
  align     Alignment (power of two)  

Return value

Returns the address of the allocated memory cluster (or 0 if allocation failed).

Description

These functions start use of a memory cluster.
Memory clusters are primarily used to prevent segmentation of heap memory.
First, a memory cluster with the specified size is allocated by a memory management function, and any subsequent memory allocation in the library will be performed with respect to this memory cluster. All allocated memory is aligned based on the value specified by align. However, if a memory cluster is insufficient, memory allocation is performed by a memory management function.
When all of the memory in a memory cluster has been released, the memory cluster is released by the memory freeing function.

sceGmoModelOpenMemoryCluster2() is the function for DMA memory. sceGmoModelOpenMemoryCluster3() is the function for eDRAM.

Example

void *addr = sceGmoModelOpenMemoryCluster( 100000, 16 ) ;
SceGmoModel *model = sceGmoModelCreate( chunk ) ;
int size = sceGmoModelCloseMemoryCluster() ;

Description

This function sets the clipping matrix. The clipping matrix is used to perform software clipping.

For matrix, specify the product of the projection matrix and the view matrix. For margin, specify the ratio of the clipping volume size to the screen volume size. When software clipping is enabled, if a triangle is placed so that it spans both the inside of the clipping volume and the outside of the screen volume, it will be clipped.

  • Decisions are made using bounding points
    (Compile with --rebuild_bounding_points on.)
  • Skinning, morphing, and 16-bit vertex color are not supported
  • Semi-transparent objects cannot be used

ScePspFVector4 margin ;

sceVfpuVector4Set( &margin, 480.0f / 4000.0f, 272.0f / 4000.0f, 1, 1 ) ;

Syntax

static __inline__ bool SCEGMO_CHUNK_ISHALF(
	const SceGmoChunk *chunk
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  chunk     Address of chunk  

Return Value

Returns true if the chunk is a half-chunk.

Description

This macro determines whether or not the specified chunk is a half-chunk.

Example

if ( SCEGMO_CHUNK_ISHALF( chunk ) ) return ;

SCEGMO_CHUNK_CHILD
 [Chunk Manipulation Macros]
Child chunk

Syntax

static __inline__ SceGmoChunk *SCEGMO_CHUNK_CHILD(
	const SceGmoChunk *chunk
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  chunk     Address of chunk  

Return Value

Returns a pointer to the first child chunk.

Description

This macro obtains a pointer to the first child chunk of the specified chunk. If the specified chunk does not have any child chunks, the macro returns a pointer to the next chunk. To determine whether or not the specified chunk has a child chunk name, get the size using SCEGMO_CHUNK_CHILDSIZE().

Example

SceGmoChunk *child = SCEGMO_CHUNK_CHILD( chunk ) ;
See Also

SCEGMO_CHUNK_CHILDSIZE()

SCEGMO_CHUNK_SKIPSTRING
 [Chunk Manipulation Macros]
Skip string

Syntax

static __inline__ void *SCEGMO_CHUNK_SKIPSTRING(
	const char *str,
	int align
) ;
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  str     Address of string argument  
  align     Alignment of next argument  

Return Value

Returns a pointer to the next argument.

Description

This macro obtains a pointer to the next argument in the string arguments.
For the alignment, specify a power of two.

Example

char *str_arg = (char *)SCEGMO_CHUNK_ARGS( chunk ) ;
int *int_arg = (int *)SCEGMO_CHUNK_SKIPSTRING( str_arg, sizeof( int ) ) ;
