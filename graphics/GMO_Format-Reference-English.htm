Arrays
 [Block]
Vertex Data

Syntax

Arrays "name" format n_verts n_morphs format2 {
	block-body
{
Description

Arrays is a subblock that represents vertex data in a Part block.

Subblocks

None

Commands

None

Arguments

 format   enum   Vertex data element. The following constants can be specified. 
 VERTEX   Vertex vector (floating-point x 3) 
 NORMAL   Normal vector (floating-point x 3) 
 COLOR   Vertex color (32-bit integer x 1) 
 POINTSIZE   Point size (floating-point x 1) 
 TEXCOORDn   Texture coordinates (floating-point x 2n) 
 WEIGHTn   Vertex weight values (floating-point x n ) 
 n_verts   int   Number of vertices. The morph shape also has the same 
 number of vertices. 
 n_morphs   int   Number of morph shapes, including the basic shape. 
 format2   enum   Vertex format of the morph shape. 

Data Format

The Arrays subblock first stores the vertex data of the basic shape.
Vertex data is structured as (data of one vertex x number of vertices).
The arrangement of data for one vertex conforms to the vertex format.
Example:
  vertex, normal, color, point-size, UV, weight // Data for one vertex  
  vertex, normal, color, point-size, UV, weight // Data for one vertex  
  vertex, normal, color, point-size, UV, weight // Data for one vertex  
  ...  

When the morph-count is 2 or more, vertex data for the morph shapes are stored consecutively.
Vertex data is structured as (data for one vertex x vertex-count x (morph-count - 1)).
The arrangement of data for one vertex conforms to the morph vertex format.
Example:
  vertex, normal   
  vertex, normal   
  vertex, normal   
  ...  

Vertex color is represented by the following bit structure.

FCurve
 [Block]
Function Data

Syntax

Fcurve "name" format n_dims n_keys{
	block-body
}
Description

Fcurve is a subblock that represents the data for one function in a Motion block.

Subblocks

None

Commands

None
Arguments

 Interpolation Format   enum   Key interpolation method (the following constants can be used) 
 CONSTANT   No interpolation 
 LINEAR   Linear interpolation 
 HERMITE   HERMITE interpolation 
 CUBIC   CUBIC interpolation 
 SPHERICAL   Spherical interpolation (for quaternions) 
 Key extrapolation method (the following constants can be used) 
 HOLD   Hold time constant 
 CYCLE   Cycle time 
 SHUTTLE   Shuttle time 
 REPEAT   Repeat (accumulate values) 
 EXTEND   Linear extension 
 n_dims   int   Number of dimensions for a key frame. Does not  
 include frame values. 
 n_keys   int   Number of key frames. 
Data Format

A function is represented by interpolating multiple key frames.
Data is structured as (key-frame x key-count).
A key frame is structured as a frame plus function values.
A function value is a vector and has just enough data for the dimension-count.
Key frames must be ordered from smallest frame value to largest.

Example:
  Frame function-value function-value function-value ... // Key frame  
  Frame function-value function-value function-value ... // Key frame  
  Frame function-value function-value function-value ... // Key frame  
   .  
   .  
   .  
    
    


However, since the HERMITE interpolation and the CUBIC interpolation include handles, the HERMITE interpolation has just enough data for (dimension-count x 3) and the CUBIC interpolation has just enough data for (dimension-count x 5).
Example: HERMITE interpolation
  Frame function-value in_dy out_dy function-value in_dy out_dy ...  
  Frame function-value in_dy out_dy function-value in_dy out_dy ...  
  Frame function-value in_dy out_dy function-value in_dy out_dy ...  
  ···  

Example: CUBIC interpolation
  Frame function-value in_dx in_dy out_dx out_dy function-value ...  
  Frame function-value in_dx in_dy out_dx out_dy function-value ...  
  Frame function-value in_dx in_dy out_dx out_dy function-value ...  
  ···  


in_dx and in_dy are for the left handle. out_dx and out_dy are for the right handle.

DefineCommand
 [Definition Commands]
Command Definition

Syntax

DefineCommand type_ID "type_name" argument_type···
Arguments

 type_ID   int   Command type ID 
 type_name   string   Command type name 
 argument_type   enum   Argument type  
 (the following constants can be used) 
 char   Signed 8-bit integer 
 short   Signed 16-bit integer 
 int   Signed 32-bit integer 
 u_char   Unsigned 8-bit integer 
 u_short   Unsigned 16-bit integer 
 u_int   Unsigned 32-bit integer 
 float   32-bit floating-point 
 string   Character string 
 Block type name   Block reference 
 Enumerated constant  
 type name 		  Enumerated constant 
 void   Any of the above types 
 ...   Variable-length argument 

Description

This command defines a command.
Type ID and type name should not overlap.
For type ID, values from 0x1000 to 0x3FFF can be used.
For type name, it is recommended to put an appropriate prefix.
When argument_type is void, argument type is estimated from the description format.
In this case, explicitly specify type qualifiers for block reference and enumerated constant such as "block type name::block name" and "enumerated constant type name::symbol name".

Example

Example 1:

 
DefineCommand 0x1020 "MyCommand" float float float

MyCommand 1.0 2.0 3.0

Example 2:

 
DefineEnum "DEBUG_MODE" "OFF" 0
DefineEnum "DEBUG_MODE "ON" 1
DefineCommand 0x1020 "MyCommand" Bone DEBUG_MODE

MyCommand "bone-0" OFF

Example 3:

 
DefineEnum "DEBUG_MODE" "OFF" 0
DefineEnum "DEBUG_MODE "ON" 1
DefineCommand 0x1020 "MyCommand" void ...

MyCommand "Bone::bone-0" DEBUG_MODE::OFF

Convert Option

Description

This command specifies an option to be directly passed to the converter. It can be specified at the beginning of a file and for certain options, can also be specified inside of a given block.

BoundingBox
 [Common Commands]
Bounding Box

Syntax

BoundingBox min_x min_y min_z max_x max_y max_z
Arguments

  min     float     Minimum value of vertex coordinate  
  max     float     Maximum value of vertex coordinate  

Description

This command defines a rectangle that encloses shape data. It is used by models, bones, parts and meshes.

The bounding box for each type of block is calculated as follows.
  • For a mesh, the local coordinate bounding box of the vertices contained in the mesh is used
  • For a part, the local coordinate bounding box for vertices contained in the part is used
  • For a bone, the combined local coordinate bounding box for the drawing parts is used
  • For a model, the combined world coordinate bounding box for all bones is used

The bounding point array for each type of block is calculated as follows.

  • For a part, the local coordinate bounding point array for vertices contained in the part is used
  • For a bone, the combined local coordinate bounding point array for the drawing parts is used
  • For a model, the combined world coordinate bounding point array for all bones is used

MorphWeights
 [Bone Commands]
Morph Weights

Syntax

MorphWeights n_weights weight ...
Arguments

  n_weights     int     Number of morph weight values  
  weight     float     Morph weight value (x number of weights)  

Description

This command specifies weights for performing morphing deformations.
A shape is deformed by animating weight values.
It is calculated as follows.

Σ (Nth morph shape) x (Nth weight)

Either MorphWeights or MorphIndex can be specified.
When both commands are omitted, no morphing is performed.

MorphIndex
 [Bone Commands]
Morph Shape Index

Syntax

MorphIndex index
Arguments

  index     float     Morph shape index  

Description

This command specifies morph shapes for performing morphing deformations.
A shape is deformed by animating indexes.

It is calculated as follows (N is the integer part and F is the decimal part of an index).

(Nth morph shape) x (1 - F) + (N+1th morph shape) x F

Either MorphWeights or MorphIndex can be specified.
When both commands are omitted, no morphing is performed.

RotateZYX / RotateYXZ / RotateQ
 [Bone Commands]
Rotation

Syntax

RotateZYX rotate_x rotate_y rotate_z
RotateYXZ rotate_x rotate_y rotate_z
RotateQ rotate_x rotate_y rotate_z rotate_w
Arguments

  rotate     float     Bone rotation angle  

Description

This command specifies the rotation angle for hierarchical coordinate calculations.
RotateZYX performs rotation by the angle specified in Z/Y/X - axis order.
RotateYXZ performs rotation by the angle specified in Y/X/Z - axis order.
RotateQ performs rotation using the quaternion (X, Y, Z, W).

Only one of RotateZYX, RotateYXZ, and RotateQ can be specified.
If none of these commands is specified, rotation calculations will not be performed.

Scale / Scale2 / Scale3
 [Bone Commands]
Scaling

Syntax

Scale scale_x scale_y scale_z
Scale2 scale_x scale_y scale_z
Scale3 scale_x scale_y scale_z
Arguments

  scale     float     Bone scaling value  

Description

This command specifies the scaling amount for hierarchical coordinate calculations.
With Scale, the scaling values directly affect the matrix stack.
With Scale2, the scaling values do not directly affect the matrix stack, but the final result after the scaling values are independently accumulated will affect the matrix stack.

Scale3 only has an effect on matrix stacks whose scaling value is at the current level. Its effect is canceled out in lower-level calculations.

Only one of Scale, Scale2 or Scale3 can be specified.
If neither of these commands is specified, scaling calculations will not be performed.

DrawBSpline
 [Mesh Commands]
Draw Spline Surface

Syntax

DrawBSpline "arrays" mode width height index ...
Arguments

 arrays   ref   Reference to vertex data to be used 
 mode   enum   Primitive mode (same as DrawArrays) 
 Edge mode can be specified as a logical OR of the  
 following constants 
 CLOSE_U   U-direction is closed 
 OPEN_U   U-direction is open 
 CLOSE_V   V-direction is closed 
 OPEN_V   V-direction is open

 height   int   Number of vertices in vertical direction 
 index   u_short   Vertex number list (width x height) 

Description

This command draws a spline curved surface.
If a knot vector has not been specified, a uniform B-spline will be used.
In this case, the state of the edges of the curved surface can be specified with Edge mode.

If a knot vector has been specified, a non-uniform B-spline will be used.
In this case, Edge mode is not used.

Animate
 [Motion Commands]
Animation

Syntax

Animate "block" command index "fcurve"
Arguments

  mode     enum     Animation control mode

This command animates other block commands according to the specified fcurve.

A block that is directly subordinate to the model can be specified for block. A Bone block or Material block is usually specified. (Which blocks are supported can vary depending on the display program.)
