FCurve
 [Blocks]
Function data

Syntax

FCurve "name" format n_dims n_keys {
	body
}
Description

This is a subblock for representing one function data sequence in a motion.

Subblocks

None

Commands

None

Arguments

 Interpolation format   enum   Key interpolation method  
 (the following constants can be used) 
 CONSTANT   No interpolation 
 LINEAR   Linear interpolation 
 HERMITE   Hermite interpolation 
 CUBIC   Cubic interpolation 
 SPHERICAL   Spherical linear interpolation (for quaternions) 
 Key extrapolation method  
 (the following constants can be used) 
 HOLD   Fix the time 
 CYCLE   Cause time to cycle 
 SHUTTLE   Cause time to shuttle 
 REPEAT   Repeat (accumulate value) 
 EXTEND   Linearly extend 
 Dimensionality   int   This is the keyframe dimensionality. The frame value is not included. 
 Number of keys   int   Number of keyframes. 
  
Data Format

A function is represented by the interpolation of multiple keyframes.
Data consists of (keyframes x number of keys).
A keyframe consists of a frame and function values.
A function value, which is a vector, has data corresponding to the dimensionality.
Keyframes must be arranged (in ascending order) from small value frames to large value frames.

Example:
  frame function_value function_value function_value . . .    // keyframe   
  frame function_value function_value function_value . . .    // keyframe  
  frame function_value function_value function_value . . .    // keyframe  
  . . .  


However, since handles are included for HERMITE interpolation and CUBIC interpolation, each of these kinds of interpolation has data corresponding to the (dimensionality x 3) or (dimensionality x 5).

Example: HERMITE interpolation
  frame function_value in_dy out_dy function_value in_dy out_dy . . .   
  frame function_value in_dy out_dy function_value in_dy out_dy . . .  
  frame function_value in_dy out_dy function_value in_dy out_dy . . .  
  . . .  

Example: CUBIC interpolation
  frame function_value in_dx in_dy out_dx out_dy function_value . . .   
  frame function_value in_dx in_dy out_dx out_dy function_value . . .  
  frame function_value in_dx in_dy out_dx out_dy function_value . . .  
  . . .  


in_dx and in_dy represent the left handle, and out_dx and out_dy represent the right handle.

DefineEnum
 [Definition Commands]
Enumeration constant definition

Syntax

DefineEnum "type_name" "symbol_name" symbol_value
Arguments

  type_name     string     Enumeration constant type name  
  symbol_name     string     Symbol name  
  symbol_value     int     Symbol value  

Description

This command defines a symbol for an enumeration constant.
Make sure that the type name does not duplicate the name of a block or command.
A suitable prefix should be appended to the type name.

Example

DefineEnum "DEBUG_MODE" "OFF" 0x0000
DefineEnum "DEBUG_MODE" "ON" 0x0001
DefineBlock
 [Definition Commands]
Block definition

Syntax

DefineBlock type_id "type_name"
Arguments

  type_id     int     Block type ID  
  type_name     string     Block type name  

Description

This command defines a block.
Make sure that the type ID or type name does not duplicate another ID or name.
The values 0x1000 to 0x3FFF can be used for type IDs.
A suitable prefix should be appended to the type name.

Example

DefineBlock 0x1010 "MyBlock"

MyBlock "block-0" {
	...
}
DefineCommand
 [Definition Commands]
Command definition

Syntax

DefineCommand type_id " type_name " arg_type . . .
Arguments

 type_id   int   Command type ID 
 type_name   string   Command type name 
 arg_type   enum   Argument type (the following constants can be used) 
 char   Signed 8-bit integer 
 short   Signed 16-bit integer 
 int   Signed 32-bit integer 
 u_char   Unsigned 8-bit integer 
 u_short   Unsigned 16-bit integer 
 u_int   Unsigned 32-bit integer 
 float   32-bit floating-point number 
 string   String 
 Block type name   Block reference 
 Enumeration constant type name   Enumeration constant 
 void   Any of the above types 
 ...   Variable-length argument 

Description

This command defines a command.
Make sure that the type ID or type name does not duplicate another ID or name.
The values 0x1000 to 0x3FFF can be used for type IDs.
A suitable prefix should be appended to the type name.

When the argument type is void, the argument type is inferred from the description format.
In that case, for a block reference or enumeration constant, you should explicitly specify a type qualifier such as "block type name::block name" or "enumeration constant type name::symbol name."

Example

Example 1:

 
DefineCommand 0x1020 "MyCommand" float float float

MyCommand 1.0 2.0 3.0

Example 2:

 
DefineEnum "DEBUG_MODE" "OFF" 0
DefineEnum "DEBUG_MODE "ON" 1
DefineCommand 0x1020 "MyCommand" Node DEBUG_MODE

MyCommand "node-0" OFF

Example 3:

 
DefineEnum "DEBUG_MODE" "OFF" 0
DefineEnum "DEBUG_MODE "ON" 1
DefineCommand 0x1020 "MyCommand" void ...

MyCommand "Node::node-0" DEBUG_MODE::OFF

FogRange
 [Fog Commands]
Fog range

Syntax

FogRange min max
Arguments

  min     float     Fog starting distance  
  max     float     Fog ending distance  

Description

This command specifies the fog range. It is used when the fog type is LINEAR.
The default value differs according to the display program.

The fog mixing coefficient f is calculated as follows (where the distance from the camera is denoted by D).

f = ( D - min ) / ( max - min )

(However, when D < min, f = 0.0, and when D > max, f = 1.0.)

FogDensity
 [Fog Commands]
Fog density

Syntax

FogDensity density
Arguments

  density     float     Fog density  

Description

This command specifies the fog density. It is used when the fog type is EXP or EXP2.
The default value differs according to the display program.

The fog mixing coefficient f is calculated as follows (where the distance from the camera is denoted by D).

f = 1 - exp( - D * density ) (when type is EXP)
f = 1 - exp( - D * D * density * density ) (when type is EXP2)

LightAtten
 [Light Commands]
Light attenuation coefficient

Syntax

LightAtten atten0 atten1 atten2
Arguments

  atten0     float     0th order attenuation coefficient  
  atten1     float     1st order attenuation coefficient  
  atten2     float     2nd order attenuation coefficient  

Description

This command specifies the attenuation coefficients of the light. It is used when the light type is POINT or SPOT.
The default value differs according to the display program.

The light intensity f is calculated as follows (where the distance from the light is denoted by D).

f = 1 / ( atten0 + atten1 * D + atten2 * D * D )

LightCone / LightCone2
 [Light Commands]
Light cone coefficient

Syntax

LightCone falloff field
LightCone2 falloff field beam
Arguments

  falloff     float     Attenuation coefficient  
  field     float     Vertex angle with minimum brightness (units = radians)  
  beam     float     Vertex angle of cone with maximum brightness (units = radians)  

Description

This command specifies the cone coefficients of the light. It is used when the light type is SPOT.
The default value differs according to the display program.

Each light intensity is calculated as follows (where the angle with the forward axis is denoted by R).

f = pow( cos( R ), falloff )
f = pow( ( cos( R ) - cos( field / 2 ) ) / ( cos( beam / 2 ) - cos( field / 2 ) ), falloff )

(However, when R > field / 2, the intensity f = 0.0.)

 [Node Commands]
Rotation

Syntax

RotateZYX rotate_x rotate_y rotate_z
RotateYXZ rotate_x rotate_y rotate_z
RotateQ quat_x quat_y quat_z quat_w
Arguments

  rotate     float     Rotation angle (units = radians)  
  quat     float     Quaternion  

Description

This command specifies the rotation angle for hierarchical coordinate calculations.
RotateZYX performs rotations according to the specified angle in Z-axis, Y-axis, X-axis order.
RotateYXZ performs rotations according to the specified angle in Y-axis, X-axis, Z-axis order.
RotateQ performs rotation according to the (X, Y, Z, W) quaternion.

Any one of RotateZYX, RotateYXZ, and RotateQ can be specified.
If they are all omitted, no rotation calculation is performed.

Scale / Scale2 / Scale3
 [Node Commands]
Scaling

Syntax

Scale scale_x scale_y scale_z
Scale2 scale_x scale_y scale_z
Scale3 scale_x scale_y scale_z
Arguments

  scale     float     Scaling value  

Description

This command specifies scaling for hierarchical coordinate calculations.
For Scale, the scaling value directly affects the matrix stack.
For Scale2, the scaling value does not directly affect the matrix stack, but the result when scaling values are independently added ultimately affects it.
For Scale3, the scaling value only affects the matrix stack for the current level. The effect is offset in lower level calculations.

Any one of Scale, Scale2, and Scale3 can be specified.
If they are all omitted, no scaling calculation is performed.

Animate
 [Motion Commands]
Animation

Syntax

Animate "block" command mode "fcurve"
Arguments

  block     ref     Reference to block to be animated  
  command     enum     Type of command to be animated  
  mode     enum     Animation control mode  
  fcurve     ref     Reference to function data  

Description

This command animates the commands of another block according to the specified function data.

The block and command arguments specify the block and commands to be animated. The combination of blocks and commands that can be animated differ according to the display program.

The mode argument specifies the subblock number and subchannel number to be animated.
The subblock is represented by INDEXn, and the subchannel is represented by OFFSETn (n = 0 to 255).
These values can be specified according to a logical OR operation. They should always be set to 0.

The fcurve argument specifies function data within the same motion.
