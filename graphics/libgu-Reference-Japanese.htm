sceGuStart
 [関数]
ディスプレイリストの作成開始

形式

#include <libgu.h>
int sceGuStart (
	int mode,
	void *p,
	int size
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

  mode     作成するディスプレイリストのモード  
  p     ディスプレイリスト作成領域の先頭アドレス  
  size     ディスプレイリスト作成領域のサイズ  

modeには以下のマクロを指定します。

  マクロ     説明  
  SCEGU_IMMEDIATE     ディスプレイリストを作成しながら描画  
  SCEGU_CALL     CALL用のディスプレイリストを非キャッシュで作成  
  SCEGU_LIST     sceGuSendList()用のディスプレイリストを  
  非キャッシュで作成  
  SCEGU_CALL_CACHED     CALL用のディスプレイリストをDキャッシュ経由で作成  
  SCEGU_LIST_CACHED     sceGuSendList()用のディスプレイリストを  
  Dキャッシュ経由で作成  


返り値

エラーの場合、負の値が返ります。
  値     結果  
  SCE_OK     正常終了  
  <0     エラー  


解説

ディスプレイリストの作成を開始する関数です。この関数に続いて、モード設定関数やプリミティブ関数などを用いてディスプレイリストを作成していきます。
modeにSCEGU_IMMEDIATEを指定した場合、ディスプレイリストを作成しながら同時にGraphics Engineに転送して描画するモードとなります。
modeにSCEGU_LISTを指定した場合は、リストを作成するだけで描画は行われません。作成したディスプレイリストは、sceGuFinish()で作成終了した後で、あらためてsceGuSendList()を使用して描画することになります。
modeにSCEGU_CALLを指定した場合も、ディスプレイリストを作成するだけで描画は行われません。作成したディスプレイリストは、sceGuFinish()で作成終了した後、親となる他のディスプレイリストを作成中にsceGuCallList()を使用して呼び出すように設定してください。親のディスプレイリストを描画するときに、呼び出されて描画されることになります。
modeにSCEGU_LIST_CACHEDを指定した場合はSCEGU_LISTと同じですが、キャッシュアドレス経由でリストを生成しますので、sceGuSendList()を使用して描画する前に明示的にDキャッシュをライトバックする必要があります。
modeにSCEGU_CALL_CACHEDを指定した場合はSCEGU_CALLと同じですが、キャッシュアドレス経由でリストを生成しますので、sceGuCallList()を使用して呼び出すリストの描画が開始されるよりも前に明示的にDキャッシュをライトバックする必要があります。

pには作成したディスプレイリストを格納する領域の先頭アドレスを、sizeにはその領域のサイズを指定してください。なおディスプレイリストはリングバッファになっていません。サイズをオーバーするとメモリ破壊を起こしますので、作成中は適宜sceGuCheckList()などを用いて使用サイズをチェックしてください。

ディスプレイリストのモードごとに別々にコンテキストを保持しているため、modeが異なれば多重にsceGuStart()を呼ぶことができます。たとえば、SCEGU_IMMEDIATEでディスプレイリストを作成・描画している間に、SCEGU_LISTで別のディスプレイリストを作成することが可能です。
一方、コンテキストを変更する次の関数は、ディスプレイリスト作成時にモードごとに呼び出す必要があります。
  • sceGuScissor()
  • sceGuEnable(SCEGU_SCISSOR_TEST)
  • sceGuDisable(SCEGU_SCISSOR_TEST)
  • sceGuEnable(SCEGU_COLOR_DOUBLE)
  • sceGuDisable(SCEGU_COLOR_DOUBLE)
  • sceGuDepthRange()
  • sceGuDepthOffset()
  • sceGuSpriteMode()
  • sceGuClearColor()
  • sceGuClearStencil()
  • sceGuClearDepth()
  • sceGuTexMode()
  • sceGuTexFunc()
  • sceGuTexMapMode()
  • sceGuTexProjMapMode()

使用例

static char disp_list[0x10000];	// 領域の確保
sceGuStart( SCEGU_IMMEDIATE, (void *)disp_list, sizeof(disp_list) );	// 領域の設定

sceGuSendList
 [関数]
ディスプレイリストのキューイング

形式

#include <libgu.h>
int sceGuSendList (
	int mode,
	const void *p,
	SceGuContext *ctx,
	int nStackDepth,
	SceGeStack *stack
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

  mode     キューイングモード  
  p     ディスプレイリスト  
  ctx     コンテキスト保存領域のポインタ  
  nStackDepth     SIGNAL CALLスタック段数  
  stack     SIGNAL CALLスタックバッファ  

modeには以下のマクロを指定します。

  マクロ     説明  
  SCEGU_QUEUE_TAIL     キューの最後に追加  
  SCEGU_QUEUE_HEAD     キューの先頭に追加  


返り値

エラーの場合、負の値が返ります。
  値     結果  
  SCE_OK     正常終了  
  <0     エラー  


解説

作成済みのディスプレイリストをGraphics Engineに転送して描画する関数です。
pには、SCEGU_LISTモードで作成してあるディスプレイリストの先頭アドレスを指定します。
modeにSCEGU_QUEUE_TAILを指定すると、pで指定したディスプレイリストは描画キューの末尾に加えられ、すでにキューイングされている他のディスプレイリストの描画終了後に描画されることになります。
modeにSCEGU_QUEUE_HEADを指定すると、pで指定したディスプレイリストは描画キューの先頭に加えられ、すでにキューイングされている他のディスプレイリストよりも先に描画されることになります。
ctxにコンテキスト保存領域のポインタを指定すると、pで指定したディスプレイリストを描画する直前にGraphics Engineの内部状態が保存され、描画終了後に復帰されるようになります。コンテキストを破壊してもかまわない場合、またはpで指定したディスプレイリストがコンテキストを変更しない場合は、ctxにNULLを指定してください。

使用例

static char disp_list[0x1000];
SceGeStack stk [32];

sceGuStart(SCEGU_LIST, disp_list, sizeof(disp_list));	// ディスプレイリストの開始
	:
sceGuFinish();					// リスト終了
sceGuSendList(SCEGU_QUEUE_TAIL, disp_list, NULL, 32, stk);
						// リストのキューイング

sceGuFlush
 [関数]
ディスプレイリストの途中まで実行

形式

#include <libgu.h>
int sceGuFlush (
	void
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

なし

返り値

エラーの場合、負の値が返ります。
  値     結果  
  SCE_OK     正常終了  
  <0     エラー  


解説

SCEGU_IMMEDIATEモードで作成中であるディスプレイリストを、現在の位置まで実行させます。
通常、SCEGU_IMMEDIATEモードでディスプレイリストを生成している場合は、sceGuDrawArray()など描画命令に対応するディスプレイリストが出力されたタイミングで、自動的にlibguがsceGeListUpdateStallAddr()関数を呼び出します。sceGuFlush()関数を使用すると、任意のタイミングでsceGeListUpdateStallAddr()関数の呼び出しを発生させる事ができます。
sceGuFlush()は、SCEGU_IMMEDIATEモードのsceGuStart()〜sceGuFinish()間で使用で来ます。他のリストモードの場合には使用できません。またSCEGU_IMMEDIATEモードであっても、sceGuBeginObject()〜sceGuEndObject()間ではストールアドレスを更新できないので、sceGuFlush()関数は何もしません。

使用例

static char disp_list[0x1000];

sceGuStart(SCEGU_IMMEDIATE, disp_list, sizeof(disp_list));
						// ディスプレイリストの開始

  SCEGU_SIGNAL_PAUSE     描画を中断する

解説

シグナル割り込みコマンドを作成中のディスプレイリストに挿入する関数です。
ディスプレイリストの描画時に、このコマンドに達した時点でCPUに対して割り込みが発生します。modeはそのときの描画動作を指定します。idはシグナルIDとして、シグナルコールバックに渡されます。

備考

ひとつのディスプレイリストに複数のシグナル割り込みコマンドを挿入する場合、それらのシグナルIDは互いに異なっていなければなりません。
また、16個以上のシグナル割り込みコマンドを挿入した場合、sceGuSignalSync()による割り込み待ちが正しく動作しない場合があります。

SCEGU_SIGNAL_WAITはGraphics EngineマネージャのSCE_GE_SIGNAL_SIGNAL_WAITによるシグナルを使って実現されます。この場合SIGNALハンドラ呼び出しはディスプレイリスト走査が一時停止した状態のまま行われます。ディスプレイリストが停止していても停止前に発効された描画コマンドによってGraphics Engineの描画動作自体は継続中であるため、SIGNALハンドラ内で描画フレームバッファの内容
を読み書きすることはできません。SIGNALハンドラを終了するとGraphics Engineマネージャ内で自動的にディスプレイリストの動作が再開します。

SCEGU_SIGNAL_NOWAITはGraphics EngineマネージャのSCE_GE_SIGNAL_SIGNAL_NOWAITによるシグナルを使って実現されます。この場合SIGNALハンドラ呼び出しは、Graphics Engineマネージャがディスプレイリストを動作再開してから行いますので、SIGNALハンドラ内で描画フレームバッファの内容を読み書きすることはできません。

SCEGU_SIGNAL_PAUSEはGraphics EngineマネージャのSCE_GE_SIGNAL_SIGNAL_PAUSEによるシグナルを使って実現されます。この場合SIGNALハンドラ呼び出しは、Graphics Engineマネージャは描画処理が一時的に完了した状態を確認してから行いますので、SIGNALハンドラ内で描画フレームバッファの内容を読み書きする事ができます。また、sceGuSendList()のSCEGU_QUEUE_HEADを使って作成済みの別のリストを割り込ませることも可能です。
SIGNALハンドラを終了してもGraphics Engineマネージャ内で自動的にディスプレイリストの動作が再開しないため、明示的にsceGuContinue()を呼び出して動作を再開させる
必要があります。

modeに指定するマクロは以下のとおりです。
  マクロ     説明  
  SCEGU_PF5650     16bit 5:6:5:0 フォーマット  
  SCEGU_PF5551     16bit 5:5:5:1 フォーマット  
  SCEGU_PF4444     16bit 4:4:4:4 フォーマット  
  SCEGU_PF8888     32bit 8:8:8:8 フォーマット  


返り値

なし

解説

フレームバッファ上の矩形領域を各引数の指定に従ってコピーするコマンドシーケンスを、作成中のディスプレイリストに挿入する関数です。

sx, sy, dx, dyは0〜1023の値を、width, heightは1〜1023の値をとりますが、ソースおよびデスティネーションバッファの横幅をラップアラウンドするような転送、sx + widthが1023を超える転送、dx + widthが1023を超える転送は行えません。
sbw, dbwは、8の倍数で8〜1024の値をとります。
sbp, dbpは、16バイトアラインメントである必要があります。

フレームバッファ上にコピーされた内容をテクスチャとして参照するためには、参照に先立ってsceGuTexFlush()およびsceGuTexSync()を実行する必要があります。sceGuTexFlush()はテクスチャキャッシュの内容を無効化し、新しいデータをテクスチャとして認識させるために必要です。sceGuTexSync()はsceGuCopyImage()によって実行される矩形転送が完了するまでテクスチャ読み出しをブロックするために必要です。
フレームバッファ上にコピーされた内容をテクスチャ以外の目的（頂点データ, 頂点インデックスデータ, ディスプレイリストなど）としてGraphics Engineが参照する場合は、ディスプレイリストがCMD_FINISH + CMD_ENDによっていったん終了される必要があり、libguではSCEGU_CALLモード, SCEGU_IMMEDIATEモードのsceGuFinish()による終了、またはsceGuSignalDrawSync()による同期が必要になります。つまり、sceGuCopyImage()を含むディスプレイリストの中で、コピー先をそのままテクスチャ以外の目的で使用してはいけません。また、SCEGU_LISTモードの場合はsceGuFinish()はCMD_RETによって実行されるため同期にならない点にも注意してください。

使用例

sceGuCopyImage(SCEGU_PF5551, 0, 0, 128, 128, 512, (void *)0x10000000,
	0, 0, 512, (void *)0x20000000);
sceGuTexFlush();
sceGuTexSync();
