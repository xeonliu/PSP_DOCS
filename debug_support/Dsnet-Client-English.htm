PSP™ Programmer Tool Runtime Library Release 6.3.0

dsnet for PSP™ Overview

1 Overview

1. 1 DECI2P Debugging Environment

"dsnet for PSP™" is a debugging environment that uses the DECI2P protocol. It enables the host and the PSP™ tool to work together so that applications that will run on the target can be developed more efficiently.

In the DECI2P debugging environment, communication with the target is performed using DECI2P packets. These packets conform to the DECI2P protocol and higher level protocols. Figure 1 shows a block diagram of the DECI2P debugging environment.



Figure 1 DECI2P Debugging Environment


A client application on the host PC communicates with "dsnetm_psp", which runs on
the PSP™ tool. Communication is performed using TCP/IP sockets

"dsnet for PSP™" is a client application that runs on the host PC and communicates with dsnetm_psp.

2 Using Each Client

2. 1 -d option

For each client program of "dsnet for PSP™", the -d option is used to specify the host name and port to connect to.

  -d Option     Host Name     Port Number  
  -d <host>     <host>     8510  
  -d <host>[:<port>]     <host>     <port>  
  -d :<port>     default-host     <port>  


The -d option specifies the host name and port of the dsnetm_psp that will be the communication target.
If the -d option is not specified, the default port (8510) on the same machine (default-host) as the one where the client program was executed will be used.

2. 2 DSNETM_PSP Environment Variable

By using the environment variable DSNETM_PSP, the connection target can be specified without having to specify the -d option for every "dsnet for PSP™" command.

export DSNETM_PSP=psptool.rd.scei.sony.co.jp

It is convenient to use this in combination with the "send host name to DHCP server" setting on the network configuration page of the DTL-T1000.

2. 3 dstdb - DS Thread debugger

dstdb [<options>] <cmd> [<arg>]...]
dstdb is a source-level/instruction-level debugger.
It supports both standard input/output and file access from the target.
For details about how to use dstdb, refer to the "dstdb Command Reference."

  Option     Function  
  -d <host>[:<port>]     dsnetm_psp host and port specification  
  -ncmv     Do not check version of manager (default)  
  -cmv     Check version of manager  
  -r     Reset target during startup  
  -nr     Do not reset target during startup (default)  
  -nx     Do not execute autorun files during startup  
  -sbootp <sbootp>     Boot parameter initial value specification  
  -nokbd     Do not use keyboard when specifying <arg>...  
  -noexit     Do not exit when specifying <arg>...  
  -p <port>     Remote console port specification  
  -c <crash dump file>     Post-mortem debug specification  
  -j <device>     Controller operation relay specification (linux)  
  -j     Controller operation relay specification (Win32)  


Normally, dstdb is started up and debugging is performed interactively as shown below.

$ dstdb                                 # Start up debugger
dstdb ORO> bloadp abooti.bin            # Load boot code
dstdb ORO> reset                        # Reset target
   ...
dstdb ORO> help                         # Display command help
   ...
dstdb ORO> mload sample.prx             # Load program
   ...
dstdb ORO> bp 0x8807e30c                # Set breakpoint at 0x8807e30c
dstdb ORO> mstart -d                    # Execute program in debug mode
   ...
at=deadbeef v0-1==deadbeef,...
   ...
dstdb ORO> db 0x8825f774                # Dump address 0x8825f774
   ...
dstdb ORO> step                         # Perform step execution
   ...
dstdb S> cont                           # Perform continuous execution
   ... ^C                               # Forcibly interrupt execution with CTRL-C
dstdb S> dr                             # Display registers
   ...
dstdb S> quit                           # Terminate dstdb
$
When you have a crash dump file that was generated by a BSOD, you can start up dstdb with the -c option as shown below to perform postmortem debugging.

$ dstdb -c pspcore.elf                 # Start up the debugger
If the -j option of dstdb is specified, the operations of a PlayStation®3 SIXAXIS™ wireless controller or DUALSHOCK®3 wireless controller that is connected to a host PC via USB can be relayed to the target. However, the wireless feature of the SIXAXIS™ wireless controller or DUALSHOCK®3 wireless controller cannot be used when running in this mode.

In linux, use the -j option to specify the device file of the joystick device (such as /dev/js0). The linux kernel must be 2.6.2 or later.

$ dstdb -j /dev/js0                    # Start up debugger
In Windows, the libpad_win.dll file of libpad for Windows must be installed in the same directory as dstdb.exe. The -j option is specified by itself; you do not need to specify the device file.

$ dstdb -j                             # Start up debugger

2. 4 dspcons

dspcons [-d <host>[:<port>]] [-e <escape>] -nokbd -ns [0-9k]...
dspcons is a console program that is the destination of TTYP input/output of the target program.

TTYP has the following 11 protocol numbers.

S0TTYP, S1TTYP, ..., S9TTYP, SKTTYP

When the argument [0-9k]... is specified, dspcons receives only the specified protocols and displays data it receives on-screen. If the argument is not specified, dspcons receives all 11 TTYP protocols and displays the data it receives on-screen.
The protocol number that is specified first is used to send key input data to the target.
If this argument is not specified, S0TTYP is used to send key input data.
If the -nokbd option is specified, all keyboard input other than <escape> is ignored.
If the -ns option is specified, error messages from the Remote File Access server are not displayed.
The method of specifying [0-9k]... is shown below.

dspcons 1 2 3          # S1TTYP,S2TTYP,S3TTYP
dspcons 1-3 k          # S1TTYP,S2TTYP,S3TTYP,SKTTYP

When the key input is <escape>, the console program terminates. All other key input is sent to the target program as key input.
The default for <escape> is CTRL-] (0x1d). To specify a control code for <escape>, append a carat "^" in front of the character. For example, specify "-e ^C".

2. 5 dsxfilesv

dsxfilesv [-d <host>[:<port>]] [-i] [-c <cmd>] [-t <sec>]
dsxfilesv is a file server program for processing Remote File Access requests that are sent from the target program. Since dstdb normally processes Remote File Access requests, dsxfilesv need not be used except when you want to make the current directory during Remote File Access a fixed location that does not depend on the current directory of dstdb.

To use dsxfilesv, you must prohibit Remote File Access handling by dstdb. Set the following line in the dstdb startup file (~/.dstdbrc or ./.dstdbrc).

option file_priority=-1

Unless the -i option is specified, dsxfilesv will operate in the background.
To terminate dsxfilesv, use the kill command.
If the -c <cmd> option is specified, the specified shell command (<cmd>) will be executed, and the program will terminate when the following conditions are satisfied.

  1. All files that were opened according to Remote File Access are closed.
  2. The shell command (<cmd>) has ended.
  3. The state described in (1) has continued for the number of seconds (default: 1 second) that was specified according to -t <sec>.

2. 6 dspstat

dspstat [-d <host>[:<port>]]
dspstat is a client program for displaying a list of client programs that are connected to the entire target. It is used to identify people who are currently using the target.
dspstat is used together with dspkill.

2. 7 dspkill

dspkill [-d <host>[:<port>]] [protocol]
dspkill is a program for forcibly disconnecting a client that is connected to a specific protocol from the client programs that are connected to the entire target. dspkill is used together with dspstat to forcibly remove people who are currently occupying the target without notice.

2. 8 dspmcons

dspmcons [-d <host>[:<port>]] ...
dspmcons is a dedicated console program for running the dsnetm_psp debug console, which runs as CP on the PSP™ TOOL. Normally it is not necessary to use dspmcons, however, it can be used for debugging when communicating with an original protocol on top of the DECI2p protocol and when referencing the packet log recorded by dsnetm_psp.

2. 9 dsbloadp

dsbloadp [-d <host>[:<port>]] <fname>
dsbloadp is a client program for sending the PSP™ boot code to the target. It corresponds to dstdb's bloadp command.
dsbloadp is used together with dspreset.

2. 10 dspreset

dspreset [-d <host>[:<port>]] [sbootp]
dspreset is a client program for resetting the entire target.
It is used together with dsbloadp.

Boot parameters can be specified by using sbootp. The following boot parameters are shared with dstdb's reset command.

Boot Parameters
  bit     Description  
  bit 10     Memory selection 32MiB/64MiB switching bit  
  0 = 32MiB  
  1 = 64MiB  
  Note that within the memory area described above, 8 MiB are also  
  used by the system. Also, in 64 MiB mode, an additional 8 MiB are  
  set aside as a reserved area. Therefore, in each mode, the size that  
  can be freely used by the user is 24 MiB when this is OFF and 48  
  MiB when it is ON.  
  bit 11     Startup switching bit for PLL clock that can coexist with WLAN  
  0 = 222 MHz (corresponds to PSP-1000)  
  1 = 333 MHz (corresponds to PSP-2000 and later)  
  bit 12     UMD™ cache switching bit  
  0 = Disable UMD™ cache  
  1 = Enable UMD™ cache  
  Note that the bit 12 setting is disabled when bit 10 is 1.  
  bit 15     Ether over DECI2p switching bit  
  0 = Use WLAN  
  1 = Use Ether over DECI2p  
  If Ether over DECI2p is selected, Ethernet packets will be sent and  
  received via DECI2p instead of via the WLAN.  
  bit 17     Memory Stick™ LOCK switch emulation according to GPI6  
  0 = No LOCK switch emulation  
  1 = LOCK switch emulation is enabled   
  bit 18     Read speed when using DVD  
  0 = Read speed is equivalent to UMD™  
  1 = Use maximum speed of DVD drive  
  bit 19     UMD™/DVD read random delay  
  0 = No random delay  
  1 = Random delay  
  bit 20     WLAN packet loss  
  0 = No random packet loss  
  1 = Random packet loss  
  bit 22     Profiler mode switching bit  
  0 = Global mode (thread manager does not manage profiler register)  
  1 = Thread mode (profiler register is managed for each thread)  
  bit 23     Profiler enable/disable switching bit  
  0 = Profiler disable  
  1 = Profiler enable  
  bit 24     Memory block overflow check enable/disable switching bit  
  0 = Disable overflow check  
  1 = Enable overflow check  
  bit 29     Development environment/system software switching bit  
  0 = Development environment  
  1 = System software  
  Other bits     Reserved  


2. 11 dsxstart

dsxstart [-d <host>[:<port>]] -sc [-c <cmd>] [-t <sec>] <fname> [<arg>…]
dsxstart is a client program for loading modules on the target. It operates in a similar manner as dstdb's mstart command.

2. 12 dsxlist

dsxlist [-d <host>[:<port>]] -sc [-c <cmd>] [-t <sec>]
dsxlist is a client program for displaying a list of modules that are loaded on the target. It operates in a similar manner as dstdb's minfo command.

2. 13 dsxping

dsxping [-d <host>[:<port>]] -sc [-c <cmd>] [-t <sec>]
dsxping is a client program for diagnosing whether or not there is a response from the target program. Normally, this client program need not be used.

2. 14 dsdcons

dsdcons [-d <host>[:<port>]] [-e <escape>] -nokbd -ns [0-9k]...
dsdcons is a console program that is the companion to dstdb's remote console input/output function.

If the remote console is specified with the -p option of dstdb, dstdb opens a DTTYP protocol server on the specified port number so that it can be used by the remote console. By using dsdcons to connect to that port, you can perform dstdb commands remotely.

$ dstdb -p 1234                        # Start up debugger with a
                                       # remote console at port 1234
dstdb 0R0> 
   ...

From this state, dsdcons can be used to connect to the machine that is executing dstdb. Note that the -d option of dsdcons is used at this time to specify the host name where dstdb is being executed and the port number.

$ dsdcons -d localhost:1234            # Connect to remote console
dsdcons (Version 2.24.6 Sat Feb 18 22:09:20 JST 2006)
target=HOST display=DTTYP keyboard=DTTYP escape=^]
dstdb 0R0>                              # dstdb console appears.
dstdb 0R0> dr                           # dstdb can be manipulated.
   ...

When <escape> is input from the keyboard, processing ends. Otherwise, all key input is sent to the target program.
The default for <escape> is CTRL-] (0x1d). To specify a different control code for <escape>, use the -e option and append a caret (^) before a character such as "-e ^C."
Even when dsdcons terminates the connection, dstdb is not terminated. Note that dstdb commands themselves are terminated when the dstdb quit command is executed from dsdcons.

2. 15 dsptun

dsptun [-d <host>[:<port>]] -tapif <tapif>
dsptun is a program that relays packets wrapped in the ETHERP protocol from the target to a linux tun device, when the Ether over DECI2p feature is enabled by setting bit 15 of the boot parameter. dsptun requires linux kernel 2.6.23.1 or later.
