PSP™ Programmer Tool Runtime Library Release 6.3.0

dstdb Command Reference

1 Line Input

When entering commands, the following editing keys can be used besides the arrow keys.

  Editing Key     Function  
  CTRL-P     Move to the previous line  
  CTRL-N     Move to the next line  
  CTRL-B     Move back to the previous character (left)  
  CTRL-F     Move forward to the next character (right)  
  CTRL-A     Move to the start of the current line  
  CTRL-E     Move to the end of the current line  
  BS, DEL     Delete the previous character  
  CTRL-D     Delete the current character  
  CTRL-X     Delete the entire current line  
  CTRL-U     Delete the entire current line  
  CTRL-K     Delete from the current character to the end of the current line  
  CTRL-C     Suspend input (with ^C echo)  
  CTRL-G     Suspend input (without ^G echo)  
  LF, CR     End input and execute current line as a command  
  TAB     Complete command name, file name, or symbol name  
  CTRL-R     Reverse search  
  CTRL-S     Forward search  
  CTRL-Y     Insert string that was deleted by CTRL-K  


TAB can be used to complete a line, but only when the TAB is input at the end of the line. This applies to the following four cases.
  1. String that does not contain a space
    Attempts to complete as a command name.
  2. Argument of the set command
    Attempts to complete as an option name.
  3. Argument of the help command
    Attempts to complete as a help keyword.
  4. Argument of other commands
    If the string contains a "/" or ".", attempts to complete as a filename. Otherwise, attempts to complete as as a symbol.
During program execution, entering CTRL-C sends a break command to the target. Entering CTRL-G does not sent a break command, but simply returns control to dstdb.
The RETURN key enables the previously executed command (such as display memory contents or perform step execution) to be reexecuted. When the command is reexecuted (such as display memory contents), the address is incremented at the same time.

2 Expressions

The following expressions (expr) can be used for command arguments that take a general value other than a filename.

  Expression     Description  
  Hexadecimal number     # 0x can be omitted as long as the value does not duplicate a symbol  
  symbol     # Symbol  
  <fname>:<lineno>     # Specification by line number   
  <fname>:<func>     # Specification of static function with duplicate name   
  .     # Current address (dot address)  
  .<digit>     # Numeric value (octal, decimal, or hexadecimal) in any  
  of the formats used in the C language  
  $<reg>     # Current register value or user variable value  
  (expr)     # Parentheses used in a formula  
  +expr     # Explicit plus sign  
  -expr     # Sign inversion  
  ~expr     # Bit inversion  
  *expr     # Indirect address specification (destination pointed to by  
  address is always a word)  
  expr1[expr2]     # Equivalent to *(expr1 + ((expr2) << 2))  
  expr1[expr2,b]     # expr1 + ((expr2) << 0) indirect address specification (byte)  
  expr1[expr2,h]     # expr1 + ((expr2) << 1) indirect address specification (half)  
  expr1[expr2,w]     # expr1 + ((expr2) << 2) indirect address specification (word)  
  expr1[expr2,d]     # expr1 + ((expr2) << 3) indirect address specification (double)  
  expr1[expr2,q]     # expr1 + ((expr2) << 4) indirect address specification (quad)  
  expr1[expr2,expr3]     # Address indirect specification (expr3 is byte count)  
  !expr     # Logical inversion  
  expr1 || expr2     # Logical OR  
  expr1 && expr2     # Logical AND  
  expr1 | expr2     # Bitwise OR  
  expr1 ^ expr2     # Bitwise XOR  
  expr1 & expr2     # Bitwise AND  
  expr1 == expr2     # True (= 1) when expressions are equal; false (= 0) otherwise  
  expr1 != expr2     # True (= 1) when expressions are not equal; false (= 0) otherwise  
  expr1 < expr2     # Unsigned comparison (less than)  
  expr1 <= expr2     # Unsigned comparison (less than or equal)  
  expr1 > expr2     # Unsigned comparison (greater than)  
  expr1 >= expr2     # Unsigned comparison (greater than or equal)  
  expr1 >> expr2     # Shift expr1 to the right by expr2 bits (both values are unsigned)  
  expr1 << expr2     # Shift expr1 to the left by expr2 bits (both values are unsigned)  
  expr1 + expr2     # Arithmetic addition of expr1 and expr2 (both values are unsigned)  
  expr1 - expr2     # Arithmetic subtraction of expr1 and expr2 (both values are unsigned)  
  expr1 * expr2     # Arithmetic multiplication of expr1 and expr2 (both values are unsigned)  
  expr1 / expr2     # Arithmetic division of expr1 and expr2 (both values are unsigned)  
  expr1 % expr2     # Arithmetic modulus of expr1 and expr2 (both values are unsigned)  


The precedence of parentheses and unary and binary operators is the same as in C, as shown below.

  Parentheses and Unary and Binary Operators     Priority  
  () []     High  
  ! + - ~ * (unary operators)     ↑  
  * / %     ↑  
  + - (binary operators)     ↑  
  << >>     ↑  
  < <= > >=     ↑  
  == !=     ↑  
  &     ↑  
  ^     ↑  
  |     ↑  
  &&     ↑  
  ||     Low  


Unless otherwise specified, a numeric value within an expression (expr) is interpreted as a hexadecimal number. Exceptions are the "10" of $10 or a line number within a symbol.

The definition of a hexadecimal number is [0[xX]][0-9a-fA-F]+. No distinction is made between uppercase and lowercase letters. To specify a decimal number at a location where a hexadecimal number can be used, place a "." before the digits. To specify an octal number, prefix it with ".0".
No space can be inserted between "." and the digit.

Either of the following can be used for the symbol specification format.

  1. <symbol>
  2. :<symbol>

An error occurs for a value that can be interpreted as a hexadecimal number or as a symbol such as "a123" if that symbol actually exists. In this case, be sure to explicitly indicate which it is by prefixing the value with "0x" if it is a hexadecimal number or prefixing it with ":" as in ":a123" if it is a symbol.

Besides a symbolic expression such as "$a0," a numeric specification such as "$4" can also be used for a register name. The numeric value portion in a register number specification is a decimal number.

A "." within an expression is a special symbol that has the value of the "current address" during the execution of various commands such as di, as, and commands that display and set memory. After these commands complete, the "." takes on the value of "next address".

The following are examples showing how expressions are used.

db 80100000
di main+0x100
di (main + 100 + *(1002 + 4) * 5)
di $epc-8
db $t0+0x124+$v0+buf

4 Command Details

4. 1 dr [-hf] [<reg>]...

For <reg>, specify a register name or register group name.
The following register group names can be specified.

  Register Group Name     Meaning  
  all     All registers  
  gpr     General Purpose Registers  
  hl     HI, LO  
  scc     System Coprocessor Registers  
  fpr     FPU registers (c1r)  
  fcr     FPU control registers (c1c)  
  fpu     All FPU registers (cop1, cp1)  
  vfr     VFPU registers (c2r)  
  vfc     VFPU control registers (c2c)  
  vfpu     All VFPU registers (cop2, cp2)  
  ge     All Graphics Engine registers  
  ge_reg     Graphics Engine control registers  
  ge_cmd     Graphics Engine command registers  
  ge_bone     Graphics Engine bone matrix registers  
  ge_world     Graphics Engine world matrix registers  
  ge_view     Graphics Engine view matrix registers  
  ge_proj     Graphics Engine projection matrix registers  
  ge_tgen     Graphics Engine texture generation matrix registers  


Note that register number format can be used for a general-purpose register (example: $a0 is $4), and for a VFPU matrix register, scalar format (example: s000), vector format (example: c000, r000), or matrix format (example: m000, e000) can be used.
If <reg> is omitted, the general-purpose registers and $lo, $hi, $PC, $bada, and the instructions before and after the $PC are disassembled and displayed.
$PC is a pseudo register that holds a program counter value that was calculated from the values of the two registers $epc and $cause.
To display a VFPU matrix register in s*** / c*** / r*** / m*** / e*** formats, enter the commands shown below. For formats other than scalar, if no dimension is specified and it is not self evident, it will be 4D by default.
Examples:
  dstdb 0R0> dr s000       
  dstdb 0R0> dr c000     → Displayed as 4D  
  dstdb 0R0> dr r000.p     ← 2D specification  
  dstdb 0R0> dr m000.t     ← 3D specification  
  dstdb 0R0> dr e000.q     ← 4D specification  

The -h or -f option can be specified when displaying register values using a register group specification.

  Option     Meaning  
  -h     Display 32-bit values by using hexadecimal numbers. [default]  
  -f      Display 32-bit values as hexadecimal numbers and  
  floating-point numbers. (fpr, vfr, gpr)  


For a VFPU matrix register, floating-point can be set as the default display by setting vfpu_display_float in .dstdbrc. Also, the -v option displays the vfr* register corresponding to each element of a vector or matrix.

4. 2 sr [<reg> [<val>]]...

The value <val> is set for the register specified by <reg>.

4. 3 rload <fname>

Load register values from the specified file <fname>. Generally, the filename of a file that was saved by using the rsave command is specified.

4. 4 rsave <fname> [<reg>]...

Saves the register names and values of the specified registers <reg>... in the specified file <fname>.
If <reg>... is omitted, the same registers as the ones described when <reg>... is omitted in the dr command are saved.

4. 5 dq [<adr> [<cnt>]]
dd [<adr> [<cnt>]]
dw [<adr> [<cnt>]]
dh [<adr> [<cnt>]]
db [<adr> [<cnt>]]
df [<adr> [<cnt>]]

Memory contents are displayed beginning with the specified address <adr>.
The default <adr> is the dot address (current address).
<cnt> indicates the number of bytes to be displayed at one time. If a fraction is specified with the dq command, for example, it is rounded up. If a command other than dw is used to access an I/O register by an amount other than 32 bits, the value of the I/O register will be unpredictable.

4. 6 sq <adr> <val>...
sd <adr> <val>...
sw <adr> <val>...
sh <adr> <val>...
sb <adr> <val>...
sf <adr> <fval>...

The specified data values <val>... are stored in memory beginning with the specified address <adr>.
Only the following floating-point expressions can be specified for <fval> when the sf command is used.
<floating_point_number>
(<fexpr>)
+<fexpr>
-<fexpr>
*<fexpr>
<fexpr1> + <fexpr2>
<fexpr1> - <fexpr2>
<fexpr1> * <fexpr2>
<fexpr1> / <fexpr2>

4. 7 bload <fname> <adr>

The contents of the file named <fname> are loaded into memory as binary data in the area specified by the starting address <adr>.

4. 8 bloadp <fname>

The contents of the file named <fname> in which the boot code is stored are loaded into memory as binary data in the stipulated area.

4. 9 start <fname>

The contents of the file named <fname> in which the boot code is stored are loaded into memory as binary data in the stipulated area and the target is reset.
Issuing this command is equivalent to executing the bloadp command and the reset command.

4. 10 bsave <fname> <adr> <cnt>

The contents of the area having a size in bytes indicated by <cnt> beginning at the address <adr> are saved as binary data in the file named <fname>.

4. 11 di [-m <mark>] [<adr> [<cnt>]]

Instructions are disassembled and displayed beginning with the specified address <adr>.
The default <adr> is the dot address (current address).
<cnt> indicates the number of instructions to be displayed at one time. The default is 20 instructions.
If -m <mark> is specified, the marker "->" is displayed at the line corresponding to address <mark>.

4. 12 as <adr> <inst>

The instruction <inst> is written to the specified address <adr>.
For example, the following can be specified.

dstdb > as 10000 lw $v0,10($s0)

4. 13 list [<adr> [<cnt> [<back>]]]

Source lines before and after the line corresponding to the specified address <adr> are displayed.
The default value of <adr> is $PC.
<cnt> indicates the number of source lines to be displayed. The default is 21 lines.
<back> is the difference between the source line at the specified address and the starting line that is displayed. The default is 10 (display starts 10 lines before the specified line).
If a return is entered after the list command is executed, the same number of source lines are displayed as during the previous invocation of the command beginning with the source line following the source line that was displayed last.
The marker "->" displayed at the left side of the source line indicates the source line corresponding to $PC.
To display source lines, the executable file must have been created using gcc -g.

4. 14 bt [<cnt>]

Stack frames are backtraced beginning with the current program counter. The program counter values for each stack frame are displayed.
To display a specific number of stack frames, specify a value for <cnt> is specified. If <cnt> is not specified, all stack frames are displayed.
When a backtrace is displayed, the following pseudo variables are set and can be referenced by subsequent commands.

  Pseudo Variable     Description  
  $BT<n>     Backtrace address  
  When <n> is greater than or equal to 1, this is the address of  
  the instruction where the function was called from that stack  
  frame. When <n> is 0, it is the same as the $PC value.  
  $SFA<n>     Stack frame address  
  Minimum value of the <n>-th stack frame.  
  $SFS<n>     Stack frame size  
  Difference when minimum value is subtracted from maximum  
  value of <n>-th stack frame.  


For example, to set a breakpoint at the location returned in the 2nd stack frame position, specify bp $BT2+0x8.

To use the subst command to define and execute an instruction named dsf that will use the dw command to dump the <n>-th stack frame, specify the following

dstdb > subst "^dsf  *\([0-9][0-9]*\) *$" dw $SFA\1 $SFS\1
dstdb > dsf 2

4. 15 bp [<adr>[,<cnt>]]...

A breakpoint is set at the specified address <adr>.
If the argument is omitted, all breakpoints that are currently set will be displayed.
The count value <cnt> indicates the specified number of times the breakpoint is passed. The default value is 1.
When a breakpoint is set, its address is set in the breakpoint pseudo variable $BP<n> that corresponds to that address, and the variable $BP<n> can be specified in subsequent ub and other commands.

4. 16 ub [all|<adr>]...

The breakpoint at the specified address <adr> is canceled.
If the argument is omitted or all is specified, all breakpoints are canceled.

4. 17 be [<adr>]...

The breakpoint at the specified address <adr> is enabled.
If <adr> is omitted, all breakpoints are enabled.

4. 18 bd [<adr>]...

The breakpoint at the specified address <adr> is disabled.
If <adr> is omitted, all breakpoints are disabled.

4. 19 hbp pc<nt>[:<adr>[,<amsk>]]]...
hbp {da|dr|dw}<nt>[:<adr>[,<amsk>][=<val>[,<vmsk>]]]

Displays the hardware breakpoint settings and status for the specified address <adr>.
If all the arguments are omitted, the current hardware breakpoints will be displayed.
pc,da,dr,dw are access types for the hardware breakpoints and their meaning is shown below.

  pc     Program counter      (during DF stage)  
  da     Data access      (dr or dw)  
  dr     Data read access      (during DF stage)  
  dw     Data write access      (during WB stage)  
Only one of da,dr,dw can be specified at a time.

<amsk> is the mask value for the specified address.
(The default value is 0xffffffff which means that all values are compared with <adr>)
<nt> specifies whether or not a trigger signal will be generated and if a break exception will occur.

  Option     Description  
  Not specified     A break exception will occur. A trigger signal will not be generated.  
  t     A break exception will occur and a trigger signal will be generated.  
  nt     A break exception will not occur, but a trigger signal will be generated.  

If <adr> is omitted, the previous address value for that type will be used.
If da, dr or dw is used, the specified data value <val> and its associated mask value <vmsk> can be specified.
The hbp and hub commands can be used only with the dstdb which was started first.

Example: Data read to address 0x08000000 
  dstdb > hbp dr:0x08000000
If execution is stopped by a hardware breakpoint, it cannot be re-executed by cont.

4. 20 hub {pc|da|dr|dw}...

Removes hardware breakpoints of the specified type {pc|da|dr|dw}.
If all the arguments are omitted, all hardware breakpoints will be removed.

4. 21 cont [<cnt>]

Continuous execution of the target program is performed starting from the current state.
The debugger waits until a breakpoint is reached or some exception is generated.
The argument <cnt> specifies the continuation count. The default is 1.
Standard output from the target program will be displayed on the screen.
To suspend the target program, enter CTRL-C. To return control to the debugger without suspending the target program, enter CTRL-G.

4. 22 step [<cnt>]

Step execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $PC.
The step count matches the number of CPU instructions. Even if delayed branch instructions are included, step execution will be performed directly in the order that the CPU executes them.

4. 23 next [<cnt>]

Next execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $PC.
The next count matches the number of CPU instructions. Even if delayed branch instructions are included, next execution will be performed directly in the order that the CPU executes them.
The only difference between next and step execution is with step execution, instruction tracing will be performed into a function which is entered using the jal or jalr instructions.

4. 24 lstep [<cnt>]

Step execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $PC.
If $PC does not contain a line number when a command is executed or after internal step execution of one instruction, a message indicating this is displayed, and the command ends (default).
If the option variable lstep_stop_at_no_line is set, an error does not occur even at a location that does not contain line number information and step execution is performed internally until a location is reached that contains line number information.
To perform lstep, the executable file must be created with gcc -g.

4. 25 lnext [<cnt>]

Next execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $PC.
If $PC does not contain a line number when a command is executed or after internal next execution of one instruction, a message indicating this is displayed, and the command ends (default).
If the option variable lstep_stop_at_no_line is set, an error does not occur even at a location that does not contain line number information and next execution is performed internally until a location is reached that contains line number information.
To perform lnext, the executable file must be created with gcc -g.

4. 26 until <adr>...

Temporary breakpoints are used to perform continuous execution from the current $PC until the specified addresses <adr>... in the order they are specified. The until command is not affected by breakpoints that were set by the bp command.

4. 27 break

Program execution is suspended.

4. 28 wait

The debugger waits for an exception to be generated while the program is being executed.

4. 29 gdi [-m <mark>] [<adr> [<cnt>]]

Instructions are disassembled and displayed as a Graphics Engine display list beginning with the specified address <adr>.
The default <adr> is the dot address (current address).
<cnt> indicates the number of instructions to be displayed at one time. The default is 20 instructions.
If -m <mark> is specified, the marker "->" is displayed on the line corresponding to the <mark> address.

4. 30 gbp [<adr>[,<cnt>]]...

A display list breakpoint is set at the specified address <adr>.
If the argument is omitted, all breakpoints that are currently set will be displayed.
The count value <cnt> indicates the specified number of times the breakpoint is passed. The default value is 1.
When a breakpoint is set, its address is set in the breakpoint pseudo variable $GBP<n> that corresponds to that address, and $GBP<n> can be used in specifications in subsequent gub and other commands.

4. 31 gub [all|<adr>]...

The display list breakpoint at the specified address <adr> is canceled.
If the argument is omitted or all is specified, all breakpoints are canceled.

4. 32 gbe [<adr>]...

The display list breakpoint at the specified address <adr> is enabled.
If <adr> is omitted, all breakpoints are enabled.

4. 33 gbd [<adr>]...

The display list breakpoint at the specified address <adr> is disabled.
If <adr> is omitted, all breakpoints are disabled.

4. 34 gcont [<cnt>]

Continuous execution of the display list is performed starting from the current state.
The debugger waits until a breakpoint is reached or some kind of exception occurs.
The argument <cnt> specifies the continuation count. The default is 1.
Standard output from the target program will be displayed on the screen.
To interrupt the target program, enter CTRL-C. To return control to the debugger without interrupting the target program, enter CTRL-G.

4. 35 gstep [<cnt>]

Step execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $ge_madr.
The step count roughly matches the number of display list instructions.

4. 36 gnext [<cnt>]

Next execution is performed the number of times specified by <cnt> (default is 1) beginning with the current $ge_madr.
The next count roughly matches the number of display list instructions.
The only difference between next and step execution is with next execution, subroutines will not be traced when the CMD_CALL instruction is executed.

4. 37 guntil <adr>...

4. 49 subst [-r] [<pattern> <replace>...]

The command input line is handled as a string, and all patterns indicated by <pattern> contained within that string are replaced by the replacement strings indicated by <replace>.
To display all replacements that have currently been set, specify only subst.
To delete all replacements that have currently been set, specify subst -r.
To delete the replacements indicated by <pattern> <replace>..., which are currently set, specify subst -r <pattern> <replace>....
This command differs from alias in that the previous setting is not deleted even if a replacement was set for the same <pattern>. Also, when <replace>... is omitted, the setting will be replaced by a blank string (in other words, it will be deleted).
If there are multiple substitution definitions, they are all applied in the order they were specified.

Although <pattern> is generally specified by using a string that is enclosed in double quotes, if the string does not have any embedded spaces, the surrounding double quotes may be omitted.
<pattern> is specified by using the following regular expressions.

  X     Character X (when X is not included in the following representations)  
  \X     Character X (when X is not "(" or ")")  
  .     Arbitrary one character  
  X*     Repetition of X (including zero repetitions)  
  [abc]     Any one character of "a", "b", and "c".  
  [x-y]     Any one character from "x" to "y".  
  [^abc]     One character that is not "a", "b", or "c".  
  ^     Beginning of a line (matches only at the beginning of <pattern>)  
  $     End of a line (matches only at the end of <pattern>)  
  \(     Start recording of tag  
  \)     End recording of tag  

Besides normal strings, the following special characters can also be specified for <replace>...

  \X     Character X (when X is not "1"-"9")  
  &     Entire currently matched string  
  \1     1st tag  
  \2     2nd tag  
  ...       
  \9     9th tag  

Some example specifications are shown below.
  • Set that breakpoint numbers can be entered by using $bn<n> 

dstdb> subst "$bp\([0-9]* \)" $BP\1
  • Set that show history can be executed by using only history 
dstdb> subst "^ *history *$" show history
When a command input line begins with "\", no substitution processing will be performed.
If a setting such as subst ".*" is made by mistake, quit can always be executed by entering \quit, for example.

4. 55 if <expr> <cmd> [elif <expr> <cmd>]... [else <cmd>]

A command is executed conditionally.
First, the expression <expr> of if <expr> <cmd> is evaluated, and if its value is not 0, <cmd> is executed.
If the value of the expression <expr> of if <expr> <cmd> is 0, the elif or else part is evaluated. For else, <cmd> is executed unconditionally, and for elif, <cmd> is executed conditionally.

4. 56 repeat [-c <cnt>] [-i <sec>] [-while <expr>] <cmd>...

The specified commands <cmd>... are repeatedly executed.
If the repetition count <cnt> is specified by the -c option, the commands are repeated only the specified number of times. If no repetition count is specified, the commands are repeated indefinitely until an error occurs or CTRL-C is entered.

Processing such as packet send/receive is performed once between repeated executions.
If no packet send/receive is performed or no keyboard entry is made within the interval in seconds specified by <sec> (default value is 1 [second]) according to the -i option, the next repetition begins.
If packet send/receive is performed or a keyboard entry is made, the next repetition is executed without waiting for that interval.
If the -while <expr> option is specified, <expr> is evaluated at such a time before the commands <cmd>... are executed, and if its value is 0, the repeat command ends. Also, if the -while option is specified and the -i option is not specified, the timeout interval will be 0.
To execute multiple commands repeatedly, use a format such as repeat (dr;dr).

6 Option Details

6. 1 target_name

Any of the following four methods can be used to set the host name and port of dsnetm_psp.
  1. Value of environment variable "DSNETM_PSP"
  2. Value of environment variable "DSNETM"
  3. option statement within auto-execution-on-startup file
  4. dstdb startup option -d <host>[:<port>]
If more than one of these is specified, the setting is overridden in the order 1) -> 2) -> 3) -> 4).
The default value of this option, which is "" (null string), has the meaning of the default port specification of the same host on which dstdb was executed.

6. 2 file_priority

This option sets whether or not Remote File Access from the target is to be handled by dstdb. If Remote File Access from the target is not to be handled by dstdb, specify -1.
The default value of this option is 0xd0.

6. 3 reset_on_start

This option specifies whether or not to reset the target when dstdb is started up.
This specification is effective when neither the -r nor the -nr command line option has been specified. The default value of this option is false.

6. 4 check_manager_version

This option specifies whether or not to check dsnetm_psp version information when dstdb is started up. If this option is true, a check is performed, and if the versions differ, the debugger ends with an error. If this option is false, the version information is not checked.
The default value of this option is false. If -cmv is specified on the command line, it is overridden to true.

6. 5 target_exec_ctl_config

This option indicates the execution control status of an execution command such as cont, step, or next.
When it is 0, it means host-side control. Otherwise, it means target-side control.

6. 6 target_exec_ctl_override

This option specifies an override for the target_exec_ctl_config value. That value is overridden when the value of this option is greater than or equal to 0.

6. 7 dr_default_format

The default value, which is option dr_default_format="", means that the internal display format of the debugger is always used. An explicit setting that is equivalent to this is as follows.
option dr_default_format="\
at=%at  v0-1=%v0,%v1  a0-3=%a0,%a1,%a2,%a3\n\
t0-7=%t0,%t1,%t2,%t3, %t4,%t5,%t6,%t7\n\
s0-7=%s0,%s1,%s2,%s3, %s4,%s5,%s6,%s7\n\
t8=%t8 t9=%t9   k0=%k0 k1=%k1   gp=%gp sp=%sp\n\
fp=%fp ra=%ra   lo=%lo hi=%lo   PC=%PC bada=%bada\n\
$cr=0x%cr [%symcr]\n\
$fcr31=0x%fcr31 [%symfcr31]\n\
$sr=0x%sr [%symsr]\n"

Values that are subject to conversion are limited as follows.
%%          - Means "%".

%<reg>      - Displays the register name <reg> according to printf("%08x",...).
None of the following can be specified for <reg>.

  • Register group name
  • Value corresponding to the numeric value representation $<n>
  • Pseudo register name
  • Prefix of "$"
  • Prefix of "_"
  • sr -> status alias (dstdb)

If a name that cannot be specified is specified, %<reg> is displayed without conversion.
%symcr, %symsr, and %symfcr31 can be specified for the symbolic display of the cause, status, and fcr31 registers as in the example shown above. One space is always displayed before and after them.
Also, a 128-bit register other than %08x or 16-bit display is not supported, and no dstdb setting option exists for dr -vu[01].

6. 8 dr_default_di

When the dr command is specified without an argument, the instructions before and after the current $PC are disassembled and displayed. The display method to be used is determined by the value of this option.
The default value used when there is no setting for this option is "\\di -m $PC $PC-8 7" (where the leading "\" is a specification that no alias expansion is to be performed).

6. 9 drge_default_gdi

When the -ge option is specified in the dr command, instructions before and after the current $ge_madr are disassembled and displayed as a display list. The display method to be used is determined by the value of this option.
The default value used when there is no setting for this option is "\\gdi -m $ge_madr $_ge_madr-10 8."
(The leading '\' specifies not to expand the alias.)

6. 10 ex_default_dr

When execution that was started by a command such as cont, step, or next is stopped due to a breakpoint or the generation of an exception, the dr information is displayed. The display method to be used is determined by the value of this option. The default value used when there is no setting for this option is "\\dr".

6. 11 exge_default_drge

When execution is started by the cont, step, or next commands, and then stopped by a display list breakpoint, dr -ge is displayed. The display method to be used is determined by the value of this option. The default value used when there is no setting for this option is "\\ dr -ge."

6. 12 lstep_default_list

This option specifies the list display method to be executed when lstep or lnext is normally terminated.
The default value used when there is no setting for this option is "\\list".

6. 13 lstep_stop_at_no_line

This option specifies whether or not to terminate the command at a location where $PC has no line number during execution or at the start of lstep or lnext.

  When this option is 0:     step or next operation is continued internally up to a location that has a line number  
  When this option is 1:     A message is displayed and the command is terminated. (default)  

6. 14 source_directories

This option specifies the source file search directories that are to be referenced by the di and list commands.
If the beginning of the source filename included in the debug data of the executable file is "/" (absolute path) or if the source_directories variable is "", only the source filename is referenced directly.
Otherwise, a reference is attempted by appending one or more (delimited by ":") directory names in front of the source filename. An empty delimiter means an empty prefix. For example, ".:" corresponds to a "" prefix following ".".
When the specified directory name starts with "@", the "@" part is replaced by the directory name converted to the absolute path of the object file name.
The default value of this option is "".

  • Specification example for only the directory in which the object exists 
set source_directories="@"

  • Specification example for the order: current directory -> object position -> ABC -> DEF 
set source_directories=".:@:ABC:DEF"

6. 15 initial_sbootp/current_sbootp

These options are boot parameter values that are referenced by the reset command.
For the meaning of each of these options, see the explanation of the reset command.
The default value of initial_sbootp is "-1".
The default value of current_sbootp is "-1".
("-1" means that the current value is not changed.)

6. 16 boot_filename

This option specifies the filename of the boot code to be loaded to reset the target when dstdb is started up.

6. 17 automatic_prefix_breakpoint

Although the argument of the ub, be, or bd command is an address, a pseudo register having $BP<n> format can be specified within an expression in that address specification.
If this automatic_prefix_breakpoint variable is true, "$BP" is automatically appended as a prefix when the argument of the ub, be, or bd command consists only of a decimal numeric value.
The default value is false.

6. 18 describe_ub_all

If the argument is omitted in the ub command, it indicates that all breakpoints will be deleted. However, if the describe_ub_all variable is true, the deletion of all breakpoints is prohibited.
The default value is false.

6. 19 di_address

This option specifies the display mode for the address (label) part of the di command.

  For 0x0"     Display addresses by using both a hexadecimal number and symbol.  
  For 0x1:     Display addresses by using only a hexadecimal number.  
  For 0x2:     Display addresses by using only a symbol if a symbol exists; otherwise,  
   display addresses by using a hexadecimal number.  
  For 0x3:     Display addresses only once on an independent line in relation to the same symbol.  
  For 0x4:     Also display line number in addition to the display described for 0x0.  
  For 0x5:     Also display line number in addition to the display described for 0x1.  
  For 0x6:     Also display line number in addition to the display described for 0x2.  
  For 0x7:     Also displays line number in addition to the display described for 0x3.  
  For 0x8:     Also display source line in addition to the display described for 0x4.  
  For 0x9:     Also display source line in addition to the display described for 0x5.  
  For 0xa:     Also display source line in addition to the display described for 0x6.  
  For 0xb:     Also display source line in addition to the display described for 0x3.  
  When source line cannot be accessed, this is the same as described for 0x7 (default).  
To display the line number of source line, be sure to create an executable file by using gcc -g.

6. 20 di_instruction_word

This option specifies the display mode for the instruction word part of the di command.

  For 0:     Instruction word is displayed as a hexadecimal number (default).  
  For 1:     Instruction word is not displayed as a hexadecimal number.  

6. 21 di_branch_address

This option specifies the display mode for the branch address part of the di command.

  For 0:     Display branch address by using both a hexadecimal number and symbol. (default)  
  For 1:     Display branch address by using only a hexadecimal number.  
  For 2:     Display branch address by using only a symbol if a symbol   
  exists; otherwise, display branch address by using a   
  hexadecimal number.  

6. 22 di_macro

This option specifies whether or not a macro instruction such as li or move is to be handled by the di or as command.

  For 0:      Macro instructions are not handled.  
  For 1:      The li, move, b, bal, and dmove macro instructions are   
  handled. (default)  

6. 23 help_lang

This option specifies the language used for displaying help output by the help command.

  "eng" :      English (default)  
  "euc" :      Japanese (EUC)  
  "jis" :      Japanese (JIS)  
  "sjis":      Japanese (shift JIS)  
  "utf-8":     Japanese (UTF-8)  

6. 24 help_pager

This option specifies the pager used for displaying help output by the help command.
The default is "" (NULL string), which means that a pager is not used.

6. 25 hex_radix

This option sets the method of interpreting a numeric value specification in dstdb.
The default value is true, and a numeric value is interpreted as a hexadecimal number without 0x.
If false is specified, a hexadecimal number must be prefixed by 0x as in C.
If there are no symbols, true is more convenient. However, if db abc is specified, for example, when there are symbols and if the symbol abc also exists, this must be reentered as db 0xabc, and some ambiguity remains. If false is specified, since only symbols will be searched for db abc, there is no need to reenter the specification depending on the situation.

6. 26 log_total_size

By default, the DECI2 packet log is saved with a maximum total number of bytes of 128 kilobytes. If the send/receive log is unnecessary, set 0.

6. 27 log_packet_size

By default, up to 128 bytes of data from the beginning of a packet is saved in the log. To save all data in the log, set a value of at least 65535.

6. 28 histfile

By default, this option is "~/.dstdb_history". A history entry is saved in the file indicated by this string when dstdb is terminated, and the history is loaded from that file when dstdb is started up.

6. 29 histfilesize

This option indicates the number of history entries to be saved in histfile. The default value is 256.

6. 30 histsize

This option indicates the upper bound for the number of history entries to be stored during command operations. The default value is 256.

6. 31 tty_max_size

This option indicates the maximum number of pending bytes in a TTY packet to be sent by dstdb. The default value is 8 MiB.
