GPI / GPO Manipulation Utility Reference


sceKernelGetGPI
 [GPI Manipulation]
Get GPI switches

Syntax

#include <kernel.h>
unsigned char sceKernelGetGPI (
	void
);
Calling Conditions

Can be called from an interrupt handler (does not depend on interrupt-disabled or enabled state)
Can be called from a thread (does not depend on interrupt-disabled or enabled state)
Multithread safe

Arguments

None

Return value

Returns the GPI switch settings.

Description

This function retrieves the GPI switch settings on the front panel of the PSP™ development TOOL. Switches 7 to 0 correspond to bits 7 to 0 of the return value; the bits for switches which are on will be 1.

GPI switches are hardware switches that can be used for debugging. They can be used in any arbitrary manner by game developers. On an actual PSP™ device, the sceKernelGetGPI() function will always return 0.

sceKernelSetGPO
 [GPO Manipulation]
Set GPO light-emitting diodes

Syntax

#include <kernel.h>
void sceKernelSetGPO (
	unsigned char gpo
);
Calling Conditions

Can be called from an interrupt handler (does not depend on interrupt-disabled or enabled state)
Can be called from a thread (does not depend on interrupt-disabled or enabled state)
Multithread safe

Arguments

  gpo     GPO setting value  

Return value

None

Description

This function sets the GPO LEDs on the front panel of the PSP™ development TOOL. LEDs 7 to 0 correspond to bits 7 to 0 of the gpo argument; LEDs for bits which are 1 will light up.

GPO LEDs are light-emitting diodes that can be used for debugging. They can be used in any arbitrary manner by game developers. On an actual PSP™ device, using this function will do nothing.
