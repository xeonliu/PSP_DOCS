libsfmt2281


SCE_SFMT2281_ARRAY_SIZE
 [定数定義]
SFMT2281擬似乱数計算の配列サイズ

構造

#include <libsfmt2281.h>

#define SCE_SFMT2281_ARRAY_SIZE	18	/* (2281 / 128) + 1 */
解説

SFMT2281による擬似乱数の配列サイズを定義します。
SceSfmt2281Context構造体内にステートとして保持される配列サイズを示すほか、sceSfmt2281FillArray32(), sceSfmt2281FillArray64()関数で、乱数を生成する場合の最小サイズも、この定数によって示されています。

関連項目

SceSfmt2281Context, sceSfmt2281FillArray32(), sceSfmt2281FillArray64()

SceSfmt2281Context
 [構造体]
SFMT2281擬似乱数計算のコンテキスト情報

構造

#include <libsfmt2281.h>
typedef struct SceSfmt2281Context {
	unsigned int idx;
	unsigned int sfmt[SCE_SFMT2281_ARRAY_SIZE][4];
} SceSfmt2281Context;
解説

SFMT2281による擬似乱数を計算するためのワークエリアです。
このワークエリアは、乱数系列1つにつき1個ずつ用意する必要があります。

関連項目

解説

SceSfmt2281Context構造体によって表現されるSFMT2281乱数系列を32ビットのシード値によって初期化します。この関数は、sceSfmt2281GenRand32(), sceSfmt2281GenRand64(), sceSfmt2281FillArray32(), sceSfmt2281FillArray64()関数に先立って実行しておく必要があります。

初期化されるのは、pCtxで示されるSceSfmt2281Context構造体だけですので、SceSfmt2281Context構造体を複数個持つことにより複数の乱数系列を同時に扱うことができます。

解説

SFMT2281による64ビットの擬似乱数を生成します。
この関数に先立ってSceSfmt2281Context構造体は、sceSfmt2281InitGenRand()関数またはsceSfmt2281InitByArray()関数によって初期化されている必要があります。

sceSfmt2281GenRand32()関数とsceSfmt2281GenRand64()関数を併用する場合、sceSfmt2281GenRand32()関数を奇数回呼び出した状態で、sceSfmt2281GenRand64()を呼び出すと、64ビットの乱数値が得られないことに注意してください。この場合、上位32ビットがゼロに固定された64ビット値が得られます。

解説

SFMT2281による32ビットの擬似乱数の任意個数、生成します。引数arrayの要素数を指定する引数sizeは、4の倍数値で、(SCE_SFMT2281_ARRAY_SIZE * 4)よりも大きな値でなくてはなりません。
この関数に先立ってSceSfmt2281Context構造体は、sceSfmt2281InitGenRand()関数またはsceSfmt2281InitByArray()関数によって初期化されている必要があります。

sceSfmt2281GenRand32()関数と併用する場合、sceSfmt2281GenRand32()関数が(SCE_SFMT2281_ARRAY_SIZE * 4)回呼び出された状態の場合のみ、sceSfmt2281FillArray32()関数を呼び出すことができます。
sceSfmt2281GenRand64()関数の場合、sceSfmt2281GenRand64()関数が(SCE_SFMT2281_ARRAY_SIZE * 2)回呼び出された状態の場合のみ、sceSfmt2281FillArray32()関数を呼び出すことができます。
