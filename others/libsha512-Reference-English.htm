SceSha512Context
 [Structure]
Context information for SHA-512 digest value computation

Structure

#include <libsha512.h>
typedef struct SceSha512Context {
	unsigned long long h[8];
	unsigned int pad;
	unsigned short usRemains;
	unsigned short usComputed;
	unsigned long long ulTotalLen;
	unsigned char buf[SCE_SHA512_BLOCK_SIZE];
} SceSha512Context;
Members

  h     Work area  
  pad     Padding for adjusting alignment  
  usRemains     Less than 128 bytes of remaining data, which was temporarily copied within the SceSha512Context structure  
  usComputed     Digest value computed flag  
  ulTotalLen     Total data size (bytes)  
  buf     Temporary copy of less than 128 bytes of data  

Description

This structure is used as a work area when computation of the SHA-512 digest value is divided. Since the sceSha512BlockInit(), sceSha512BlockUpdate(), and sceSha512BlockResult() functions use this structure as a work area, applications must not directly access the members of this structure.

  Value     Result

sceSha512BlockUpdate
 [Function]
SHA-512 digest value computation processing

Syntax

#include <libsha512.h>
int sceSha512BlockUpdate (
	SceSha512Context *pContext,
	const void *plain,
	unsigned int len
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pContext     Address of digest value computation work area.  
  plain     Pointer to cleartext data for which digest value is to be computed.  
  len     Data size (bytes) of cleartext data for which digest value is to be computed.  

Return Value

If an error occurs, a negative value is returned.
  Value     Result  
  SCE_OK     Normal completion  
  SCE_ERROR_BUSY     Digest value was already computed for pContext  
  SCE_ERROR_INVALID_POINTER     pContext or plain address is invalid  


Description

This function uses the cleartext specified by plain and len to update the work area within the SceSha512Context structure. The sceSha512BlockUpdate() function, which can be called any number of times between the sceSha512BlockInit() function and sceSha512BlockResult() function, enables the digest value to be computed even for a large amount of data that cannot fit in memory.

sceSha512BlockResult
 [Function]
Get computed SHA-512 digest

Syntax

include <libsha512.h>
int sceSha512BlockResult (
	SceSha512Context *pContext,
	unsigned char *digest
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  pContext     Address of digest value computation work area.  
  digest     Returns the computed digest value (64 bytes).  

Return Value

If an error occurs, a negative value is returned.
  Value     Result  
  SCE_OK     Normal completion  
  SCE_ERROR_INVALID_POINTER     pContext ordigest address is invalid  


Description

This function retrieves the computed digest value from the SceSha512Context structure. The SHA-512 algorithm computes a digest value in units of 128 bytes, so a remaining amount less than 128 bytes may have been temporarily copied within the SceSha512Context structure by the sceSha512BlockUpdate() function. If this remaining data exists, the final digest value is determined by calling the sceSha512BlockResult() function. Always use the sceSha512BlockResult() function to obtain the digest value.
The digest value of the SceSha512Context structure is valid until the next call to the sceSha512BlockInit() function.
