解説

libfontは、外部からキャッシュシステムを与える事ができます。
libfontは、このキャッシュシステムとSceFont_t_cacheSystemInterfaceで与えられた関数群を通じて通信します。libfontがこの関数群をコールするときは、最初の引数にcacheInstanceを渡します。

libfontが期待するキャッシュシステムの関数群の仕様は以下の通りです。
キャッシュシステムの初期化と終了処理はlibfontの中からは行いません。libfontの初期化前と終了処理の後に、必要に応じアプリケーションが処理してください。

result = lockFunc ( cacheInstance )

キャッシュをロックします。
キャッシュシステムは、キャッシュがロックされている間、他からの要求は受け付けてはいけません。
ロックに成功したときはresultに0を、ロックに失敗したときには-1を返します。

result = unlockFunc ( cacheInstance )

キャッシュのロックを解除します。
ロックの解除に成功したときはresultに0を、ロックの解除に失敗したときには-1を返します。

cacheSlot = findFunc ( cacheInstance, hashValue, key, result )

キャッシュの中からhashValueの値からkeyで示されたデータの有無を調べます。
hashValueはlibfontが生成したハッシュ値です。この値を使うとキャッシュ内にのデータとの照合処理を素早く行う事ができます。しかしながら、libfontは必ずこの値をキャッシュシステムが使って実装されるという事を期待していません。キャッシュシステム自身がkeyからハッシュ値を生成する事が許容されています。

sceFontSetAltCharacterCode
 [関数]
代替文字のコードの設定

形式

#include <libfont.h>
SceFont_t_error errorCode = sceFontSetAltCharacterCode (
	SceFont_t_libId libID,
	SceFont_t_charCode charCode
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフでない

引数

  libID     ライブラリインスタンスのポインタ  
  charCode     代替文字の文字コード  

返り値

正常終了の場合、SCE_FONT_NOERRORが返されます。
エラーの場合、以下のいずれかが返されます。
SCE_FONT_ERR_LIBID, SCE_FONT_ERR_ARG, SCE_FONT_ERR_NOSUPPORT, SCE_FONT_ERR_UNKNOWN

解説

グリフが存在しない文字コードに対してsceFontGetCharGlyphImage()等でグリフイメージの取得を行おうとしたときに、代替グリフとして使用する文字の文字コードを指定します。
libfontのライブラリインスタンスが生成された直後は、デフォルトとしてU+005Fが設定されます。
sceFontGetCharGlyphImage()等の関数でフォントへアクセスする場合、その関数へ渡された文字コードの文字がアクセス対象のフォントに含まれていない場合に、libfontは自動的にsceFontSetAltCharacterCode()で設定された文字コードの文字に置き換えて処理を行います。
ただし、これはsceFontGetCharGlyphImage()等でアクセスするフォントの文字集合にsceFontSetAltCharacterCode()で指定した文字コードcharCodeの文字が含まれているという事が前提条件となります。
sceFontSetAltCharacterCode()で指定した文字コードcharCodeを含んでいないフォントにおいては代替文字による置き換えは行われず、結果として幅高さともに0の文字として扱われます。

デフォルトの代替文字のコードU+005Fは、内蔵欧文フォントと内蔵日本語フォントのどちらにも含まれていますが、内蔵ハングルフォントには含まれていません。即ちこれは、デフォルトの代替文字の設定において内蔵ハングルフォントを使う場合には代替文字による置き換えは行われないという事を意味します。

内蔵フォント文字集合の詳細は、別にリリースされております「PSP® 搭載フォント文字一覧表」をご覧ください。

ちなみに内蔵ハングルフォントの文字集合に含まれる文字の範囲は下に示した表の通りです。
ハングルフォントに関しては、この表の範囲に無い文字はlibfontへアクセスする前にあらかじめ除外するか何らかの文字へ置き換えを行う事をおすすめします。

  U+3131〜U+318E  
  U+3200〜U+321C  
  U+3260〜U+327B  
  U+327F  
  U+AC00〜U+D7A3  
  U+FFE6  

使用例

SceFont_t_error errorCode;
SceFont_t_f32 pixel;

errorCode = sceFontSetAltCharacterCode (libID, (SceFont_t_charCode)0x22a0);
if ( errorCode != SCE_FONT_NOERROR ) {
	printf ("Error (sceFontSetAltCharacterCode): 0x%8.8x\n", (int)errorCode);
}
