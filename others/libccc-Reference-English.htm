Character Code Conversion Library Reference


sceCccJISCS
 [Character Code Types]
JIS X 0213 character set represented using 16 bits

Syntax

#include <libccc.h>
typedef unsigned short sceCccJISCS;
Description

This is the JIS X 0213 character set represented in 16 bits.
bit 15 is the plane (men). Bits 8 to 14 are the value of the row (ku) + 0x20, and bits 0 to 7 are the value of the cell (ten) + 0x20.

sceCccSetTable
 [Function]
Set JIS and UCS character code conversion tables

Syntax

#include <libccc.h>
void sceCccSetTable (
	sceCccUCS2 const* jis2ucs,
	sceCccJISCS const* ucs2jis
);
Arguments

  jis2ucs     Pointer to JIS to UCS-2 correspondence table (JIS order)  
  ucs2jis     Pointer to UCS-2 to JIS correspondence table (UCS-2 order)  

Return Value

None

Description

This function sets tables for converting between the JIS and UCS-2 character codes, and vice versa.
These tables must be set before character conversion is performed.
The default size is 65536 entries. A range that exceeds that of UCS-2 is not supported.

sceCccSwprintfUTF8
 [Function]
Format as wide string

Syntax

#include <libccc.h>
int sceCccSwprintfUTF8 (
	SceWChar16 * wcs,
	SceSize maxlen,
	const SceWChar16 *format,
	...
);
Arguments

  wcs     Output buffer for formatted string  
  maxlen     Number of characters in output buffer  
  format     Formatting string  

Return Value

The number of formatted characters is returned.

Description

This function uses the SceWChar16 type as wide characters to perform string format processing. A buffer that can store a formatted string of at least maxlen characters must be specified for wcs.

A format specification string that begins with % can be specified as the format string in format. A format specification string has the following format. e, f, g, and a are not supported as types for handling floating-point numbers.

%[flags][width][.precision][modifier]type

Any of the following flag characters can be specified for flags. This can also be omitted.
  -     Output with left justification.  
  +     Output '+' or '-' for a signed value.  
  ' ' (space)     Output ' ' (space) for a signed positive number.  
  #     When used together with each of the types o, x, or X, output 0, 0x, or 0X.  
  0     When used together with width, 0 is output up to the specified minimum width.  

width is a field for specifying the character width to be output. It is specified by a decimal number. This field can be omitted.

.precision is a field for specifying the output precision. It is specified by a '.' (period) followed by a decimal number. This field can be omitted. Its effect differs according to the type.

modifier specifies a correction to the format that is specified by type. This field can be omitted. Its effect differs according to the type.
  hh     The argument that is to be converted to an integer is considered to be a signed char or unsigned char.  
  h     The argument that is to be converted to an integer is considered to be a short int or unsigned short int.  
  l     The argument that is to be converted to an integer is considered to be a long int or unsigned long int.  
  ll     The argument that is to be converted to an integer is considered to be a long long int or unsigned long long int.  
  j     The argument that is to be converted to an integer is considered to be an intmax_t or uintmax_t.  
  z     The argument that is to be converted to an integer is considered to be a size_t or ssize_t.  
  t     The argument that is to be converted to an integer is considered to be a ptrdiff_t.  

Any of the following output formats can be specified for type. The type specification cannot be omitted.
  d     An int argument is converted to a signed decimal number.  
  i     An int argument is converted to a signed decimal number.  
  o     An int argument is converted to an unsigned octal number.  
  u     An int argument is converted to an unsigned decimal number.  
  x     An int argument is converted to an unsigned hexadecimal number.  
  X     An int argument is converted to an unsigned hexadecimal number.  
  lc     An int argument is converted to wide characters as unsigned char and output.  
  c     An SceWChar16 argument is output as a multibyte character.  
  ls     A const char* pointer argument is converted to wide characters as a UTF-8 format multibyte string and output.  
  s     A const SceWChar16* pointer argument is output as a multibyte string.  
  p     A void * pointer argument is converted to a hexadecimal number as an address.  
  n     The number of characters that were output up to this entry is saved in a variable pointed to by an int * argument.  
  %     The % itself is output.  

This function differs from sceCccSwprintfSJIS() in that a const char * multibyte string that is referenced by the format specification character %ls is considered to be in UTF-8 format.

See Also

sprintf(), SceWChar16, sceCccSwprintfSJIS()
