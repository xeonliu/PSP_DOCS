sceFpuRound
 [Function]
Round to nearest

Syntax

#include <libfpu.h>
float sceFpuRound (
	float s
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  s     Floating-point value  

Return Value

The floating-point value representing the nearest integer to s is returned.

Description

This function uses the FPU round.w.s instruction to round s to the nearest integer.

round is the process in which a value is rounded to the nearest integer, but in cases such as 0.5, 1.5, 2.5, etc., in which the distance to adjacent integers is the same, the behavior as to which integer to round to differs between the round() and roundf() functions in libc, and the sceFpuRound() function. The round() and roundf() functions round to the integer which is farthest from 0, but the sceFpuRound() function corresponds directly to the round.w.s instruction of the ALLEGREX FPU, which is IEEE754 compliant, and rounds toward the even integer.

See Also

roundf(), __builtin_allegrex_round_w_s()

sceFpuPositiveSNaN
 [Function]
Signaling NaN (positive SNaN)

Syntax

#include <libfpu.h>
float sceFpuPositiveSNaN (
	unsigned int uiSignal
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  uiSignal     Signal value  

Return Value

A Signaling NaN (non-numeric NaN) is returned with the signal value specified by bits 21 to 0 of the uiSignal argument.

Description

This function returns a non-numeric NaN. In IEEE754, the sign bit and bits 21 to 0 of the mantissa of a NaN have no meaning. However, this function, sceFpuPositiveSNaN(), returns 0 (positive) for the sign bit and 0 (Signaling NaN) for high-order bit 22 of the mantissa. The values of bits 21 to 0 of the mantissa can be specified with the uiSignal argument.

 [Function]

	float s
);

Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  s     Floating-point value  

Return Value

sin(s) is returned.

Description

This function performs polynomial expansion in FPU to calculate sin(s).
If the absolute value of s is or less, the error compared to the value of sin that was obtained precisely will be 1LSB or less.
If the absolute value of s exceeds about , the correct value will not be returned. In addition, there is a possibility that a FPU exception occurs.
Since the calculation is performed only in FPU, this function can be called from a thread that cannot use VFPU.

float sceFpuExp2 (
	float x
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  x     Floating-point value  

Return Value

2x is returned.

Description

This function uses the FPU to calculate 2 to the x power using a polynomial expansion.

sceFpuPow
 [Function]
Power function

Syntax

#include <libfpu.h>
float sceFpuPow (
	float x,
	float y
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Multithread safe

Arguments

  x     Floating-point value  
  y     Floating-point value  

Return Value

xy is returned.
When x is negative and y is not an integer value, the return value is NaN.

Description

This function uses the FPU to calculate x to the y power using a polynomial expansion.
When x is negative and y is not an integer value, a NaN is generated. To determine whether or not y is an integer value, calculate the floor and ceil values of y and see if the results are the same. Be careful when the absolute value of y exceeds the precision of a float.

See Also

powf()
