2 RSS Channel Overview

The RSS channel feature implemented in PSP™ system software enables updated content on websites to be distributed to users relatively easily using RSS 2.0 (RSS = Really Simple Syndication).

2. 1 Overview of the Relationship Between a Game, the RSS Reader Utility, the RSS Channel Subscription Utility, PSP™ System Software (RSS Channel Feature), and the Distribution Server

Figure 1 shows the relationship between the RSS channel subscription utility, the RSS Reader Utility, a game, PSP™ system software (RSS channel feature), and the distribution server. The usage procedure is discussed in more detail below.

(1) [game]: Register a Channel

A game program uses the RSS channel subscription utility to register the URL of an RSS feed distributed from a server operated by the game developer. Registration information is stored on a Memory Stick™ as subscription channel data, along with the title, thumbnail image and other information.

(2) [system software]: Reference the URL of the RSS feed

PSP™ system software (RSS channel feature) accesses the Memory Stick™ and references the URL of the RSS feed that was registered in step (1).

(3) [system software]: Download the RSS information

PSP™ system software (RSS channel feature) accesses the URL that was referenced in step (2) and downloads the RSS information which contains the URL of the distributed content. The downloaded RSS information is not saved on the Memory Stick™.

(4) [system software]: Reference the URL of the content

PSP™ system software (RSS channel feature) accesses the RSS information that was downloaded in step (3) and references the URL of the distributed content.

(5) [system software]: Download the content

PSP™ system software (RSS channel feature) accesses the content referenced by the URL in step (4) and downloads it. The downloaded content is saved on the Memory Stick™.

(6) [system software]: Use or view the content

The content that was saved in step (5) can be used in a game or it can be viewed using PSP™ system software.
Steps (2) to (5) above can also be implemented by using the RSS Reader Utility in a proprietary manner by the game program.
For more details, please refer to the "RSS Reader Utility Overview".

2. 2 Content That Can Be Distributed Via an RSS Channel

The following two types of content can be distributed over an RSS channel.

(1) Audio, video, and image files

These files can be used for purposes unrelated to a game such as for podcasting. They can also be used to promote game content.

(2) Save data files

Data that is used in a game such as map data or character data can be updated daily.

2. 3 Pros and Cons of Distributing Content Via an RSS Channel

The pros and cons of distributing content via an RSS channel are listed below for users and content providers.

(1) Users

Pros

  • Using a simple procedure, users can confirm whether or not they have the latest content without the need for an Internet browser.
  • Similarly, using a simple procedure, the latest content can be saved without the need for an Internet browser.
  • A timer feature is available to automatically save the latest content via a wireless LAN without having to perform any specific action.

Cons

  • The information that enables the user to understand what the content contains consists only of a limited character string and a thumbnail image.

(2) Content providers

Pros

  • Chances are better that users will verify your content as well as be aware that new content exists.
  • The time required to develop your website can be reduced.

Cons

  • Since users can obtain desired objects directly, they will have fewer opportunities to see content other than the specific content they are obtaining.

3 RSS Channel Subscription Utility Overview

The RSS channel subscription utility provides a means of subscribing to an RSS channel from a game program so that game-related content can be downloaded and provided to the user via the Internet using the RSS channel feature, even after a game ends.
By using this utility, saved data files used by games, game trailers for promoting games, wallpaper files, sound tracks, radio programs, and other files can be provided to users through the RSS channel feature provided by PSP™ system software. For details, see various sections in this document and refer to the "Utility Reference."

3. 1 Calling Types

The utility can be called using the following two methods.

Calling the utility without a user interface
This calling method is used when the user interface for subscribing to an RSS channel is provided by the game program. In this case, the call cannot be canceled.

Calling the utility with a user interface
This calling method is used when the user interface provided by the utility for subscribing to an RSS channel is used. In this case, the user can cancel the call.

Which of the calling methods described above is used is determined by the type member of the SceUtilityRssSubscriberParam structure as shown below.

  Calling Method     type member of SceUtilityRssSubscriberParam structure  
  Without user interface     SCE_UTILITY_RSS_SUBSCRIBER_AUTOADD_CHANNEL  
  With user interface     SCE_UTILITY_RSS_SUBSCRIBER_ADD_CHANNEL  

3. 2 User Interface (when Calling the Utility With a User Interface)
