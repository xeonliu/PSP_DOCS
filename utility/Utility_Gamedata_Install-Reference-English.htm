About the abortedStatus member variable
If the game data install utility is aborted and SCE_UTILITY_COMMON_RESULT_ABORTED is returned in the result member variable of the SceUtilityParamBase structure, then the status of the game data install utility immediately before it was aborted will be returned in abortedStatus. As a result, the actual status will not be overwritten by the aborted status.
For example, if the utility is aborted (suspended), SCE_UTILITY_COMMON_RESULT_ABORTED is returned in result. However, this information alone does not enable you to know whether or not the install has completed. If SCE_UTILITY_COMMON_RESULT_OK is returned in abortedStatus at this time, then you know that the abort occurred after installation and the install has actually finished.
If abortedStatus is SCE_UTILITY_COMMON_RESULT_CANCELED, then you know that the abort occurred before the install actually finished.
If result is any other value besides SCE_UTILITY_COMMON_RESULT_ABORTED, status will not be output to abortedStatus.

Description

This structure is used for installing game data. It is passed to sceUtilityGamedataInstallInitStart().

sceUtilityGamedataInstallInitStart
 [Function]
Start game data install utility initialization

Syntax

#include <utility_gamedata_install.h>
int sceUtilityGamedataInstallInitStart(
	SceUtilityGamedataInstallParam *param
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  param     Game data install utility start-up parameters  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, one of the values SCE_UTILITY_COMMON_ERROR_xxx is returned.

Description

This function starts game data install utility initialization processing.
To use the game data install utility, this function must be called first.
If this function is successful, the game data install utility status becomes SCE_UTILITY_COMMON_STATUS_INITIALIZE, and then transitions to SCE_UTILITY_COMMON_STATUS_RUNNING.

Example

SceUtilityGamedataInstallParam param;

memset(&param, 0x00, sizeof(SceUtilityGamedataInstallParam));
param.size = sizeof(SceUtilityGamedataInstallParam);

<<Set required values in SceUtilityGamedataInstallParam>>
<<For details, see the sample>>

ret = sceUtilityGamedataInstallInitStart(&param);
Note

Since the game data install utility uses a common memory area with other utility services, if another utility service is already running, this function call will fail.

If installed data already exists on the Memory Stick™, the game data install utility will delete that data first before proceeding with the install.
If the game opens a file that was installed on the Memory Stick™, the file must be closed before the game data install utility can be used.

After the game data install utility has been started and its operation is finished, this function must be called to release the resources that were used by the game data install utility.

sceUtilityGamedataInstallUpdate
 [Function]
Game data install utility per-frame processing

Syntax

#include <utility_gamedata_install.h>
int sceUtilityGamedataInstallUpdate(
	int animation_speed
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  animation_speed     Game data install utility animation speed  
  (60 times per second = 1)  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, one of the values SCE_UTILITY_COMMON_ERROR_xxx is returned.

Description

This function performs the game data install utility's internal per-frame processing such as event processing, animation processing, and drawing processing.
The animation speed is passed in the argument animation_speed. For example, to call this function 60 times per second, specify 1, and to call it 30 times per second, specify 2.

When this function is called, the drawing context is saved within the utility, the utility performs drawing, and then the drawing context is restored.
Make sure that the game program waits for the end of drawing packet transfer before calling this function.

This function should be called when the game data install utility status that is obtained from sceUtilityGamedataInstallGetStatus() is SCE_UTILITY_COMMON_STATUS_RUNNING.

Example

ret = sceUtilityGamedataInstallUpdate(1);

Description

To detect when the game data install utility is finished, call this function every frame and check whether the return value is SCE_UTILITY_COMMON_STATUS_RUNNING or SCE_UTILITY_COMMON_STATUS_FINISHED.

		:
		:
}

#include <utility_common.h>

#define SCE_UTILITY_COMMON_RESULT_CANCELED	(1)
#define SCE_UTILITY_COMMON_RESULT_ABORTED	(2)
Description

This is the utility service execution result that is returned in the result member variable of SceUtilityParamBase. Note that there may be cases when the game data install utility returns execution results using a special method.

Cancel termination SCE_UTILITY_COMMON_RESULT_CANCELED
If installation is canceled by the user or by calling sceUtilityGamedataInstallAbort(), the game data install utility will return SCE_UTILITY_COMMON_RESULT_CANCELED in result.
However, if SCE_UTILITY_COMMON_RESULT_ABORTED is returned in result, SCE_UTILITY_COMMON_RESULT_CANCELED may be returned in abortedStatus.
For details, see "About the abortedStatus member variable" in the section about SceUtilityGamedataInstallParam.
