Reading Content Information Files
Even if the members of SceUtilitySDExtFile are appropriately set, types that can read content information files will not read them in the following cases.
  • When the file is not found
  • When the actual file size is greater than the buffer size (dataBufSize).
At these times, no error is returned. Note that "File not found" and "File is corrupted" are only for data files and not for content information files.

Overwrite Mode (member variable overWriteMode)
This is a valid parameter when saving save data (all types).
The parameter affects the handling of the content information files ICON0.PNG, ICON1.PMF, PIC1.PNG, and SND0.AT3.
When overwrite mode is OFF, file writing is not performed for a file that already exists. No decision is made concerning whether or not the contents are the same. Member Variable abortedStatus
When the Save Data utility is forcibly terminated and SCE_UTILITY_COMMON_RESULT_ABORTED is returned in member variable result of SceUtilityParamBase, the status of the Save Data utility immediately before it was forcibly terminated is returned in abortedStatus. As a result, the actual status will not be overwritten by the forced termination status.
For example, if the Save Data utility is forcibly terminated (suspended) during fixed loading, SCE_UTILITY_COMMON_RESULT_ABORTED is returned in result. If only this happened, you would not know whether or not the save data was actually read. If SCE_UTILITY_COMMON_RESULT_OK is returned in abortedStatus at this time, it indicates that the Save Data utility was forcibly terminated after the data was read. Therefore, the read buffer in memory can be used.
If abortedStatus is SCE_UTILITY_COMMON_RESULT_CANCELED, it indicates that the Save Data utility was forcibly terminated before the data was read, and no data is loaded in the buffer.
If result is not SCE_UTILITY_COMMON_RESULT_ABORTED, no status is output to abortedStatus.

This is a member variable for specifying the save data format (version).

  • Old format 2: Formats for Runtime Library Release 2.0.0, 2.5.0 and 2.6.0

Member Variable secureFileId
This is a parameter for strengthening protection against illegal modification or inspection of protected data files.
Although the ID itself can be specified for an individual protected data file, the application range is limited according to the Save Data utility type that is used.
  • For list-selection saving and loading types, an ID is applied to all list-specified data. It cannot be specified individually.
  • For data file writing and reading types, an ID can be applied to individual files.
Since a data file will no longer be able to be decoded if the ID is lost, you should manage IDs carefully. When operating so that ID values are saved in a separate data file, decoding will be impossible if that file is lost.
The following constraints apply to the (16-byte) binary values that are used for IDs.
  • An ID in which all 16 bytes are 0 cannot be used. The utility will return an error.
  • Avoid using ASCII character codes for all 16 bytes. (No error will occur, but this should be avoided.)
  • Avoid using the same value for all 16 bytes. (No error will occur, but this should be avoided.)

Content Information File Decision Method
Member variables icon0, icon1, pic1, and snd0 of the SceUtilitySavedataParam structure are used.
If each pDataBuf is NULL or dataBufSize is 0, the content information file is handled in the same way as when the size is not changed by overwriting or new data is not created or, in other words, as when the content information file does not appear in the list for a situation involving a data file list.
To decide correctly, appropriate values must be specified for pDataBuf and dataBufSize. The buffer specified by pDataBuf is not accessed.
Note: If the save data saving decision type is used, save data will be created or overwritten by the next startup type that is used. Therefore, specifications of this kind are the result of a judgment that setting appropriate values for pDataBuf and dataBufSize will not be troublesome since a required buffer should be expected for content information. Unfortunately, extra effort is required for uses that are incompatible with this assumption.
Note: During overwriting, when overWriteMode is referenced for calculating the size and if the calculated size is insufficient, a judgment is made to determine whether to overwrite the content information file.
Therefore, when writing, if an insufficient size is determined for the following types, which reference overWriteMode, overWriteMode must match the input when these calls are made.
SCE_UTILITY_SAVEDATA_TYPE_AUTOSAVE
SCE_UTILITY_SAVEDATA_TYPE_SAVE
SCE_UTILITY_SAVEDATA_TYPE_LISTSAVE
SCE_UTILITY_SAVEDATA_TYPE_MC_WRITE_SECUREFILE
SCE_UTILITY_SAVEDATA_TYPE_MC_WRITE_NORMALFILE

When the save data to be overwritten is not found, it is simply handled as saving new data.

Member Variables overwriteNeedSizeKB and overwriteNeedSizeStr
These member variables return the amount of free space lacking for the save data to be overwritten when overwriting is performed in terms of individual files.
When there is sufficient free space, 0 is output for overwriteNeedSizeKB, but overwriteNeedSizeStr is not output.

When no Memory Stick™ is inserted
The save data size is based on the assumption of a Memory Stick™ having a cluster size of 32KiB and is returned forcreateNeedSizeKB and createNeedSizeStr.
This can be used for displaying a message such as "Insert a Memory Stick™ with at least xx KB of free space" on the game screen.
