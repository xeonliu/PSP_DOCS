PSP™ Programmer Tool Runtime Library Release 6.3.0

Utility Services Overview


1 Utility Overview

Utilities are services that make certain functions of PSP™ system software available to a game program.

The following services are provided.

  • Setting and getting system configuration information
  • Saving and reading save data
  • Entering strings
  • Configuring the network
  • Displaying message dialogs
  • Game sharing server functions
  • Internet browser
  • Functions for loading modules in flash memory
  • Screenshots
  • NP sign-in
  • PS3™ scan
  • RSS channel subscription
  • RSS Reader

System Information Setting / Getting Utility
The service for setting and getting system information is provided to set and get system information saved in the flash memory of the PSP™.

Save Data Utility
The save data utility provides a means of manipulating, loading, saving, and deleting save data. For details, refer to the "Save Data Utility Overview" document.

On-Screen Keyboard Utility
The on-screen keyboard utility provides a means of entering and editing text.

Network Configuration Utility
The network configuration utility provides a means of configuring network connections that use a wireless LAN device.

Message Dialog Utility
The message dialog utility provides a means of displaying on screen dialogs that contain system messages, or messages provided by a game program.

Game Sharing Utility
The game sharing utility is a function for sending external bootable binaries to PSP™ peers, which are clients, using ad hoc mode communication. For details, refer to the "Game Sharing Overview" document.

Internet Browser Utility

The Internet browser utility provides a means of calling the Internet browser from a game program. For details, refer to the "Internet Browser Utility Overview" document.

Module Utility
This utility loads and unloads modules in flash memory.

Screenshot Utility
This utility captures game screens and saves them as image files on a Memory Stick™.

NP Sign-In
Sign in to the PlayStation®Network. For details, refer to the "PlayStation®Network Overview" document.

PS3™ Scan Utility
The PS3™ scan utility provides a means of connecting a PSP™ and PlayStation®3 via a wireless LAN.

RSS Channel Subscription
The RSS channel feature provided by PSP™ system software is used to save channel information on a Memory Stick™ so that later, game-related data can be saved via the network even after a game ends. For details, refer to the "RSS Channel Subscription Utility" document.

RSS Reader Utility
The RSS reader utility provides the functions of an RSS reader but without a user interface. For details, refer to the "RSS Reader Utility Overview" document.

1. 2 Using Utilities

To use a service that has a user interface from a game program, or to use a service that does not have a user interface but runs asynchronously with the game program, the four interfaces shown below should be called.
  1. Intialize and start utility [ sceUtility*InitStart() ]
  2. Shut down utility [ sceUtility*ShutdownStart() ]
  3. Update utility drawing [ sceUtility*Update() ]
  4. Get utility status [ sceUtility*GetStatus() ]



For other services, call the interfaces defined by each utility.


1. 3 Utility Initialization Parameters

A set of utility initialization parameters are defined by SceUtility*Param for each utility. When a utility is initialized, use these initialization parameters when calling sceUtility*InitStart() for the utility you wish to use.

When a utility is in use, these initialization parameters must be a maintained in a persistent memory area. (Note: The declaration or use of utility initialization parameters by automatic variables is strictly prohibited.)

2 Utility Services and Sleep Mode

The utilities below will exit automatically when suspend processing occurs.

  • Save data utility
  • On-screen keyboard utility
  • Network configuration utility
  • Message dialog utility
  • Game sharing utility
  • NP sign-in
  • Screenshot utility
  • PS3™ scan utility
  • RSS channel subscription utility
  • RSS reader utility
  • Store checkout utility
  • PSN utility

If suspend processing causes these utility services to be automatically terminated, then SCE_UTILITY_COMMON_RESULT_ABORTED will be stored as the termination status.

Suspend processing is kept waiting until the utilities terminate. Note that, for this reason, game applications must call the sceUtility*Update() and sceUtility*ShutdownStart() functions so that the utilities will be able to perform termination processing even in the midst of suspend processing. During suspend processing, if sceUtility*Update() and sceUtility*Shutdown function calls are stopped, suspend processing may end up in a deadlock.

Network Configuration Overview
Multiple network configurations exist, and one network configuration contains various configuration items. Each network configuration is identified by a network configuration ID (integer value greater than or equal to 1). Currently, at most 10 network configurations can be saved, and the IDs range from 1 to 10.

Library Usage Example
A network configuration is read by specifying a configuration item and calling sceUtilityGetNetParam(). One configuration item is read each time this function is called, and the value that is obtained is saved in the SceUtilityNetParamEntry union.
The following example gets the SSID.

int print_ssid(int id)
{
	union SceUtilityNetParamEntry entry;
	int ret = 0;

	/* Read SSID */
	ret = sceUtilityGetNetParam(id, SCE_UTILITY_NET_PARAM_CODE_SSID, &entry);
	if(ret < 0){
		/* Error handling */
	}
	printf("SSID: %s\n", entry.ssid);

	return ret;
}

4 Display Languages and Button Assignments

The display language and the console's button assignments for all of the utilities, as well as for Impose (the game exit confirmation screen displayed when the PS button (HOME button) is pressed or when the disc cover OPEN switch is operated), can be set from a game program.
It is recommended that they be explicitly set to the same assignments as those used in the game.

4. 1 Relationship with PSP™ System Software

Utilities and Impose do not provide the same settings as PSP™ system software. This is due to the fact that, from the perspective of the PSP™ user, the utilities and Impose are a part of the game, and there is no need for parameters which may cause the game to perform in unintended ways.
However, in terms of the game specifications, this does not necessarily mean that the game itself has to have the same language and button assignments as PSP™ system software. Using the system information acquisition utility function sceUtilityGetSystemParamInt(), PSP™ system software settings can be obtained, providing a structure in which the game can run with the same language and button assignments as PSP™ system software, and also, the parameters which represent the settings used by the game can furthermore be passed to the utilities and to Impose.

4. 2 Settings Passed to the Utilities

The display language and button assignments are respectively set using the message_lang and ctrl_assign members of the SceUtilityParamBase structure.
Note that if an initialization function is called with the SceUtilityParamBase structure cleared to 0 using memset() or the like, the above members will not be set, and the display language will default to Japanese, while the ? (circle) button will become the Enter button.

4. 3 Settings Passed to Impose

The display language and button assignments are respectively set using the language and button_assign arguments to the sceImposeSetLanguageMode() function.
Since it is not known when the Impose screen will be called, set these as soon as possible after the game program is executed. Of course, this does not include cases in which a game's display language can be changed.
Note that if the function is not called, the settings will be the same as those of PSP™ system software.

5 On-Screen Keyboard Utility

The on-screen keyboard utility provides a means of entering or editing text.

The on-screen keyboard has a mechanism for entering and editing text and several keys for entering characters on the PSP™ screen. A PSP™ user can enter characters or numbers to edit text by manipulating keys that are displayed on this screen and hardware buttons on the PSP™ console.

English letters (uppercase or lowercase, and letters with phonetic symbols), Cyrillic letters (uppercase or lowercase), English symbols, numbers, hiragana, and katakana can be entered from the keys that are displayed on the screen. To enable entry of characters that cannot be directly entered from the keys on the screen (such as kanji), the on-screen keyboard internally maintains a kana-kanji conversion mechanism. The PSP™ console hardware buttons are used to manipulate the buttons on the screen, move the cursor that indicates the text editing position, and to perform operations such as kana-kanji conversion.

When multiple text entries are required, input processing can be performed collectively without starting up the on-screen keyboard numerous times. An on-screen keyboard utility user (application) can specify several options when starting up the on-screen keyboard. If there are multiple inputs, options can be specified independently for each one.

When the on-screen keyboard is started up, the controller is set to the mode for using both the digital buttons and analog pad, and when the keyboard is shut down, the controller is returned to the original mode.
The on-screen keyboard utility user (application) should not change the controller mode while the on-screen keyboard is running.

The options are summarized below.
  • Specifying the text to be edited, which is set when the on-screen keyboard is started
  • Specifying an area for storing the edited string
  • Specifying the type of characters to be entered by using the on-screen keyboard

6 Network Configuration Utility

The network configuration utility is used to configure settings needed to establish a network connection using the wireless LAN device. The two types of network connections are infrastructure mode and ad hoc mode.

With infrastructure mode, at most 10 configurations can be saved. A network connection can be established by selecting a saved configuration. Using the reconnection service, a connection can be automatically established using the previous settings from the last connection. However, multiple connections cannot be controlled simultaneously. While an infrastructure mode connection is in progress, a function for displaying the connection status can be used.

Ad hoc mode has no function for editing or saving a configuration. A network connection can only be established. A connection can be established using the connect, create, or join functions.

The network configuration utility requires a network library that corresponds to the connection mode. Before using this utility, be sure to load and initialize the network library. (For details, refer to the PSP™ Network Libraries.)

The following services are provided.
Since these services use a common memory area, multiple services cannot be used at the same time.
  • Starting an infrastructure mode connection and editing a configuration
  • Displaying the status of an infrastructure mode connection
  • Starting an ad hoc mode connection
  • Starting an infrastructure mode reconnection
  • Starting an ad hoc mode connection (create)
  • Starting an ad hoc mode connection (join)

9. 6 Cont Mode Startup

Cont mode provides a faster way to perform module loading, initialization, and function execution during utility startup.
Cont mode is used by performing the following steps.
  1. Start Cont mode
  2. Execute Cont mode functions (can be done multiple times)
  3. End Cont mode
First, call sceUtilityScreenshotInitStart() and specify a startup type of "SCE_UTILITY_SCREENSHOT_TYPE_CONT_START" to start Cont mode and perform setup processing. Next, call sceUtilityScreenshotContStart() to execute Cont mode functions. This can be done multiple times. After function execution has completed, call sceUtilityScreenshotContStart() with a startup type of "SCE_UTILITY_SCREENSHOT_TYPE_CONT_FINISH" to end Cont mode.
Note that different functions are called at different times. sceUtilityScreenshotInitStart() is called when starting Cont mode, but sceUtilityScreenshotContStart() should be called subsequently. If these functions are not used in the proper order, the utility will not work properly.
After the utility is started, sceUtilityScreenshotContStart() can be called once the status returned by sceUtilityScreenshotGetStatus() has transitioned to SCE_UTILITY_COMMON_STATUS_WAITING. If the status has not transitioned to this value, it means that initialization processing for starting the utility has not completed and calling sceUtilityScreenshotContStart() at this time will result in an error. Also, when the status is SCE_UTILITY_COMMON_STATUS_WAITING, the system appears to behave as if it has exited from the utility, however, loop processing for the utility must continue until the utility has terminated its processing, as is the case with a normal utility.
(For specific implementation examples, see the implementation of the utility_update_screenshot() function in screenshot.c.)
The functions that use Cont mode are explained below.

SCE_UTILITY_SCREENSHOT_TYPE_CONT_AUTO

9. 14 Calculating the Space Required to Save a Screenshot

This section briefly explains the method used to calculate the space required by the screenshot utility for saving a screenshot image.

PSCM.DAT

  iconFileSize     : Icon image file size (bytes)  
  bgFileSize     : Background image file size (bytes)  

Captured Image Size
Since the size of the captured image is determined as a result of performing actual save processing (compression processing), its precise size cannot be known in advance. By estimating the size, you can generally avoid errors due to insufficient space when the image is saved.

Size of the Screenshot Directory screenshot_dir
Since the size of this directory depends on the format of the destination where the screenshot images will be saved, a constant value cannot be used. However, you can generally create this directory by estimating the size to be 1 cluster (32KiB), Note that if you want to calculate the size more precisely, you must obtain cluster information for the destination and take into consideration the size of one cluster.

15 Precautions

15. 1 Initialization and Shutdown Status While a Utility Service is Initializing or Shutting Down

Although the return value of sceUtility*GetStatus() while a utility service is being initialized or shut down is SCE_UTILITY_COMMON_STATUS_INITIALIZE or SCE_UTILITY_COMMON_STATUS_SHUTDOWN, respectively, when initialization processing or shutdown processing is terminated in a short time between two calls of sceUtility*GetStatus(), SCE_UTILITY_COMMON_STATUS_INITIALIZE or SCE_UTILITY_COMMON_STATUS_SHUTDOWN is not returned by sceUtility*GetStatus().

In other words, SCE_UTILITY_COMMON_STATUS_RUNNING (or SCE_UTILITY_COMMON_STATUS_WAITING depending on the utility type and startup type) may be returned after SCE_UTILITY_COMMON_STATUS_NONE, or SCE_UTILITY_COMMON_STATUS_NONE may be returned after SCE_UTILITY_COMMON_STATUS_FINISHED.

In this situation, be sure to call the utility service so that the game program runs normally.

15. 2 Exclusive Control of the Utility Service's sceUtility*Update() Function Call and the Game Program's Drawing Operation

The following processing is performed within the sceUtility*Update() function
  1. Save the drawing context of the application
  2. Create, transfer, and wait for the end of transfer of the utility module's drawing packets
  3. Restore the drawing context of the application
This processing is performed without any knowledge of Vblank.

Therefore, if the frame buffers are swapped by a call to sceGuSwapBuffers() or the game program's drawing packet transfer processing is executed simultaneously during sceUtility*Update() processing, drawing by the utility may be corrupted.

Be sure to wait for the end of transfer of the game program's drawing packets before calling sceUtility*Update(). Also, be sure to perform processing that delays frame buffer swapping during a call to sceUtility*Update().

15. 3 Checking Data Related to Utility Services

A method of checking the validity of data (images, sound, configuration information, etc.) related to utility services is provided as a system software debug support feature.
For details, refer to the "Debugging Support Functions Provided by System Software" chapter of the "PSP™ Setup Guide".
