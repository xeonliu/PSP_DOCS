 [定数定義]
ユーザIDをNULL指定するマクロ

定義

#include <utility_savedata.h>
#define SCE_UTILITY_SAVEDATA_USERID_NULL  	"<>"
解説

リスト選択タイプの場合、ユーザIDがNULLのセーブデータは、この文字列を使ってリストに指定することができます。
ユーザIDを入力する際にこのマクロを指定すると、NULL文字を指定するのと等価となります。
ただし、リスト選択タイプ以外で、このマクロでの入力が必須となる場合はありません。
ユーティリティからの出力の際に、このマクロが返されることはありません。
各パラメータの解説も参照下さい。

メンバ変数abortedStatusについて
セーブデータユーティリティが強制終了され、SceUtilityParamBaseのメンバ変数resultにSCE_UTILITY_COMMON_RESULT_ABORTEDが返った場合、強制終了直前のセーブデータユーティリティのステータスがabortedStatusに返されます。これにより、強制終了ステータスに実際のステータスが書き潰されてしまうことがありません。
例えば、固定ロード時に強制終了（サスペンド）されると、resultにはSCE_UTILITY_COMMON_RESULT_ABORTEDが返ります。ただこれだけだと、実際にセーブデータが読み込まれたかどうかわかりません。この時abortedStatusに、SCE_UTILITY_COMMON_RESULT_OKが返っている場合は、読み込まれた後に強制終了されていますので、メモリの読み込みバッファは使用可能ということになります。
abortedStatusがSCE_UTILITY_COMMON_RESULT_CANCELEDの場合は、読み込まれる前に強制終了となり、バッファへ読み込みは行われていません。
resultがSCE_UTILITY_COMMON_RESULT_ABORTED以外の場合には、abortedStatusへステータスは出力されません。

Memory Stick™空き容量及びセーブデータサイズ取得タイプ時のセーブデータユーティリティ実行結果について
result及びabortedStatusにエラーが返った場合でも、正常に出力されるメンバが存在する場合があります。
SCE_UTILITY_SAVEDATA_ERROR_SIZES_NO_MS、SCE_UTILITY_SAVEDATA_ERROR_SIZES_EJECT_MS、SCE_UTILITY_SAVEDATA_ERROR_SIZES_ACCESS_ERRORが返っても、pUtilityData以下のメンバにはMemory Stick™が不要なものがあり、そのメンバには正しく出力されています（詳しくはメンバの説明を参照して下さい）。
その他のエラーについては、全てのメンバの結果は破棄して下さい。

メンバ変数secureFileIdについて
保護ありデータファイルの不正変更や閲覧に対する防御をより強固にするためのパラメータです。
ID自体は保護ありデータファイル個別に指定できるものですが、使用するセーブデータユーティリティのタイプによっては適用範囲に制限があります。
  • リスト選択セーブ及びロードタイプの場合、IDはリスト指定されたデータ全てに適用されます。別個に指定はできません。
  • データファイル書き込み及び読み込みタイプの場合、それぞれファイル別にIDを適用できます。
IDが失われるとデータファイルを復号化することができなくなりますので、管理には十分注意して下さい。ID値を別のデータファイルに保存するような運用をしますと、そのファイルが失われた時、復号が不可能となります。
IDに使用するバイナリ値（16bytes）については、以下の制約があります。
  • 16bytes全てが0は使用できません。ユーティリティがエラーとなります。
  • 16bytes全てがASCII文字コードは避けて下さい。（エラーとはなりませんが）
  • 16bytes全てが同じ値は避けて下さい。（エラーとはなりませんが）

不足サイズが返るので、ゲーム画面では「空き容量が足りません。あと〜KB必要です」のように使用できます。

コンテンツ情報ファイルの判定法
SceUtilitySavedataParam構造体のメンバ変数icon0、icon1、pic1、snd0が使用されます。
それぞれのpDataBufがNULL、またはdataBufSizeが0の場合は、上書きによるサイズ変更や新規作成が行われない扱い、つまりデータファイルのリストで言うところの、リストに載っていない状態と等価となります。
正しく判定するためには、pDataBufとdataBufSizeには適当な値を指定する必要があります。もちろんpDataBufで指定したバッファにアクセスすることはありません。
※セーブデータ保存判定タイプを使用すると言うことは、次回起動タイプで、セーブデータの作成や上書きが行われるということです。よって、コンテンツ情報に必要なバッファはすでに想定されているはずですので、pDataBufとdataBufSizeに適切な値を設定することが煩雑になることはない、との判断でこのような仕様となっています。この想定にそぐわないご使用の場合はお手数をおかけしますが、ご了承下さい。
ご注意：上書きする場合のサイズ計算にはoverWriteModeを参照し、不足サイズを算出する際に、コンテンツ情報ファイルの上書きを行うかを判定します。
この為、書き込み時にoverWriteModeを参照する以下のタイプ
SCE_UTILITY_SAVEDATA_TYPE_AUTOSAVE
SCE_UTILITY_SAVEDATA_TYPE_SAVE
SCE_UTILITY_SAVEDATA_TYPE_LISTSAVE
SCE_UTILITY_SAVEDATA_TYPE_MC_WRITE_SECUREFILE
SCE_UTILITY_SAVEDATA_TYPE_MC_WRITE_NORMALFILE
の不足サイズを判定する場合は、overWriteModeをこれらの呼び出す際の入力と一致させておく必要があります。

メンバ変数createNeedSizeKB、createNeedSizeStrについて
上書きするセーブデータを一旦削除し、その後新規保存する場合の不足サイズを返します。
不足しない場合はcreateNeedSizeKBには0が、createNeedSizeStrは出力されません。
上書きするセーブデータが見つからない場合は、単に新規保存として扱います。

メンバ変数overwriteNeedSizeKB、overwriteNeedSizeStrについて
上書きするセーブデータに対し、ファイル単位で上書きする場合の不足サイズを返します。
不足しない場合はoverwriteNeedSizeKBには0が、overwriteNeedSizeStrは出力されません。

Memory Stick™が差されていない時は
クラスタサイズ32KiBのMemory Stick™を仮定した上でのセーブデータサイズを、createNeedSizeKB、createNeedSizeStrに返します。
ゲーム画面では「〜KB以上の空きのあるMemory Stick™を差して下さい」のように使用できます。
