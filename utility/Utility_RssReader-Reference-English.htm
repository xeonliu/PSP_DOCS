sceUtilityRssReaderInitStart
 [Function]
Start initialization of RSS reader utility

Syntax

#include <utility_rss_reader.h>
int sceUtilityRssReaderInitStart( 
	SceUtilityRssReaderParam *param
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  param     RSS reader utility startup parameters  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, SCE_UTILITY_COMMON_ERROR_xxx is returned.

Description

This function starts RSS reader utility initialization processing.
To use the RSS reader utility when it is in SCE_UTILITY_COMMON_STATUS_NONE state, this function must be called.
The following call type is functionally enabled with this function.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE
Other calling types either return an error or are functionally disabled.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE type

If the call is successful, the utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_WAITING state.
If 0 is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, an internal thread is started up to read the RSS feed, the retrieving member of the SceUtilityRssReaderRetrieveArgs structure becomes true, and the utility enters "reading RSS feed" state.
If an error value is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, the retrieving member of the SceUtilityRssReaderRetrieveArgs structure becomes false, and the utility enters "RSS feed not read, waiting" state.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL, SCE_UTILITY_RSS_READER_TYPE_GET_INFO, SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD, or SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL type

If the call is successful, the utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_WAITING state.
Then, an error is returned in the result member of the SceUtilityParamBase structure.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

If the call is successful, the utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

	ret = sceUtilityRssReaderContStart(&rss_reader_param);
}
else {
	ret = sceUtilityRssReaderInitStart(&rss_reader_param);
}

sceUtilityRssReaderContStart
 [Function]
Start continuation processing of RSS reader utility

Syntax

#include <utility_rss_reader.h>
int sceUtilityRssReaderContStart( 
	SceUtilityRssReaderParam *param
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

  param     RSS reader utility startup parameters  

Return Value

When the function completes normally, 0 is returned.
If an error occurs, SCE_UTILITY_COMMON_ERROR_xxx is returned.

Description

This function starts continuation processing for the RSS reader utility.
To use the RSS reader utility when it is in SCE_UTILITY_COMMON_STATUS_WAITING state, this function must be called.

The following call types are functionally enabled with this function.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL
  • SCE_UTILITY_RSS_READER_TYPE_GET_INFO
  • SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD
  • SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL
  • SCE_UTILITY_RSS_READER_TYPE_FINISH

When the function is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE type

If the function is successful, the RSS reader utility remains in SCE_UTILITY_COMMON_STATUS_WAITING state.
If 0 is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, an internal thread is started up to read the RSS feed, the retrieving member of the SceUtilityRssReaderRetrieveArgs structure that was passed when the function was called becomes true, and the utility enters "reading RSS feed" state. While retrieving is true, the internal thread of the utility will reference the SceUtilityRssReaderParam structure that was passed when the function was called. Therefore, it should not be changed until retrieving becomes false.
If an error value is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, the retrieving member of the SceUtilityRssReaderRetrieveArgs structure becomes false, and the internal state does not change.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL type

If the function is successful, the RSS reader utility remains in SCE_UTILITY_COMMON_STATUS_WAITING state.
If an internal thread has been started up to read the RSS feed, it is terminated. At this time, the retrieving member of the SceUtilityRssReaderRetrieveArgs structure that was passed when the request was made to read the RSS feed becomes false, and the utility enters "RSS feed not read, waiting" state.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_GET_INFO type

If the function is successful, the RSS reader utility remains in SCE_UTILITY_COMMON_STATUS_WAITING state.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD type

If the function is successful, the RSS reader utility remains in SCE_UTILITY_COMMON_STATUS_WAITING state.
If 0 is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, an internal thread is started up for downloading, the downloading member of the SceUtilityRssReaderDownloadArgs structure that was passed when the function was called becomes true, and the utility enters "downloading" state. While downloading is true, the internal thread of the utility will reference the SceUtilityRssReaderParam structure that was passed when the function was called. Therefore, it should not be changed until downloading becomes false.
If an error value is returned in the result member of the SceUtilityParamBase structure that was passed when the function was called, the downloading member of the SceUtilityRssReaderDownloadArgs structure becomes false, and the internal state does not change.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL type

If the function is successful, the RSS reader utility remains in SCE_UTILITY_COMMON_STATUS_WAITING state and if an internal thread has been started up for downloading, it is terminated. At this time, the downloading member of the SceUtilityRssReaderDownloadArgs structure that was passed when downloading was requested becomes false, and the utility enters "finished reading RSS feed, waiting" state.

When the function is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

If the function is successful, the RSS reader utility transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

sceUtilityRssReaderGetStatus
 [Function]
Get RSS reader utility status

Syntax

#include <utility_rss_reader.h>
int sceUtilityRssReaderGetStatus( 
	void
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (does not depend on interrupt-disabled or -enabled state)
Not multithread safe

Arguments

None

Return Value

When the function completes normally, the RSS reader utility state or SCE_UTILITY_COMMON_STATUS_xxx are returned.
If an error occurs, SCE_UTILITY_COMMON_ERROR_xxx is returned.

Description

To detect that the RSS reader utility has finished, this function is called every frame to check whether the return value is SCE_UTILITY_COMMON_STATUS_RUNNING or SCE_UTILITY_COMMON_STATUS_FINISHED.
To continuously call the RSS reader utility, this function should be called every frame to check whether the return value is SCE_UTILITY_COMMON_STATUS_RUNNING or SCE_UTILITY_COMMON_STATUS_WAITING.

Example

while(1) {
	int status = sceUtilityRssReaderGetStatus();

	if(status == SCE_UTILITY_COMMON_STATUS_NONE) {
		break;
	}

	switch(status) {
	case SCE_UTILITY_COMMON_STATUS_RUNNING:
	case SCE_UTILITY_COMMON_STATUS_WAITING:{
		ret = sceUtilityRssReaderUpdate(1);
		break;
	}
	case SCE_UTILITY_COMMON_STATUS_FINISHED:{
		ret = sceUtilityRssReaderShutdownStart();
		break;
	}
	case SCE_UTILITY_COMMON_STATUS_SHUTDOWN: {
		break;
	}
}
