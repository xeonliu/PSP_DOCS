PSP® Programmer Tool Runtime Library Release 6.6.0

ユーティリティサービス

1 ユーティリティ概要

ユーティリティは、PSP®のシステムソフトウェアにおける機能の一部をゲームプログラムに提供するサービスです。

提供するサービスは以下の通りです。

  • システム設定情報の設定と取得
  • セーブデータの保存と読み込み
  • 文字列の入力
  • ネットワーク設定
  • メッセージダイアログの表示
  • ゲームシェアリングサーバー機能
  • インターネットブラウザ
  • フラッシュメモリ上のモジュールロード機能
  • スクリーンショット
  • NPサインイン
  • PS3®スキャン
  • RSSチャンネル登録
  • RSSリーダー

システム情報の設定/取得ユーティリティ
本体フラッシュメモリに保存されているシステム設定情報の設定と取得の方法を提供します。

セーブデータユーティリティ
セーブデータに関する操作、ロード、セーブ、削除などを行う手段を提供します。詳細は、「セーブデータユーティリティ」概要ドキュメントを参照してください。

オンスクリーンキーボードユーティリティ
テキストの入力・編集手段を提供します。

ネットワーク設定ユーティリティ
無線LANデバイスを利用したネットワーク接続のための設定を行う手段を提供します。

メッセージダイアログユーティリティ
システム規定あるいはゲームプログラムが指定するメッセージを含んだダイアログを画面に表示する手段を提供します。

ゲームシェアリングユーティリティ
クライアントとなるPSP®端末に対して、外部起動用バイナリをアドホックモード通信によって送信するための機能です。詳細は「ゲームシェアリング概要」ドキュメントを参照してください。

インターネットブラウザユーティリティ
ゲームプログラムからインターネットブラウザを呼び出すための手段を提供します。詳細は、「インターネットブラウザユーティリティ」概要ドキュメントを参照してください。

モジュールユーティリティ
フラッシュメモリ上のモジュールのロード・アンロードを行います。

スクリーンショットユーティリティ
ゲーム画面をキャプチャし、画像ファイルとしてMemory Stick™に保存します。

NPサインイン
PlayStation®Networkへのサインインを行います。詳細は「PlayStation®Network 概要」ドキュメントを参照してください。

PS3®スキャンユーティリティ
PSP®とPlayStation®3を無線LANで接続するための手段を提供します。

RSSチャンネル登録
PSP®システムソフトウェア上のRSSチャンネル機能を使って、ゲームの終了後もネットワークを通じてゲームに関係するデータを保存できるように、Memory Stick™上にチャンネル情報を保存します。詳細は、「RSSチャンネル登録ユーティリティ」ドキュメントを参照してください。

RSSリーダーユーティリティ
ユーザインタフェースのない、RSSリーダー機能を提供します。詳細は、「RSSリーダーユーティリティ」概要ドキュメントを参照してください。

1. 2 ユーティリティの使用

ユーザインタフェースを含むサービス、あるいはユーザインタフェースを含まないがゲームプログラムと非同期に動作するサービスをゲームプログラムから使用するには、次に示す4つのインタフェースを呼び出します。
  1. ユーティリティの初期化と起動 [ sceUtility*InitStart() ]
  2. ユーティリティの終了 [ sceUtility*ShutdownStart() ]
  3. ユーティリティの描画の更新 [ sceUtility*Update() ]
  4. ユーティリティの状態取得 [ sceUtility*GetStatus() ]



それ以外のサービスについては、それぞれのユーティリティが定義するインタフェースを呼び出します。


1. 3 ユーティリティ初期化パラメータ

SceUtility*Paramで示されるユーティリティ初期化パラメータは、ユーティリティ単位に定義されています。ユーティリティ初期化時は、使用するユーティリティのsceUtility*InitStart()に対して、このユーティリティ初期化パラメータを使用します。

ユーティリティ使用中において、この初期化パラメータは永続的メモリ領域に確保されていなければなりません（注：自動変数によるユーティリティ初期化パラメータ宣言・使用は絶対に行わないでください）。

2 ユーティリティサービスとスリープモードについて

以下のユーティリティは、サスペンド処理が発生すると自動的に終了します。

  • セーブデータユーティリティ
  • オンスクリーンキーボードユーティリティ
  • ネットワーク設定ユーティリティ
  • メッセージダイアログユーティリティ
  • ゲームシェアリングユーティリティ
  • NPサインイン
  • スクリーンショットユーティリティ
  • PS3®スキャンユーティリティ
  • RSSチャンネル登録ユーティリティ
  • RSSリーダーユーティリティ
  • ストアチェックアウトユーティリティ
  • PSNユーティリティ

これらのユーティリティサービスがサスペンド処理によって自動終了した場合、終了ステータスとしてSCE_UTILITY_COMMON_RESULT_ABORTEDが格納されます。

ユーティリティが終了するまで、サスペンド処理は待たされます。このため、サスペンド処理中にもユーティリティが終了処理を実行できるように、ゲームアプリケーションはsceUtility*Update()関数およびsceUtility*ShutdownStart()関数を呼び出す必要があることに注意してください。サスペンド処理中にsceUtility*Update()およびsceUtility*ShutdownStart()関数の呼び出しを停止すると、サスペンド処理がデッドロックに陥る可能性があります。

3. 3 ネットワーク設定

PSP®ネットワークライブラリでアクセスポイントに接続したり通信を行うには、接続環境にしたがったネットワーク設定が必要となります。システム情報の設定/取得ユーティリティでは、フラッシュメモリに保存されているネットワーク設定を取得することができます。ネットワーク設定の編集は、ネットワーク設定ユーティリティを用いて行います。

システム情報の設定/取得ユーティリティにおいて、ネットワーク設定を取得するために必要なファイルは次の通りです。
  カテゴリ     ファイル名  
  ヘッダファイル     utility_netparam.h  
  スタブライブラリファイル     utility_stub.a  

ネットワーク設定概要
ネットワーク設定は複数存在し、1つのネットワーク設定には様々な設定項目が含まれています。
各ネットワーク設定は、ネットワーク設定ID(1以上の整数値)によって識別されます。現状、ネットワーク設定は最大10個保存可能で、ID範囲も1〜10となります。

ライブラリ使用例
ネットワーク設定の読み出しは、項目を指定してsceUtilityGetNetParam()を呼び出すことで行います。1回の呼び出しで1つの設定項目を読み出し、取得した値はSceUtilityNetParamEntry共用体に格納します。
以下、SSIDを取得する例です。

int print_ssid(int id)
{
	union SceUtilityNetParamEntry entry;
	int ret = 0;

	/* SSID を読み出す */
	ret = sceUtilityGetNetParam(id, SCE_UTILITY_NET_PARAM_CODE_SSID, &entry);
	if(ret < 0){
		/* エラー処理 */
	}
	printf("SSID: %s\n", entry.ssid);

	return ret;
}

4 表示言語とボタンアサインについて

すべてのユーティリティと、インポーズ（PSボタン(HOMEボタン)押下時もしくはディスクカバーOPENスイッチスライド時に表示されるゲーム終了確認画面）では、ゲームプログラムから表示言語と本体のボタンアサインを設定することができます。
できうる限りゲーム内で使用しているものと同じになるように、明示的に設定することを推奨します。

4. 1 PSP®システムソフトウェアとの関連

ユーティリティおよびインポーズが、PSP®システムソフトウェアと同じ設定となるパラメータは用意しておりません。これは、PSP®ユーザにとって、ユーティリティおよびインポーズはゲームの一部であり、ゲームが意図しない動作となる可能性のあるパラメータは不要であるというのが理由です。
ただし、あくまでゲームの仕様として、ゲーム自体が、PSP®システムソフトウェアと同じ言語やボタンアサインで動作するという場合はその限りではありません。システム情報取得ユーティリティ関数sceUtilityGetSystemParamInt()によって、PSP®システムソフトウェアの設定が取得できますので、これで得た言語およびボタンアサインでゲームが動作し、かつ、ユーティリティおよびインポーズへゲームと同じ設定となるパラメータを渡すような造りは可能となっています。

4. 2 ユーティリティへの設定

表示言語、ボタンアサインはそれぞれ、SceUtilityParamBase構造体のメンバmessage_langとctrl_assignで設定します。
SceUtilityParamBase構造体をmemset()などで0クリアしたまま、上記メンバを設定せずに初期化関数を呼び出すと、表示言語は日本語、決定ボタンは○ボタンとなってしまいますのでご注意ください。

4. 3 インポーズへの設定

表示言語、ボタンアサインはそれぞれ、sceImposeSetLanguageMode()関数の、language、button_assign引数で指定します。
インポーズ画面はいつ呼び出されるか不明なため、ゲームプログラム実行後なるべく早い時点で設定してください。もちろん、ゲームの表示言語を変更できる場合などは除きます。
関数の呼び出しを行わない場合は、PSP®システムソフトウェアと同じ設定となりますのでご注意ください。

5 オンスクリーンキーボードユーティリティ

オンスクリーンキーボードユーティリティは、テキストの入力・編集手段を提供します。

オンスクリーンキーボードは、テキストの入力と編集を行うための機構と文字を入力するためのいくつかのキーをPSP®の画面に有します。PSP®ユーザは、この画面に表示されたキーとPSP®本体のハードウェアボタンを操作する事により、文字や数字を入力しテキストを編集する事ができます。

画面上に表示されるキーからは、英語アルファベット（大文字・小文字、発音記号付きの文字）、ロシア語アルファベット（大文字・小文字）、英記号、数字、ひらがな、カタカナを入力する事ができます。画面上のキーから直接入力できない文字（漢字等）の入力を可能にするために、オンスクリーンキーボードは仮名漢字変換機構を内部に保持しています。PSP®本体のハードウェアボタンからは、画面上のボタンの操作やテキスト編集位置を示すカーソル移動の操作、仮名漢字変換等の操作を行います。

複数のテキスト入力が必要な場合は、何度もオンスクリーンキーボードを起動する事なく一括して入力を処理する事ができます。オンスクリーンキーボードユーティリティの利用者（アプリケーション）は、オンスクリーンキーボード起動時にいくつかのオプションを指定する事ができます。入力対象が複数ある場合はその個々に対して独立して指定する事ができます。

オンスクリーンキーボードは、起動時にデジタルボタンとアナログパッドの両方を使用するモードにコントローラを設定し、終了時に元のモードへ復元します。
オンスクリーンキーボード動作中は、オンスクリーンキーボードユーティリティの利用者（アプリケーション）からコントローラのモードを変更しないようにしてください。

オプションの概略は以下の通りです。
  • オンスクリーンキーボード開始時にセットされる編集対象テキストの指定
  • 編集された文字列をストアする領域の指定
  • オンスクリーンキーボードで入力する文字の種別指定

6 ネットワーク設定ユーティリティ

ネットワーク設定ユーティリティは、無線LANデバイスを利用したネットワーク接続を行うための設定を行います。ネットワーク接続には、インフラストラクチャモードとアドホックモードがあります。

インフラストラクチャモードでは最大で10設定を保存できます。また、保存した設定を選択して接続を行うことができます。再接続サービスでは、最後の接続で使用したある設定で自動的に接続を行うことができます。ただし、いずれも同時に複数の接続を制御することはできません。インフラストラクチャモード接続中は、接続状態を表示する機能を利用できます。

アドホックモードでは設定を編集・保存する機能はありません。接続のみを行うことができます。接続方法として、接続、作成、参加の3つの機能を利用することができます。

ネットワーク設定ユーティリティは、接続モードに応じたネットワークライブラリを必要とします。このユーティリティを使用する前に、ネットワークライブラリの読込み、および初期化を行ってください。（詳細はPSP®ネットワークライブラリを参照してください。）

提供するサービスは以下の通りです。
これらのサービスは共通のメモリ領域を使用するため、複数のサービスを同時に使用することはできません。

  • インフラストラクチャモード接続の開始、および設定の編集
  • インフラストラクチャモード接続の状態を表示
  • アドホックモード接続の開始
  • インフラストラクチャモード再接続の開始
  • アドホックモード接続(作成)の開始
  • アドホックモード接続(参加)の開始

9. 6 Contモードによるユーティリティ起動

Contモードは、ユーティリティ起動において、モジュールロード、初期化、機能実行までの処理をより高速に起動するためのしくみです。
Contモードは以下のような順序で使用します。
  1. Contモードの開始
  2. Contモード機能の実行（複数回呼び出し可）
  3. Contモードの終了
はじめに、ContモードをsceUtilityScreenshotInitStart()により開始します。このときに指定する起動タイプは"SCE_UTILITY_SCREENSHOT_TYPE_CONT_START"です。これによりユーティリティの準備処理が行われます。次に、sceUtilityScreenshotContStart()を使用してContモード機能を実行します。この実行は複数回可能です。
機能の実行が終了したら、sceUtilityScreenshotContStart()に、起動タイプ"SCE_UTILITY_SCREENSHOT_TYPE_CONT_FINISH"を指定してContモードを終了します。
呼び出しに際し、注意しなければならないことは、使用する関数が変化することです。Contモードの開始では、sceUtilityScreenshotInitStart()を使用しますが、それ以降は、sceUtilityScreenshotContStart()を使用する点です。呼び出し順序が正しくないと期待した動作にはなりません。
ユーティリティ起動を行い、sceUtilityScreenshotContStart()を呼出しを行うためには、sceUtilityScreenshotGetStatus()で返される状態が、SCE_UTILITY_COMMON_STATUS_WAITINGに遷移していなければなりません。この状態に遷移していない場合は、ユーティリティ起動のための初期化処理が継続中であることを意味しており、sceUtilityScreenshotContStart()での呼出しはエラーとなります。また、SCE_UTILITY_COMMON_STATUS_WAITINGにおいて、ユーティリティから抜け出たような動作に見えますが、通常のユーティリティと同様に、ユーティリティ処理のために終了状態にとなるまで、ユーティリティのためのループ処理を継続する必要があります。
（具体的な実装例については、screenshot.cのutility_update_screenshot()関数の実装を参照してください）
以下、Contモードを使用した機能について説明します。

SCE_UTILITY_SCREENSHOT_TYPE_CONT_AUTO
スクリーンショットの保存をContモードで実行します。この機能を使用することで、完全な同期は難しいですが、カウントダウンなどを使用した記念撮影タイミングを演出することが可能となります。また、複数回呼び出し可能であるため、任意枚数のスクリーンショットを保存することも可能となり、取り直しなどを含めた総合的なシーンへの応用が可能です。

9. 7 Contモードとユーティリティアップデート処理

Contモードは他のユーティリティと異なり、見掛け上、ユーティリティ処理の途中状態であるにもかかわらず、ユーティリティ処理が完了したかのような振る舞いで動作します。そのため、Contモード終了処理が呼び出されるまでは、ユーティリティ機能を正常に動作させるために、通常起動のユーティリティ処理で呼び出しているアップデート処理を同様に必要とします。このアップデート処理によって、Contモード中に発生する予期せぬエラー状態に対し、適切に対応することができます。
具体的な実装については、スクリーンショットユーティリティのサンプルプログラム("devkit/sample/utility/basic/screenshot.c")を参照してください。

  名前付けルール     説明

  付加してファイル名にします

9. 14 スクリーンショット保存に必要なサイズ計算

スクリーンショットユーティリティを使用して、スクリーンショット画像を保存するにあたり、保存先に必要とされる容量計算について簡単に説明します。
下記に示すサイズ計算は、大まかではありますが、スクリーンショット画像保存のための最低限の容量を算出するための目安となります。

PSCM.DAT
PSCM.DATファイルのサイズ計算は下記の計算式で算出されます。
PSCM.DATサイズ = PSCM.DATベース(276 byte) + iconFileSize + bgFileSize
  iconFileSize     : アイコン画像ファイルのサイズ(byte)  
  bgFileSize     : 背景画像ファイルのサイズ(byte)  

撮影画像サイズ
撮影画像サイズは、実際に保存処理（圧縮処理）を行った結果の値であるため、事前にその詳細を知ることはできません。大まかな目安として、圧縮しないRAW状態での画像サイズを基準にしていただくことは可能です。これは、圧縮処理後は一般的にRAW状態の画像サイズより小さくなるためです。このサイズで見積もっていただくことで、画像保存において、保存先の票量不足によるエラーの事前回避が概ね可能となります。

スクリーンショットディレクトリscreenshot_dirのサイズ
保存先のフォーマット状態に依存するので固定値として使用することはできませんが、一般に1クラスタ（32KiB）を見積もっていただくと、保存のためのディレクトリ作成が可能です。ただし、厳密に計算したい場合は、保存先のクラスタ情報を取得し、1クラスタのサイズを意識していただく必要がありますので、その点をご注意ください。

12. 3 サインインモード

サインインモードはサインアップとサインインをまとめて実行します。
以下に処理の流れを示します。


必要なユーザメモリ空間のメモリ
サインインモードを使用するためには、ユーザメモリ空間から以下のメモリを与える必要があります。
  • モジュールのロードに使用するメモリ
    ユーザメモリ空間にSCE_UTILITY_PSN_MODULE_SIZE_SIGNIN(512KiB)以上の連続領域を空けておく。
  • PSNユーティリティがヒープに使用するメモリ
    ユーザメモリ空間のSCE_UTILITY_PSN_HEAP_SIZE_SIGNIN(1MiB)以上の連続領域をSceUtilityPsnParamのメンバheap_addr、heap_sizeで指定する。

NPサインインユーティリティとの違い
PSNユーティリティのサインインモードとNPサインインユーティリティの違いは以下の通りです。
  • PSNユーティリティのサインインモード
  • サインアップしていない場合は、その場でサインアップすることができる
  • システムソフトウェア上のサインインと同等の機能が利用できる
  • ユーザメモリ空間のメモリをユーティリティに与える必要がある
  • NPサインインユーティリティ
  • サインアップしていない場合は、エラー画面が表示される
  • パスワードリセット等、サインインの一部の機能が利用できない
  • ユーザメモリ空間のメモリが必要ない

PSNユーティリティのサインインモードはシステムソフトウェア上のサインインと同等の機能が利用できる代わりにメモリを必要とし、逆にNPサインインユーティリティはメモリを必要としない代わりに機能が制限されます。

12. 4 ストアブラウズモード

ストアブラウズモードはサインアップ、サインイン、タイトルストアのブラウズをまとめて実行します。SceUtilityPsnParamのメンバtarget_type、target_idでブラウズ開始時にユーザに提示するカテゴリもしくはプロダクトを指定することができます。
以下に処理の流れを示します。


必要なユーザメモリ空間のメモリ
ストアブラウズモードを使用するためには、ユーザメモリ空間から以下のメモリを与える必要があります。
  • モジュールのロードに使用するメモリ
    ユーザメモリ空間にSCE_UTILITY_PSN_MODULE_SIZE_STORE(1MiB)以上の連続領域を空けておく。
  • PSNユーティリティがヒープに使用するメモリ
    ユーザメモリ空間のSCE_UTILITY_PSN_HEAP_SIZE_STORE(3MiB)以上の連続領域をSceUtilityPsnParamのメンバheap_addr、heap_sizeで指定する。多めにメモリを与えた場合には、ストアブラウズ中の画像のキャッシュに利用され、一度表示した画像の再取得を減らすことができます。

ダウンロードリストのサービスID指定
SceUtilityPsnParamのメンバservice_idにサービスIDを指定することで、ストアブラウズ中のダウンロードリストで指定したサービスIDに該当する購入済みの商品のみを表示することができます。service_idにNULLを指定した場合には、すべての購入済みの商品が表示されます。

終了確認ダイアログの設定
ストアブラウズ開始時にユーザに提示する画面で、戻るにアサインされているボタンを押した場合にストアブラウズの終了確認ダイアログを表示するかどうかを、SceUtilityPsnParamのメンバexit_dialogで指定することができます。SCE_UTILITY_PSN_STORE_USE_EXIT_DIALOGを指定した場合にはダイアログを表示し、SCE_UTILITY_PSN_STORE_NO_EXIT_DIALOGを指定した場合にはダイアログを表示せずにストアブラウズを終了します。



視聴年齢制限チェック
SceUtilityPsnParamのメンバmin_ageにゲームを視聴可能な最低年齢を指定することで、sceNpGetContentRatingFlag()による視聴年齢制限のチェックを行うことができます。ユーザのPlayStation®Networkアカウントの視聴年齢制限機能が有効で、かつ、アカウントの年齢が指定された最低年齢に満たない場合にはエラーを表示し、PSNユーティリティを終了します。
視聴年齢に制限がないゲームの場合、もしくは、あらかじめゲームプログラム側で視聴年齢制限のチェックを行っている場合には、min_ageに0を指定してください。

制限事項
PSNユーティリティのストアブラウズモードには以下の制限があります。
  • ゲームストアの商品のみブラウズ可能
  • カートに追加できる商品の数は最大16個
  • 検索機能は利用不可
  • 画像はPNGのみサポート

12. 5 処理結果の通知

PSNユーティリティが終了すると、SceUtilityPsnParamのメンバbase.resultに以下の返り値が設定されます。
  • SCE_UTILITY_COMMON_RESULT_OK (0)
    サインインモードの場合にはサインイン、ストアブラウズモードの場合にはストアブラウズが正常に終了した場合に返ります。
  • SCE_UTILITY_COMMON_RESULT_CANCELED (1)
    サインアップもしくはサインイン処理がユーザ操作によりキャンセルされた場合に返ります。
  • SCE_UTILITY_COMMON_RESULT_ABORTED (2)
    サスペンドにより処理が強制終了した場合に返ります。
  • SCE_UTILITY_PSN_ERROR_PARAM (0x80111800)
    ユーティリティ起動時に指定したSceUtilityPsnParamの値が不正な場合に返ります。
  • SCE_UTILITY_PSN_ERROR_INTERNAL (0x80111801)
    ネットワークエラー等により、処理が継続できなくなった場合に返ります。

返り値がSCE_UTILITY_PSN_ERROR_PARAMもしくはSCE_UTILITY_PSN_ERROR_INTERNALの場合には、PSNユーティリティ内でエラー画面を表示してから終了するため、ゲームプログラム側でエラー画面を表示する必要はありません。

13. 6 内部状態

自動接続ユーティリティの内部状態には、ユーティリティの状態として次の状態があります。
  • SCE_UTILITY_COMMON_STATUS_NONE
  • SCE_UTILITY_COMMON_STATUS_INITIALIZE
  • SCE_UTILITY_COMMON_STATUS_RUNNING
  • SCE_UTILITY_COMMON_STATUS_FINISHED

SCE_UTILITY_COMMON_STATUS_RUNNING状態には、さらにその内部状態として次の6つの内部状態があります。
  • SCE_UTILITY_AUTO_CONNECT_STATUS_CONNECTING
    アクセスポイントへの接続中。この状態のときは接続中を示すGUIがユーティリティによってオーバーレイされます。キャンセルボタンでユーザーは自動接続処理をキャンセルすることができます。
  • SCE_UTILITY_AUTO_CONNECT_STATUS_TESTING_HTTP
    インターネット接続テスト中。この状態の時にはGUIはオーバーレイされないため、ユーザー操作によって処理を中断するには、ゲームプログラム側でユーザーの入力によって、sceUtilityAutoConnectAbort()を呼び出すことが必要になります。
  • SCE_UTILITY_AUTO_CONNECT_STATUS_ABORTING
    アボート処理中
  • SCE_UTILITY_AUTO_CONNECT_STATUS_FINISHING
    自動接続処理終了処理中
  • SCE_UTILITY_AUTO_CONNECT_STATUS_ESTABLISHED
    自動接続処理は終わり、インターネット接続が確立された
  • SCE_UTILITY_AUTO_CONNECT_STATUS_FAILED
    自動接続処理は終わり、インターネット接続が確立されなかった

13. 7 強制終了

自動接続ユーティリティは、内部状態がSCE_UTILITY_COMMON_STATUS_RUNNINGのとき、sceUtilityAutoConnectAbort()を呼び出すことで強制終了することができます。
処理結果の通知自動接続ユーティリティが終了すると、SceUtilityAutoConnectParamのメンバbase.resultに以下の返り値が設定されます。
  • SCE_UTILITY_COMMON_RESULT_OK(0)
    正常にインターネット接続が確立された場合に返ります。
  • SCE_UTILITY_COMMON_RESULT_CANCELED(1)
    自動接続処理がユーザー操作によりキャンセルされた場合に返ります。
  • SCE_UTILITY_COMMON_RESULT_ABORTED(2)
    サスペンド、またはゲームプログラム側からsceUtilityAutoConnectAbort()を呼び出すことで自動接続処理が強制終了された場合に返ります。
  • SCE_UTILITY_AUTO_CONNECT_ERROR_INVALID_ARGS (0x80111A01)
    ユーティリティ起動時に指定したSceUtilityAutoConnectParamの値が不正な場合に返ります。
  • SCE_UTILITY_AUTO_CONNECT_ERROR_NOT_ESTABLISHED (0x80111A02)
    設定されているインフラストラクチャモードのネットワーク接続をすべて試したものの、自動接続処理でインターネット接続が確立されなかった場合に返ります。
  • SCE_UTILITY_AUTO_CONNECT_ERROR_WIRELESS_OFF (0x80111A03)
    ワイヤレススイッチがオフになっているために、自動接続処理でインターネット接続が確立されなかった場合に返ります。
  • SCE_UTILITY_AUTO_CONNECT_ERROR_INTERNAL (0x80111AFF)
    内部エラーにより自動接続処理が終了した場合に返ります。

15 注意事項

15. 1 ユーティリティサービスが、初期化処理中および終了処理中である間の、初期化処理中ステータスおよび終了処理中ステータスについて

ユーティリティサービスが、初期化処理中、および、終了処理中である間、sceUtility*GetStatus()の返値は、それぞれ、SCE_UTILITY_COMMON_STATUS_INITIALIZE、SCE_UTILITY_COMMON_STATUS_SHUTDOWNとなりますが、初期化処理、終了処理が、2回のsceUtility*GetStatus()の呼び出しの間に、短期間で終了した場合、sceUtility*GetStatus()は、SCE_UTILITY_COMMON_STATUS_INITIALIZE、SCE_UTILITY_COMMON_STATUS_SHUTDOWNを返しません。

つまり、SCE_UTILITY_COMMON_STATUS_NONEの次にSCE_UTILITY_COMMON_STATUS_RUNNING（Utilityの種類と起動タイプによってはSCE_UTILITY_COMMON_STATUS_WAITING）、SCE_UTILITY_COMMON_STATUS_FINISHEDの次にSCE_UTILITY_COMMON_STATUS_NONEが返る可能性があります。

このような場合でもゲームプログラムが正常に動作するように、ユーティリティサービスの呼び出しを行ってください。

15. 2 ユーティリティサービスのsceUtility*Update()関数の呼び出しとゲームプログラム側の描画処理の排他について

sceUtility*Update()関数内では、
  1. アプリケーションの描画コンテキストを退避
  2. ユーティリティモジュールの描画パケットの作成／転送／転送終了待ち
  3. アプリケーションの描画コンテキストの復帰
を行っており、Vblankを意識せずに処理を行っています。

従って、sceUtility*Update()の処理中にsceGuSwapBuffers()等によるフレームバッファのスワップ処理、またはゲームプログラム側の描画パケット転送処理が同時に実行されると、ユーティリティの描画が乱れることがあります。
sceUtility*Update()の前までにゲーム側の描画パケットの転送終了を待つようにしてください。また、sceUtility*Update()の呼び出し中にフレームバッファのスワップ処理を待たせる処理をしてください。

15. 3 ユーティリティサービスに関連するデータのチェック方法について

ユーティリティサービスに関連するデータ（画像、サウンド、各種設定情報など）に関して、その妥当性を確認するためのチェック方法が、システムソフトウェアのデバッグ支援機能として提供されております。
詳細については、「PSP®セットアップガイド」ドキュメントの「システムソフトウェアのデバッグ支援機能」章を参照してください。
