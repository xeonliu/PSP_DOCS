RSS Reader Utility Overview


1 About Utilities

2 RSS Channel Overview

The RSS channel feature implemented in PSP™ system software enables updated content on websites to be distributed to users relatively easily using RSS 2.0 (RSS = Really Simple Syndication).

2. 1 Overview of the Relationship Between a Game, the RSS Reader Utility, the RSS Channel Subscription Utility and the Distribution Server

Figure 1 shows the relationship between the RSS reader utility, the RSS channel subscription utility, a game and the distribution server. The RSS channel subscription utility and PSP™ system software (RSS channel feature) are not directly affected by the use of the RSS reader utility.

(1) [game]: Register a Channel

A game program uses the RSS channel subscription utility to register the URL of an RSS feed distributed from a server. Registration information is stored on a Memory Stick™ as subscription channel data, along with the title, thumbnail image and other information.
A game program must perform this step to use the RSS channel feature of PSP™ system software, however, it is not needed when RSS is used to distribute content within a game only.

(2) [game]: Request the RSS reader utility to read an RSS feed

A game program requests the RSS reader utility to read an RSS feed by passing it the URL of the RSS feed distributed from a server.

(3) [utility]: Read the RSS feed

The RSS reader utility accesses the URL of the RSS feed from step 2 and reads RSS information into RSS reader utility memory.

(4) [game]: Request the RSS reader utility to get information about the RSS feed

A game program requests the RSS reader utility to obtain information about the RSS feed that was read. This includes information under the <title> and <description> tags of the channel or item.

(5) [game]: Request the RSS reader utility to download content

A game program requests the RSS reader utility to download content referenced in the RSS information that was read.

(6) [utility]: Reference the URL of the content

The RSS reader utility references the URL of the content from the RSS information that was read, for content that the game program requested to be downloaded,.

(7) [utility]: Download the content

The RSS reader utility downloads the content from the URL that was referenced in step (6).

(8) [game]: Use the content

The content that was saved in step (7) can be used in a game or it can be viewed using PSP™ system software.



Figure 1 Relationship Between a Game, the RSS Reader Utility, the RSS Channel Subscription Utility, the Distribution Server, and System Software



2. 2 Content That Can Be Distributed Via an RSS Channel

The following two types of content can be distributed over an RSS channel.

(1) Audio, video, and image files

These files can be used for purposes unrelated to a game such as for podcasting. They can also be used to promote game content.

(2) Save data files

Data that is used in a game such as map data or character data can be updated daily.

2. 3 Pros and Cons of Distributing Content Via an RSS Channel

The pros and cons of distributing content via an RSS channel are listed below for users and content providers.

(1) Users

Pros

  • Using a simple procedure, users can confirm whether or not they have the latest content without the need for an Internet browser.
  • Similarly, using a simple procedure, the latest content can be saved without the need for an Internet browser.
  • A timer feature is available to automatically save the latest content via a wireless LAN without having to perform any specific action.

Cons

  • The information that enables the user to understand what the content contains consists only of a limited character string and a thumbnail image.

(2) Content providers

Pros

  • Chances are better that users will verify your content as well as be aware that new content exists.
  • The time required to develop your website can be reduced.

Cons

  • Since users can obtain desired objects directly, they will have fewer opportunities to see content other than the specific content they are obtaining.

3 RSS Reader Utility Overview

The RSS reader utility has the following characteristics.
  • No user interface required.
  • Can be called from two types of APIs (sceUtilityRssReaderInitStart() and sceUtilityRssReaderContStart()) using a specified startup type.
  • Besides SCE_UTILITY_COMMON_STATUS_NONE, SCE_UTILITY_COMMON_STATUS_INITIALIZE, SCE_UTILITY_COMMON_STATUS_RUNNING, and SCE_UTILITY_COMMON_STATUS_FINISHED, the RSS reader utility also has SCE_UTILITY_COMMON_STATUS_WAITING as an internal state.
  • If the utility is started up (sceUtilityRssReaderInitStart()) with a startup type other than SCE_UTILITY_RSS_READER_TYPE_FINISH, the utility's state will transition from SCE_UTILITY_COMMON_STATUS_RUNNING to SCE_UTILITY_COMMON_STATUS_WAITING.
  • If the utility is called with sceUtilityRssReaderContStart() when its state is SCE_UTILITY_COMMON_STATUS_WAITING, and the next startup type is specified, the result obtained will be that of the startup type from the previous call.
  • When the utility's state is SCE_UTILITY_COMMON_STATUS_WAITING, the utility must be explicitly called with sceUtilityRssReaderContStart() using startup type SCE_UTILITY_RSS_READER_TYPE_FINISH to cause a transition to SCE_UTILITY_COMMON_STATUS_FINISHED state.
  • When the utility's state is SCE_UTILITY_COMMON_STATUS_WAITING, processing associated with the startup type from the previous call (either sceUtilityRssReaderInitStart() or sceUtilityRssReaderContStart()) may be continued by the internal thread of the utility (when the previous startup type was SCE_UTILITY_RSS_READER_TYPE_RETRIEVE or SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD).

3. 1 Internal States

The RSS reader utility's internal states include the following.
  • SCE_UTILITY_COMMON_STATUS_NONE
  • SCE_UTILITY_COMMON_STATUS_INITIALIZE
  • SCE_UTILITY_COMMON_STATUS_RUNNING
  • SCE_UTILITY_COMMON_STATUS_FINISHED
  • SCE_UTILITY_COMMON_STATUS_WAITING
In addition, SCE_UTILITY_COMMON_STATUS_WAITING has the following four internal states.
  • RSS feed not read, waiting
  • Reading RSS feed
  • Finished reading RSS feed, waiting
  • Downloading
Figure 2 shows the internal states and state transitions. Details about state transitions are described later. The prefixes "sceUtilityRssReader," "SCE_UTILITY_COMMON_STATUS_," and "SCE_UTILITY_RSS_READER_TYPE_" have been omitted from the figure.



Figure 2 RSS Reader Utility's Internal States and State Transitions


3. 2 Calling Method

The function used for calling the RSS reader utility depends on the utility's internal state.

When the utility is in SCE_UTILITY_COMMON_STATUS_NONE state
The utility should be called with the sceUtilityRssReaderInitStart() function.

When the utility is in SCE_UTILITY_COMMON_STATUS_WAITING state
The utility should be called with the sceUtilityRssReaderContStart() function.

3. 3 Calling Types

The following types of calls are provided for the RSS reader utility.

Read an RSS feed
Asynchronously read RSS information into the utility's internal memory by specifying the URL of an RSS feed.

Cancel the reading of an RSS feed
Cancel the reading of an RSS feed, which is currently in process.

Get information about an RSS feed that was read
Get information about the RSS feed, which was read into the utility's internal memory.

Download the content of an RSS feed that was read
Asynchronously download a content file that was referenced from the RSS feed, which was read into the utility's internal memory.

Cancel downloading of the content of an RSS feed that was read
Cancel the downloading of the content file, which is currently in process.

Finish
Cause the RSS reader utility, which is in SCE_UTILITY_COMMON_STATUS_WAITING state, to transition to SCE_UTILITY_COMMON_STATUS_FINISHED state.

The six calling methods described above can be set in the type member of the SceUtilityRssReaderParam structure.
  Calling Method     Type Name  
  Read RSS feed     SCE_UTILITY_RSS_READER_TYPE_RETRIEVE  
  Cancel reading of RSS feed     SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL  
  Get information about RSS feed that was read     SCE_UTILITY_RSS_READER_TYPE_GET_INFO  
  Download content of RSS feed that was read     SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD  
  Cancel downloading of content of RSS feed that was read     SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL  
  Finish     SCE_UTILITY_RSS_READER_TYPE_FINISH  

3. 4 State Transitions

This section explains the internal states and state transitions shown in Figure 2. The prefixes "sceUtilityRssReader", "SCE_UTILITY_COMMON_STATUS_" and "SCE_UTILITY_RSS_READER_TYPE_" have been omitted from the figure.

3. 5 State Transitions from SCE_UTILITY_COMMON_STATUS_NONE State

When the utility is called from sceUtilityRssReaderInitStart(), the utility's state will change.
The following calling type is functionally enabled.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE
Other calling types either return an error or are functionally disabled.

(1) When the utility is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE type

The utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_WAITING state.
If the call is successful, the retrieving member variable of the SceUtilityRssReaderRetrieveArgs structure that was passed when the utility was called will change to true, and the utility will enter "Reading RSS feed" state.
If the call fails, the retrieving member variable of the SceUtilityRssReaderRetrieveArgs structure that was passed when the utility was called remains false, and the utility enters "RSS feed not read, waiting" state.

(2) When the utility is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL, SCE_UTILITY_RSS_READER_TYPE_GET_INFO, SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD, or SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL type

The utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_WAITING state. At this time, an error is returned in the result member of the SceUtilityParamBase structure.

(3) When the utility is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

The utility passes through the SCE_UTILITY_COMMON_STATUS_INITIALIZE and SCE_UTILITY_COMMON_STATUS_RUNNING states and transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

3. 6 State Transitions from SCE_UTILITY_COMMON_STATUS_WAITING State

When the utility is called from sceUtilityRssReaderContStart() and RSS reading or downloading has completed, the utility's state will change.

(1) RSS feed not read, waiting

The following calling types are functionally enabled.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE
  • SCE_UTILITY_RSS_READER_TYPE_FINISH

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. If the call is successful, the retrieving member variable of the SceUtilityRssReaderRetrieveArgs structure will change to true, and an internal thread of the utility is started up which begins reading the RSS feed into internal memory via the network. If the call fails, the retrieving member variable remains false.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

The utility transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

(2) Reading RSS feed

The following calling types are functionally enabled.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL
  • SCE_UTILITY_RSS_READER_TYPE_FINISH

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE_CANCEL type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. When the call is completed, the retrieving member variable of the SceUtilityRssReaderRetrieveArgs structure changes to false, and the internal thread of the utility, which had been reading the RSS feed into internal memory, stops.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

The internal thread of the utility, which had been reading the RSS feed into internal memory stops, and the utility transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

When reading of the RSS feed has completed, the retrieving member changes to false, and the utility enters "RSS feed not read, waiting" state. At this time, an error value that represents the result is set in the result member of the SceUtilityRssReaderRetrieveArgs structure.

(3) Finished reading RSS feed, waiting

The following calling types are functionally enabled.
  • SCE_UTILITY_RSS_READER_TYPE_RETRIEVE
  • SCE_UTILITY_RSS_READER_TYPE_GET_INFO
  • SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD
  • SCE_UTILITY_RSS_READER_TYPE_FINISH

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_RETRIEVE type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. If the call is successful, the retrieving member variable of the SceUtilityRssReaderRetrieveArgs structure changes to true, and an internal thread of the utility is started up which begins reading the RSS feed into internal memory via the network. If the call fails, the retrieving member variable remains false.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_GET_INFO type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. No internal thread runs.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. If the call is successful, the downloading member variable of the SceUtilityRssReaderDownloadArgs structure changes to true, and an internal thread of the utility is started up to begin downloading the content file that was referenced in the RSS feed that was read, via the network. If the call fails, the downloading member variable remains false.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

The internal thread of the utility, which had been reading the RSS feed into internal memory, stops and the utility transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.

(4) Downloading

The following calling types are functionally enabled.
  • SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL
  • SCE_UTILITY_RSS_READER_TYPE_FINISH

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_DOWNLOAD_CANCEL type

The utility does not transition from SCE_UTILITY_COMMON_STATUS_WAITING state. When the call is completed, the downloading member variable of the SceUtilityRssReaderDownloadArgs structure changes to false, and the internal thread of the utility, which had been downloading the content file, stops.

When the utility is called with SCE_UTILITY_RSS_READER_TYPE_FINISH type

The internal thread of the utility, which had been downloading the content file, stops and the utility transitions to SCE_UTILITY_COMMON_STATUS_FINISHED state.
When downloading is completed, the downloading member is set to false and the utility enters "Finished reading RSS feed, waiting" state. At this time, an error value that represents the result is set in the result member of the SceUtilityRssReaderDownloadArgs structure.
