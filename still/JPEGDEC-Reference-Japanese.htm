sceJpegCreateMJpeg
 [関数]
モーションJPEGデコードのコンテキスト作成

形式

#include <scejpeg.h>
int sceJpegCreateMJpeg(
	int iFrameWidth,
	int iFrameHeight
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  iFrameWidth     出力バッファの最大幅(ピクセル数)  
  iFrameHeight     出力バッファの最大高さ(ピクセル数)  

返り値

  値     説明  
  0     初期化成功  
  負数     エラー値(詳細は「エラーコード一覧」参照)  


解説

出力バッファサイズを指定してデコードコンテキストを作成します。バッファサイズはピクセル数で指定し、また1ピクセルは常に4バイト(ARGB8888フォーマット)となります。
ここで指定したバッファサイズは、デコード関数sceJpegDecodeMJpeg()にて、出力サイズのチェックに使われます。
iFrameWidthの値は、1024以下でなければなりません。つまり、それよりも大きなMotion JPEGデータのデコードはできません。また、色変換機能の限界より、実際のJPEG画像サイズの幅は720以下でなければなりません。

使用例

int res;
res = sceJpegCreateMJpeg(320, 240);
備考

MPEGライブラリのような出力フォーマット変更(16bitカラーにするなど)はサポートされていません。

sceJpegDecodeMJpeg
 [関数]
モーションJPEGデータのデコードと高速色変換

形式

#include <scejpeg.h>
int sceJpegDecodeMJpeg(
	unsigned char *pJpeg,
	SceSize isize,
	void *pRGBA,
	int omitDht
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  pJpeg     デコードしたいJPEGデータへのポインタ  
  isize     JPEGデータのサイズ  
  pRGBA     出力先バッファへのポインタ  
  omitDht     DHTセグメント省略を許すかどうかのフラグ  

返り値

  値     説明  
  >0     出力データのピクセル数  
  負数     エラー値(詳細は「エラーコード一覧」参照)  


解説

JPEGデータをデコードし、指定領域にRGBA8888形式で格納します。デコードしようとした画像のサイズが(縦横のどちらか一方だけでも)sceJpegCreateMJpeg()で指定したサイズを超えた場合、エラーとなります。なお、デコードする画像データの縦または横方向ピクセル数が16の倍数で無かった場合、内部的に16ピクセルの倍数に切り上げて処理が行われるため、切り上げたピクセル数以上のバッファが必要になります。
正常終了時には、デコードしたデータの(x方向のピクセル数)*65536＋(y方向のピクセル数)が返ります。

Motion JPEGデータの種類によっては、JPEGデータにハフマン符号定義セグメント(DHTセグメント)が存在せず、常時IJGの定める推奨ハフマン符号を使用するタイプのものがあります。引数omitDhtに対して1 (SCE_JPEG_MJPEG_WITHOUT_DHT)を指定することでこのようなデータのデコードも可能です。この引数を設定すると、DHTセグメントがなかったときは推奨テーブルが使用され、DHTセグメントがあった場合はそのテーブルの内容が使用されます。ただし、SCE_JPEG_MJPEG_WITHOUT_DHTを指定した場合、そうでないときに比べてほんの少しJPEGデコードにかかる時間は増えます。DHTセグメントがあることがわかっているデータの場合は、このomitDhtには0(SCE_JPEG_MJPEG_WITH_DHT)を指定するようにしてください。

この関数はパフォーマンスを優先しているため、Media Engine側で一括してデコード処理を行います。このため、他のMedia Engineを用いる音声合成などの処理と同時に使用すると、処理落ちなどの現象が発生することがあります。このような場合は、sceJpegDecodeMJpegSuccessively()を用いてください。

使用例

int res;
char jpegData[100*1024];
char framebuf[320*240*4];


res = sceJpegDecodeMJpeg(jpegData, sizeof(jpegData), framebuf,
			SCE_JPEG_MJPEG_WITH_DHT);

この関数はパフォーマンスを優先しているため、Media Engine側で一括してデコード処理を行います。このため、他のMedia Engineを用いる音声合成などの処理と同時に使用すると、処理落ちなどの現象が発生することがあります。このような場合は、sceJpegDecodeMJpegYCbCrSuccessively()を用いてください。

使用例

int res;
char jpegData[100*1024];
char ycbcr[320*240*3/2];

res = sceJpegDecodeMJpeg(jpegData, sizeof(jpegData), ycbcr, sizeof(ycbcr),
			SCE_JPEG_MJPEG_WITH_DHT);

  pYCbCr     YCbCr形式のデータが格納されている入力バッファへのポインタ  
  xysize     入力バッファの縦横のピクセルサイズ数  
  iFrameWidth     出力フレームバッファの横幅  

返り値

  値     説明  
  >0     正常終了  
  負数     エラー値(詳細は「エラーコード一覧」参照)  


解説

sceJpegDecodeMJpegYCbCr()でデコードした結果の色変換を行い、RGBA8888形式に変換します。sceJpegDecodeMJpegYCbCr()関数でデコードされた結果以外に対してこの関数を用いたときの挙動は保障されません。また、iFrameWidthの値は1024以下でなければなりません。
pYCbCrにはsceJpegDecodeMJpegYCbCr()関数に与えたデコード結果用のバッファへのポインタを、xysizeには同関数の返り値を与えます。
なおこのとき、出力バッファサイズに関するエラーチェックは特に行われません。xysizeの値を見て、アプリ側でサイズのチェックを行う必要があります。

使用例

int res;
char jpegData[100*1024];
char ycbcr[320*240*3/2];
char framebuf[320*240*4];

res = sceJpegDecodeMJpeg(jpegData, sizeof(jpegData), ycbcr, sizeof(ycbcr),
			SCE_JPEG_MJPEG_WITH_DHT);
if (res < 0) {
	/* エラー処理 */
}
res = sceJpegMJpegCsc(framebuf, ycbcr, res, 320);

sceJpegGetOutputInfo
 [関数]
JPEGデコード結果の確認

形式

#include <scejpeg.h>
int sceJpegGetOutputInfo(
	char *pJpeg,
	int isize,
	int *pCs,
	int omitDht
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  pJpeg     デコードしたいJPEGデータへのポインタ  
  isize     JPEGデータのサイズ  
  pCs     色空間情報を格納する変数へのポインタ  
  omitDht     DHTセグメント省略を許すかどうかのフラグ  
  (現状何を指定しても同じ結果となります)  

返り値

  値     説明  
  ≧0     色変換無しでデコードしたときに必要な出力バッファのサイズ  
  負数     エラー値(詳細は「エラーコード一覧」参照)  


解説

指定されたJPEGデータをデコードしたとき、必要になる出力バッファのサイズを返します。また、pCsがNULLで無かった場合、色空間の種別と色空間サンプリングを*pCsの指す先に格納します。色空間の種別は以下のような形式で示されます。色空間サンプリングは色空間がYCbCrのときのみ有効な値となり、それ以外のときは不定となります。

  値     説明  
  16:24ビット目     色空間種別、0:不明、1:グレイスケール、2:YCbCr  
  8:16ビット目     垂直方向の空間サンプリング  
  0:8ビット目     水平方向の空間サンプリング  


例えば、*pCS=0x020201となった場合、いわゆるYUV422形式のJPEGデータということになります。
JPEGライブラリでは、色空間の第一次元の空間サンプリングが1:1であり、二次元目以降の色次元の空間サンプリングは全て同じであるという、JPEGデータしか取り扱うことができません。このため、この関数で取得できる情報も、このようなJPEGデータに関連するものだけとなります。未対応のデータであった場合、エラーが返ります。

使用例

int res;
res = sceJpegGetOutputInfo(JpegBuffer, sizeof(JpegBuffer), NULL, 0);
sceJpegMJpegCscWithColorOption
 [色変換機能]
モーションJPEG向け色変換(オプション付き)

形式

#include <scejpeg.h>
int sceJpegMJpegCscWithColorOption (
	void *pRGBA,
	const unsigned char *pYCbCr,
	int xysize,
	int *iFrameWidth,
	int colorOption
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼ぶこと）
マルチスレッドセーフである

引数

  pRGBA     色変換結果を出力したい先へのポインタ  
  pYCbCr     色変換したいデータへのポインタ  
  xysize     色変換したいデータのピクセルサイズ(上位16ビットに横方向ピクセル数、  
  下位16ビットに縦方向ピクセル数を入れてください)  
  iFrameWidth     色変換結果を格納するバッファの幅  
  (単位はピクセル、0以上1024以下であること)  
  colorOption     入出力データの形式に関するパラメータ  

返り値

  値     説明  
  ≧0     正常終了  
  負数     エラー値(詳細は「エラーコード一覧」参照)  


解説

YCbCr形式からRGB形式への色変換を行います。入力データの空間サンプリングはYUV420形式、画像サイズも720*480ピクセル以下に限られますが、変換時の色変換係数および出力RGBAデータの形式を選択することが可能です。

出力フォーマットに関する指示はcolorOptionで行います。以下の中から選択してください。
  • SCE_JPEG_PIXEL_RGB565
  • SCE_JPEG_PIXEL_RGBA5551
  • SCE_JPEG_PIXEL_RGBA4444
  • SCE_JPEG_PIXEL_RGBA8888
また、colorOptionに以下の値を加算して渡すことにより、入力データの色空間を指定することができます。
  • SCE_JPEG_COLORSPACE_JFIF
  • SCE_JPEG_COLORSPACE_BT601
デフォルトの色空間はSCE_JPEG_COLORSPACE_JFIFです。

YCbCr形式からRGB形式への色変換を行います。色空間サンプリングとしてYUV444,422,420に対応します。Cb, Crが空間サンプリングされていたとき(YUV444以外のとき)には、バイリニアフィルタによりアップサンプリングをかけることを選択できます。また、縦横65535ピクセルまでの画像データに対応します。ただし、入力画像の色空間はJFIF形式、出力形式もRGBA8888に限定され、速度も遅くなります。

modeには
(垂直方向の色空間サンプリング係数) * 256
(水平方向の色空間サンプリング係数)
を加えた値に、必要に応じてSCE_CSC_BILINEAR_ENABLEおよびSCE_CSC_SPLIT_ENABLEを加えた値を指定します。上記の色空間サンプリングの値はsceJpegGetOutputInfo()を用いて取得した値の下位16ビット値をそのまま使うことができます。
