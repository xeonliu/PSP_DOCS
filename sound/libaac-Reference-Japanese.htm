SceAacInitArg
 [構造体]
デコード資源の初期化変数

構造

#include <libaac.h>
typedef struct {
	SceOff aacHeadPos;
	SceOff aacTailPos;
	SceUChar8 *aacBuf;
	SceInt32 aacBufByte;
	SceUChar8 *pcmBuf;
	SceInt32 pcmBufByte;
	SceInt32 fs;
	SceInt32 reserved;
} SceAacInitArg;
メンバ

  aacHeadPos     AACファイル最初のオーディオデータの先頭位置  
  aacTailPos     AACファイル最後のオーディオデータの末尾位置  
  aacBuf     入力バッファの先頭アドレス  
  aacBufByte     入力バッファのバイト数  
  pcmBuf     出力バッファの先頭アドレス  
  pcmBufByte     出力バッファのバイト数  
  fs     サンプリング周波数（Hz）  
  reserved     予約領域  

解説

AACハンドルの初期化に利用する構造体です。sceAacInit()の引数として利用されます。入出力バッファのアドレスと大きさ、ループ再生のためのファイル情報、デコーダ初期化のためのサンプリング周波数から構成されます。
aacHeadPosには、ヘッダを除く最初のオーディオデータの先頭位置を指定してください。aacTailPosには、ファイル末尾となるオーディオデータの末尾位置を指定してください。
入出力バッファのメモリアラインメントは、64バイトアラインメントである必要があります。また、必要な入出力バッファサイズは、以下のようになります。

  • 入力(AAC)バッファサイズ
    8192バイト以上、かつ64バイトの倍数
  • 出力(PCM)バッファサイズ
    8192バイト(1024サンプル×2バイト×2チャネル×2バッファ)

fsにはサンプリング周波数をHzで指定します。reservedには必ず0を代入してください。

解説

libaacに必要な計算資源を確保します。libaacの各機能を利用するためには、まず、本関数を実行する必要があります。
この計算資源によって複数個のAACストリームデータを並列でデコードすることが可能です。個々のデコード処理に必要な計算資源をlibaacでは、デコード資源と呼びます。同時に利用可能なデコード資源の数を引数numに指定します。デコード資源は後述するAACハンドルと一対一に対応するため、デコード資源の数は、AACハンドルの数に一致します。

使用例

/* libaacに必要な計算資源を2つ確保 */
sceAacInitResource (2);

sceAacDecode
 [関数]
AACデータのデコード

形式

#include <libaac.h>
SceInt32 sceAacDecode(
	const SceInt32 handle,
	SceShort16 **pcmSrcPtr
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

  handle     デコード資源管理用AACハンドル  
  pcmSrcPtr     デコード結果の出力先アドレスを指すポインタ  

返り値

  値     説明  
  非負(>=0)     *pcmSrcPtrから読み出し可能なバイト数  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

入力バッファに読み込まれたAACデータを1フレームだけリニアPCMデータにデコードし、出力バッファに書き込みます。入力のAACデータ形式に依らず、出力のPCMデータのフォーマットは、ビット深度16ビットのステレオ形式になります。このPCMデータは、pcmSrcPtrが指すアドレスを先頭アドレスとし、左、右、左、……のようにインタリーブ形式で左右交互に出力されます。出力されるPCMデータのサイズは、返り値によって取得することができます。出力されるPCMデータは、次にsceAacDecode()を実行するまでの間だけ有効です。
デコード処理を実行することで入力バッファ内のAACデータが消費されます。デコード処理を行う前に必ずsceAacCheckStreamDataNeeded()を実行し、必要に応じてAACデータを追加してください。AACデータの読み込みは、同期または非同期に行うことができます。また、AACデータの追加完了が遅く、入力バッファ内のAACデータが残り1フレームもない場合、デコード呼び出しの１回目にはsceAacDecode()は無音のPCMデータを出力します。

使用例

デコードされたPCMデータをvaudio経由でオーディオ出力する例を以下に示します。

/* AACデータをデコード */
out_bytes = sceAacDecode(handle, &pcmSrcPtr);
/* vaudioにPCMデータを受け渡し */
sceVaudioOutputBlocking(vol, pcmSrcPtr);
備考

本関数実行前にsceAacInit()によってデコード資源の初期化およびハンドルの取得を完了させてください。
サンプリング周波数が44100Hz以外のデータを出力する場合には、事前にsceVaudioChReserve()によって、サンプリング周波数とPCMサンプル数をvaudioに指定してください。

sceAacCheckStreamDataNeeded
 [関数]
AACデータを入力バッファへ追加する必要性のチェック

形式

#include <libaac.h>
SceInt32 sceAacCheckStreamDataNeeded(
	const SceInt32 handle
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフである

引数

  handle     デコード資源管理用AACハンドル  

返り値

  値     説明  
  1     AACデータを入力バッファへ追加する必要性がある場合  
  0     AACデータを入力バッファへ追加する必要性がない場合  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

入力バッファにAACデータを追加する必要性をチェックします。具体的には、未デコードのAACデータサイズが入力バッファサイズの半分を切った場合、入力バッファにAACデータを追加する必要があると見なします。デコード処理を行う前に必ず本関数を実行し、AACファイルからAACデータを追加する必要性をチェックしてください。

使用例

/* AACデータを入力バッファに追加する必要性をチェック */
if (sceAacCheckStreamDataNeeded(handle) > 0) {
         /* 入力バッファにAACデータを読み込み */
         sceAacGetInfoToAddStreamData(handle, &aacDstPtr, &writableByte, &aacSrcPos);
         sceIoLseek(fd, aacSrcPos, SCE_SEEK_SET);
         addByte = sceIoRead(fd, aacDstPtr, writableByte);
         /* 入力バッファに読み込んだAACデータサイズをlibaacに通知 */
         sceAacNotifyAddStreamData(handle, addByte);
}
備考

本関数の引数には、あらかじめsceAacInit()によって取得しておいたAACハンドルを指定してください。

sceAacGetInfoToAddStreamData
 [関数]
AACデータを入力バッファに追加するための情報の取得

形式

#include <libaac.h>
SceInt32 sceAacGetInfoToAddStreamData (
	const SceInt32 handle,
	SceUChar8 **aacDstPtr,
	SceInt32 *writableByte,
	SceInt32 *aacSrcPos
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフである

引数

  handle     デコード資源管理用AACハンドル  
  aacDstPtr     追加のAACデータの書き込み先アドレスを指すポインタ  
  writableByte     *aacDstPtrから書き込み可能なバイト数を指すポインタ  
  aacSrcPos     AACファイルの読み出しシーク位置を指すポインタ  

返り値

  値     説明  
  0     正常終了  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

入力バッファにAACデータを追加するための情報を取得します。本関数実行後、引数を通じて追加のAACデータの書き込み先アドレス*aacDstPtr、データサイズ*writableByte、およびAACファイルの読み出しシーク位置*aacSrcPosを取得できます。これらの情報を取得後、以下の方法にしたがってAACデータを入力バッファに追加します。
まず、ファイルディスクリプタをAACファイル先頭から*aacSrcPosだけ移動させます。それから、移動させたファイルディスクリプタを利用し、*writableByteバイトのAACデータを追加先アドレス*aacDstPtrに読み込みます。非同期に読み込むことも可能です。
AACファイルからAACデータの追加が終わった後は、sceAacNotifyAddStreamData()によってlibaacに対しAACデータの追加を通知します。詳しくは、sceAacNotifyAddStreamData()の項を参照してください。
なお、AACファイル内のAACデータがすべて入力バッファに読み込まれている場合、またはAACデータを入力バッファに読み込む必要がない場合は、*aacDstPtrとしてNULL、*writableByteとして0が返ります。

使用例

/* AACデータを入力バッファに追加する必要性をチェック */
if (sceAacCheckStreamDataNeeded(handle) > 0) {
         /* 入力バッファにAACデータを読み込み */
         sceAacGetInfoToAddStreamData(handle, &aacDstPtr, &writableByte, &aacSrcPos);
         sceIoLseek(fd, aacSrcPos, SCE_SEEK_SET);
         addByte = sceIoRead(fd, aacDstPtr, writableByte);
         /* 入力バッファに読み込んだAACデータサイズをlibaacに通知 */
         sceAacNotifyAddStreamData(handle, addByte);
}
備考

本関数の引数には、あらかじめsceAacInit()によって取得しておいたAACハンドルを指定してください。

sceAacResetPlayPosition
 [関数]
再生位置のリセット

形式

#include <libaac.h>
SceInt32 sceAacResetPlayPosition(
	const SceInt32 handle
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない
引数

  handle     デコード資源管理用AACハンドル  

返り値

  値     説明  
  0     正常終了  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。
解説

再生位置をAACファイルの先頭にリセットします。同時に、ループ再生回数消費による再生停止状態であれば再生を再開します。本関数の実行によって既に設定されていた「ループ再生回数」の値は、変化しません。また、本関数の実行によってsceAacGetSumDecodedSample()で取得可能な「デコードされた累計出力サンプル数」の値が0にリセットされます。
また、ループ再生を実行する際に明示的に本関数を呼ぶ必要はありません。sceAacSetLoopNum()によってループ再生回数に非零が指定されている場合、AACファイル最後のデータのデコード終了時に、ファイル先頭から再生し直すようにsceAacGetInfoToAddStreamData()の出力が変化し、ループ再生を実現します。

使用例

/* 再生位置をAACファイル先頭にリセット */
status = sceAacResetPlayPosition(handle);
/* AACデータを入力バッファに追加する必要性をチェック */
if (sceAacCheckStreamDataNeeded(handle) > 0) {
         /* 入力バッファにAACデータを読み込み */
         sceAacGetInfoToAddStreamData(handle, &aacDstPtr, &writableByte, &aacSrcPos);
         sceIoLseek(fd, aacSrcPos, SCE_SEEK_SET);
         addByte = sceIoRead(fd, aacDstPtr, writableByte);
         /* 入力バッファに読み込んだAACデータサイズをlibaacに通知 */
         sceAacNotifyAddStreamData(handle, addByte);
}
備考

本関数実行前にsceAacInit()によってデコード資源の初期化とハンドルの取得を完了させてください。

sceAacSetLoopNum
 [関数]
残りループ再生回数の設定

形式

#include <libaac.h>
SceInt32 sceAacSetLoopNum (
	const SceInt32 handle,
	const SceInt32 loopNum
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

  handle     デコード資源管理用AACハンドル  
  loopNum     残りループ再生回数  

返り値

  値     説明  
  0     正常終了  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

残りループ再生回数を設定します。ループ再生回数は、AACハンドルごとに独立に管理されます。デフォルトのループ再生回数は、「無限回」です。
引数loopNumの指定値と解釈されるループ再生回数の対応関係については、以下の表を参照してください。

  値     説明  
  正(>0)     loopNum回  
  0     ループなし  
  負(<0)     無限回  


本関数によってループ再生回数に正数が指定されている場合、AACファイル最後のデータのデコード終了時に、ファイル先頭から再生し直すようにsceAacGetInfoToAddStreamData()の出力が変化します。
また、ループ開始・終了位置は、それぞれAACファイル先頭および末尾のタグとAACデータの境界です。その他の位置をループ再生の開始・終了位置に設定することはできません。

使用例

/* ループ再生回数を10回に設定する */
status = sceAacSetLoopNum(handle, 10);
備考

本関数の引数には、あらかじめsceAacInit()によって取得しておいたAACハンドルを指定してください。
loopNumに任意の負数をセットすれば無限回のループを実現できますが、libaacは、SCE_AAC_LOOP_INFINITYという負数のマクロを提供しています。ループ再生回数を無限回に設定する際にご利用ください。

  非負(>=0)     一回のデコードにおける最大出力サンプル数  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

sceAacDecode()を一回実行した際に出力される1チャネルあたりの最大サンプル数を取得します。
MPEG4 AAC-LCのデータに対する具体的な出力サンプル数は、1024サンプルになります。また、libaacは出力バッファをダブルバッファとして扱います。このため、本関数によって取得した値を8倍(2バイト×2チャネル×2バッファ)することでlibaacに必要な出力バッファサイズを計算することができます。SceAacInitArgの項も併せて参照してください。

使用例

/* libaacに必要な最大出力サンプル数を取得 */
maxOutputSample = sceAacGetMaxOutputSample(handle);
備考

本関数実行前にsceAacInit()によってデコード資源の初期化とハンドルの取得を完了させてください。
同一のAACファイルフォーマットに対して常に同じ値を返します。
