sceAtracGetAtracID
 [関数]

形式

#include <libatrac3plus.h>

sceAtracReinit
 [関数]
ATRAC™ ID再初期化

形式

#include <libatrac3plus.h>

SceInt32 sceAtracReinit(
	int at3_entry,
	int at3plus_entry
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフである

引数

  at3_entry     同時に使用したいATRAC3™用ATRAC™ IDの数  
  at3plus_entry     同時に使用したいATRAC3plus™用ATRAC™ IDの数  

返り値

  値     説明  
  0     正常終了  
  <0     エラー  


エラー時の返り値については巻末のエラーコード一覧をご覧ください。

解説

ライブラリの再初期化を行い、同時に使用できるATRAC™ IDの数を変更します。
ATRAC™ IDの同時使用可能数はデフォルトではATRAC3™が2、ATRAC3plus™が2です。この関数を使用することにより、
(ATRAC3™用IDの数) + (ATRAC3plus™用IDの数)×2 ≦ 6
を満たす範囲で変化させることが可能になります。上記条件を満たさない場合は、エラーが返ります。
なお、この関数を呼び出す際には、全てのATRAC™ IDが解放されている必要があります。

使用例

res = sceAtracReinit(4,1);

sceAtracSetData
 [関数]
初期データセット関数

形式

#include <libatrac3plus.h>

SceInt32 sceAtracSetData(
	SceInt32 atracID,
	SceUChar8 *pucBufferAddr,
	SceUInt32 uiBufferByte
);
呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み許可状態で呼び出すこと）
マルチスレッドセーフでない

引数

  atracID     ATRAC3™/ATRAC3plus™データのデコードを管理するためのID  
  pucBufferAddr     入力データの先頭アドレス  
  uiBufferByte     入力バッファのサイズ  

返り値

  値     説明  
  0     正常終了  
  <0     エラー  


エラー時の返り値については巻末のエラーコード一覧をご覧ください。

解説

ATRAC™ IDと入力バッファを関連付けてライブラリに登録します。この関数を呼ぶ際には、前もってATRAC™ IDを確保しておく必要があります。また、入力バッファには、ファイルの先頭からのデータをセットしてください。
この関数は入力データとしてモノラル及びステレオデータをうけつけます。この関数で初期化を行なった後、デコード関数sceAtracDecodeData()を呼び出すと常に2ch分のデータを出力します。
この関数により、libatrac3plusが動作するために必要とするメモリの初期化、pucBufferAddrとして設定されたバッファの設定、pucBufferAddrにセットされたATRAC3™/ATRAC3plus™データのヘッダの解析が行なわれます。

使用例

/* "ATRAC" IDと入力バッファの関連付けを行います*/

備考

入力バッファにデータが正しく読み込めていなかった場合、pucBufferAddrからuiBUfferByte+2バイト先までアクセスすることがあります。
したがって、読みだしバッファの最終アドレスから2バイト、アクセス可能な領域がなければなりません。

	/*データの追加*/
}

SCE_ATRAC_NONLOOP_STREAM_DATA_IS_ON_MEMORYが設定されている場合には、ストリーミング再生中に現在デコードしている位置以降のデータがすべて用意された状態ですので、バッファへのデータの継ぎ足しは必要ありません。

なお、SCE_ATRAC_LOOP_STREAM_DATA_IS_ON_MEMORYが設定された場合であっても、バッファへの継ぎ足し処理は可能です。

注意
セカンドバッファを利用している場合に、SCE_ATRAC_LOOP_STREAM_DATA_IS_ON_MEMORYが設定された場合には注意が必要です。SCE_ATRAC_LOOP_STREAM_DATA_IS_ON_MEMORYは、ループ回数を変更しなければ、現在使用しているループデータ用のバッファ(ファーストバッファ)に継ぎ足し処理が必要ないことを示す識別子です。ループ回数が変化するなどして、ファーストバッファの再生処理が終了し、利用するバッファがセカンドバッファに移った場合には、セカンドバッファへの継ぎ足し処理が必要になることがあります。
