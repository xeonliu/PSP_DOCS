sceVaudioChReserve
 [Function]
Reserve a virtual audio output channel

Syntax

#include <vaudio.h>
int sceVaudioChReserve(
	int len,
	int frequency,
	int format
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  len     Data length per channel (specified in number of samples)  
  frequency     Output data sampling frequency  
  format     Data format  

Return Value

If the function completes successfully, 0 is returned. Otherwise, one of the following error values is returned.

  Macro     Value     Description  
  SCE_ERROR_INVALID_SIZE     0x80000104     len is invalid  
  SCE_ERROR_INVALID_FORMAT     0x80000108     format is invalid  
  SCE_AUDIO_ERROR_INVALID_FREQUENCY     0x8026000a     frequency is invalid  
  SCE_ERROR_BUSY     0x80000021     Channel already reserved  


Description

This function reserves and configures a virtual audio output channel.

The following values can be specified for frequency.
48000, 44100, 32000, 24000, 22050, 16000, 12000, 11025, 8000
The following values can be specified for len.
256, 576, 1024, 1152, 2048

sceVaudioOutputBlocking
 [Function]
Output audio data (blocking)

Syntax

#include <vaudio.h>
int sceVaudioOutputBlocking( 
	int iVol,
	void* pBuf
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  iVol     Volume (0 - 1048575)  
  pBuf     Pointer to output data buffer  

Return Value

When the return value is non-negative, the number of samples that could be registered in the audio driver is returned. Otherwise, one of the following error values is returned.

  Macro     Value     Description  
  SCE_AUDIO_ERROR_INVALID_VOLUME     0x8026000B     Volume is invalid  
  SCE_ERROR_PRIV_REQUIRED     0x80260004     Pointer to output buffer is invalid  


Description

This function performs audio output to a logical channel.
The output data buffer specified by pBuf is registered in the audio driver. The audio driver outputs the registered sound data to the DAC. If the output data buffer could not be registered in the audio driver, the thread that called this function is placed in a WAIT state until the buffer can be registered. When the output data buffer can be registered in the audio driver, the thread switches from READY to RUN state, and processing for registering the output buffer in the audio driver is performed. When registration in the audio driver is completed, control returns to the calling process.

Example

/* Output the data in pcmBuf without changing the volume */
sceVaudioOutputBlocking(
	SCE_VAUDIO_VOLUME_0dB,
	pcmBuf
);
sceVaudioSetEffectType
 [Function]
Set effect type during virtual audio output

Syntax

#include <vaudio.h>
int sceVaudioSetEffectType(
	int type,
	int vol
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  type     Effect type  
  vol     Analog volume setting  

Return Value

When the function completes successfully, the updated analog volume setting is returned. Otherwise, one of the following error values is returned.

  Macro     Value     Description  
  SCE_ERROR_INVALID_INDEX     0x80000102     Effect type is invalid  


Description

This function sets the effect type to be applied when virtual audio output is performed.
The sound pressure level will change with effect processing, so the analog volume level must be adjusted accordingly. This function returns this adjusted volume. This value should be used to set the volume.
sceVaudioSetAlcMode
 [Function]
Set dynamic normalizer use

Syntax

#include <vaudio.h>
const char *sceVaudioSetAlcMode(
	int mode
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  mode     ALC mode  

Return Value

If the function completes successfully, 0 is returned. Otherwise, a negative number is returned.

Description

This function sets whether or not the ALC (dynamic normalizer) mode is used.

Example

sceVaudioSetAlcMode(SCE_VAUDIO_ALC_MODE1);
