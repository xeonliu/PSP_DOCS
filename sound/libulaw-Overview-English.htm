u-law Overview

1 Library Overview

1. 1 Function Overview and Features

The u-law library provides an encoder and decoder for the audio codec G711 standard (u-law compression format).
It has functions for compressing (encoding) a 16-bit PCM waveform into compressed u-law format and functions for expanding (decoding) compressed u-law data back into a PCM waveform.

2 Overview of the Usage Procedure

The following procedures can be used for encoding and decoding.
No initialization or termination processing is required.
Perform appropriate error checking for each function call.

2. 1 Encoding

(1) Prepare 16-bit signed linear PCM data to be encoded and the buffer for storing the encoded result

The input data is 16-bit signed linear PCM data.
For the output buffer, allocate at least the same number of bytes as the number of samples of PCM data.

(2) Encode data

Call sceUlawEncode() to perform encoding.
The specified number of samples of data that was compressed to u-law compressed format is stored in the output buffer.

2. 2 Decoding

(1) Prepare compressed u-law data to be decoded and the buffer for storing the decoded result

The input data is compressed u-law data.
For the output buffer, allocate at least twice the number of bytes as the number of samples of u-law data.

(2) Decode data

Call sceUlawDecode() to perform decoding.
The specified number of samples of data that was expanded to 16-bit signed linear PCM format is stored in the output buffer.
