SCE_WAVE_LOOP
 [Constant Definitions]
Loop specification flag

Definition

#include <libwave.h>
#define  SCE_WAVE_LOOP			(1 << 0)
Description

Specify this flag to play a continuous loop in the libwave ADPCM (VAG) playback sound system. Specify the flag with sceWaveSetVoice() or sceWaveSetLoopMode().
If SCE_WAVE_LOOP is specified and a loop is also specified in the VAG data, a continuous loop is played based on the loop specification in the VAG data. If there is no loop specified in the VAG data, a continuous loop is played over the entire VAG data.

SCE_WAVE_LOOP cannot be specified at the same time as SCE_WAVE_VAGFLAG.

See Also

sceWaveSetVoice(), sceWaveSetLoopMode(), SCE_WAVE_VAGFLAG


SCE_WAVE_VAGFLAG
 [Constant Definitions]
Use loop seting within VAG file

Definition

#include <libwave.h>
#define  SCE_WAVE_VAGFLAG		(1 << 1)
Description

Specify this flag to play a loop using the loop flag contained in the VAG file in the libwave ADPCM (VAG) playback sound system. Specify the flag with sceWaveSetVoice() or sceWaveSetLoopMode().
If SCE_WAVE_VAGFLAG is specified and a loop is also specified in the VAG data, a continuous loop is played based on the loop specification in the VAG data. If there is no loop specified in the VAG data, oneshot playback is performed.

SCE_WAVE_VAGFLAG cannot be specified at the same time as SCE_WAVE_LOOP.

See Also

sceWaveSetVoice(), sceWaveSetLoopMode(), SCE_WAVE_LOOP

sceWaveSetVoice
 [Function]
Set sound data

Syntax

int sceWaveSetVoice( 
	int iVoice, 
	const void* pBuf, 
	SceSize size, 
	int iMode 
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupted-enabled state)
Not multithread safe

Arguments

  iVoice     Voice number (0 - 7)  
  pBuf     Pointer to buffer containing sound data   
  size     Size of sound data (bytes)  
  iMode     Mode  
  SCE_WAVE_LOOP: Loop continuously   
  SCE_WAVE_VAGFLAG: Use loop setting in VAG file  
  0:oneshot playback  

Return Value

  Macro     Value     Meaning  
  SCE_OK     0     Normal completion  
  SCE_WAVE_ERROR_VOICENUM     0x80440005     Invalid voice number  
  SCE_WAVE_ERROR_BUFFPTR     0x80440006     Invalid buffer pointer   
  SCE_WAVE_ERROR_SIZE     0x80440007     Invalid data size  
  SCE_WAVE_ERROR_MODE     0x80440008     Invalid mode  
  SCE_WAVE_ERROR_VOICEPLAY     0x80440009     Specified voice is currently being played  


Description

This function sets VAG data within the buffer specified by pBuf and iSize for the voice having the number specified by iVoice. The VAG data that was set can be played by calling sceWavePlay() and specifying the voice number.
Various settings are possible for iMode. To use the loop setting that is contained in the VAG data, specify SCE_WAVE_VAGFLAG for iMode, and to explicitly perform one-shot playback of VAG data, specify 0 for iMode. Also, to explicitly perform loop playback, specify SCE_WAVE_LOOP for iMode.
Note that sound data cannot be set if the specified voice is being played. Before setting sound data, check the status of the voice to make sure that it is stopped (SCE_WAVE_STAT_STOP).

sceWaveSetVoice() does not create a copy of the VAG data within the buffer specified by pBuf. Instead, it directly plays the VAG data within the specified buffer. Be sure to save the data within the buffer while the VAG data is playing.
Do not release the buffer specified by pBuf while VAG data within the buffer is being played. Proper operation cannot be guaranteed if the buffer is released during playback. To release the buffer, first confirm that the voice is in stopped state (SCE_WAVE_STAT_STOP). Likewise, proper operation cannot be guaranteed if the buffer contents are changed during playback. To change the contents of the buffer, first make sure that the voice is in stopped state (SCE_WAVE_STAT_STOP).

Example

/* Set pVagBuf for voice 0 and perform one-shot playback */
sceWaveSetVoice(0,pVagBuf,VagSize,0);

/* Set pVagBuf for voice 0 and perform endless-loop playback */
sceWaveSetVoice(0,pVagBuf,VagSize,SCE_WAVE_LOOP);

/* Set pVagBuf for voice 0 and perform playback according to the loop */
/* setting within the VAG data */
sceWaveSetVoice(0,pVagBuf,VagSize,SCE_WAVE_VAGFLAG);
See Also

sceWavePlay()

sceWaveSetVolume
 [Function]
Set volume

Syntax

int sceWaveSetVolume(
	int iVoice, 
	int iVolL, 
	int iVolR 
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupted-enabled state)
Not multithread safe

Arguments

  iVoice     Voice number (0 - 7)  
  iVolL     Left volume (0 - 127)  
  iVolR     Right volume (0 - 127)  

Return Value

  Macro     Value     Meaning  
  SCE_OK     0     Normal completion  
  SCE_WAVE_ERROR_VOICENUM     0x80440005     Invalid voice number  
  SCE_WAVE_ERROR_VOL     0x8044000A     Invalid volume value  


Description

This function sets the volume of the specified voice.
When the maximum volume is specified, the volume will be equal to the peak value of the original waveform.
Also, since the volume values maintained internally by the library range from 0 to 4096, values in the range 0 to 127 are converted to 0 to 4096 before they are set. If you want to set the volume to 0 to 4096, please use sceWaveSetVolumeEx().

Example

/* Set the volume so that voice 0 is output from the center of the maximum volume */
sceWaveSetVolume(0,SCE_WAVE_VOL_MAX, SCE_WAVE_VOL_MAX);
See Also

sceWavePlay(), sceWaveGetVolume()

sceWavePlay
 [Function]
Play sound

Syntax

int sceWavePlay( 
	int iVoice, 
	int iVolL, 
	int iVolR
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupted-enabled state)
Not multithread safe

Arguments

  iVoice     Voice number (0 - 7)  
  iVolL     Left volume (0 - 127)  
  iVolR     Right volume (0 - 127)  

Return Value

  Macro     Value     Meaning  
  SCE_OK     0     Normal completion  
  SCE_WAVE_ERROR_VOICENUM     0x80440005     Invalid voice number  
  SCE_WAVE_ERROR_BUFFPTR     0x80440006     Invalid buffer pointer  
  SCE_WAVE_ERROR_VOL     0x8044000A     Invalid volume value  


Description

This function starts playback from the beginning of the sound data for the voice specified by iVoice.
If the voice specified for iVoice is being played, playback starts from the beginning after the sound that is currently being played has stopped.
If sceWavePlay() is executed without having specified an appropriate buffer using sceWaveSetVoice(), SCE_WAVE_ERROR_BUFFPTR is returned. Be sure to specify an appropriate buffer by using sceWaveSetVoice() before executing sceWavePlay().

Example

static char vagBuf[];
int iVagSize;

/* Initialization */
sceWaveInit();

/* Start sound system */
sceWaveStart();

/* Set data for voice 0 */
sceWaveSetVoice( 0, vagBuf, iVagSize, SCE_WAVE_VAGFLAG );

/* Play the sound that was set for voice 0 at maximum volume */
sceWavePlay( 0, SCE_WAVE_VOL_MAX, SCE_WAVE_VOL_MAX );

/* Various processing */

/* Stop sound */
sceWaveStop( 0 );

/* Stop sound system */
sceWaveEnd();

/* Exit */
sceWaveExit();
See Also

sceWaveSetVoice(), sceWaveStop()

 [Function]

Syntax

int sceWaveAudioWrite(

 )

sceWaveAudioWriteBlocking
 [Function]
Output audio data (blocking)

Syntax

int sceWaveAudioWriteBlocking( 
	int iCh,
	int iVolL,
	int iVolR,
	const void* pBuf 
)
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be called in an interrupt-enabled state)
Multithread safe

Arguments

  iCh     Audio output-only channel number (0 - 2)  
  iVolL     Left channel volume (0 - 32768)  
  iVolR     Right channel volume (0 - 32768)  
  pBuf     Pointer to output data buffer  

Return Value

If the return value is greater than or equal to zero, the number of samples that could be registered in the audio driver is returned, If the value is less than zero, the following errors are returned.

  Macro     Value     Meaning  
  SCE_WAVE_ERROR_AUDIOCH     0x80440010     Channel number is invalid  
  SCE_WAVE_ERROR_VOL     0x8044000A     Volume value is invalid  
  SCE_AUDIO_ERROR_PRIV_REQUIRED     0x80260004     Pointer to output buffer is invalid  


Description

This function performs low-level audio output. Specifically, it registers the output data buffer specified by pBuf in the audio driver. If the output data buffer could not be registered in the audio driver, the thread that called this API enters a wait state and waits until it can be registered. When the output data buffer can be registered in the audio driver, the thread transitions from READY to RUN state, and processing to register the output buffer in the audio driver is performed. After the output buffer has been registered in the audio driver, the function returns to the caller.
After the data is registered, the audio driver starts outputting the registered sound data to the DAC according to the settings that were specified in advance using sceWaveAudioSetSample(), sceWaveAudioSetFormat(), and sceWaveAudioSetVolume().
Note that if NULL is specified for pBuf, without performing actual audio output, processing waits until audio data currently registered in the audio driver that has not yet been output is completely output. However, this wait will occur only until the "currently" registered data is output. If another thread is performing audio output, there is a possibility that new audio will be output by the other thread after the wait state is entered. As a result, this feature cannot be used to wait for all audio output that is being performed by other threads to end. (For this purpose, use a thread synchronization function such as an event flag.)

Notes

Do not change the contents of the output data buffer until the audio driver has completed output processing.

Example

/* Output 16-bit PCM STEREO data that is in pcmBuf to channel 0 */
sceWaveAudioWriteBlocking(
	0,
	SCE_WAVE_AUDIO_VOL_MAX, SCE_WAVE_AUDIO_VOL_MAX,
	pcmBuf
);
