Audio Routing Driver Reference


SCE_AUDIOROUTING_NORMAL
 [Constant Definitions]
Normal speaker output switching mode

Definition

#include <audiorouting.h>
#define  SCE_AUDIOROUTING_NORMAL		0
Description

This constant represents the normal speaker output switch state (which automatically enables or disables the speaker output depending on whether a device is connected to the headphone jack of the PSP™).

See also

sceAudioRoutingSetMode()

sceAudioRoutingSetMode
 [Function]
Set automatic speaker output switching

Definition

#include <audiorouting.h>
int sceAudioRoutingSetMode(
	SceInt32 mode
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be in an interrupt-enabled state)
Multithread safe

Arguments

  mode     Switching mode, which can be one of the following:  
  SCE_AUDIOROUTING_MODE0(=SCE_AUDIOROUTING_NORMAL)  
  SCE_AUDIOROUTING_MODE1  
  SCE_AUDIOROUTING_MODE2  
  SCE_AUDIOROUTING_MODE3  

Return Values

When the function completes normally, 0 is returned. If the mode argument is invalid, SCE_AUDIOROUTING_ERROR_INVAL is returned.

Description

This function changes the behavior of the PSP™ built-in software that automatically switches the speaker output on and off. Specifically, this function can set to a mode in which the speaker output is turned on only when there is no device connected to the headphone jack, a mode in which the speaker output will always be on regardless of whether a device is connected to the headphone jack, a mode in which output is only sent to the speaker and not to the headphone jack or a mode in which output is sent only to the headphone jack and not to the speaker.

sceAudioRoutingGetMode
 [Function]
Get speaker output automatic switching mode

Definition

#include <audiorouting.h>
int sceAudioRoutingGetMode(
	void
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (must be in an interrupt-enabled state).
Multithread safe

Arguments

None

Return Values

The current speaker-switching state is returned, which will be one of the following: SCE_AUDIOROUTING_MODE0, SCE_AUDIOROUTING_MODE1, SCE_AUDIOROUTING_MODE2 or SCE_AUDIOROUTING_MODE3.

Description

This function returns the speaker output switching behavior that is currently being performed by the built-in software of the PSP™.

sceAudioRoutingSetVolumeMode
 [Function]
Set volume mode

Definition

#include <audiorouting.h>
int sceAudioRoutingSetVolumeMode(
	SceInt32 mode
);
Calling Conditions

Cannot be called from an interrupt handler
Can be called from a thread (must be in an interrupt-enabled state).
Multithread safe

Arguments

  mode     Volume mode, which can be one of the following:  
  SCE_AUDIOROUTING_VOLUME_MODE_NORMAL  
  SCE_AUDIOROUTING_VOLUME_MODE_FORCE_MAX  

Return Values

When the function completes normally, 0 is returned. If the mode argument is invalid, SCE_AUDIOROUTING_ERROR_INVAL is returned.

Description

This function sets the PSP™ volume mode. Specifically, it can set either a mode in which sound output is controlled by the volume buttons on the PSP™, or a mode in which sound is always output at the maximum level.


sceAudioRoutingGetVolumeMode
 [Function]
Get volume mode

Definition

#include <audiorouting.h>
int sceAudioRoutingGetVolumeMode(
	void
);
Calling Conditions

Can be called from an interrupt handler
Can be called from a thread (must be in an interrupt-enabled state).
Multithread safe

Arguments

None

Return Values

The current volume-mode setting is returned, which will be either SCE_AUDIOROUTING_VOLUME_MODE_NORMAL or SCE_AUDIOROUTING_VOLUME_MODE_FORCE_MAX.

Description

This function returns the current volume-mode setting of the PSP™.
