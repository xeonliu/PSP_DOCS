sceSasInit
 [関数]
初期化処理

形式

#include <libsas.h>

int sceSasInit(
	void
);
呼出条件

割り込みハンドラから呼び出し可能
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

なし

返り値

正常終了の場合、SCE_OK
返り値が負（<0）の場合エラー
（詳細はエラーコード一覧を参照してください）

解説

SASの状態を初期化します。
SASを使用する場合、必ずこの関数またはsceSasInitWithGrain()を呼ぶ必要があります。
この関数を用いてSASを初期化した場合、一度のsceSasCore()およびsceSasCoreWithMix()関数呼び出しにより生成されるPCMサンプル数（粒度）はSCE_SAS_GRAIN_SAMPLES（=256）となります。また、出力モードはステレオモードとして初期化されます。

使用例

#include <libsas.h>

int main(void)
{
	int   result;
	
	result = sceSasInit();
	
	return 0;
}

SASを使用する場合、必ずこの関数またはsceSasInit()を呼ぶ必要があります。
この関数を用いてSASを初期化することで、一度のsceSasCore()およびsceSasCoreWithMix()関数呼び出しにより生成されるPCMサンプル数（粒度）をSCE_SAS_GRAIN_SAMPLES（=256）以外とすることが可能です。また、出力モードはステレオモードとして初期化されます。

sceSasCore
 [関数]
SAS定期処理

形式

#include <libsas.h>

int sceSasCore(
	short *pOut
);
呼出条件

割り込みハンドラから呼び出し不可能
スレッドから呼び出し可能（割り込み許可状態で呼び出すこと）
マルチスレッドセーフである

引数

  pOut     波形出力バッファへのポインタ  

返り値

正常終了の場合、SCE_OK
返り値が負（<0）の場合エラー
（詳細はエラーコード一覧を参照してください）
解説

当関数をコールする事で一粒度分の波形を出力します。一粒度のサンプル数は、sceSasInit()関数で初期化を行ったときはSCE_SAS_GRAIN_SAMPLESに、sceSasInitWithGrain()関数で初期化を行ったときはそのとき指定した値となります。
pOutは64バイトアラインメントが要求されます。
この関数はマルチスレッドセーフですので、この関数の呼び出し中にたとえばsceSasSetKeyOn()関数などを呼ぶことは可能ですが、sceSasCore()およびsceSasCoreWithMix()そのものを複数スレッドから同時に呼び出すことはできません（SCE_SAS_ERROR_BUSYが返ります）。

使用例

#include<libsas.h>

int main(void)
{
	int   result;
	int iSide = 0;
	/* ダブルバッファにする */
	static short pcmBuf[2][256 * 2] __attribute__((aligned(64)));
	
	/* 1粒度分波形出力処理 */
	result = sceSasCore(pcmBuf[iSide]);
	iSide ^= 1;
	
	return 0;
}
sceSasCoreWithMix
 [関数]
外部PCMミキシング機能つきSAS定期処理

形式

#include <libsas.h>

int sceSasCoreWithMix(
	short *pInOut,
	int lvol,
	int rvol
);
呼出条件

割り込みハンドラから呼び出し不可能
スレッドから呼び出し可能（割り込み許可状態で呼び出すこと）
マルチスレッドセーフである

引数

  pInOut     波形出力バッファへのポインタ  
  lvol     外部PCM入力 左チャネルのボリューム値  
  rvol     外部PCM入力 右チャネルのボリューム値  

返り値

正常終了の場合、SCE_OK
返り値が負（<0）の場合エラー
（詳細はエラーコード一覧を参照してください）

解説

当関数をコールする事で一粒度分の波形を合成し、さらに指定された領域にあるPCMデータとミキシングを行った結果が上書きされて出力されます。一粒度のサンプル数は、sceSasInit()関数で初期化を行ったときはSCE_SAS_GRAIN_SAMPLESに、sceSasInitWithGrain()関数で初期化を行ったときはそのとき指定した値となります。
pInOutは64バイトアラインメントが要求されます。
ミキシング時に、入力PCMデータに対して音量変化をさせることが可能です。ボリューム値として設定できる範囲は

0 ≦ lvol, rvol ≦ SCE_SAS_VOLUME_MAX

となります。
この関数はマルチスレッドセーフですので、この関数の呼び出し中にたとえばsceSasSetKeyOn()関数などを呼ぶことは可能ですが、sceSasCore()およびsceSasCoreWithMix()そのものを複数スレッドから同時に呼び出すことはできません（SCE_SAS_ERROR_BUSYが返ります）。

使用例

#include<libsas.h>

int main(void)
{
	int   result;
	int iSide = 0;
	/* ダブルバッファにする */
	static short pcmBuf[2][256 * 2] __attribute__((aligned(64)));
	
	/* 1粒度分波形出力処理 */
	result = sceSasCoreWithMix(pcmBuf[iSide],
	SCE_SAS_VOLUME_MAX, SCE_SAS_VOLUME_MAX);
	iSide ^= 1;
	
	return 0;
}

sceSasConcatenateATRAC3
 [関数]
ATRAC3™ストリーミングにおけるバッファ追加

形式

#include <libsas.h>

int sceSasConcatenateATRAC3 (
	int iVoiceNum,
	const void *buffer,
	unsigned int size
);
呼出条件

割り込みハンドラから呼び出し不可能
スレッドから呼び出し可能（割り込み許可状態で呼び出すこと）
マルチスレッドセーフである

引数

  iVoiceNum     ボイス番号（0 -31）  
  buffer     追加したいバッファへのポインタ  
  size     追加したいバッファのサイズ  

返り値

正常終了時にSCE_OK
返り値が負（<0）の場合エラー
（詳細はエラーコード一覧を参照してください）

解説

ATRAC3™を再生しているボイスに対して、続くデータを指定するのに使う関数です。

ATRAC3™のストリーミング再生を行う場合、再生が進むよりも早くデータを読み込み、この関数を使ってSASにそのデータの位置とサイズを通知する必要があります。もし、データの追加が間に合わなかった場合、該当するボイスは無音になり、追加された後に続きから発音されます。

SASの内部にはバッファは2つ（現在再生しているバッファと次に使うバッファの合計2つ）しか登録できません。3つ以上登録しようとするとエラーになります。

また、短い間隔（高い頻度）でデータの継ぎ足しを行うと実行効率が低下して期待する性能が出ないことがあります。このような事態を防ぐため、データ終端以外の場所において一度に継ぎ足しするデータのサイズが1024バイト未満だった場合にはエラーが返ります。エラーにならないケースであっても、ある程度以上のサイズのバッファ（ビットレートに対して8倍程度）を用意することを推奨します。

なお、すべてのデータを完全に読み込み終わった（ATRAC3™データファイルを終端まで読み込み終わった）後、データが本当に終了したことを伝えるために、ダミーで一度データを継ぎ足す必要があります。これを行わなかった場合、SASの内部で終端を認識できないため、ATRAC3™ストリーミング再生を行っているボイスは再生中のままとなり、EndFlagはアサートされないままになります。

全ボイスのエンベロープ値を取得します。SASは全部で32(SCE_SAS_VOICE_MAX)ボイスの音声を取り扱うことができますので、取得されるデータが格納される先である、pHeightsは32以上の要素が必要です。
一度にまとめて情報を取得できるので、sceSasGetEnvelope()を何度も呼び出すより、パフォーマンスが向上することが期待されます。

使用例

#include <libsas.h>

int main(void)
{
	int   result, i;
	SceInt32 envs[SCE_SAS_VOICE_MAX];
	
	/* ボイス3のエンベロープ値を調査 */
	result = sceSasGetEnvelope( envs );
	if (result < 0)
		return -1;
	for (i = 0; i < SCE_SAS_VOICE_MAX; i++) {
		printf("voice %d env = %d\n", i, envs[i]);
	}
	return 0;
}

sceSasSetOutputmode
 [関数]
出力モードの設定

形式

#include <libsas.h>

int sceSasSetOutputmode(
	int outputmode
);
呼出条件

割り込みハンドラから呼び出し不可能
スレッドから呼び出し可能（割り込み許可状態で呼び出すこと）
マルチスレッドセーフである

引数

  outputmode     出力モード  
  （SCE_SAS_OUTPUTMODE_STEREOまたは  
  SCE_SAS_OUTPUTMODE_MULTI）  

返り値

正常終了の場合、SCE_OK
返り値が負（<0）の場合エラー
（詳細はエラーコード一覧を参照してください）

解説

出力モードを変更します。引数で与える値であるSCE_SAS_OUTPUTMODE_STEREOはステレオモード（2チャネル出力、エフェクトあり）、SCE_SAS_OUTPUTMODE_MULTIはマルチチャネルモード（4チャネル出力、エフェクトなし）に対応します。各出力モードの詳細は別資料「SAS概要」を参照してください。
マルチチャネルモード時には、エフェクトを付加する前の4チャネルのモノラル音声が出力されます。そのため、エフェクト処理は行われません。このとき、エフェクト設定関数を呼ぶと、エフェクト設定関数はエラーを返します。
ステレオモードに比べ、マルチチャネルモードは出力チャネル数が2倍になるため、sceSasCore()およびsceSasCoreWithMix()に必要な出力バッファサイズも2倍必要になります。バッファサイズが不足している場合には、不正な領域に出力データが書き込まれる恐れがありますので注意してください。
各ボイスのマルチチャネルモードの4チャネルへの割り振りは、sceSasSetVolume()を利用してください。
マルチチャネルモード時には、出力PCMデータの並びがステレオモードと異なり、sceSasCore()およびsceSasCoreWithMix()で得られた出力PCMデータを、そのままlibwaveなどのサウンド出力ライブラリに渡すことはできません。マルチチャネルモードの出力PCMデータは、独自のエフェクト、ミキシング、インタリーブ処理を行った後で、libwaveなどに渡してください。

使用例

#include <libsas.h>

int main(void)
{
	int   result;
	
	result = sceSasSetOutputmode(SCE_SAS_OUTPUTMODE_MULTI);
	
	return 0;
}
