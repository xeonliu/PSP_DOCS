SceMp3InitArg
 [構造体]
デコード資源の初期化変数

構造

#include <libmp3.h>
typedef struct {
	SceOff mp3HeadPos;
	SceOff mp3TailPos;
	SceUChar8 *mp3Buf;
	SceInt32 mp3BufByte;
	SceUChar8 *pcmBuf;
	SceInt32 pcmBufByte;
} SceMp3InitArg;
メンバ

  mp3HeadPos     MP3ファイル最初のオーディオデータの先頭位置  
  mp3TailPos     MP3ファイル最後のオーディオデータの末尾位置  
  mp3Buf     入力バッファの先頭アドレス  
  mp3BufByte     入力バッファのバイト数  
  pcmBuf     出力バッファの先頭アドレス  
  pcmBufByte     出力バッファのバイト数  

解説

MP3ハンドルの初期化に利用する構造体です。sceMp3ReserveMp3Handle()の引数として利用されます。入出力バッファのアドレスと大きさ、ループ再生のためのファイル情報から構成されます。
mp3HeadPosには、ファイル先頭のタグを除く最初のオーディオデータの先頭位置を指定してください。mp3TailPosには、ファイル末尾のタグを除く最後のオーディオデータの末尾位置を指定してください。タグの有無の判断、タグのサイズを求める方法に関しては、「libmp3概要」を参照してください。
入出力バッファのメモリアラインメントは、64バイトアラインメントである必要があります。また、必要な入出力バッファサイズは、以下のようになります。

  • 入力(MP3)バッファサイズ
    8192バイト以上、かつ64バイトの倍数
  • 出力(PCM)バッファサイズ
    9216バイト(最大1152サンプル×2バイト×2チャネル×2バッファ)

解説

libmp3に必要な計算資源を確保します。libmp3の各機能を利用するためには、まず、本関数を実行する必要があります。
この計算資源によって、複数個のMP3ストリームデータを並列でデコードすることが可能です。個々のデコード処理に必要な計算資源をlibmp3では、デコード資源と呼びます。同時に利用可能なデコード資源の数は、2です。デコード資源は後述するMP3ハンドルと一対一に対応するため、デコード資源の数は、MP3ハンドルの数に一致します。

使用例

/* libmp3に必要な計算資源を確保 */
sceMp3InitResource ();

解説

sceMp3InitResource()によって確保されたデコード資源の使用権を取得します。この使用権は、MP3ハンドルというデコード資源と一対一に対応するIDによって管理されます。MP3データのデコード処理を実行する前には、必ずMP3ハンドルを取得する必要があります。
引数mp3InitArgの設定方法は2通りあり、それぞれ得られるMP3ハンドルの種別が異なります。1つは、MP3ハンドルに与える初期値を設定したSceMp3InitArg変数へのポインタを与える方法です。この場合、低レベルAPIであるsceMp3LowLevelXxx()を除く、すべてのAPIを利用することができます。もう1つは、NULLを与え、MP3ハンドルに初期値を与えない方法です。この場合、低レベルAPIであるsceMp3LowLevelXxx()および一部のAPIしか利用することができません。そのため、基本的には、初期値を与える方法を推奨します。API利用制限については、備考を参照してください。
また、libmp3には、同時に利用可能なデコード資源に制限があるため、利用可能なデコード資源の数だけ使用権を取得した後では、本関数を実行してもエラーが返り、新たにデコード資源を取得することができません。その場合には、sceMp3ReleaseMp3Handle()を呼び、利用していないMP3ハンドルをlibmp3に返還してください。
本関数のMP3ハンドルを取得する手順に関しては、マルチスレッドセーフです。したがって、複数スレッドから本関数を呼び出しても、各スレッドで重複するMP3ハンドルが割り当てられることはありません。しかし、デコード処理など、その後の処理に関しては、マルチスレッドセーフではありません。取得したMP3ハンドル以外のMP3ハンドルを引数としてデコード処理などを行ったときの動作は、保証されませんので注意してください。

使用例

SceInt32 handle;
SceMp3InitArg mp3InitArg;
/* mp3InitArg のメンバに初期値をセット*/
/* デコード資源の使用権を管理するMP3ハンドルを取得 */
handle = sceMp3ReserveMp3Handle(&mp3InitArg);

解説

指定されたMP3ハンドルに対応するデコード資源に対して、入力バッファに読み込まれたMP3データのMPEG/Audioフレームヘッダを元に初期化処理を行います。本関数の実行前にsceMp3ReserveMp3Handle()の引数として与えた入力バッファにMP3データを読み込んでおく必要があります。
MP3ファイルからMP3データを入力バッファに読み込む方法は、以下の通りです。まず、sceMp3GetInfoToAddStreamData()を実行し、MP3データを入力バッファに読み込むための情報を取得します。次にこれらの情報から入力バッファにMP3データを読み込みます。そして、sceMp3NotifyAddStreamData()を実行し、読み込んだMP3データサイズをlibmp3に通知します。

使用例

/* 入力バッファにMP3データを読み込み */
sceMp3GetInfoToAddStreamData(handle, &mp3DstPtr, &writableByte, &mp3SrcPos);
sceIoLseek(fd, mp3SrcPos, SCE_SEEK_SET);
mp3AddByte = sceIoRead(fd, mp3DstPtr, writableByte);
/* 入力バッファに読み込んだMP3データサイズをlibmp3に通知 */
sceMp3NotifyAddStreamData(handle, mp3AddByte);
/* デコード資源を初期化 */
sceMp3Init(handle);
備考

本関数の引数には、あらかじめsceMp3ReserveMp3Handle()によって取得しておいたMP3ハンドルを指定してください。

解説

入力バッファに読み込まれたMP3データを1フレームだけリニアPCMデータにデコードし、出力バッファに書き込みます。入力のMP3データ形式に依らず、出力のPCMデータのフォーマットは、ビット深度16ビットのステレオ形式になります。このPCMデータは、pcmDstPtrが指すアドレスを先頭アドレスとし、左、右、左、……のようにインタリーブ形式で左右交互に出力されます。出力されるPCMデータのサイズは、返り値によって取得することができます。出力されるPCMデータは、次にsceMp3Decode()を実行するまでの間だけ有効です。
デコード処理を実行することで入力バッファ内のMP3データが消費されます。デコード処理を行う前に必ずsceMp3CheckStreamDataNeeded()を実行し、必要に応じてMP3データを追加してください。MP3データの読み込みは、同期または非同期に行うことができます。また、MP3データの追加完了が遅く、入力バッファ内のMP3データが残り1フレームもない場合、sceMp3Decode()は無音のPCMデータを出力します。

使用例

デコードされたPCMデータをvaudio経由でオーディオ出力する例を以下に示します。

/* MP3データをデコード */
sceMp3Decode(handle, &pcmDstPtr);
/* vaudioにPCMデータを受け渡し */
sceVaudioOutputBlocking(vol, pcmDstPtr);
備考

本関数実行前にsceMp3Init()によってデコード資源の初期化を完了させてください。
事前に、sceVaudioChReserve()によって、サンプリング周波数とPCMサンプル数をvaudioに指定してください。

使用例

/* MP3データを入力バッファに追加する必要性をチェック */
if (sceMp3CheckStreamDataNeeded(handle) > 0) {
         /* 入力バッファにMP3データを読み込み */
         sceMp3GetInfoToAddStreamData(handle, &mp3DstPtr, &writableByte, &mp3SrcPos);
         sceIoLseek(fd, mp3SrcPos, SCE_SEEK_SET);
         mp3AddByte = sceIoRead(fd, mp3DstPtr, writableByte);
         /* 入力バッファに読み込んだMP3データサイズをlibmp3に通知 */
         sceMp3NotifyAddStreamData(handle, mp3AddByte);
}

解説

入力バッファにMP3データを追加するための情報を取得します。本関数実行後、引数を通じて追加のMP3データの書き込み先アドレス*mp3DstPtr、データサイズ*writableByte、およびMP3ファイルの読み出しシーク位置*mp3SrcPosを取得できます。これらの情報を取得後、以下の方法にしたがってMP3データを入力バッファに追加します。
まず、ファイルディスクリプタをMP3ファイル先頭から*mp3SrcPosだけ移動させます。それから、移動させたファイルディスクリプタを利用し、*writableByteバイトのMP3データを追加先アドレス*mp3DstPtrに読み込みます。非同期に読み込むことも可能です。
MP3ファイルからMP3データの追加が終わった後は、sceMp3NotifyAddStreamData()によってlibmp3に対しMP3データの追加を通知します。詳しくは、sceMp3NotifyAddStreamData()の項を参照してください。
なお、MP3ファイル内のMP3データがすべて入力バッファに読み込まれている場合、またはMP3データを入力バッファに読み込む必要がない場合は、*mp3DstPtrとしてNULL、*writableByteとして0が返ります。

使用例

/* MP3データを入力バッファに追加する必要性をチェック */
if (sceMp3CheckStreamDataNeeded(handle) > 0) {
         /* 入力バッファにMP3データを読み込み */
         sceMp3GetInfoToAddStreamData(handle, &mp3DstPtr, &writableByte, &mp3SrcPos);
         sceIoLseek(fd, mp3SrcPos, SCE_SEEK_SET);
         mp3AddByte = sceIoRead(fd, mp3DstPtr, writableByte);
         /* 入力バッファに読み込んだMP3データサイズをlibmp3に通知 */
         sceMp3NotifyAddStreamData(handle, mp3AddByte);
}
備考

指定されたMP3ハンドルに対応するデコード資源に対して、初期化処理を行います。ただし、sceMp3ReserveMp3Handle()の引数としてNULLを与えて得た低レベルAPI用MP3ハンドルのみ、本関数を実行可能です。
低レベルAPI用MP3ハンドルでは利用可能なAPIが制限されますが、本関数の引数mp3DstPtrにMP3ファイル先頭のMP3データを渡すことで、利用可能なAPIを増やすことができます。この操作によって、利用可能となるAPIは以下の通りです。
  • sceMp3GetMaxOutputSample()
  • sceMp3GetMPEGVersion()
  • sceMp3GetSamplingRate()
  • sceMp3GetBitRate()
  • sceMp3GetMp3ChannelNum()
この操作は任意であり、mp3DstPtrにNULLを渡すことも可能です。また、MP3ファイル先頭にタグ情報が付加されている場合、タグ情報を除いたMP3データを渡す必要がありますので、ご注意ください。

使用例

/* 低レベルAPIにおけるデコード資源を初期化 */
sceMp3LowLevelInit(handle, NULL);
備考

本関数の引数には、あらかじめsceMp3ReserveMp3Handle()によって取得しておいたMP3ハンドルを指定してください。

 [関数]
低レベルAPIにおけるMP3データのデコード

形式

#include <libmp3.h>
SceInt32 sceMp3LowLevelDecode(
	SceInt32 handle,
	const SceUChar8 *mp3DstPtr,
	SceUInt32 *mp3UsedByte,
	SceShort16 *pcmDstPtr,
	SceUInt32 *pcmOutByte

呼出条件

割り込みハンドラから呼び出し不可
スレッドから呼び出し可能（割り込み禁止 / 許可状態に依存しない）
マルチスレッドセーフでない

引数

  handle     デコード資源管理用MP3ハンドル  
  mp3DstPtr     デコードしたいMP3データを指すポインタ  
  mp3UsedByte     デコードによって消費されたMP3データサイズ  
  pcmDstPtr     デコードによって出力されたPCMデータの書き込み先アドレスを指すポインタ

返り値

  値     説明  
  0     正常終了  
  負(<0)     エラー  

エラー時の返り値については、巻末の「エラーコード一覧」をご覧ください。

解説

mp3DstPtrで指定した先にあるMP3データ1フレームだけリニアPCMデータにデコードし、にpcmDstPtrの先に書き込みます。入力のMP3データ形式に依らず、出力のPCMデータのフォーマットは、ビット深度16ビットのステレオ形式になります。このPCMデータは、pcmDstPtrが指すアドレスを先頭アドレスとし、左、右、左、……のようにインタリーブ形式で左右交互に出力されます。デコードによって消費されたMP3データサイズは、mp3UsedByteによって取得することができます。出力されるPCMデータサイズは、pcmOutByteによって取得することができます。

使用例

デコードされたPCMデータをvaudio経由でオーディオ出力する例を以下に示します。

/* MP3データをデコード */
sceMp3LowLevelDecode(handle, mp3DstPtr, &mp3UsedByte, pcmDstPtr, &pcmOutByte);
/* vaudioにPCMデータを受け渡し */
sceVaudioOutputBlocking(vol, pcmDstPtr);
備考

本関数実行前にsceMp3LowLevelInit()によってデコード資源の初期化を完了させてください。
事前に、sceVaudioChReserve()によって、サンプリング周波数とPCMサンプル数をvaudioに指定してください。
